# Simple CRUD Using an In-Memory JSON Object (Express Only)

Before introducing databases, it is important to understand **how CRUD works logically**.

Here, we will:

* Use **one JSON array**
* Keep everything in **a single file**
* Perform **basic CRUD operations**
* Focus on Express routing and REST methods

---

### Step 1: Sample Data (In-Memory JSON)

```js
const users = [
  { id: 1, name: "Arjun", role: "student" },
  { id: 2, name: "Priyesha", role: "mentor" }
];
```

Important:

* This data exists **only in memory**
* Restarting the server will reset it
* This is intentional for learning

---

### Step 2: GET – Fetch All Users

```js
app.get("/users", (req, res) => {
  res.status(200).json(users);
});
```

What this demonstrates:

* REST resource: `/users`
* HTTP method: GET
* Returning JSON data

---

### Step 3: GET – Fetch User by ID

```js
app.get("/users/:id", (req, res) => {
  const userId = Number(req.params.id);
  const user = users.find(u => u.id === userId);

  if (!user) {
    return res.status(404).json({ message: "User not found" });
  }

  res.status(200).json(user);
});
```

Key learning points:

* Route parameters (`:id`)
* Searching data
* Proper `404` handling

---

### Step 4: POST – Create a New User

```js
app.use(express.json());

app.post("/users", (req, res) => {
  const newUser = {
    id: users.length + 1,
    name: req.body.name,
    role: req.body.role
  };

  users.push(newUser);

  res.status(201).json({
    message: "User created",
    user: newUser
  });
});
```

Key points:

* Using request body
* Creating new resource
* Returning `201 Created`

---

### Step 5: PUT – Replace a User

```js
app.put("/users/:id", (req, res) => {
  const userId = Number(req.params.id);
  const index = users.findIndex(u => u.id === userId);

  if (index === -1) {
    return res.status(404).json({ message: "User not found" });
  }

  users[index] = {
    id: userId,
    name: req.body.name,
    role: req.body.role
  };

  res.status(200).json({
    message: "User replaced",
    user: users[index]
  });
});
```

Important distinction:

* PUT replaces the **entire object**
* Missing fields are overwritten

---

### Step 6: PATCH – Update Part of a User

```js
app.patch("/users/:id", (req, res) => {
  const userId = Number(req.params.id);
  const user = users.find(u => u.id === userId);

  if (!user) {
    return res.status(404).json({ message: "User not found" });
  }

  if (req.body.name) user.name = req.body.name;
  if (req.body.role) user.role = req.body.role;

  res.status(200).json({
    message: "User updated",
    user
  });
});
```

This demonstrates:

* Partial updates
* Why PATCH exists
* Safer updates than PUT

---

### Step 7: DELETE – Remove a User

```js
app.delete("/users/:id", (req, res) => {
  const userId = Number(req.params.id);
  const index = users.findIndex(u => u.id === userId);

  if (index === -1) {
    return res.status(404).json({ message: "User not found" });
  }

  users.splice(index, 1);

  res.status(204).end();
});
```

Key learning:

* Resource deletion
* `204 No Content`
* No response body needed

---

## 13. Important Learning Notes

* This CRUD is **temporary**
* No database is involved
* Data resets on server restart
* Logic is identical when using databases later

This prepares learners for:

* MongoDB
* SQL databases
* Real-world APIs

---

## 14. Why This Example Matters

With this single example, learners now understand:

* REST methods in Express
* Route parameters
* Request bodies
* Status codes
* CRUD flow end-to-end

When databases are introduced later:

> Only the **data source changes**, not the API logic.
