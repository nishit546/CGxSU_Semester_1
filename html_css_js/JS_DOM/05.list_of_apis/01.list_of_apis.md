# List of APIs

> Note: Some APIs may have rate limits or ask for simple headers (like `Accept: application/json`), but none below require an API key for basic `GET` usage.

> **Note**
> All the APIs listed below are public, free APIs that generally do not require any access token. Because they are public and maintained by different providers, they may change over time, stop working, or return responses that are not valid JSON.
>
> If any API:
>
> * does not return the expected output, or
> * is down, or
> * its documentation link is missing or broken,
>
> simply skip that API and move on to another one from the list.
>
> Always refer to the latest official documentation of each API for up-to-date usage details before using it in your projects.
