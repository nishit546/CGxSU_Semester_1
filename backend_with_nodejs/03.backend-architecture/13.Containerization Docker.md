# 13. Containerization (Docker)

## Standardizing Application Deployment Across Environments

---

## 1. The Deployment Problem

In distributed systems, multiple services need to run:

* On different machines
* Across different environments (development, staging, production)

A common issue arises:

> “It works on my machine, but not on yours”

---

## 2. Why This Happens

Applications depend on:

* Operating system
* Installed libraries
* Runtime versions
* Environment configurations

Differences in any of these can break the application.

---

## 3. Need for Standardization

To ensure consistency:

> The application and its environment must be packaged together

---

## 4. What is Containerization

Containerization is a method of packaging an application along with:

* Its dependencies
* Runtime
* System libraries

This package is called a **container**.

---

## 5. Role of Docker

Docker is a platform used to create and run containers.

It provides:

* A consistent runtime environment
* Lightweight isolation
* Portability across systems

---

## 6. Core Concepts

---

### 6.1 Image

An image is a blueprint of the application.

It includes:

* Application code
* Dependencies
* Configuration

---

### 6.2 Container

A container is a running instance of an image.

---

### 6.3 Dockerfile

A Dockerfile defines how an image is built.

It specifies:

* Base environment
* Installation steps
* Commands to run the application

---

## 7. Example: Backend Service in Container

Instead of relying on system setup:

```txt id="traditional"
Install Node.js → Install dependencies → Run app
```

With Docker:

```txt id="docker"
Build image → Run container
```

The environment is predefined and consistent.

---

## 8. Benefits of Containerization

---

### 8.1 Consistency

Same container runs across:

* Development
* Testing
* Production

---

### 8.2 Portability

Containers can run on:

* Local machines
* Cloud platforms
* Different operating systems

---

### 8.3 Isolation

Each service runs in its own environment:

* No dependency conflicts
* No interference between services

---

### 8.4 Faster Setup

Developers do not need to manually configure environments.

---

## 9. Real System Example

On platforms like JioHotstar:

Multiple services exist:

```txt id="services-container"
Auth Service → Container  
Streaming Service → Container  
Recommendation Service → Container  
```

Each service is:

* Packaged independently
* Deployed independently
* Scaled independently

---

## 10. Running Multiple Services

In a microservices architecture:

* Each service runs in its own container
* Services communicate over the network

This creates a system composed of multiple containers.

---

## 11. Limitation of Containerization Alone

While containers solve deployment issues, they introduce new challenges:

* Managing multiple containers
* Scaling containers dynamically
* Handling failures
* Load distribution

---

## 12. Key Insight

Containerization solves:

> “How to run applications consistently”

But does not solve:

> “How to manage them at scale”

---

## 13. Summary

* Containerization packages applications with their dependencies
* Docker enables consistent and portable environments
* Each microservice can run in its own container
* Containers simplify deployment but not orchestration
