## 15.2 What Happens If Kubernetes Is Not Used? (Failure Scenario)

To understand the importance of orchestration, consider the same IPL scenario on JioHotstar without using Kubernetes.

---

### 1. Manual Scaling Limitations

During a traffic spike (7:00 PM):

* Engineers must manually start new containers
* Scaling decisions are delayed
* System cannot react instantly

Result:

* Sudden surge overwhelms existing servers
* Users experience buffering and request failures

---

### 2. Uneven Load Distribution

Without orchestration:

```txt id="no-k8s-lb"
Users → Some containers overloaded  
       → Some containers underutilized
```

Result:

* Certain containers crash due to overload
* Resources are not efficiently utilized

---

### 3. No Automatic Failure Recovery

If a container crashes:

* It remains down until manually restarted

If a machine fails:

* All services on that machine become unavailable

Result:

* Service outages
* Increased downtime
* Poor user experience

---

### 4. Deployment Risks

During updates:

* Old version must be stopped before deploying new version

```txt id="no-rolling"
Stop old containers → Deploy new containers
```

Result:

* Temporary downtime
* Risk of deployment failures affecting entire system

---

### 5. Resource Inefficiency

Without automated management:

* Over-provisioning to handle peak load
* Under-utilization during low traffic

Result:

* Increased infrastructure cost
* Inefficient resource usage

---

### 6. Operational Complexity

Managing containers manually requires:

* Continuous monitoring
* Manual intervention
* Custom scripts for scaling and recovery

Result:

* High operational overhead
* Increased chance of human error

---

### 7. Overall System Impact

Without orchestration:

* System cannot handle sudden traffic spikes effectively
* Failures are not automatically recovered
* Deployment processes introduce downtime
* Infrastructure costs increase

---

## Key Insight

Without orchestration, containerized systems remain:

> Difficult to manage, slow to scale, and prone to failure under real-world conditions
