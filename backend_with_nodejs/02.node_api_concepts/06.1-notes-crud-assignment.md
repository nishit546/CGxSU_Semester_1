## Mongoose Schema

File: `src/models/note.model.js`

```js
const noteSchema = new mongoose.Schema(
  {
    title:    { type: String, required: [true, "Title is required"] },
    content:  { type: String, required: [true, "Content is required"] },
    category: { type: String, enum: ["work", "personal", "study"], default: "personal" },
    isPinned: { type: Boolean, default: false },
  },
  { timestamps: true }
);
```

| Field | Type | Required | Details |
|-------|------|----------|---------|
| `title` | String | Yes | Cannot be empty |
| `content` | String | Yes | Cannot be empty |
| `category` | String | No | Enum: `work` / `personal` / `study` ‚Äî Default: `personal` |
| `isPinned` | Boolean | No | Default: `false` |
| `createdAt` | Date | Auto | Auto-generated by `timestamps: true` |
| `updatedAt` | Date | Auto | Auto-generated by `timestamps: true` |

## API Endpoints

Base URL: `/api/notes`

| # | Method | Endpoint | Description |
|---|--------|----------|-------------|
| 1 | `POST` | `/api/notes` | Create a single note |
| 2 | `POST` | `/api/notes/bulk` | Create multiple notes at once |
| 3 | `GET` | `/api/notes` | Get all notes |
| 4 | `GET` | `/api/notes/:id` | Get a single note by ID |
| 5 | `PUT` | `/api/notes/:id` | Replace a note completely |
| 6 | `PATCH` | `/api/notes/:id` | Update specific fields only |
| 7 | `DELETE` | `/api/notes/:id` | Delete a single note |
| 8 | `DELETE` | `/api/notes/bulk` | Delete multiple notes by IDs |

### 5. PUT `/api/notes/:id` ‚Äî Replace a note completely

PUT means **full replacement**. Every field must be provided. Fields not sent will be reset to their default value.

**Request body:**
```json
{
  "title": "Completely new title",
  "content": "Completely new content",
  "category": "personal",
  "isPinned": false
}
```

**Success response ‚Äî `200`:**
```json
{
  "success": true,
  "message": "Note replaced successfully",
  "data": { }
}
```

> Use `{ new: true, overwrite: true, runValidators: true }` options in your Mongoose call.

---

### 6. PATCH `/api/notes/:id` ‚Äî Update specific fields only

PATCH means **partial update**. Only send the fields you want to change.

**Request body:**
```json

```

**Success response ‚Äî `200`:**
```json
{
  "success": true,
  "message": "Note updated successfully",
  "data": { }
}
```

> Use `{ new: true, runValidators: true }`. Other fields must remain unchanged.

## PUT vs PATCH ‚Äî Know the Difference

This is an important concept in REST API design.

| | PUT | PATCH |
|-|-----|-------|
| **Meaning** | Replace the entire document | Update only the fields you send |
| **Fields not sent** | Reset to default | Stay unchanged |
| **Use case** | Full edit form submission | Toggle `isPinned`, change one field |

**Example ‚Äî you have this note:**
```json
{ "title": "Old title", "content": "Important content", "isPinned": true }
```

**Send PUT with only:**
```json
{ "title": "New title" }
```
**Result ‚Äî content and isPinned are reset:**
```json
{ "title": "New title", "content": "", "isPinned": false }
```

**Send PATCH with only:**
```json
{ "title": "New title" }
```
**Result ‚Äî only title changed, rest stays:**
```json
{ "title": "New title", "content": "Important content", "isPinned": true }
```

## Response Format

Every endpoint ‚Äî success or error ‚Äî must follow this exact format. No exceptions.

```json
{
  "success": true,
  "message": "...",
  "data": { }
}
```

- `success` ‚Äî always `true` or `false`
- `message` ‚Äî always a human-readable string
- `data` ‚Äî object, array, or `null`

## HTTP Status Codes

| Code | When to use |
|------|-------------|
| `200` | Successful GET, PUT, PATCH, DELETE |
| `201` | Successful POST ‚Äî note(s) created |
| `400` | Missing required fields / empty body / invalid ObjectId / empty array |
| `404` | Note not found with that `_id` |
| `500` | Unexpected server or database error |

---

## Validation Rules

- `title` and `content` required on create ‚Äî return `400` if missing
- On any `/:id` route ‚Äî validate ObjectId format first, return `400` if invalid
- On any `/:id` route ‚Äî if note not found, return `404`
- On PATCH ‚Äî if body is empty, return `400` with `"No fields provided to update"`
- On bulk create ‚Äî if `notes` array is missing or empty, return `400`
- On bulk delete ‚Äî if `ids` array is missing or empty, return `400`
