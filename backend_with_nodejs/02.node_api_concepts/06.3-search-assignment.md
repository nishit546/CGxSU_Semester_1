## Section 5 — Search

> **What is search?**
> Search uses MongoDB `$regex` to find documents where a field contains the keyword.
> The `i` option makes it case-insensitive — `"MEETING"` and `"meeting"` both match.
>
> Use `$or` when you want to search across multiple fields at once.

---

### 9. GET `/api/notes/search` — Search in title only

**Required param:**

| `q` | Yes | Keyword to search for |

## The Master Endpoint

### 18. GET `/api/notes/query` — Everything at once

This is the final endpoint. Every param is optional. When all are used together, this endpoint does search + filter + sort + paginate in a single call. This is what a real production query API looks like.

**Supported params:**

| Param | Required | Default | Description |
|-------|----------|---------|-------------|
| `q` | No | — | Search in title and content |
| `category` | No | — | Filter by category |
| `isPinned` | No | — | Filter by pinned status |
| `sortBy` | No | `createdAt` | Field to sort by |
| `order` | No | `desc` | Sort direction |
| `page` | No | `1` | Page number |
| `limit` | No | `10` | Results per page |

**Example requests — test every combination:**
```
GET /api/notes/query
GET /api/notes/query?q=react
GET /api/notes/query?category=work
GET /api/notes/query?isPinned=true&sortBy=title&order=asc
GET /api/notes/query?q=meeting&category=work&sortBy=createdAt&order=desc&page=1&limit=5
GET /api/notes/query?q=node&category=study&isPinned=false&sortBy=updatedAt&order=desc&page=2&limit=3
```

**Success — `200`:**
```json
{
  "success": true,
  "message": "Notes fetched successfully",
  "data": [],
  "pagination": {
    "total": 12,
    "page": 1,
    "limit": 5,
    "totalPages": 3,
    "hasNextPage": true,
    "hasPrevPage": false
  }
}
```

> **Full query builder — build it in this exact order:**
```js
const getAllNotesQuery = async (req, res) => {
  try {
    const { q, category, isPinned, sortBy, order, page, limit } = req.query;

    // Step 1 — Build filter
    const filter = {};

    if (q) {
      filter.$or = [
        { title:   { $regex: q, $options: "i" } },
        { content: { $regex: q, $options: "i" } }
      ];
    }
    if (category)                filter.category = category;
    if (isPinned !== undefined)  filter.isPinned  = isPinned === "true";

    // Step 2 — Sorting
    const allowedSortFields = ["title", "createdAt", "updatedAt", "category"];
    const sortField = allowedSortFields.includes(sortBy) ? sortBy : "createdAt";
    const sortOrder = order === "asc" ? 1 : -1;

    // Step 3 — Pagination
    const pageNum  = parseInt(page)  || 1;
    const limitNum = parseInt(limit) || 10;
    const skip     = (pageNum - 1) * limitNum;

    // Step 4 — Execute
    const total = await Note.countDocuments(filter);
    const notes = await Note.find(filter)
      .sort({ [sortField]: sortOrder })
      .skip(skip)
      .limit(limitNum);

    res.status(200).json({
      success: true,
      message: "Notes fetched successfully",
      data: notes,
      pagination: {
        total,
        page:        pageNum,
        limit:       limitNum,
        totalPages:  Math.ceil(total / limitNum),
        hasNextPage: pageNum < Math.ceil(total / limitNum),
        hasPrevPage: pageNum > 1,
      },
    });

  } catch (error) {
    res.status(500).json({ success: false, message: error.message, data: null });
  }
};
```
