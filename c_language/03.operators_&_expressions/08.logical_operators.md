# 06.3: Logical Operators in C (`&&`, `||`, `!`)

Logical operators allow us to **combine conditions** or **reverse a condition‚Äôs meaning**.
They are mainly used inside **`if`, `while`, and `for`** statements.

---

## 1) Operators

1. **AND (`&&`)** ‚Üí True only if **both conditions** are true.
2. **OR (`||`)** ‚Üí True if **at least one condition** is true.
3. **NOT (`!`)** ‚Üí Reverses the truth value (true ‚Üí false, false ‚Üí true).

---

## 2) Example: Checking conditions

```c
#include <stdio.h>
int main() {
    int age = 20, attendance = 85;

    printf("age >= 18 && attendance >= 75 -> %d\n", (age >= 18) && (attendance >= 75));
    printf("age < 18 || attendance >= 75  -> %d\n", (age < 18)  || (attendance >= 75));
    printf("!(age < 18)                   -> %d\n", !(age < 18));

    return 0;
}
```

### Output:

```
age >= 18 && attendance >= 75 -> 1
age < 18 || attendance >= 75  -> 1
!(age < 18)                   -> 1
```

## 3) Truth Table (Quick Reference)

<img src="./truth_table.png">

| A | `!A` |
| - | ---- |
| 0 | 1    |
| 1 | 0    |

Here‚Äôs the **truth table image** for logical operators `&&` (AND) and `||` (OR).

* `A && B` is `1` **only if both are 1**.
* `A || B` is `1` if **at least one is 1**.
* `!A` (NOT) flips the value (`1 ‚Üí 0`, `0 ‚Üí 1`).

## 4) Common Mistakes

### 4.1 Using `=` instead of `==`

```c
int x = 5;
if (x = 0) {   // ‚ùå assignment, not comparison
    printf("True\n");
}
```

‚úÖ Correct:

```c
if (x == 0) {  // comparison
    printf("True\n");
}
```

### 4.2 Not grouping with parentheses

```c
if (age >= 18 && attendance >= 75 || cgpa > 7.0)
    printf("Eligible\n");
```

This can confuse‚Äîbetter write:

```c
if ((age >= 18 && attendance >= 75) || cgpa > 7.0)
    printf("Eligible\n");
```

## 6) Key Takeaways

* `&&` ‚Üí needs **all true**.
* `||` ‚Üí needs **at least one true**.
* `!` ‚Üí flips the result.
* Use **parentheses** to make logic clear.
* Always use `==` for comparison, not `=`.

## 7) Short‚ÄëCircuit Evaluation (VERY Important)

* `A && B`: If **A is false**, C **skips** evaluating B (no need).
* `A || B`: If **A is true**, C **skips** evaluating B (already true).

This saves time and prevents errors (e.g., dividing by zero).

### 7.1 Safe division using `&&`

```c
#include <stdio.h>
int main() {
    int a = 10, b = 0;

    // B is evaluated only if (b != 0) is true
    if (b != 0 && (a / b) > 2) {
        printf("Quotient > 2\n");
    } else {
        printf("Either b is 0 or quotient <= 2\n");
    }
    return 0;
}
```

### 7.2 Avoiding function side effects via short‚Äëcircuit

```c
#include <stdio.h>

int dangerous_eval() {
    printf("This should not run!\n");
    return 1;
}

int main() {
    int ok = 0;
    // Right side won‚Äôt run because left is false
    if (ok && dangerous_eval()) {
        printf("Won't reach here\n");
    }
    return 0;
}
```

**Output**

```
(nothing printed)
```

No ‚ÄúThis should not run!‚Äù ‚Üí proof that the RHS was skipped.

## 9) Operator Precedence (Where do logicals sit?)

From high to low (partial list you‚Äôll use daily):

1. `!` (logical NOT)
2. Relational `< <= > >=`
3. Equality `== !=`
4. `&&`
5. `||`

**Example**

```c
int r = 2 + 3 * 4 > 10 && 5 == 5 || 0;
// 2 + 12 > 10 && 1 || 0
// 14 > 10 && 1 || 0
// 1 && 1 || 0
// 1 || 0 -> 1
```

üëâ When unsure, **use parentheses**. They also make your intent clear.

## 10) Common Pitfalls (with code)

### 10.1 Using `=` instead of `==` in conditions

```c
#include <stdio.h>
int main() {
    int a = 5;
    if (a = 0) {                 // ‚ùå assignment, sets a to 0, condition becomes false
        printf("True\n");
    } else {
        printf("False\n");       // prints False
    }
    return 0;
}
```

**Fix:** `if (a == 0)`

---

### 10.2 Assuming non‚Äë0 is always `1`

```c
#include <stdio.h>
int main() {
    int x = 42;                 // truthy
    printf("%d\n", x && 5);     // 1 (true && true)
    printf("%d\n", x || 0);     // 1
    printf("%d\n", !x);         // 0
    return 0;
}
```

**Note:** Any **nonzero** is treated as true when evaluated, but `x && y` / `x || y` produce **0 or 1** as results.

---

### 10.3 Short‚Äëcircuit side effects

```c
#include <stdio.h>
int inc(int *p) { (*p)++; return 1; }

int main() {
    int a = 0, b = 0;

    // Left is 0 ‚Üí RHS won‚Äôt run, so b stays 0
    if (a && inc(&b)) { }
    printf("a=%d b=%d\n", a, b); // a=0 b=0

    a = 1;
    if (a && inc(&b)) { }
    printf("a=%d b=%d\n", a, b); // a=1 b=1
    return 0;
}
```

---

### 10.4 Guarding against divide‚Äëby‚Äëzero (good pattern)

```c
if (den != 0 && (num / den) > 10) { /* safe */ }
```

Don‚Äôt write:

```c
if ((num / den) > 10 && den != 0) { /* UB/crash possible */ } // ‚ùå
```

## 11) De Morgan‚Äôs Laws (clean up complex NOTs)

* `!(A && B)`  ‚â°  `(!A) || (!B)`
* `!(A || B)`  ‚â°  `(!A) && (!B)`

**Example:**

```c
// Not (user is verified AND age >= 18)
if (!(verified && age >= 18)) { /* ... */ }

// Equivalent (often more readable)
if (!verified || age < 18) { /* ... */ }
```

## 14) Quick Checklist

* Combine conditions with `&&` and `||`; invert with `!`.
* Results are **0 or 1**; any nonzero is ‚Äútruthy‚Äù when evaluated.
* **Short‚Äëcircuit** protects you (place ‚Äúsafe‚Äù checks first).
* Avoid `=` vs `==` mistakes.
* Use parentheses for clarity & to control order.
* Apply **De Morgan‚Äôs laws** to simplify `!` of complex expressions.
