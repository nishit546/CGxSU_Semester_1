# CodingGita – Student Profile Platform

**Figma Design & Prototype Specification**

---

## 1. Objective

Design a **web-based platform** that showcases **all CodingGita students** and their complete professional profiles.
The platform will act as a **public discovery + private editing system** where:

* Visitors can **browse, search, and filter students**
* Students can **log in and manage their own profile**
* Each student profile acts as a **single source of truth** for projects, achievements, and work experience

This design must be created **entirely in Figma first**, including a **clickable prototype**.

---

## 2. Design Constraints

* Use **CodingGita.com color theme**
* Follow **Material Design principles**
* Desktop-first but **fully responsive**
* Clean, professional, and portfolio-ready UI
* Scalable for thousands of student profiles

## Usage Guidance

* **Primary Blue** (`#1565D8`) — Use for primary buttons, nav bar, links, accent UI elements.
* **Dark Navy** (`#1E2A38`) — Headlines, footers, dark text UI elements.
* **Teal/Green** (`#00BFA5`) — Achievements, positive badges, highlights.
* **Success / Info / Warning** — Standard semantic statuses consistent with Material Design.
* **Background Gray** — Always behind cards or full sections.
* **Card White** — Used for all student profile cards, modals, and page cards.

## 4. Pages & Frames to Design

---

## 4.1 Students Profile Listing Page (Explore Page)

**Frame Name:** `Students Listing – Explore Talent`

### Purpose

Public discovery page listing all CodingGita students.

---

### Layout

#### Top App Bar

* CodingGita Logo (Left)
* Global Search Bar (Name / Skill / Project)
* Login / Profile Icon (Right)

---

#### Left Filter Panel (Sticky)

Filters should support **multi-select**:

* Skills (Chip based)
* Certificates
* Internships
* Hackathons
* Freelancing Experience
* LeetCode Rating Range
* YouTube Content (Yes / No)
* Clear Filters Button

---

#### Main Content Area

* Grid layout (3–4 cards per row on desktop)

---

### Student Profile Card

Each card must contain:

* Profile Photo
* Full Name
* Primary Role (Frontend / Backend / Full Stack / AI etc.)
* Top 3 Skills (Chips)
* Mini Stats Row:

  * Projects Count
  * Internships
  * Achievements
* CTA Buttons:

  * View Profile
  * View Projects

## 5. Prototype Requirements

Create a **clickable prototype** with:

* Student Card → Profile Page
* Profile Tabs interaction
* Login → Dashboard
* Dashboard → Public Profile Preview
* Filters → Live results (visual only)

## 6. Responsive Frames Required

Design these breakpoints:

* Desktop (1440px)
* Tablet (768px)
* Mobile (375px)

Mobile priority:

* Filter as bottom sheet
* Cards as single column
* Tabs as horizontal scroll
