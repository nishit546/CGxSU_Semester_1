# **SEO Guidelines for index.html (Personal Portfolio – React)**

## **1. Role of index.html in React**

In a React app:

* `index.html` is the **entry point**
* It is the **first thing crawlers see**
* It provides **default SEO metadata**

Even if you use React Helmet later, this file must be properly configured.

---

## **2. File Location**

```bash
public/index.html
```

---

## **3. Basic Structure (Must Have)**

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />

    <meta name="viewport" content="width=device-width, initial-scale=1.0" />

    <title>Your Name | Software Engineer Portfolio</title>

    <meta
      name="description"
      content="Portfolio of Your Name, showcasing projects, hackathons, achievements, and certifications in software development."
    />

  </head>

  <body>
    <div id="root"></div>
  </body>
</html>
```

---

## **4. Title Tag (Most Important)**

```html
<title>Your Name | Software Engineer Portfolio</title>
```

Guidelines:

* Include your name
* Include role (Software Engineer / Frontend Developer)
* Keep within ~60 characters

---

## **5. Meta Description**

```html
<meta
  name="description"
  content="Portfolio of Your Name, showcasing projects, hackathons, achievements, and certifications in software development."
/>
```

Guidelines:

* 140–160 characters
* Mention key highlights:

  * Projects
  * Hackathons
  * Certifications

---

## **6. Keywords (Optional but Useful for Learning)**

```html
<meta
  name="keywords"
  content="Your Name, portfolio, software engineer, react developer, hackathon winner, coding projects"
/>
```

---

## **7. Author Tag**

```html
<meta name="author" content="Your Name" />
```

---

## **8. Open Graph Tags (Social Sharing)**

These control how your portfolio appears when shared.

```html
<meta property="og:title" content="Your Name Portfolio" />
<meta
  property="og:description"
  content="Explore projects, hackathons, and achievements of Your Name"
/>
<meta property="og:image" content="http://codinggita.com/portfolio.png" />
<meta property="og:url" content="http://codinggita.com/" />
<meta property="og:type" content="website" />
```

---

## **9. Twitter Card Tags**

```html
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content="Your Name Portfolio" />
<meta
  name="twitter:description"
  content="Projects, hackathons, and achievements"
```

## **10. Canonical URL**

```html
<link rel="canonical" href="http://codinggita.com/" />
```

Prevents duplicate content issues.

---

## **11. Favicon**

```html

```

## **12. Structured Data (Advanced but Powerful)**

This helps search engines understand your profile.

---

### **Example: Person Schema**

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "Your Name",
  "url": "http://codinggita.com",
  "sameAs": [
    "https://github.com/yourprofile",
    "https://linkedin.com/in/yourprofile"
  ],
  "jobTitle": "Software Engineer",
  "worksFor": {
    "@type": "Organization",
    "name": "Freelancer"
  }
}
</script>
```

## **15. Complete Example (Production-Ready index.html with Detailed SEO Setup)**

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <!-- Character Encoding -->
    <meta charset="UTF-8" />

    <!-- Mobile Responsiveness -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />

    <!-- Browser Compatibility -->
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />

    <!-- Primary Title (Most Important for SEO) -->
    <title>
      Your Name | Software Engineer | React Developer | Portfolio
    </title>

    <!-- Primary Meta Description -->
    <meta
      name="description"
      content="Your Name is a Software Engineer specializing in React development. Explore projects, hackathon wins, certifications, and technical achievements."
    />

    <!-- Keywords (Optional, for completeness) -->
    <meta
      name="keywords"
      content="Your Name, software engineer, react developer, portfolio, hackathon winner, coding projects, certifications"
    />

    <!-- Author -->
    <meta name="author" content="Your Name" />

    <!-- Theme Color (Mobile Browsers) -->
    <meta name="theme-color" content="#0f172a" />

    <!-- Canonical URL (Avoid duplicate content issues) -->
    <link rel="canonical" href="http://codinggita.com/" />

    <!-- Favicon -->
    <link rel="icon" href="/favicon.ico" />

    <!-- Open Graph (For WhatsApp, LinkedIn, Facebook sharing) -->
    <meta property="og:type" content="website" />
    <meta property="og:title" content="Your Name | Developer Portfolio" />
    <meta
      property="og:description"
      content="Portfolio showcasing projects, hackathons, certifications, and achievements."
    />
    <meta property="og:url" content="http://codinggita.com/" />
    <meta
      property="og:image"
      content="http://codinggita.com/portfolio-preview.png"
    />

    <!-- Twitter Card (For Twitter/X sharing) -->
    <meta name="twitter:card" content="summary_large_image" />
    <meta name="twitter:title" content="Your Name Portfolio" />
    <meta
      name="twitter:description"
      content="Explore my projects, achievements, and certifications."
    />
    <meta
      name="twitter:image"
      content="http://codinggita.com/portfolio-preview.png"
    />

    <!-- Robots Instruction -->
    <meta name="robots" content="index, follow" />

    <!-- Structured Data (Schema.org for better indexing) -->
    <script type="application/ld+json">
      {
        "@context": "https://schema.org",
        "@type": "Person",
        "name": "Your Name",
        "url": "http://codinggita.com",
        "image": "http://codinggita.com/profile.jpg",
        "sameAs": [
          "https://github.com/yourprofile",
          "https://linkedin.com/in/yourprofile"
        ],
        "jobTitle": "Software Engineer",
        "alumniOf": {
          "@type": "EducationalOrganization",
          "name": "Your College Name"
        },
        "knowsAbout": [
          "React",
          "JavaScript",
          "Node.js",
          "MongoDB",
          "Frontend Development"
        ]
      }
    </script>

    <!-- Preconnect for Performance (Optional but Good Practice) -->
    <link rel="preconnect" href="https://fonts.googleapis.com" />
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />

  </head>

  <body>
    <!-- Root Element for React -->
    <div id="root"></div>

    <!-- Optional: Fallback content for SEO (if JS fails) -->
    <noscript>
      <h1>Your Name - Software Engineer Portfolio</h1>
      <p>
        This portfolio includes projects, hackathons, certifications, and
        achievements in software development.
      </p>
    </noscript>
  </body>
</html>
```
