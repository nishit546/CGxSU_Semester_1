
# Core Data Structures in Redis

One of Redis's most powerful features is its support for multiple data structures.

Unlike traditional databases where everything is stored in tables, Redis provides specialized structures designed for specific use cases.

Understanding these structures is essential because choosing the correct structure can significantly improve application performance.

The most commonly used Redis data structures are:

## 1. Strings

Strings are the simplest and most frequently used data structure in Redis.

A string stores a single value against a key.

Examples:

* OTP codes
* User tokens
* Cached responses
* Visitor counters

Strings are often the first Redis structure developers learn because many caching scenarios use them.

---

**Example 1: OTP Storage**

```text
otp:987654 ‚Üí 123456
```

**Example 2: Visitor Counter**

```text
website_visitors ‚Üí 10542
```

**Example 3: Cached API Response**

```text
student:101 ‚Üí {"name":"Arjun","department":"CSE"}
```

They are commonly used to represent objects.

Hashes provide a convenient way to store structured data inside Redis.

## 3. Lists

Lists store ordered collections of elements.

Items can be added to the beginning or end of the list.

Common use cases include:

* Notifications
* Task queues
* Message queues
* Activity logs

Lists are useful whenever the order of items matters.

## 5. Sorted Sets

Sorted Sets are similar to Sets but include an associated score for each value.

Redis automatically maintains the correct ordering based on these scores.

Common use cases include:

* Leaderboards
* Student rankings
* Gaming scores
* Product rankings

Sorted Sets are one of the most powerful features offered by Redis.
