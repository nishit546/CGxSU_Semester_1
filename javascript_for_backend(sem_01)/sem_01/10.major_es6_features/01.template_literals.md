# **Template Literals (Backticks ` )**

Template Literals (also called **Template Strings**) were introduced in ES6 to make working with strings **simpler, cleaner, and more powerful**.

They solve many problems that existed with traditional string handling in JavaScript.

---

# **1. What Are Template Literals?**

Template literals are **strings written inside backticks** instead of quotes.

Traditional string:

```javascript
var msg = "Welcome to CodingGita";
```

Template literal:

```javascript
var msg = `Welcome to CodingGita`;
```

Backtick key is usually **below the Esc key** on your keyboard: `

---

# **2. Why Template Literals Were Introduced? (The Problem They Solve)**

Traditional strings had major limitations:

* Hard to write multiline text
* Hard to add variables inside strings
* Hard to format readable code
* Many + operators make code messy
* No clean way to format messages dynamically

Template literals remove all these pain points.

---

# **3. Feature 1 — String Interpolation (Insert Variables Easily)**

Template literals allow inserting variables using:

```
${variableName}
```

### Before ES6 (old way)

```javascript
var name = "Arjun";
var course = "JavaScript";
var msg = "Hello, " + name + ". Welcome to " + course + "!";

console.log(msg);
```

Messy and hard to read.

---

### ES6 Template Literal (clean way)

```javascript
var name = "Arjun";
var course = "JavaScript";

var msg = `Hello, ${name}. Welcome to ${course}!`;

console.log(msg);
```

Much cleaner and easier to maintain.

---

# **4. Feature 2 — Insert Expressions (Not just variables)**

You can insert **any JavaScript expression** inside `${ }`:

```javascript
var a = 10, b = 20;

console.log(`Sum: ${a + b}`);
console.log(`Product: ${a * b}`);
console.log(`Random: ${Math.random()}`);
```

Output example:

```
Sum: 30
Product: 200
Random: 0.82374212
```

Expressions inside `${ }` are evaluated automatically.

---

# **5. Feature 3 — Multiline Strings Without `\n`**

Traditional string:

```javascript
var msg = "Hello\nWelcome\nto CodingGita";
```

Template literal ( easier ):

```javascript
var msg = `
Hello
Welcome
to CodingGita
`;

console.log(msg);
```

Multiline formatting becomes natural.

---

# **6. Feature 4 — HTML Templates (Very Useful in JS + React)**

Template literals make creating long HTML strings easy.

Example: Building a student card:

```javascript
var name = "Yashvi";
var age = 18;

var card = `
<div class="student-card">
    <h2>${name}</h2>
    <p>Age: ${age}</p>
</div>
`;

console.log(card);
```

Clean, readable, easy to maintain.

---

# **7. Feature 5 — Function Calls Inside Templates**

```javascript
function greet() {
    return "Good Morning";
}

var name = "Mahir";

console.log(`${greet()}, ${name}!`);
```

Output:

```
Good Morning, Mahir!
```

---

# **8. Feature 6 — Using Ternary Operator Inside Templates**

```javascript
var marks = 92;

console.log(`
Student Result:
Status: ${marks >= 90 ? "Topper" : "Regular"}
`);
```

Output:

```
Student Result:
Status: Topper
```

Useful for formatting dashboards.

---

# **9. Feature 7 — Template Literals With Objects & Arrays**

### Objects:

```javascript
var student = {
    name: "Krishna",
    branch: "CSE"
};

console.log(`Student: ${student.name} - ${student.branch}`);
```

---

### Arrays:

```javascript
var topics = ["HTML", "CSS", "JS"];

console.log(`We teach: ${topics.join(", ")}`);
```

---

# **10. Feature 8 — Nesting Template Literals**

```javascript
var username = "Jenil";
var status = "Active";

var msg = `
User Info:
    Name: ${username}
    Status: ${status}
    Message: ${
        status === "Active" 
        ? `Welcome back, ${username}!`
        : `Please activate your account`
    }
`;

console.log(msg);
```

---

# **11. Tag Functions**

A special feature of template literals is **Tagged Templates**.

Tagged templates allow you to **process** template literal parts through a function.

Example:

```javascript
function highlight(strings, ...values) {
    console.log(strings);
    console.log(values);
}

highlight`Hello ${"Arjun"} you scored ${95}`;
```

Output shows how strings and values are separated.

This is used in:

* Styling libraries
* Localization (multiple languages)
* Safe HTML sanitization

We will revisit this later.

# **13. Common Mistakes**

### ❌ Mistake 1 — Using quotes instead of backticks

```javascript
var msg = "Hello ${name}"; // wrong
```

It will print literally:

```
Hello ${name}
```

---

### ❌ Mistake 2 — Forgetting `${ }`

```javascript
var msg = `Hello name`; // wrong
```

---

### ❌ Mistake 3 — Using ${ } outside backticks

```javascript
var x = 10;
console.log("Value is ${x}"); // wrong
```

---

### ❌ Mistake 4 — Incorrect nesting

```javascript
var msg = `Result: ${a > b ? "A" : "B"}`; // correct
```
