# **Enhanced Object Literals**

Enhanced Object Literals were introduced in ES6 to make writing objects **shorter, cleaner, smarter, and more expressive**.

---

# **1. What Are Enhanced Object Literals?**

Before ES6, creating objects in JavaScript required long, repetitive syntax.

Enhanced Object Literals allow:

* Shorter syntax for assigning properties
* Shorter syntax for writing methods
* Automatically computed property names
* Better object construction patterns

These features make objects more flexible and easier to read.

---

# **2. Feature 1 — Property Shorthand**

When the **variable name** and **property name** are the same, you can write:

### Before ES6 (old way)

```javascript
var name = "Arjun";
var age = 19;

var student = {
    name: name,
    age: age
};
```

### ES6 Enhanced Object Literal

```javascript
var name = "Arjun";
var age = 19;

var student = {
    name,
    age
};
```

Cleaner and shorter.

---

# **3. Feature 2 — Method Shorthand**

Writing functions inside objects becomes shorter.

### Before ES6

```javascript
var student = {
    greet: function() {
        console.log("Hello!");
    }
};
```

### After ES6

```javascript
var student = {
    greet() {
        console.log("Hello!");
    }
};
```

No need to write `function`.

---

# **4. Feature 3 — Computed Property Names**

You can dynamically create property names using `[]`.

### Example 1 — Using a variable

```javascript
var key = "score";

var student = {
    name: "Mahir",
    [key]: 95
};

console.log(student.score); // 95
```

---

### Example 2 — Using expressions

```javascript
var index = 1;

var obj = {
    ["item" + index]: "Laptop",
    ["item" + (index + 1)]: "Keyboard"
};

console.log(obj);
// { item1: "Laptop", item2: "Keyboard" }
```

---

### Example 3 — Based on function return

```javascript
function getKey() {
    return "status";
}

var order = {
    id: 101,
    [getKey()]: "Delivered"
};

console.log(order.status);
```

Dynamic, powerful, used widely in frameworks.

---

# **5. Feature 4 — Returning Object Without Repetition**

Very useful when creating objects based on parameters.

### Before ES6

```javascript
function createStudent(name, age, branch) {
    return {
        name: name,
        age: age,
        branch: branch
    };
}
```

### ES6

```javascript
function createStudent(name, age, branch) {
    return { name, age, branch };
}
```

Much cleaner.

---

# **6. Feature 5 — Use In Function Parameters (Object Return Patterns)**

Enhanced object literals work beautifully with destructuring and function returns.

### Example:

```javascript
function getMarks() {
    var math = 90;
    var physics = 88;
    var chemistry = 92;

    return { math, physics, chemistry };
}
```

---

# **7. Feature 6 — Merging Objects with Spread**

Enhanced object literals + spread = very powerful.

```javascript
var personal = { name: "Krishna", age: 18 };
var academic = { branch: "CSE", marks: 88 };

var student = {
    ...personal,
    ...academic
};

console.log(student);
```

---

# **8. Feature 7 — Prototype Assignment Shorthand**

Advanced but important.

### Before ES6

```javascript
var obj = {
    __proto__: someProto,
    name: "Arjun"
};
```

### ES6 (still possible)

```javascript
var obj = {
    __proto__: someProto,
    name: "Arjun"
};
```

Prototype linking is now standardized.

(Not required for beginners, but good to know.)

# **9. Use Cases (Real Examples)**

---

## **9.1 Student Profile Creation**

```javascript
function createProfile(name, age, branch) {
    return {
        name,
        age,
        branch,
        show() {
            console.log(`${name} (${branch})`);
        }
    };

var p = createProfile("Jenil", 19, "CSE");
p.show();
```

## **9.2 Dynamic Form Field Generation**

```javascript
var inputName = "email";
var form = {
    [inputName]: "student@gmail.com"
};
```

## **9.3 REST API Response Formatting (Node.js style)**

```javascript
function sendResponse(status, data) {
    return {
        status,
        data,
        timestamp() {
            return Date.now();
        }
    };
}
```

# **10. Common Mistakes**

---

### ❌ Mistake 1 — Forgetting brackets in computed keys

```javascript
var obj = {
    key: "value",
    key2: "value"    // wrong intention
};
```

You must use:

```javascript
var obj = {
    [key]: "value"
};
```

---

### ❌ Mistake 2 — Misusing method shorthand inside nested objects

```javascript
var obj = {
    inner: {
        greet() { } // works but beginners confuse scope
    }
};
```

---

### ❌ Mistake 3 — Overwriting properties unintentionally

```javascript
var student = {name: "Arjun"};
var update = {name: "Krishna"};

var result = {...student, ...update};

console.log(result.name); // Krishna
```

Spread order matters.

# **13. MCQs**

### **Q1. What does this create?**

```javascript
var name = "Krishna";
var obj = { name };
```

1. { name: undefined }
2. { Krishna: name }
3. { name: "Krishna" }
4. Error

**Answer: 3**

---

### **Q2. Which one is valid method shorthand?**

1. `method: function() {}`
2. `method() {}`
3. `function method() {}`
4. `method: () => {}`

**Answer: 2**

---

### **Q3. What does this output?**

```javascript
var key = "score";
var obj = {
  [key]: 88
};
console.log(obj.score);
```

1. undefined
2. 88
3. "score"
4. Error

**Answer: 2**
