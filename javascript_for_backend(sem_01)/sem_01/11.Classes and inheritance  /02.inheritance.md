
# **Inheritance in JavaScript (extends, super, overriding)**

Inheritance is the second major concept of Object-Oriented Programming after classes.
It allows one class to **acquire** the properties and methods of another class.

---

# **1. What is Inheritance?**

**Inheritance** means:

* One class (child) can reuse the code of another class (parent)
* The child class *extends* features of the parent
* The child class can *add its own features*
* The child class can *modify* parent features

### Real-life example (CodingGita context)

* **Student** → common features
* **CSEStudent** → inherits Student, adds CSE-specific features
* **AIMLStudent** → inherits Student, adds AIML features

All departments share common properties like name, age, branch — no need to rewrite them.

This is the purpose of inheritance.

---

# **2. Basic Syntax of Inheritance**

The keyword used in JavaScript is:

```javascript
class Child extends Parent {}
```

### Example:

```javascript
class Student {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
}

class CSEStudent extends Student {}
```

Now the child class automatically gets:

* constructor
* properties
* methods

from the parent class.

---

# **3. Creating Objects With Inheritance**

```javascript
var s1 = new CSEStudent("Arjun", 19);
console.log(s1.name); // Arjun
console.log(s1.age);  // 19
```

Even though `CSEStudent` has **no constructor**, it uses the parent constructor automatically.

---

# **4. Adding Extra Properties in Child Class**

If the child needs extra data, we add a constructor.

But **when we add a constructor in child → we must call `super()`**.

---

# **5. What is super()?**

`super()` is used to call the parent class constructor.

Without calling `super()`, the child class cannot use `this`.

### Example:

```javascript
class CSEStudent extends Student {
    constructor(name, age, programmingLanguage) {
        super(name, age);   // Call parent constructor
        this.programmingLanguage = programmingLanguage;
    }
}
```

### Why must super() come first?

Because JavaScript needs to create the parent part of the object *before* the child adds its own properties.

---

# **6. Adding Extra Methods in Child Class**

```javascript
class CSEStudent extends Student {
    constructor(name, age, programmingLanguage) {
        super(name, age);
        this.programmingLanguage = programmingLanguage;
    }

    showTech() {
        console.log(`${this.name} is learning ${this.programmingLanguage}`);
    }
}

var c = new CSEStudent("Krishna", 18, "JavaScript");
c.showTech();
```

Child class has:

* Parent properties
* Parent methods
* Child methods

---

# **7. Method Overriding (Very Important)**

Child class can replace (override) a parent method with its own version.

### Parent class:

```javascript
class Student {
    introduce() {
        console.log("I am a student.");
    }
}
```

### Child class overriding:

```javascript
class CSEStudent extends Student {
    introduce() {
        console.log("I am a CSE student.");
    }
}
```

### Result:

```javascript
var s = new CSEStudent();
s.introduce();  // I am a CSE student.
```

The child method replaces the parent’s method.

---

# **8. Calling Parent Method From Child (super.methodName)**

Sometimes child wants to use parent behavior **and** add something new.

```javascript
class CSEStudent extends Student {
    introduce() {
        super.introduce(); // Call parent’s method
        console.log("I specialize in Computer Science.");
    }
}
```

---

# **9. Multi-Level Inheritance**

JavaScript supports chain inheritance.

```javascript
class Person {
    constructor(name) {
        this.name = name;
    }
}

class Student extends Person {
    constructor(name, branch) {
        super(name);
        this.branch = branch;
    }
}

class CSEStudent extends Student {
    constructor(name, branch, programmingLanguage) {
        super(name, branch);
        this.programmingLanguage = programmingLanguage;
    }
}
```

### Creating object:

```javascript
var c = new CSEStudent("Mahir", "CSE", "JavaScript");
console.log(c);
```

Chain:

`CSEStudent → Student → Person`

---

# **10. Static Methods in Inheritance**

Static methods are also inherited.

```javascript
class A {
    static greet() {
        console.log("Hello from A");
    }
}

class B extends A {}

B.greet(); // Hello from A
```

# **11. Practical Example**

## **Student → CSEStudent → AIMLStudent**

```javascript
class Student {
    constructor(name, year) {
        this.name = name;
        this.year = year;
    }

    details() {
        console.log(`${this.name} - Year ${this.year}`);
    }
}
```

### Level 1 Child

```javascript
class CSEStudent extends Student {
    constructor(name, year, subject) {
        super(name, year);
        this.subject = subject;
    }

    showSubject() {
        console.log(`${this.name} studies ${this.subject}`);
    }
}
```

### Level 2 Child

```javascript
class AIMLStudent extends CSEStudent {
    constructor(name, year, subject, aiProject) {
        super(name, year, subject);
        this.aiProject = aiProject;
    }

    showProject() {
        console.log(`${this.name}'s AI Project: ${this.aiProject}`);
    }
}
```

### Creating object:

```javascript
var a = new AIMLStudent("Jenil", 2, "Data Structures", "Face Recognition");
a.details();
a.showSubject();
a.showProject();
```

# **12. Common Mistakes**

---

### ❌ Mistake 1: Using `this` before `super()`

```javascript
constructor() {
    this.name = "A";  // ❌ ERROR
    super();
}
```

Correct:

```javascript
constructor() {
    super();
    this.name = "A";  
}
```

---

### ❌ Mistake 2: Forgetting to pass arguments to super()

```javascript
super(name); // correct
```

---

### ❌ Mistake 3: Overriding method without calling parent when needed

Child may accidentally remove important parent logic.

---

### ❌ Mistake 4: Calling static methods from objects

```javascript
obj.method();  // ❌
ClassName.method(); // ✔️
```
