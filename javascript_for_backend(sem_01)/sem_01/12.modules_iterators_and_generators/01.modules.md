# **Modules in JavaScript (import / export)**

(ES6 Module System)

This topic is extremely important because **modern JavaScript, React, Node.js, and all real projects use modules**.

---

# **1. What Are Modules?**

A **module** is simply a JavaScript file.

Each file can:

* export values
* import values
* keep its own private variables
* organise code into clean, maintainable pieces

### Why modules?

Without modules:

* All code stays in one file
* Hard to maintain
* Hard to reuse
* Hard to debug
* Naming conflicts happen easily

Modules solve all these problems.

---

# **2. How ES6 Modules Work**

There are two key keywords:

* **export** → send something out of a file
* **import** → bring something into a file

That’s it.

---

# **3. Types of Exports**

There are **two kinds**:

1. **Named Export**
2. **Default Export**

We MUST understand both clearly.

---

# **4. Named Exports**

Use when a file exports **multiple** functions/variables.

### Example (math.js):

```javascript
export const PI = 3.14;

export function add(a, b) {
    return a + b;
}

export function subtract(a, b) {
    return a - b;
}
```

### Importing named exports

```javascript
import { PI, add, subtract } from "./math.js";

console.log(add(10, 20));
```

Important:

* Names MUST match exactly
* Order does not matter
* You must use `{ }`

---

## **4.1 Renaming Named Exports**

Sometimes the imported name conflicts with something else.

```javascript
import { add as sum } from "./math.js";

console.log(sum(10, 20));
```

---

# **5. Default Export**

Use default export when:

* A module should export **one main thing**
* You want a simpler import syntax

### Example (student.js):

```javascript
export default function getStudent() {
    return "Krishna";
}
```

### Importing default:

```javascript
import getStudent from "./student.js";
```

Important:

* No `{ }` required
* Name during import can be ANYTHING
* Only one default export per file

---

# **6. Mixing Default + Named Exports**

A single file may contain both.

### Example (course.js):

```javascript
export default function mainCourse() {
    return "JavaScript Basics";
}

export const duration = "40 hours";
export function getFees() {
    return 5000;
}
```

### Importing:

```javascript
import mainCourse, { duration, getFees } from "./course.js";

console.log(mainCourse());
console.log(duration);
console.log(getFees());
```

---

# **7. Exporting Variables, Functions, Classes**

Modules can export:

### **Variables**

```javascript
export const username = "Jenil";
```

### **Functions**

```javascript
export function greet() {
    console.log("Hello!");
}
```

### **Classes**

```javascript
export class Student {
    constructor(name) {
        this.name = name;
    }
}
```

### Importing all:

```javascript
import { username, greet, Student } from "./file.js";
```

---

# **8. Exporting Multiple Items at Once**

Instead of adding `export` in front of each item:

### File: utils.js

```javascript
function sayHi() {
    console.log("Hi");
}

function sayBye() {
    console.log("Bye");
}

const company = "CodingGita";

export { sayHi, sayBye, company };
```

---

# **9. Exporting Everything Using**

Useful when you want to re-export items.

### File: core.js

```javascript
export * from "./utils.js";
```

This collects all exports from utils.js.

---

# **10. Importing Everything Using**

```javascript
import * as tools from "./utils.js";

tools.sayHi();
console.log(tools.company);
```

Now `tools` becomes an object containing all exports.

---

# **11. How Modules Work in Browsers**

In the browser, you must use:

```html
<script type="module" src="main.js"></script>
```

This tells the browser:

* This file uses import/export
* Load it as a module

Example structure:

```
index.html
main.js
utils.js
```

### index.html:

```html
<script type="module" src="main.js"></script>
```

### main.js:

```javascript
import { sayHi } from "./utils.js";
sayHi();
```

---

# **12. Important Module Rules**

1. **Modules are always in strict mode**
2. **Imports work only at the top level, not inside conditions**

   ```javascript
   if (true) {
       import ... // ❌ not allowed
   }
   ```
3. **Imports are read-only**

   ```javascript
   import { count } from "./file.js";
   count = 10; // ❌ error
   ```
4. **Modules run once**
   Importing a module multiple times does NOT run it again.

# **14. Common Mistakes**

---

### ❌ Mistake 1: Missing .js in import path

```javascript
import student from "./student"; // ❌
```

Must be:

```javascript
import student from "./student.js"; // ✔️
```

---

### ❌ Mistake 2: Trying to import default using {}

```javascript
import { student } from "./student.js"; // ❌ wrong for default export
```

---

### ❌ Mistake 3: Wrong file path

```javascript
import x from "/student.js"; // ❌ absolute path wrong
```

Use relative:

```javascript
import x from "./student.js"; // ✔️
```

---

### ❌ Mistake 4: Mixing CommonJS and ES modules (Node.js)

This is covered later when we start Node.js.
