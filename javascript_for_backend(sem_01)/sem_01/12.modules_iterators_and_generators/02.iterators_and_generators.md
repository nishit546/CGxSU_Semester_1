# **Iterators & Generators in JavaScript**

# **1. What Is an Iterator?**

Think of an iterator like a **remote control** that gives you the “next item” every time you press the button.

It does **not give all values at once**, but **one value at a time**.

Example idea:

* You have a list: [10, 20, 30]
* An iterator gives:

  * First press → 10
  * Second press → 20
  * Third press → 30
  * Fourth press → “I’m done”

In JavaScript, this “press the button” is done using:

```javascript
iterator.next()
```

---

# **2. How an Iterator Works**

Every iterator must return an object like:

```javascript
{
  value: something,
  done: true/false
}
```

---

# **3. Arrays Already Have Iterators (You Don’t Create Them)**

```javascript
var arr = ["Jenil", "Mahir", "Krishna"];

var it = arr[Symbol.iterator]();

console.log(it.next());
console.log(it.next());
console.log(it.next());
console.log(it.next());
```

Output:

```
{ value: 'Jenil', done: false }
{ value: 'Mahir', done: false }
{ value: 'Krishna', done: false }
{ value: undefined, done: true }
```

So arrays are *already iterable* in JavaScript.

---

# **4. Using Iterators With for...of**

Because arrays have iterators, this works:

```javascript
for (var name of arr) {
    console.log(name);
}
```

Output:

```
Jenil
Mahir
Krishna
```

**for…of** uses iterators behind the scenes.

---

# **5. Creating Your Own Simple Iterator**

Let’s create an iterator that counts from 1 to 3.

```javascript
function createCounter() {
    let number = 1;

    return {
        next() {
            if (number <= 3) {
                return { value: number++, done: false };
            } else {
                return { done: true };
            }
        }
    };
}

var counter = createCounter();

console.log(counter.next());
console.log(counter.next());
console.log(counter.next());
console.log(counter.next());
```

Output:

```
{ value: 1, done: false }
{ value: 2, done: false }
{ value: 3, done: false }
{ done: true }
```

This shows how iterators work internally.

# **GENERATORS**

Generators make iterators **super easy**.

---

# **6. What Is a Generator?**

A **generator** is a special function that can:

* Pause
* Return a value
* Resume again

Generators use:

### `function*` ← star makes it a generator

### `yield` ← gives the next value

---

# **7. Generator Example**

```javascript
function* numbers() {
    yield 1;
    yield 2;
    yield 3;
}

var it = numbers();

console.log(it.next());
console.log(it.next());
console.log(it.next());
console.log(it.next());
```

Output:

```
{ value: 1, done: false }
{ value: 2, done: false }
{ value: 3, done: false }
{ value: undefined, done: true }
```

**This produces the same behavior as our custom iterator—but much simpler.**

---

# **8. Looping Over a Generator**

```javascript
for (var n of numbers()) {
    console.log(n);
}
```

Output:

```
1
2
3
```

---

# **9. Generator With a Loop**

```javascript
function* countToFive() {
    for (let i = 1; i <= 5; i++) {
        yield i;
    }
}

for (var n of countToFive()) {
    console.log(n);
}
```

---

# **10. Infinite Generators**

Generators can produce **endless** values without storing everything in memory.

```javascript
function* infinite() {
    let num = 1;
    while (true) {
        yield num++;
    }
}

var it = infinite();

console.log(it.next());
console.log(it.next());
console.log(it.next());
```

They never run out of values.

---

# **11. Example — Roll Number Generator**

```javascript
function* rollNumbers() {
    let roll = 1;
    while (true) {
        yield roll++;
    }
}

var rolls = rollNumbers();

console.log("Roll:", rolls.next().value); // 1
console.log("Roll:", rolls.next().value); // 2
console.log("Roll:", rolls.next().value); // 3
```

Perfect for generating IDs, roll numbers, ticket numbers, etc.

---

# **12. Common Mistakes**

### Mistake 1

Forgetting `*` in generator:

```javascript
function generator() { } // ❌ NOT a generator
```

Correct:

```javascript
function* generator() { }
```

---

### Mistake 2

Using `return` instead of `yield`

`return` ends the generator permanently.

---

### Mistake 3

Trying to loop with `for...in` instead of `for...of`

* `for...in` = object keys
* `for...of` = iterables (arrays, strings, generators, sets, maps)
