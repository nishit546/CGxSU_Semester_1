# **Sitemap.xml in React**

## **1. What is Sitemap.xml**

A **sitemap.xml** is a structured file that tells search engines:

* Which pages exist on your website
* How those pages are organized
* Which pages are more important
* When pages were last updated

It acts as a roadmap for crawlers like Googlebot.

---

## **2. Why Sitemap is Important**

* Helps faster indexing of pages
* Ensures no important page is missed
* Useful for React apps due to client-side rendering
* Improves crawl efficiency and structure

---

## **3. Where to Keep Sitemap.xml in React**

Sitemap must be placed inside the **public folder**.

---

### **Correct Folder Structure**

```bash
your-react-app/
│
├── public/
│   ├── sitemap.xml
│   ├── robots.txt
│   ├── index.html
│
├── src/
│   ├── App.js
│   ├── pages/
│
├── package.json
```

---

### **Final Access URL**

```bash
http://codinggita.com/sitemap.xml
```

---

## **4. What to Write Inside sitemap.xml**

A sitemap contains:

* One root tag: `<urlset>`
* Multiple `<url>` entries
* Each `<url>` contains structured tags

---

## **5. Basic Structure**

```xml
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">

  <url>
    <loc>http://codinggita.com/</loc>
    <lastmod>2026-04-01</lastmod>
    <changefreq>daily</changefreq>
    <priority>1.0</priority>
  </url>

</urlset>
```

---

## **6. Understanding Each Tag**

### **1. `<loc>` (Required)**

Defines the page URL.

```xml
<loc>http://codinggita.com/courses</loc>
```

Guidelines:

* Use full URLs
* Ensure the page exists
* Prefer HTTPS in production

---

### **2. `<lastmod>`**

Indicates when the page was last updated.

```xml
<lastmod>2026-04-01</lastmod>
```

* Format: `YYYY-MM-DD`

---

### **3. `<changefreq>`**

Specifies how often the page changes.

```xml
<changefreq>weekly</changefreq>
```

Possible values:

* always
* hourly
* daily
* weekly
* monthly
* yearly
* never

---

### **4. `<priority>`**

Defines relative importance of the page.

```xml
<priority>0.8</priority>
```

Range:

* 0.0 to 1.0

---

## **7. Pages to Include**

### **Core Pages**

```xml
<loc>http://codinggita.com/</loc>
<loc>http://codinggita.com/about</loc>
<loc>http://codinggita.com/contact</loc>
```

---

### **Feature Pages**

```xml
<loc>http://codinggita.com/courses</loc>
<loc>http://codinggita.com/programs</loc>
```

---

### **Dynamic Pages**

```xml
<loc>http://codinggita.com/course/react</loc>
<loc>http://codinggita.com/course/node</loc>
```

---

### **Blog Pages**

```xml
<loc>http://codinggita.com/blog/react-seo</loc>
```

---

## **8. Pages to Exclude**

Do not include:

* `/admin`
* `/login`
* `/dashboard`
* `/cart`
* `/private`
* API routes (`/api/...`)

---

## **9. Complete Example**

```xml
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">

  <url>
    <loc>http://codinggita.com/</loc>
    <lastmod>2026-04-01</lastmod>
    <changefreq>daily</changefreq>
    <priority>1.0</priority>
  </url>

  <url>
    <loc>http://codinggita.com/about</loc>
    <changefreq>monthly</changefreq>
    <priority>0.7</priority>
  </url>

  <url>
    <loc>http://codinggita.com/courses</loc>
    <changefreq>weekly</changefreq>
    <priority>0.9</priority>
  </url>

  <url>
    <loc>http://codinggita.com/course/react</loc>
    <changefreq>weekly</changefreq>
    <priority>0.8</priority>
  </url>

  <url>
    <loc>http://codinggita.com/course/node</loc>
    <changefreq>weekly</changefreq>
    <priority>0.8</priority>
  </url>

</urlset>
```

---

## **10. Priority Guidelines**

* Homepage → 1.0
* Main listing pages → 0.8 – 0.9
* Content pages → 0.6 – 0.8
* Less important pages → 0.3 – 0.5

---

## **11. Manual Workflow**

1. Create file in:

```bash
public/sitemap.xml
```

2. Add all important URLs
3. Save the file
4. Run the project
5. Verify at:

```bash
http://codinggita.com/sitemap.xml
```

---

## **12. Common Mistakes**

### **1. Using Relative URLs**

```xml
<loc>/about</loc>
```

Correct:

```xml
<loc>http://codinggita.com/about</loc>
```

---

### **2. Including Restricted Pages**

Avoid adding private or protected routes.

---

### **3. Wrong File Location**

Placing sitemap inside `src/` will not make it accessible.

---

### **4. Missing Root Tag**

Ensure `<urlset>` is present.

---

## **13. Key Takeaways**

* Place sitemap.xml inside `public/`
* Use complete URLs
* Include only important pages
* Maintain proper structure and tags
* Keep it updated
