
# **robots.txt in React (Complete Guide with Codebase Placement)**

## **1. What is robots.txt**

A **robots.txt** file is a simple text file that tells search engine crawlers:

* Which pages they are allowed to access
* Which pages they should not crawl

It acts as a **set of instructions for bots like** Googlebot.

---

## **2. Why robots.txt is Important**

robots.txt helps you:

* Prevent indexing of sensitive pages (admin, dashboard, etc.)
* Save crawl budget
* Avoid duplicate content issues
* Guide search engines efficiently

---

## **3. Where to Keep robots.txt in React Project**

In a React app (like Create React App or Vite), the file must be placed in the **public folder**.

### **Correct Folder Structure**

```
your-react-app/
│
├── public/
│   ├── robots.txt   ✅ (IMPORTANT)
│   ├── sitemap.xml
│   ├── index.html
│
├── src/
│   ├── App.js
│   ├── pages/
│
├── package.json
```

---

### **Why inside `public/`?**

Because:

* Files inside `public/` are served directly by the server
* robots.txt must be accessible at:

```
http://codinggita.com/robots.txt
```

---

## **4. Basic robots.txt Example (Must Know)**

```
User-agent: *
Allow: /

Sitemap: http://codinggita.com/sitemap.xml
```

---

### **Explanation**

* `User-agent: *` → applies to all crawlers
* `Allow: /` → allows full website access
* `Sitemap:` → tells where sitemap is located

---

## **5. Blocking Specific Pages**

### Example:

```
User-agent: *
Disallow: /admin
Disallow: /dashboard
Disallow: /private
```

---

### What this means:

* Crawlers will NOT visit:

  * `/admin`
  * `/dashboard`
  * `/private`

---

## **6. Allow + Disallow Together**

```
User-agent: *
Allow: /
Disallow: /admin
```

---

## **7. Blocking Entire Website (Rare but Important)**

```
User-agent: *
Disallow: /
```

This tells search engines:

* Do NOT crawl anything

---

### When to use:

* Staging environment
* Testing builds

---

## **8. Allow Only Specific Bot (Advanced)**

```
User-agent: Googlebot
Allow: /

User-agent: *
Disallow: /
```

---

### Meaning:

* Only Googlebot is allowed
* All other bots are blocked

---

## **9. Linking Sitemap in robots.txt**

Always include sitemap:

```
Sitemap: http://codinggita.com/sitemap.xml
```

---

## **10. Real Production robots.txt Example**

```
User-agent: *
Allow: /

Disallow: /admin
Disallow: /user-dashboard
Disallow: /api

Sitemap: http://codinggita.com/sitemap.xml
```

---

## **11. Common Mistakes (Very Important)**

### 1. Blocking Entire Website by Mistake

```
Disallow: /
```

This can completely remove your site from search results.

---

### 2. Not Adding Sitemap

Missing:

```
Sitemap: ...
```

---

### 3. Wrong File Location

Placing robots.txt inside:

```
src/
```

❌ This will NOT work

---

## **12. How to Test robots.txt**

You can test using:

* Google Search Console
* Manual check:

```
http://codinggita.com/robots.txt
```

---

## **13. Key Takeaways**

* robots.txt controls crawler access
* Must be placed in `public/` folder
* Accessible at root URL
* Helps manage crawl budget and SEO
* Always include sitemap link
