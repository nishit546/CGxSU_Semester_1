# Frontend Development Checklist (Full Stack Project – 2026)

## 0. Design Approval (MANDATORY FIRST STEP)

* [ ] Figma design created before starting development

* [ ] Design includes all major screens/pages

* [ ] Desktop version design is complete (mobile not required in Figma)

* [ ] Consistent design system followed (colors, typography, spacing)

* [ ] Components planned:

  * [ ] Buttons
  * [ ] Forms
  * [ ] Cards
  * [ ] Tables
  * [ ] Navigation (Navbar/Sidebar)

* [ ] User flow is clearly defined (navigation between pages)

* [ ] Design reviewed and approved before development starts

### Important Instructions

* Development should **NOT begin without design approval**
* Figma is required only for **desktop layout planning**
* Responsive behavior (tablet/mobile) will be handled during development

---

## 1. Project Setup & Structure

* [ ] Project created using Vite
* [ ] Tailwind CSS configured properly
* [ ] MUI integrated 
* [ ] Clean folder structure implemented:

  * [ ] components
  * [ ] pages
  * [ ] features
  * [ ] hooks
  * [ ] services
  * [ ] utils

## 2. Routing System

* [ ] React Router implemented
* [ ] Public routes configured
* [ ] Protected routes (authentication required)
* [ ] Role-based routes (Admin/User if applicable)

## 3. State Management (Redux Toolkit)

* [ ] Redux Toolkit setup completed
* [ ] Store properly configured
* [ ] Slices created:

## 4. API Integration

* [ ] Loading states handled
* [ ] Error states handled
* [ ] Retry / fallback mechanism (recommended)

## 17. Local Storage & Session Storage (IMPORTANT)

### Must Have

* [ ] localStorage used for persistent data:

  * [ ] Theme preference
  * [ ] Auth token (if applicable)
  * [ ] User preferences

* [ ] sessionStorage used for temporary/session-based data:

  * [ ] Form progress (multi-step forms)
  * [ ] Temporary filters/search states

### Best Practices

* [ ] Sensitive data NOT stored insecurely
* [ ] Data cleared on logout (localStorage/sessionStorage)
* [ ] Utility/helper functions created for storage handling
* [ ] Fallback handling if storage is unavailable

# Final Evaluation Criteria (Important)

A project will be considered **complete and industry-ready only if**:

* [ ] UI is polished and responsive
* [ ] State management is properly implemented
* [ ] API integration is clean and scalable
* [ ] UX is smooth (loading, error, empty states)
* [ ] Performance optimizations are applied
* [ ] SEO basics are implemented
* [ ] Storage (localStorage/sessionStorage) is used correctly
