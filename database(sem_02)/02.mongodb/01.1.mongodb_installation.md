# **Setting Up MongoDB**

Before working with MongoDB, you need a running MongoDB environment. There are **two standard ways** to use MongoDB:

1. **Local Installation** – MongoDB runs on your own system
2. **MongoDB Atlas** – MongoDB runs on the cloud

In this article, you will learn **both approaches**, understand **when to use each**, and verify your setup correctly.

---

## **1. Choosing Between Local MongoDB and MongoDB Atlas**

| Option            | When to Use                                 |
| ----------------- | ------------------------------------------- |
| **Local MongoDB** | Learning, practice, offline work, exams     |
| **MongoDB Atlas** | Real-world projects, cloud apps, deployment |

For beginners, **local installation is recommended first**.
MongoDB Atlas is essential later when building real applications.

---

## **2. Installing MongoDB Locally**

### **A. Windows Installation**

#### Step 1: Download MongoDB

* Visit the official MongoDB Community Server page
* Download the **Windows MSI installer**

#### Step 2: Run the Installer

* Launch the installer
* Choose **Complete Installation**
* Keep default settings

#### Step 3: Install MongoDB as a Service

* Select **Install MongoDB as a Service**
* Choose **Run service as Network Service user**
* This ensures MongoDB starts automatically

#### Step 4: Verify Installation

Open **Command Prompt** and run:

```bash
mongosh
```

If MongoDB is installed correctly, you will enter the MongoDB shell.

---

### **B. macOS Installation**

#### Step 1: Install Homebrew (If Needed)

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

#### Step 2: Install MongoDB

```bash
brew tap mongodb/brew
brew install mongodb-community
```

#### Step 3: Start MongoDB Service

```bash
brew services start mongodb/brew/mongodb-community
```

#### Step 4: Verify Installation

```bash
mongosh
```

You should now be inside the MongoDB shell.

---

### **C. Linux Installation (Ubuntu/Debian Based)**

MongoDB is **not installed correctly using default apt packages**.
Use MongoDB’s official repository.

#### Step 1: Import MongoDB Public Key

```bash
curl -fsSL https://pgp.mongodb.com/server-7.0.asc | sudo gpg -o /usr/share/keyrings/mongodb-server.gpg --dearmor
```

#### Step 2: Add MongoDB Repository

```bash
echo "deb [ signed-by=/usr/share/keyrings/mongodb-server.gpg ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org.list
```

#### Step 3: Install MongoDB

```bash
sudo apt update
sudo apt install -y mongodb-org
```

#### Step 4: Start MongoDB Service

```bash
sudo systemctl start mongod
sudo systemctl enable mongod
```

#### Step 5: Verify Installation

```bash
mongosh
```

## **3. Using MongoDB Atlas (Cloud MongoDB)**

MongoDB Atlas is a **fully managed cloud MongoDB service**.

You do not need to install or maintain MongoDB servers manually.

---

### **A. Creating an Atlas Account**

* Visit the MongoDB Atlas website
* Sign up using email or GitHub
* Log in to the dashboard

---

### **B. Creating a Free Cluster**

* Click **Create Cluster**
* Choose **Free Tier (M0)**
* Select cloud provider and region
* Create the cluster

---

### **C. Configure Access**

#### Network Access

* Add your current IP address
* For learning, you may allow access from anywhere

#### Database User

* Create a database user
* Set username and password

---

### **D. Connecting to Atlas Using MongoDB Shell**

Click **Connect → MongoDB Shell**, copy the connection string:

```bash
mongosh "mongodb+srv://<username>:<password>@cluster0.mongodb.net/"
```

Replace credentials and run it in terminal.

If successful, you are now connected to MongoDB Atlas.

---

## **4. How to Verify MongoDB Is Working**

Inside `mongosh`, run:

```bash
show dbs
```

If no error appears, MongoDB is running correctly.

---

## **5. Common Beginner Mistakes**

* Running `mongod` manually (not needed)
* Forgetting to start the MongoDB service
* Blocking IP access in MongoDB Atlas
* Mixing local and cloud connection strings

---

You now know:

* How to install MongoDB locally
* How to start and verify MongoDB
* How to use MongoDB Atlas
* When to use local vs cloud MongoDB
