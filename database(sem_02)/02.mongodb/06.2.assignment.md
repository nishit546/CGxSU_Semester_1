## Mandatory Instructions (Must Be Followed)

### Data Source

- All questions must be solved using the provided `jobs.json` dataset only.
- Creating new data is strictly prohibited.
- Modifying existing documents is not allowed.
- Inserting additional records into the collection is not permitted.

### Database Configuration

- Database Name: `codinggita`
- Collection Name: `jobs`
- Data File: `jobs.json`

### Usage Rules

- Update operations are strictly not allowed.
- Delete operations are strictly not allowed.
- Aggregation operations are strictly not allowed.
- Only the `find()` method must be used for all queries.
- Students must use only the MongoDB query operators covered in class.

### Submission Process

- Every query must first be written and executed in Mongo Shell (`mongosh`).
- The output of each query must be carefully verified:
  - The result should correctly satisfy the logical condition of the question.
  - If the output is incorrect, the query must be corrected and re-tested.
- Only after a query produces the correct output in Mongo Shell:
  - The final verified query should be written on paper.
- Writing queries directly on paper without testing is not acceptable.
- Queries submitted on paper must exactly match the verified Mongo Shell queries.

These instructions are mandatory and are designed to reflect professional backend development practices. All learners are expected to follow them strictly.
