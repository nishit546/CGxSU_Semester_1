
# React Native – Lists

**Description**: Lists are one of the most commonly used UI patterns in mobile apps. This module covers everything you need to know about efficiently rendering lists in React Native.

---

## 1. What are Lists?

Lists are used to display collections of similar data items.

**Common Examples**:
- Product catalogs
- Contact lists
- News feeds
- Chat messages
- User profiles
- Order history
- Movie/TV show lists
- Settings menus

React Native provides two main optimized components:
- **`FlatList`** – Best for simple or long flat lists
- **`SectionList`** – Best for grouped/sectioned lists

---

### **When to Use Lists?**

| Use Case                    | Recommended Component |
|----------------------------|-----------------------|
| Long feed of items         | `FlatList`            |
| Grouped data (A-Z, Categories) | `SectionList`     |
| Small static content       | `ScrollView`          |
| Very large lists (1000+)   | `FlatList` + optimizations or `FlashList` |

---

### **Core Philosophy**

- **Performance First**: Always prefer `FlatList` over `ScrollView` for dynamic data.
- **Virtualization**: Only render what’s visible on screen.
- **Reusable Components**: Keep list items clean and memoized.

---

## 2. FlatList – The Go-To List Component

**Description**: `FlatList` is a high-performance component for rendering long lists of data. It uses **virtualization**, meaning it only renders the items currently visible on the screen, making it very efficient even with thousands of items.

---

### **Basic Complete Example**
```jsx
import React from 'react';
import {
  FlatList,
  Text,
  View,
  StyleSheet,
  Pressable,
} from 'react-native';

const DATA = [
  { id: '1', title: 'iPhone 15', price: 899 },
  { id: '2', title: 'MacBook Air', price: 1299 },
  { id: '3', title: 'AirPods Pro', price: 249 },
  { id: '4', title: 'iPad Air', price: 599 },
];

const App = () => {
  const renderItem = ({ item }) => (
    <Pressable style={styles.item}>
      <Text style={styles.title}>{item.title}</Text>
      <Text style={styles.price}>${item.price}</Text>
    </Pressable>
  );

  return (
    <View style={styles.container}>
      <FlatList
        data={DATA}
        keyExtractor={(item) => item.id}
        renderItem={renderItem}
        contentContainerStyle={styles.list}
        showsVerticalScrollIndicator={false}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8f9fa',
  },
  list: {
    padding: 16,
  },
  item: {
    backgroundColor: '#fff',
    padding: 20,
    marginBottom: 12,
    borderRadius: 12,
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 2 },
    shadowOpacity: 0.1,
    shadowRadius: 8,
    elevation: 3,
  },
  title: {
    fontSize: 18,
    fontWeight: '600',
  },
  price: {
    fontSize: 16,
    color: '#007AFF',
    marginTop: 4,
  },
});

export default App;
```

---

### **Important Props**

| Prop                    | Description                                      | Example |
|-------------------------|--------------------------------------------------|--------|
| `data`                  | Array of data to render                          | `DATA` |
| `renderItem`            | Function that renders each item                  | `({item}) => <View>...</View>` |
| `keyExtractor`          | Unique key for each item                         | `(item) => item.id` |
| `horizontal`            | Renders list horizontally                        | `true` |
| `numColumns`            | Grid layout                                      | `2` or `3` |
| `ListHeaderComponent`   | Component rendered at the top                    | `<SearchBar />` |
| `ListFooterComponent`   | Component at the bottom (e.g. loading)           | `<ActivityIndicator />` |
| `ListEmptyComponent`    | Shown when data is empty                         | `<Text>No results</Text>` |
| `refreshing`            | Pull-to-refresh state                            | `true` / `false` |
| `onRefresh`             | Pull-to-refresh handler                          | `() => fetchData()` |

---

### **More Examples**

#### **1. Horizontal FlatList**
```jsx
<FlatList
  horizontal
  data={DATA}
  renderItem={...}
  showsHorizontalScrollIndicator={false}
/>
```

#### **2. Grid Layout**
```jsx
<FlatList
  data={DATA}
  numColumns={2}
  keyExtractor={item => item.id}
  renderItem={...}
/>
```

#### **3. Pull-to-Refresh**
```jsx
<FlatList
  data={data}
  refreshing={refreshing}
  onRefresh={onRefresh}
  renderItem={...}
/>
```

#### **4. With Header & Footer**
```jsx
<ListHeaderComponent={<Text style={styles.header}>Featured Products</Text>}
<ListFooterComponent={isLoading && <ActivityIndicator />}
<ListEmptyComponent={<Text>No items found</Text>}
/>
```

## 3. SectionList – Grouped Lists

**Description**: `SectionList` is a performant component for rendering lists with grouped sections. It’s ideal for data that needs to be categorized, such as contacts by alphabet, settings grouped by category, or products by brand.

---

### **Basic Complete Example**
```jsx
import React from 'react';
import {
  SectionList,
  Text,
  View,
  StyleSheet,
  Pressable,
} from 'react-native';

const SECTIONS = [
  {
    title: 'A',
    data: ['Alice', 'Aman', 'Ankit', 'Arjun'],
  },
  {
    title: 'B',
    data: ['Bharat', 'Bobby', 'Brian'],
  },
  {
    title: 'C',
    data: ['Chetan', 'Chirag', 'Clara'],
  },
  {
    title: 'D',
    data: ['Deepak', 'Diana'],
  },
];

const App = () => {
  const renderItem = ({ item }) => (
    <Pressable style={styles.item}>
      <Text style={styles.itemText}>{item}</Text>
    </Pressable>
  );

  const renderSectionHeader = ({ section }) => (
    <View style={styles.header}>
      <Text style={styles.headerText}>{section.title}</Text>
    </View>
  );

  return (
    <View style={styles.container}>
      <SectionList
        sections={SECTIONS}
        keyExtractor={(item, index) => item + index}
        renderItem={renderItem}
        renderSectionHeader={renderSectionHeader}
        stickySectionHeadersEnabled={true}
        showsVerticalScrollIndicator={false}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8f9fa',
  },
  header: {
    backgroundColor: '#f0f0f0',
    padding: 16,
    borderBottomWidth: 1,
    borderBottomColor: '#ddd',
  },
  headerText: {
    fontSize: 18,
    fontWeight: 'bold',
    color: '#333',
  },
  item: {
    padding: 20,
    backgroundColor: '#fff',
    borderBottomWidth: 1,
    borderBottomColor: '#eee',
  },
  itemText: {
    fontSize: 17,
  },
});

export default App;
```

---

### **Important Props**

| Prop                        | Description                                      | Example |
|----------------------------|--------------------------------------------------|--------|
| `sections`                 | Array of sections (with `title` and `data`)      | `SECTIONS` |
| `renderItem`               | How to render each item                          | `({item}) => <View>...</View>` |
| `renderSectionHeader`      | Renders the header for each section              | `({section}) => <Text>{section.title}</Text>` |
| `keyExtractor`             | Unique key for each item                         | `(item, index) => item + index` |
| `stickySectionHeadersEnabled` | Makes headers stick while scrolling          | `true` |
| `ListHeaderComponent`      | Component at the very top                        | `<SearchBar />` |
| `ListFooterComponent`      | Component at the bottom                          | Loading indicator |
| `onEndReached`             | Called when scrolled near the end                | Infinite scroll |

---

### **More Examples**

#### **1. Grouped by Category (Products)**
```jsx
const sections = [
  { 
    title: 'Electronics', 
    data: [/* products */] 
  },
  { 
    title: 'Fashion', 
    data: [/* clothes */] 
  },
];
```

#### **2. With Custom Section Header**
```jsx
renderSectionHeader={({ section }) => (
  <View style={styles.sectionHeader}>
    <Text style={styles.sectionTitle}>{section.title}</Text>
    <Text style={styles.count}>{section.data.length} items</Text>
  </View>
)}
```

#### **3. Sticky Headers + Search**
```jsx
<SectionList
  stickySectionHeadersEnabled={true}
  ListHeaderComponent={<SearchBar onChangeText={handleSearch} />}
/>
```

---

**Best Practices**:
- Use `stickySectionHeadersEnabled={true}` for better UX
- Keep section data well-structured
- Consider `FlatList` instead if you don’t need sections

## 4. Infinite Scroll

**Description**: Infinite Scroll automatically loads more data when the user scrolls near the bottom of the list. It provides a smooth, continuous browsing experience commonly used in social media, e-commerce, and news apps.

---

### **Basic Complete Example**
```jsx
import React, { useState, useEffect } from 'react';
import {
  FlatList,
  View,
  Text,
  StyleSheet,
  ActivityIndicator,
} from 'react-native';

const InfiniteScrollScreen = () => {
  const [data, setData] = useState([]);
  const [page, setPage] = useState(1);
  const [loading, setLoading] = useState(false);
  const [loadingMore, setLoadingMore] = useState(false);
  const [hasMore, setHasMore] = useState(true);

  // Initial load
  useEffect(() => {
    loadData(1);
  }, []);

  const loadData = async (pageNum) => {
    setLoading(true);
    try {
      // Simulate API call
      const newData = await fetchData(pageNum);
      setData(prev => pageNum === 1 ? newData : [...prev, ...newData]);
      setHasMore(newData.length > 0);
    } catch (error) {
      console.error(error);
    } finally {
      setLoading(false);
    }
  };

  const fetchData = (pageNum) => {
    return new Promise(resolve => {
      setTimeout(() => {
        const items = Array.from({ length: 20 }, (_, i) => ({
          id: (pageNum - 1) * 20 + i + 1,
          title: `Item #${(pageNum - 1) * 20 + i + 1}`,
        }));
        resolve(items);
      }, 800);
    });
  };

  const loadMore = () => {
    if (loadingMore || !hasMore) return;
    
    setLoadingMore(true);
    const nextPage = page + 1;
    
    loadData(nextPage).then(() => {
      setPage(nextPage);
      setLoadingMore(false);
    });
  };

  const renderItem = ({ item }) => (
    <View style={styles.item}>
      <Text style={styles.title}>{item.title}</Text>
    </View>
  );

  return (
    <View style={styles.container}>
      <FlatList
        data={data}
        keyExtractor={item => item.id.toString()}
        renderItem={renderItem}
        onEndReached={loadMore}
        onEndReachedThreshold={0.5}     // Trigger when 50% from bottom
        ListFooterComponent={
          loadingMore ? (
            <View style={styles.footer}>
              <ActivityIndicator size="large" color="#007AFF" />
            </View>
          ) : null
        }
        ListEmptyComponent={
          loading ? <ActivityIndicator size="large" style={styles.loader} /> : null
        }
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8f9fa',
  },
  item: {
    padding: 20,
    backgroundColor: '#fff',
    marginVertical: 6,
    marginHorizontal: 16,
    borderRadius: 12,
  },
  title: {
    fontSize: 17,
  },
  footer: {
    padding: 30,
    alignItems: 'center',
  },
  loader: {
    marginTop: 50,
  },
});

export default InfiniteScrollScreen;
```

---

### **Important Props & Techniques**

| Prop / Technique              | Description                                      | Recommended Value |
|------------------------------|--------------------------------------------------|-------------------|
| `onEndReached`               | Called when end is reached                       | `loadMore` function |
| `onEndReachedThreshold`      | How close to bottom to trigger (0.0 - 1.0)       | `0.3` to `0.5` |
| `ListFooterComponent`        | Shown at bottom while loading more               | `ActivityIndicator` |
| `refreshing` + `onRefresh`   | Pull-to-refresh                                  | Combined with infinite scroll |

---

### **More Examples**

#### **1. Pull-to-Refresh + Infinite Scroll**
```jsx
<FlatList
  refreshing={refreshing}
  onRefresh={onRefresh}
  onEndReached={loadMore}
  onEndReachedThreshold={0.4}
/>
```

#### **2. With Search + Infinite Scroll**
```jsx
const [searchQuery, setSearchQuery] = useState('');
// Reset page and data when search changes
```

#### **3. Error Handling**
```jsx
const [error, setError] = useState(null);
// Show retry button in ListFooterComponent if error occurs
```

---

**Best Practices**:
- Add a small delay or threshold to prevent too many calls
- Show loading indicator at the bottom
- Handle cases where there is no more data (`hasMore`)
- Reset `page` and `data` when filters/search change
- Use `useCallback` for `loadMore` to avoid unnecessary re-renders

## 5. Pagination

**Description**: Pagination loads data page by page instead of all at once. It is better for very large datasets as it reduces initial load time, saves memory, and improves performance.

**Common API Pattern**:  
`/users?page=1&limit=20`

---

### **Basic Complete Example**
```jsx
import React, { useState, useEffect } from 'react';
import {
  FlatList,
  View,
  Text,
  StyleSheet,
  ActivityIndicator,
  Pressable,
} from 'react-native';

const PaginationScreen = () => {
  const [data, setData] = useState([]);
  const [page, setPage] = useState(1);
  const [loading, setLoading] = useState(false);
  const [loadingMore, setLoadingMore] = useState(false);
  const [hasMore, setHasMore] = useState(true);

  const fetchPage = async (pageNum) => {
    setLoading(pageNum === 1);
    setLoadingMore(pageNum > 1);

    try {
      // Simulate API call with pagination
      const response = await new Promise(resolve => {
        setTimeout(() => {
          const items = Array.from({ length: 20 }, (_, i) => ({
            id: (pageNum - 1) * 20 + i + 1,
            name: `User ${(pageNum - 1) * 20 + i + 1}`,
            email: `user${(pageNum - 1) * 20 + i + 1}@example.com`,
          }));
          resolve(items);
        }, 800);
      });

      if (pageNum === 1) {
        setData(response);
      } else {
        setData(prev => [...prev, ...response]);
      }

      setHasMore(response.length === 20); // If less than limit, no more data
    } catch (error) {
      console.error(error);
    } finally {
      setLoading(false);
      setLoadingMore(false);
    }
  };

  useEffect(() => {
    fetchPage(1);
  }, []);

  const loadMore = () => {
    if (loadingMore || !hasMore) return;
    const nextPage = page + 1;
    setPage(nextPage);
    fetchPage(nextPage);
  };

  const renderItem = ({ item }) => (
    <View style={styles.item}>
      <Text style={styles.name}>{item.name}</Text>
      <Text style={styles.email}>{item.email}</Text>
    </View>
  );

  return (
    <View style={styles.container}>
      <FlatList
        data={data}
        keyExtractor={item => item.id.toString()}
        renderItem={renderItem}
        onEndReached={loadMore}
        onEndReachedThreshold={0.4}
        ListFooterComponent={
          loadingMore && (
            <View style={styles.footer}>
              <ActivityIndicator size="large" color="#007AFF" />
            </View>
          )
        }
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8f9fa',
  },
  item: {
    backgroundColor: '#fff',
    padding: 20,
    marginVertical: 6,
    marginHorizontal: 16,
    borderRadius: 12,
  },
  name: {
    fontSize: 17,
    fontWeight: '600',
  },
  email: {
    fontSize: 15,
    color: '#666',
    marginTop: 4,
  },
  footer: {
    padding: 30,
    alignItems: 'center',
  },
});

export default PaginationScreen;
```

---

### **Important Concepts**

| Concept                | Description                                      | Implementation |
|-----------------------|--------------------------------------------------|----------------|
| `page` state          | Current page number                              | `useState(1)` |
| `hasMore`             | Whether more data exists                         | Check response length |
| `onEndReached`        | Trigger pagination                               | `loadMore` function |
| Initial Load          | Load first page on mount                         | `useEffect` |

---

### **More Examples**

#### **1. With Search + Pagination**
```jsx
const [search, setSearch] = useState('');
// Reset page = 1 and data when search changes
```

#### **2. Page Size Control**
```jsx

// API call: `/users?page=${page}&limit=${LIMIT}`
```

#### **3. Manual "Load More" Button**
```jsx

  <Pressable onPress={loadMore}>
    <Text>Load More</Text>
  </Pressable>
)}
```

---

**Best Practices**:
- Combine with **Infinite Scroll** (`onEndReached`)
- Show loading indicator at bottom while fetching more
- Reset `page` and `data` when filters change
- Handle network errors gracefully
- Consider caching strategies for better UX

## 6. Pull To Refresh

**Description**: Pull to Refresh allows users to swipe down on a list to reload or update its content. It’s one of the most common and intuitive interactions in mobile apps.

---

### **Basic Complete Example**
```jsx
import React, { useState, useEffect } from 'react';
import {
  FlatList,
  View,
  Text,
  StyleSheet,
  RefreshControl,
} from 'react-native';

const PullToRefreshScreen = () => {
  const [data, setData] = useState([]);
  const [refreshing, setRefreshing] = useState(false);

  const fetchData = async () => {
    // Simulate API call
    return new Promise(resolve => {
      setTimeout(() => {
        const newData = Array.from({ length: 15 }, (_, i) => ({
          id: i + 1,
          title: `Item ${i + 1}`,
          time: new Date().toLocaleTimeString(),
        }));
        resolve(newData);
      }, 1200);
    });
  };

  const onRefresh = async () => {
    setRefreshing(true);
    try {
      const freshData = await fetchData();
      setData(freshData);
    } catch (error) {
      console.error('Failed to refresh:', error);
    } finally {
      setRefreshing(false);
    }
  };

  // Initial load
  useEffect(() => {
    onRefresh();
  }, []);

  const renderItem = ({ item }) => (
    <View style={styles.item}>
      <Text style={styles.title}>{item.title}</Text>
      <Text style={styles.subtitle}>Updated at: {item.time}</Text>
    </View>
  );

  return (
    <View style={styles.container}>
      <FlatList
        data={data}
        keyExtractor={item => item.id.toString()}
        renderItem={renderItem}
        refreshing={refreshing}
        onRefresh={onRefresh}
        showsVerticalScrollIndicator={false}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8f9fa',
  },
  item: {
    backgroundColor: '#fff',
    padding: 20,
    marginVertical: 6,
    marginHorizontal: 16,
    borderRadius: 12,
  },
  title: {
    fontSize: 17,
    fontWeight: '600',
  },
  subtitle: {
    fontSize: 14,
    color: '#666',
    marginTop: 6,
  },
});

export default PullToRefreshScreen;
```

---

### **Important Props**

| Prop            | Description                                      | Usage |
|----------------|--------------------------------------------------|-------|
| `refreshing`   | Controls the refresh indicator state             | `true` / `false` |
| `onRefresh`    | Function called when user pulls down             | Async function |
| `RefreshControl` | Custom refresh indicator (optional)            | Advanced styling |

---

### **More Examples**

#### **1. With SectionList**
```jsx
<SectionList
  sections={sections}
  refreshing={refreshing}
  onRefresh={onRefresh}
  renderItem={...}
/>
```

#### **2. Custom RefreshControl**
```jsx
<FlatList
  ...
  refreshControl={
    <RefreshControl
      refreshing={refreshing}
      onRefresh={onRefresh}
      colors={['#007AFF']}
      tintColor="#007AFF"
      title="Pull to refresh..."
    />
  }
/>
```

#### **3. Combined with Infinite Scroll**
```jsx
<FlatList
  data={data}
  refreshing={refreshing}
  onRefresh={onRefresh}
  onEndReached={loadMore}
  onEndReachedThreshold={0.5}
  ListFooterComponent={loadingMore && <ActivityIndicator />}
/>
```

---

**Best Practices**:
- Always set `refreshing` to `false` in the `finally` block
- Provide visual feedback during refresh
- Combine with Infinite Scroll for modern UX
- Show a meaningful message if refresh fails
- Keep refresh delay reasonable (800ms–1500ms)

## 7. Performance Optimization Tips for Lists

**Description**: Even with `FlatList`, poor implementation can cause lag, high memory usage, and janky scrolling. These tips will help you build buttery-smooth lists in production apps.

---

### **Best Practices Overview**

Here are the most important optimization techniques:

### **1. Always Use `keyExtractor`**
```jsx
<FlatList
  data={data}
  keyExtractor={(item) => item.id.toString()}   // Very Important!
  renderItem={...}
/>
```

### **2. Memoize List Items with `React.memo()`**
```jsx
const ListItem = React.memo(({ item }) => {
  return (
    <View style={styles.item}>
      <Text>{item.title}</Text>
    </View>
  );
});
```

### **3. Use `getItemLayout` for Fixed Height Items**
```jsx
const ITEM_HEIGHT = 80;

<FlatList
  data={data}
  getItemLayout={(data, index) => ({
    length: ITEM_HEIGHT,
    offset: ITEM_HEIGHT * index,
    index,
  })}
  renderItem={...}
/>
```

---

### **Basic Complete Optimized Example**
```jsx
import React, { useCallback } from 'react';
import { FlatList, View, Text, StyleSheet } from 'react-native';

const OptimizedList = ({ data }) => {
  const renderItem = useCallback(({ item }) => (
    <ListItem item={item} />
  ), []);

  return (
    <FlatList
      data={data}
      keyExtractor={(item) => item.id.toString()}
      renderItem={renderItem}
      getItemLayout={(data, index) => ({
        length: 88,
        offset: 88 * index,
        index,
      })}
      removeClippedSubviews={true}
      windowSize={21}
      maxToRenderPerBatch={10}
      updateCellsBatchingPeriod={50}
      initialNumToRender={10}
      showsVerticalScrollIndicator={false}
    />
  );
};

// Memoized Item
const ListItem = React.memo(({ item }) => {
  return (
    <View style={styles.item}>
      <Text style={styles.title}>{item.title}</Text>
      <Text style={styles.subtitle}>{item.subtitle}</Text>
    </View>
  );
});

const styles = StyleSheet.create({
  item: {
    padding: 20,
    borderBottomWidth: 1,
    borderBottomColor: '#eee',
    backgroundColor: '#fff',
  },
  title: {
    fontSize: 17,
    fontWeight: '600',
  },
  subtitle: {
    fontSize: 15,
    color: '#666',
    marginTop: 4,
  },
});

export default OptimizedList;
```

---

### **More Optimization Tips**

| Tip                              | Benefit                                      |
|----------------------------------|---------------------------------------------|
| `removeClippedSubviews={true}`   | Unmounts off-screen items                   |
| `windowSize`                     | Controls how many items are rendered        |
| `maxToRenderPerBatch`            | Controls batch rendering                    |
| `updateCellsBatchingPeriod`      | Improves responsiveness                     |
| `initialNumToRender`             | Faster initial load                         |
| Avoid inline functions           | Prevents unnecessary re-renders             |
| Use `expo-image` instead of `Image` | Better caching & performance             |

---

**Advanced Tip**:  
For extremely large lists (1000+ items), consider using **FlashList** from Shopify (`@shopify/flash-list`) — it's significantly faster than `FlatList`.

## 8. FlatList vs ScrollView

**Description**: Understanding when to use `FlatList` versus `ScrollView` is crucial for building performant React Native apps. Choosing the wrong component can lead to laggy UI, high memory usage, and poor user experience.

---

### **Comparison Table**

| Feature              | **FlatList**                                      | **ScrollView**                                   |
|----------------------|---------------------------------------------------|--------------------------------------------------|
| Data Size            | Large / Infinite lists                            | Small / Static content                           |
| Rendering            | Virtualized (only renders visible items)          | Renders **all** children at once                 |
| Performance          | Excellent for long lists                          | Poor with large content                          |
| Memory Usage         | Low                                               | High                                             |
| Pull-to-Refresh      | Built-in support (`onRefresh`)                    | Manual implementation needed                     |
| Infinite Scroll      | Easy (`onEndReached`)                             | Difficult                                        |
| Horizontal Support   | Excellent                                         | Good                                             |
| Recommended For      | Feeds, Product lists, Chat messages, Grids        | Forms, Onboarding, Small carousels               |

---

### **Basic Examples**

#### **1. ScrollView (Good Use Case)**
```jsx
<ScrollView style={styles.container}>
  <Text>Form Title</Text>
  <TextInput placeholder="Name" />
  <TextInput placeholder="Email" />
  <TextInput placeholder="Password" />
  {/* Few components only */}
</ScrollView>
```

#### **2. FlatList (Recommended for Lists)**
```jsx
<FlatList
  data={products}
  keyExtractor={item => item.id}
  renderItem={({ item }) => <ProductCard item={item} />}
  onEndReached={loadMoreProducts}
/>
```

---

### **When to Choose Which?**

**Use ScrollView when:**
- Content is small and static (forms, terms & conditions, onboarding)
- You need smooth scrolling with mixed content (images + text + buttons)
- Number of items is very small (< 20)

**Use FlatList when:**
- You have 20+ items
- Data comes from an API
- You need infinite scroll or pull-to-refresh
- You want grid layouts (`numColumns`)
- Performance is critical

---

### **Advanced Comparison: Real-World Impact**

| Scenario                    | ScrollView Performance | FlatList Performance |
|----------------------------|------------------------|----------------------|
| 50 items                   | Acceptable             | Excellent            |
| 500+ items                 | Very Laggy             | Smooth               |
| Chat messages (1000+)      | Crashes often          | Smooth               |
| E-commerce product feed    | Not recommended        | Ideal                |

---

**Pro Tip**:
> **Rule of Thumb**: If your list can grow beyond 20–30 items — **always use `FlatList`**.
