
# React Native – Navigation

**Description**: Navigation is one of the most important parts of any mobile app. It defines how users move between different screens, pass data, and manage the overall user journey.

---

## 1. What is Navigation?

Navigation allows users to move between different screens in your app.

**Common Flows**:
- Login → Home → Profile
- Product List → Product Details
- Home → Cart → Checkout
- Onboarding → Login → Main App

**Standard Library**: **React Navigation** (Most popular and officially recommended by the React Native community).

---

## 2. Installation & Setup

### **Installation Commands**
```bash
# Core package
npm install @react-navigation/native

# For Expo users
npx expo install react-native-screens react-native-safe-area-context

# For Stack Navigator (most common)
npm install @react-navigation/native-stack
```

### **Basic Setup – Wrap Your App**
```jsx
// App.js
import React from 'react';
import { NavigationContainer } from '@react-navigation/native';

import AppNavigator from './navigation/AppNavigator';

export default function App() {
  return (
    <NavigationContainer>
      <AppNavigator />
    </NavigationContainer>
  );
}
```

---

### **Important Notes**
- `NavigationContainer` must wrap your entire navigation structure.
- Always install `react-native-screens` and `react-native-safe-area-context` for best performance.
- For bare React Native (non-Expo), additional native configuration may be required.

---

## 3. Stack Navigation

**Description**: Stack Navigator works like a stack of cards. When you navigate to a new screen, it is pushed on top of the previous one.You can go back by popping the top screen.

**Best For**: Sequential flows like:
- Login → OTP → Home
- Product List → Product Details → Cart
- Onboarding screens

---

### **Installation**
```bash
npm install @react-navigation/native-stack
# or
yarn add @react-navigation/native-stack
```

> **Note**: Make sure you have already installed `@react-navigation/native`, `react-native-screens`, and `react-native-safe-area-context`.

---

### **Basic Complete Example**
```jsx
// App.js
import React from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';

import HomeScreen from './screens/HomeScreen';
import DetailsScreen from './screens/DetailsScreen';
import ProfileScreen from './screens/ProfileScreen';

const Stack = createNativeStackNavigator();

const App = () => {
  return (
    <NavigationContainer>
      <Stack.Navigator 
        initialRouteName="Home"
        screenOptions={{
          headerStyle: { backgroundColor: '#007AFF' },
          headerTintColor: '#fff',
          headerTitleStyle: { fontWeight: 'bold' },
        }}
      >
        <Stack.Screen 
          name="Home" 
          component={HomeScreen} 
          options={{ title: 'Home' }}
        />
        <Stack.Screen 
          name="Details" 
          component={DetailsScreen} 
          options={{ title: 'Product Details' }}
        />
        <Stack.Screen 
          name="Profile" 
          component={ProfileScreen} 
          options={{ title: 'My Profile' }}
        />
      </Stack.Navigator>
    </NavigationContainer>
  );
};

export default App;
```

---

### **Navigation Methods**

| Method                    | Description                                      | Example |
|--------------------------|--------------------------------------------------|--------|
| `navigation.navigate()`  | Go to screen (reuses if already in stack)       | `navigate('Details')` |
| `navigation.push()`      | Always push a new screen (even if exists)       | `push('Details')` |
| `navigation.goBack()`    | Go back to previous screen                      | `goBack()` |
| `navigation.replace()`   | Replace current screen (no back)                | `replace('Login')` |
| `navigation.reset()`     | Reset stack to new state                        | Advanced reset |

## 4. Bottom Tab Navigation

**Description**: Bottom Tab Navigation displays a tab bar at the bottom of the screen, allowing users to quickly switch between main sections of the app.

**Best For**: Main app sections such as:
- Home, Search, Cart, Profile
- Feed, Explore, Notifications, Account

---

### **Installation**
```bash
npm install @react-navigation/bottom-tabs
# or
yarn add @react-navigation/bottom-tabs
```

---

### **Basic Complete Example**
```jsx
// App.js
import React from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';
import { Ionicons } from '@expo/vector-icons';

import HomeScreen from './screens/HomeScreen';
import SearchScreen from './screens/SearchScreen';
import CartScreen from './screens/CartScreen';
import ProfileScreen from './screens/ProfileScreen';

const Tab = createBottomTabNavigator();

const App = () => {
  return (
    <NavigationContainer>
      <Tab.Navigator
        screenOptions={({ route }) => ({
          tabBarIcon: ({ color, size }) => {
            let iconName;

            if (route.name === 'Home') iconName = 'home';
            else if (route.name === 'Search') iconName = 'search';
            else if (route.name === 'Cart') iconName = 'cart';
            else if (route.name === 'Profile') iconName = 'person';

            return <Ionicons name={iconName} size={size} color={color} />;
          },
          tabBarActiveTintColor: '#007AFF',
          tabBarInactiveTintColor: 'gray',
          headerShown: true,
        })}
      >
        <Tab.Screen name="Home" component={HomeScreen} />
        <Tab.Screen name="Search" component={SearchScreen} />
        <Tab.Screen 
          name="Cart" 
          component={CartScreen}
          options={{ tabBarBadge: 3 }} // Show badge
        />
        <Tab.Screen name="Profile" component={ProfileScreen} />
      </Tab.Navigator>
    </NavigationContainer>
  );
};

export default App;
```

---

### **Important Options**

| Option                    | Description                                      | Example |
|--------------------------|--------------------------------------------------|--------|
| `tabBarIcon`             | Icon for each tab                                | `Ionicons` |
| `tabBarActiveTintColor`  | Color of active tab                              | `'#007AFF'` |
| `tabBarBadge`            | Show notification badge                          | `3` or `'New'` |
| `headerShown`            | Show/hide header                                 | `false` |
| `tabBarStyle`            | Custom styling for tab bar                       | `{ backgroundColor: '#fff' }` |

---

### **More Examples**

#### **1. Custom Tab Bar Icon with Focused State**



#### **4. Tab Bar Label Customization**
```jsx

  tabBarLabel: 'Home',

```

#### **5. Programmatic Navigation**
```jsx

```

**Pro Tips**:
- Combine **Stack Navigator** inside tabs for better UX (e.g., Home → Details)
- Use `Ionicons` or `MaterialIcons` for consistent icons

- Use `tabBarBadge` for notifications/cart count

## 6. Nested Navigation (Most Important for Real Apps)

**Description**: Real-world React Native apps rarely use just one type of navigator. **Nested Navigation** means combining multiple navigators (Stack + Tabs + Drawer) to create complex but smooth user flows.

### **Why Nested Navigation Matters?**
- Clean separation of concerns
- Better user experience
- Authentication flow + Main app flow
- Independent navigation within tabs

---

### **Recommended Real App Structure**
```jsx
<NavigationContainer>
  <Stack.Navigator screenOptions={{ headerShown: false }}>
    {/* Authentication Flow */}
    <Stack.Screen name="Auth" component={AuthStack} />
    
    {/* Main App Flow */}
    <Stack.Screen name="Main" component={MainTabs} />
  </Stack.Navigator>
</NavigationContainer>
```

---

### **Complete Working Example**
```jsx
// App.js
import React from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';
import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';
import { Ionicons } from '@expo/vector-icons';

import LoginScreen from './screens/LoginScreen';
import HomeScreen from './screens/HomeScreen';
import SearchScreen from './screens/SearchScreen';
import CartScreen from './screens/CartScreen';
import ProfileScreen from './screens/ProfileScreen';
import ProductDetailsScreen from './screens/ProductDetailsScreen';

const Stack = createNativeStackNavigator();
const Tab = createBottomTabNavigator();

// 1. Bottom Tabs Navigator
function MainTabs() {
  return (
    <Tab.Navigator screenOptions={{ headerShown: false }}>
      <Tab.Screen 
        name="HomeTab" 
        component={HomeStack} 
        options={{
          tabBarIcon: ({ color, size }) => <Ionicons name="home" size={size} color={color} />
        }}
      />
      <Tab.Screen 
        name="Search" 
        component={SearchScreen}
        options={{
          tabBarIcon: ({ color, size }) => <Ionicons name="search" size={size} color={color} />
        }}
      />
      <Tab.Screen 
        name="Cart" 
        component={CartScreen}
        options={{
          tabBarIcon: ({ color, size }) => <Ionicons name="cart" size={size} color={color} />,
          tabBarBadge: 2,
        }}
      />
      <Tab.Screen 
        name="Profile" 
        component={ProfileScreen}
        options={{
          tabBarIcon: ({ color, size }) => <Ionicons name="person" size={size} color={color} />
        }}
      />
    </Tab.Navigator>
  );
}

// 2. Home Stack (Inside one Tab)
function HomeStack() {
  return (
    <Stack.Navigator>
      <Stack.Screen name="Home" component={HomeScreen} />
      <Stack.Screen 
        name="ProductDetails" 
        component={ProductDetailsScreen} 
        options={{ title: 'Product Details' }}
      />
    </Stack.Navigator>
  );
}

// 3. Auth Stack
function AuthStack() {
  return (
    <Stack.Navigator screenOptions={{ headerShown: false }}>
      <Stack.Screen name="Login" component={LoginScreen} />
    </Stack.Navigator>
  );
}

// 4. Root Navigator
const App = () => {
  const [isLoggedIn, setIsLoggedIn] = React.useState(false); // Replace with real auth logic

  return (
    <NavigationContainer>
      <Stack.Navigator screenOptions={{ headerShown: false }}>
        {isLoggedIn ? (
          <Stack.Screen name="Main" component={MainTabs} />
        ) : (
          <Stack.Screen name="Auth" component={AuthStack} />
        )}
      </Stack.Navigator>
    </NavigationContainer>
  );
};

export default App;
```

---

### **Key Benefits of Nested Navigation**

| Benefit                        | Explanation |
|-------------------------------|-----------|
| Independent Navigation        | Each tab can have its own stack |
| Clean Authentication Flow     | Separate Auth and Main flows |
| Better Performance            | Only renders active navigators |
| Scalable Architecture         | Easy to maintain large apps |

---

### **Common Patterns & Tips**

1. **Hide Header in Tabs**
   ```jsx
   screenOptions={{ headerShown: false }}
   ```

2. **Navigate Between Nested Navigators**
   ```jsx
   navigation.navigate('HomeTab', { 
     screen: 'ProductDetails',
     params: { id: 123 }
   });
   ```

3. **Reset Navigation on Login**
   ```jsx
   navigation.reset({
     index: 0,
     routes: [{ name: 'Main' }]
   });
   ```

4. **Common Folder Structure**
   ```
   src/
   ├── navigation/
   │   ├── AuthNavigator.js
   │   ├── MainNavigator.js
   │   └── AppNavigator.js
   ├── screens/
   └── context/
   ```

---

**Pro Tip**:  
Always keep authentication flow separate from the main app using a root Stack Navigator.

## 7. Passing Parameters (Data Between Screens)

**Description**: Passing data between screens is one of the most common tasks in mobile apps. React Navigation makes it easy to send and receive parameters between screens.

### **Why Passing Parameters Matters?**
- Send product details from list to detail screen
- Pass user data, IDs, settings, or form data
- Enable dynamic screen content

---

### **Basic Complete Example**

#### **1. Sending Data**
```jsx
// HomeScreen.js
import { useNavigation } from '@react-navigation/native';

const HomeScreen = () => {
  const navigation = useNavigation();

  const products = [
    { id: 1, title: 'iPhone 15', price: 899, category: 'Electronics' },
    { id: 2, title: 'MacBook Pro', price: 1999, category: 'Electronics' },
  ];

  const goToDetails = (product) => {
    navigation.navigate('Details', {
      id: product.id,
      title: product.title,
      price: product.price,
      category: product.category,
    });
  };

  return (
    <View style={styles.container}>
      {products.map(product => (
        <Pressable 
          key={product.id}
          style={styles.card}
          onPress={() => goToDetails(product)}
        >
          <Text>{product.title}</Text>
        </Pressable>
      ))}
    </View>
  );
};
```

#### **2. Receiving Data**
```jsx
// DetailsScreen.js
import { useRoute } from '@react-navigation/native';

const DetailsScreen = ({ navigation }) => {
  const route = useRoute();
  const { id, title, price, category } = route.params;

  return (
    <View style={styles.container}>
      <Text style={styles.title}>{title}</Text>
      <Text style={styles.price}>${price}</Text>
      <Text>Category: {category}</Text>
      <Text>Product ID: {id}</Text>

      <Pressable style={styles.button} onPress={() => navigation.goBack()}>
        <Text style={styles.buttonText}>Go Back</Text>
      </Pressable>
    </View>
  );
};
```

---

### **Important Methods**

| Method                        | Description                                      | Use Case |
|------------------------------|--------------------------------------------------|--------|
| `navigation.navigate(name, params)` | Send data while navigating                  | Most common |
| `route.params`               | Access received data                             | In target screen |
| `useRoute()`                 | Access route object (functional way)             | Modern approach |
| `navigation.setParams()`     | Update params of current screen                  | Dynamic updates |

---

### **More Examples**

#### **1. Updating Header Dynamically**
```jsx
<Stack.Screen 
  name="Details" 
  component={DetailsScreen}
  options={({ route }) => ({
    title: route.params?.title || 'Product',
  })}
/>
```

#### **2. Passing Functions (Advanced)**
```jsx
navigation.navigate('EditProfile', {
  onSave: (updatedUser) => {
    // Handle save
  }
});
```

#### **3. Default Values (Safe Access)**
```jsx
const { id = 0, title = 'No Title', price = 0 } = route.params || {};
```

#### **4. Deep Navigation with Nested Navigators**
```jsx
navigation.navigate('Main', {
  screen: 'HomeTab',
  params: {
    screen: 'Details',
    params: { id: 101 }
  }
});
```

#### **5. Passing Large Data (Best Practice)**
For large objects, consider using global state (`Context`, Redux, Zustand) instead of deep params.

---

**Best Practices**:
- Keep params lightweight (IDs + small data)
- Use global state for complex/shared data
- Always handle missing params with default values
- Clean up params when not needed

## 8. Authentication Flow

**Description**: Authentication Flow manages the user’s login state and routes them to the correct part of the app — either the authentication screens (Login, Signup, OTP) or the main app.

**Typical Flow**:
**Splash → Login/Signup → (OTP Verification) → Home**

---

### **Basic Complete Example**
```jsx
// App.js
import React, { useState, useEffect } from 'react';
import { NavigationContainer } from '@react-navigation/native';
import AsyncStorage from '@react-native-async-storage/async-storage';

import AuthNavigator from './navigation/AuthNavigator';
import MainNavigator from './navigation/MainNavigator';
import SplashScreen from './screens/SplashScreen';

const App = () => {
  const [isLoggedIn, setIsLoggedIn] = useState(null); // null = loading
  const [loading, setLoading] = useState(true);

  // Check login status on app start
  useEffect(() => {
    const checkLoginStatus = async () => {
      try {
        const token = await AsyncStorage.getItem('userToken');
        setIsLoggedIn(!!token); // true if token exists
      } catch (error) {
        console.log('Error checking login:', error);
        setIsLoggedIn(false);
      } finally {
        setLoading(false);
      }
    };

    checkLoginStatus();
  }, []);

  if (loading) {
    return <SplashScreen />;
  }

  return (
    <NavigationContainer>
      {isLoggedIn ? (
        <MainNavigator />
      ) : (
        <AuthNavigator />
      )}
    </NavigationContainer>
  );
};

export default App;
```

---

```
│   └── AppNavigator.js

│   │   ├── SignupScreen.js
│   │   └── OTPScreen.js
│   └── main/
├── context/
│   └── AuthContext.js
└── utils/
```

### **Login & Logout Logic**
```jsx
// In LoginScreen
const handleLogin = async () => {
  try {
    // Call your API
    const response = await loginAPI(email, password);
    await AsyncStorage.setItem('userToken', response.token);
    await AsyncStorage.setItem('userData', JSON.stringify(response.user));
    
    // Navigate to Main App
    navigation.reset({
      index: 0,
      routes: [{ name: 'Main' }],
    });
  } catch (error) {
    Alert.alert('Error', 'Invalid credentials');
  }
};

// Logout
const logout = async () => {
  await AsyncStorage.removeItem('userToken');
  await AsyncStorage.removeItem('userData');
  navigation.reset({
    index: 0,
    routes: [{ name: 'Login' }],
  });
};
```

---

### **Best Practices**

| Practice                     | Recommendation |
|-----------------------------|----------------|
| Persist Login               | Use `AsyncStorage` or `expo-secure-store` |
| Loading State               | Show Splash Screen while checking token |
| Reset Navigation            | Use `navigation.reset()` after login |
| Protected Routes            | Check auth state before showing screens |
| Token Expiry                | Implement refresh token logic |

---

**Security Tips**:
- Never store sensitive data in plain `AsyncStorage`
- Use `expo-secure-store` for tokens on Expo
- Always validate tokens on the backend

## 9. Protected Routes

**Description**: Protected Routes ensure that only authenticated users can access certain screens (like Home, Profile, Cart, etc.). Unauthenticated users are redirected to the Login screen.

**Best Practice**: Use a wrapper component or conditional rendering with your Auth Context.

---

### **Basic Complete Example**

#### **1. Auth Context (Recommended)**
```jsx
// context/AuthContext.js
import React, { createContext, useContext, useState, useEffect } from 'react';
import AsyncStorage from '@react-native-async-storage/async-storage';

const AuthContext = createContext();

export const AuthProvider = ({ children }) => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const loadUser = async () => {
      try {
        const userData = await AsyncStorage.getItem('userData');
        if (userData) setUser(JSON.parse(userData));
      } catch (error) {
        console.log(error);
      } finally {
        setLoading(false);
      }
    };
    loadUser();
  }, []);

  const login = async (userData) => {
    await AsyncStorage.setItem('userData', JSON.stringify(userData));
    setUser(userData);
  };

  const logout = async () => {
    await AsyncStorage.removeItem('userData');
    setUser(null);
  };

  return (
    <AuthContext.Provider value={{ user, login, logout, loading }}>
      {children}
    </AuthContext.Provider>
  );
};

export const useAuth = () => useContext(AuthContext);
```

#### **2. Protected Route Component**
```jsx
// components/ProtectedRoute.js
import React from 'react';
import { useAuth } from '../context/AuthContext';
import { View, ActivityIndicator } from 'react-native';
import LoginScreen from '../screens/auth/LoginScreen';

const ProtectedRoute = ({ children }) => {
  const { user, loading } = useAuth();

  if (loading) {
    return (
      <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
        <ActivityIndicator size="large" color="#007AFF" />
      </View>
    );
  }

  if (!user) {
    return <LoginScreen />;
  }

  return children;
};

export default ProtectedRoute;
```

#### **3. Usage in Navigator**
```jsx
// App.js
import { AuthProvider } from './context/AuthContext';
import ProtectedRoute from './components/ProtectedRoute';

const App = () => {
  return (
    <AuthProvider>
      <NavigationContainer>
        <Stack.Navigator screenOptions={{ headerShown: false }}>
          <Stack.Screen name="Auth" component={AuthNavigator} />
          
          <Stack.Screen name="Main">
            {() => (
              <ProtectedRoute>
                <MainNavigator />
              </ProtectedRoute>
            )}
          </Stack.Screen>
        </Stack.Navigator>
      </NavigationContainer>
    </AuthProvider>
  );
};
```

```jsx
const LogoutButton = ({ navigation }) => {
  const { logout } = useAuth();
  
  const handleLogout = async () => {
    await logout();
    navigation.reset({
      index: 0,
      routes: [{ name: 'Auth' }],
    });
  };

  return <Pressable onPress={handleLogout}><Text>Logout</Text></Pressable>;
};
```

#### **3. Role-Based Protection**
```jsx
const ProtectedRoute = ({ children, requiredRole }) => {
  const { user } = useAuth();
  
  if (!user || (requiredRole && user.role !== requiredRole)) {
    return <AccessDeniedScreen />;
  }
  
  return children;
};
```

---

**Best Practices**:
- Always use **Auth Context** for global auth state
- Show a **loading indicator** while checking auth
- Use `navigation.reset()` after login/logout
- Store minimal user data (avoid storing sensitive info)
- Handle token expiration gracefully

## 10. Deep Linking Basics

**Description**: Deep Linking allows users to open a specific screen in your app directly using a URL (from a web link, notification, or QR code). It provides a native-like experience and improves user engagement.

---

### **Basic Complete Example**
```jsx
// App.js
import React from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';

import HomeScreen from './screens/HomeScreen';
import DetailsScreen from './screens/DetailsScreen';
import ProfileScreen from './screens/ProfileScreen';

const Stack = createNativeStackNavigator();

const linking = {
  prefixes: ['myapp://', 'https://myapp.com', 'https://www.myapp.com'],
  config: {
    screens: {
      Home: 'home',
      Details: 'details/:id',        // Dynamic route
      Profile: 'profile/:userId?',   // Optional parameter
    },
  },
};

const App = () => {
  return (
    <NavigationContainer linking={linking}>
      <Stack.Navigator>
        <Stack.Screen name="Home" component={HomeScreen} />
        <Stack.Screen name="Details" component={DetailsScreen} />
        <Stack.Screen name="Profile" component={ProfileScreen} />
      </Stack.Navigator>
    </NavigationContainer>
  );
};

export default App;
```

### **Navigation Comparison**

| Navigator       | Best For                        | When to Use                              | Complexity |
|-----------------|---------------------------------|------------------------------------------|----------|
| **Stack**       | Linear flows                    | Login, Onboarding, Product Details       | Easy |
| **Bottom Tabs** | Main app sections               | Home, Search, Cart, Profile              | Easy |
| **Drawer**      | Side menu navigation            | Settings, More options, Admin panels     | Medium |
| **Nested**      | Complex real-world apps         | Most production applications             | Medium-High |

**Best Practices for Deep Linking**:
- Use consistent URL schemes (`myapp://`)
- Always handle missing parameters gracefully
- Test on both Android and iOS
- Combine with Universal Links / App Links for web-to-app
- Add fallback behavior for invalid links
