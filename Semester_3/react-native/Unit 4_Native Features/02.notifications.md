
# React Native – Notifications

**Description**: Notifications are essential for user engagement. They deliver timely updates, reminders, and messages even when the app is closed or in the background.

---

## 1. What are Notifications?

Notifications are messages that appear on the user’s device to inform them about events, updates, or reminders.

### **Types of Notifications**

| Type          | Origin               | Use Case |
|---------------|----------------------|----------|
| **Local**     | Device itself        | Reminders, alarms, daily tips |
| **Push**      | Remote server        | Chat messages, order status, promotions |
| **Scheduled** | Device (with logic)  | Daily reminders, habit trackers |
| **Foreground**| App is open          | In-app alerts, live updates |

---

### **Why Notifications Matter**
- Increase user retention
- Drive re-engagement
- Deliver time-sensitive information
- Support marketing & promotions
- Enable background functionality

---

### **Key Concepts**
- **Permission**: User must grant notification permission
- **Token**: Unique device token for push notifications
- **Payload**: Data sent with the notification
- **Deep Linking**: Tapping a notification can open a specific screen

---

## 2. Expo Notifications Setup

**Description**: `expo-notifications` is the official Expo library for handling local and push notifications. Proper setup is required before you can send or receive notifications.

---

### **Installation**
```bash
npx expo install expo-notifications
```

---

### **Basic Configuration**
```jsx
// App.js
import React, { useEffect } from 'react';
import * as Notifications from 'expo-notifications';
import { Provider } from 'react-redux';
import AppNavigator from './navigation/AppNavigator';

// Configure notification behavior
Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowBanner: true,
    shouldShowList: true,
    shouldPlaySound: true,
    shouldSetBadge: true,
  }),
});

const App = () => {
  // Handle notification when app is in foreground
  useEffect(() => {
    const subscription = Notifications.addNotificationReceivedListener(notification => {
      console.log('Notification received in foreground:', notification);
    });

    return () => subscription.remove();
  }, []);

  // Handle notification response (user taps notification)
  useEffect(() => {
    const subscription = Notifications.addNotificationResponseReceivedListener(response => {
      console.log('Notification tapped:', response.notification.request.content.data);
      // Navigate to specific screen based on data
    });

    return () => subscription.remove();
  }, []);

  return (
    <Provider store={store}>
      <AppNavigator />
    </Provider>
  );
};

export default App;
```

---

### **Important Setup Steps**

1. **Configure Notification Handler** (shown above)
2. **Request Permission** (see next section)
3. **Get Push Token** for remote notifications
4. **Handle Background / Quit State** (using `expo-notifications`)

## 3. Requesting Notification Permission

**Description**: Before sending notifications, you must request user permission. This should be done thoughtfully to maximize acceptance rates.

---

### **Basic Complete Example**
```jsx
import React, { useEffect } from 'react';
import { Alert } from 'react-native';
import * as Notifications from 'expo-notifications';

const requestNotificationPermission = async () => {
  const { status: existingStatus } = await Notifications.getPermissionsAsync();

  let finalStatus = existingStatus;

  if (existingStatus !== 'granted') {
    const { status } = await Notifications.requestPermissionsAsync();
    finalStatus = status;
  }

  if (finalStatus !== 'granted') {
    Alert.alert(
      'Notifications Disabled',
      'Please enable notifications in Settings for important updates.',
      [
        { text: 'Cancel' },
        { text: 'Open Settings', onPress: () => Linking.openSettings() },
      ]
    );
    return false;
  }

  return true;
};

// Usage in App.js or onboarding screen
const initializeNotifications = async () => {
  const permissionGranted = await requestNotificationPermission();
  
  if (permissionGranted) {
    console.log(' Notification permission granted');
    // Get push token for remote notifications
    const token = await Notifications.getExpoPushTokenAsync();
    console.log('Push Token:', token.data);
  }
};
```

---

### **Permission Flow**
```text
Check current status
       │
   Already granted? ── Yes ── Done
       │
       No
       │
   Show friendly explanation
       │
   Request permission
       │
   ┌─────┴─────┐
Granted?   Denied?
   │          │
 Success   Offer Settings redirect
```

## 4. Local Notifications

**Description**: Local notifications are created and scheduled by the app itself. They are useful for reminders, alerts, and updates without needing a server connection.

---

### **Basic Complete Example**
```jsx
import React from 'react';
import { View, Pressable, Text, StyleSheet, Alert } from 'react-native';
import * as Notifications from 'expo-notifications';

const LocalNotificationsScreen = () => {
  const sendImmediateNotification = async () => {
    try {
      await Notifications.scheduleNotificationAsync({
        content: {
          title: " Task Completed!",
          body: "Great job! Keep up the momentum.",
          sound: true,
          priority: Notifications.AndroidNotificationPriority.HIGH,
        },
        trigger: null, // Show immediately
      });

      Alert.alert(' Sent!', 'Immediate notification has been shown');
    } catch (error) {
      Alert.alert('Error', 'Failed to send notification');
    }
  };

  const scheduleFutureNotification = async () => {
    try {
      await Notifications.scheduleNotificationAsync({
        content: {
          title: " Drink Water Reminder",
          body: "It's been 2 hours. Stay hydrated!",
          sound: true,
        },
        trigger: {
          seconds: 7200, // 2 hours from now
        },
      });

      Alert.alert(' Scheduled', 'Notification set for 2 hours from now');
    } catch (error) {
      Alert.alert('Error', 'Failed to schedule notification');
    }
  };

  const scheduleDailyReminder = async () => {
    try {
      await Notifications.scheduleNotificationAsync({
        content: {
          title: " Good Morning!",
          body: "Don't forget to check your tasks today.",
        },
        trigger: {
          hour: 8,
          minute: 0,
          repeats: true, // Daily
        },
      });

      Alert.alert('Daily Reminder Set', 'You will receive a reminder every morning at 8 AM');
    } catch (error) {
      Alert.alert('Error', 'Failed to schedule daily reminder');
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Local Notifications</Text>

      <Pressable style={styles.button} onPress={sendImmediateNotification}>
        <Text style={styles.buttonText}>Send Immediate Notification</Text>
      </Pressable>

      <Pressable style={[styles.button, styles.scheduleButton]} onPress={scheduleFutureNotification}>
        <Text style={styles.buttonText}>Schedule in 2 Hours</Text>
      </Pressable>

      <Pressable style={[styles.button, styles.dailyButton]} onPress={scheduleDailyReminder}>
        <Text style={styles.buttonText}>Set Daily Morning Reminder</Text>
      </Pressable>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
    justifyContent: 'center',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    textAlign: 'center',
    marginBottom: 40,
  },

    backgroundColor: '#007AFF',
    padding: 18,
    borderRadius: 12,
    alignItems: 'center',
    marginVertical: 10,
  },
  scheduleButton: {
    backgroundColor: '#34C759',
  },
  dailyButton: {
    backgroundColor: '#FF9500',
  },
  buttonText: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },
});

export default LocalNotificationsScreen;
```

---

### **Key Options**

| Option              | Description |
|---------------------|-----------|
| `title`             | Notification title |

| `sound`             | Play sound |

## 5. Push Notifications

**Description**: Push notifications are sent from a server to the user’s device, even when the app is closed. They are essential for real-time updates like chat messages, order status, or promotions.

---

### **Step 1: Get Expo Push Token**
```jsx
import * as Notifications from 'expo-notifications';

const getPushToken = async () => {
  try {
    const tokenData = await Notifications.getExpoPushTokenAsync();
    console.log('Expo Push Token:', tokenData.data);
    
    // Send this token to your backend server for later use
    await sendTokenToBackend(tokenData.data);
    
    return tokenData.data;
  } catch (error) {
    console.error('Failed to get push token:', error);
    return null;
  }
};
```

---

### **Step 2: Send Push from Server**

You can use Expo’s push service or Firebase Cloud Messaging (FCM).

#### **Example using Expo Push API (from your backend)**
```jsx
// Backend (Node.js example)
const sendPushNotification = async (pushToken, message) => {
  const response = await fetch('https://exp.host/--/api/v2/push/send', {
    method: 'POST',
    headers: {
      'Accept': 'application/json',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      to: pushToken,
      title: message.title,
      body: message.body,
      data: message.data,
      sound: 'default',
    }),
  });

  return response.json();
};
```

---

### **Complete Flow**
1. User grants notification permission
2. App gets Expo Push Token
3. App sends token to your backend
4. Backend stores token associated with user
5. Backend sends push notification when needed
6. Device receives and displays notification

## 6. Firebase Cloud Messaging (FCM)

**Description**: Firebase Cloud Messaging (FCM) is Google’s reliable, cross-platform push notification service. It is widely used for sending notifications from a server to devices.

---

### **Setup Steps**

1. Create a Firebase project in the [Firebase Console](https://console.firebase.google.com)
2. Add your Android app and download `google-services.json`
3. For Expo, use EAS (Expo Application Services) or bare workflow
4. Install the Firebase Messaging library

---

### **Installation (Bare Workflow)**
```bash
npm install @react-native-firebase/messaging
```

---

### **Basic FCM Setup**
```jsx
import React, { useEffect } from 'react';
import messaging from '@react-native-firebase/messaging';
import { Alert } from 'react-native';

const FCMSetup = () => {
  // Request permission
  const requestUserPermission = async () => {
    const authStatus = await messaging().requestPermission();
    const enabled =
      authStatus === messaging.AuthorizationStatus.AUTHORIZED ||
      authStatus === messaging.AuthorizationStatus.PROVISIONAL;

    if (enabled) {
      console.log(' Notification permission granted');
      getFCMToken();
    } else {
      console.log(' Notification permission denied');
    }
  };

  // Get FCM Token
  const getFCMToken = async () => {
    try {
      const token = await messaging().getToken();
      console.log('FCM Token:', token);
      
      // Send token to your backend
      await sendTokenToBackend(token);
    } catch (error) {
      console.error('Failed to get FCM token:', error);
    }
  };

  useEffect(() => {
    requestUserPermission();

    // Handle notifications when app is in foreground
    const unsubscribe = messaging().onMessage(async remoteMessage => {
      Alert.alert(
        remoteMessage.notification.title,
        remoteMessage.notification.body
      );
    });

    return unsubscribe;
  }, []);

  return null; // This component is for setup only
};

export default FCMSetup;
```

---

### **Key Features of FCM**
- Cross-platform (Android & iOS)
- Reliable delivery
- Rich notifications (images, actions)
- Topic messaging
- Background message handling

## 7. Scheduling Notifications

**Description**: Scheduling notifications allows your app to deliver reminders, daily tips, or time-based alerts even when the app is not open.

---

### **Basic Complete Example**
```jsx
import React from 'react';
import { View, Pressable, Text, StyleSheet, Alert } from 'react-native';
import * as Notifications from 'expo-notifications';

const SchedulingScreen = () => {
  const scheduleDailyReminder = async () => {
    try {
      await Notifications.scheduleNotificationAsync({
        content: {
          title: " Study Time",
          body: "Time to continue your React Native course!",
          sound: true,
        },
        trigger: {
          hour: 9,
          minute: 0,
          repeats: true, // Daily at 9:00 AM
        },
      });

      Alert.alert(' Scheduled', 'Daily reminder set for 9:00 AM');
    } catch (error) {
      Alert.alert('Error', 'Failed to schedule notification');
    }
  };

  const scheduleIn30Minutes = async () => {
    try {
      await Notifications.scheduleNotificationAsync({
        content: {
          title: " Quick Break Reminder",
          body: "Take a short break and stretch!",

        trigger: {
          seconds: 1800, // 30 minutes
        },
      });

      Alert.alert('Scheduled', 'Notification set for 30 minutes from now');
    } catch (error) {
      Alert.alert('Error', 'Failed to schedule notification');
    }

  const cancelAllNotifications = async () => {
    await Notifications.cancelAllScheduledNotificationsAsync();
    Alert.alert('Cleared', 'All scheduled notifications have been canceled');

    <View style={styles.container}>

      <Pressable style={[styles.button, styles.dangerButton]} onPress={cancelAllNotifications}>
        <Text style={styles.buttonText}>Cancel All Notifications</Text>

};
```

## 8. Handling Notification Interactions

**Description**: Handling notification interactions allows your app to respond when the user taps on a notification — even if the app was closed or in the background.

---

### **Basic Complete Example**
```jsx
import React, { useEffect } from 'react';
import * as Notifications from 'expo-notifications';
import { Alert } from 'react-native';

const NotificationHandler = () => {
  useEffect(() => {
    // Handle notification when app is in foreground
    const foregroundSubscription = Notifications.addNotificationReceivedListener(notification => {
      console.log('Notification received in foreground:', notification);
    });

    // Handle user tapping on notification
    const responseSubscription = Notifications.addNotificationResponseReceivedListener(response => {
      const data = response.notification.request.content.data;
      console.log('User tapped notification. Data:', data);

      if (data.screen) {
        // Navigate to specific screen
        // navigation.navigate(data.screen, data.params);
        Alert.alert('Notification Tapped', `Opening screen: ${data.screen}`);
      }
    });

    return () => {
      foregroundSubscription.remove();
      responseSubscription.remove();
    };
  }, []);

  return null; // This component is for setup only
};

export default NotificationHandler;
```

---

### **Usage in App.js**
```jsx
const App = () => {
  return (
    <>
      <NotificationHandler />
      <MainNavigator />
    </>
  );
};
```

## 9. Production Tips for Notifications

**Description**: Implementing notifications in production requires careful attention to reliability, user experience, and performance.

---

### **Best Production Practices**

1. **Store Push Tokens Securely**
   - Save Expo/FCM tokens on your backend associated with user accounts
   - Handle token refresh (tokens can change)
   - Remove invalid tokens when delivery fails

2. **Handle Different App States**
   ```jsx
   // Foreground
   Notifications.addNotificationReceivedListener(...)

   // Background / Quit
   messaging().setBackgroundMessageHandler(...)
   ```

3. **Implement Notification Categories** (iOS)
   - Allow custom actions (Reply, Like, etc.)
   - Improve user engagement

4. **Use Rich Notifications**
   - Add images
   - Include action buttons
   - Use custom sounds

5. **Monitor Performance**
   - Track delivery rates
   - Monitor open rates
   - A/B test notification content
