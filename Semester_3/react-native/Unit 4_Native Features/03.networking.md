
# React Native – Networking

**Description**: Networking is a core part of most mobile applications. This module covers fetching data, uploading files, handling connectivity, and building offline-first features.

---

## 1. What is Networking?

Networking refers to communication between your React Native app and a backend server or third-party services.

### **Common Tasks**
- Fetching data (GET)
- Sending data (POST)
- Uploading files (images, documents)
- Downloading files (PDFs, reports)
- Real-time updates (WebSockets)
- Authentication (login, tokens)
- Caching responses

---

### **Why Networking Matters**
- Makes your app dynamic instead of static
- Enables data synchronization
- Supports user-generated content
- Allows integration with external services
- Powers most modern app features

---

### **Key Considerations**
- Error handling
- Loading states
- Offline support
- Security (HTTPS, tokens)
- Performance (caching, compression)

---

## 2. File Upload with FormData

**Description**: Uploading files (images, documents, videos) requires using `FormData`. This is the standard way to send binary files along with other form fields to your backend.

---

### **Basic Complete Example**
```jsx
import React, { useState } from 'react';
import { View, Image, Pressable, Text, StyleSheet, Alert } from 'react-native';
import * as ImagePicker from 'expo-image-picker';
import axios from 'axios';

const FileUploadScreen = () => {
  const [image, setImage] = useState(null);
  const [uploading, setUploading] = useState(false);

  const pickImage = async () => {
    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      allowsEditing: true,
      quality: 0.8,
    });

    if (!result.canceled) {
      setImage(result.assets[0].uri);
    }
  };

  const uploadImage = async () => {
    if (!image) {
      Alert.alert('No Image', 'Please select an image first');
      return;
    }

    setUploading(true);

    const formData = new FormData();
    formData.append('profileImage', {
      uri: image,
      name: 'profile-picture.jpg',
      type: 'image/jpeg',
    });
    formData.append('userId', '12345');
    formData.append('description', 'User profile picture');

    try {
      const response = await axios.post(
        'https://api.example.com/upload-profile',
        formData,
        {
          headers: {
            'Content-Type': 'multipart/form-data',
          },
        }
      );

      Alert.alert('Success', 'Profile picture uploaded successfully!');
      console.log('Response:', response.data);
    } catch (error) {
      Alert.alert('Upload Failed', error.message);
      console.error(error);
    } finally {
      setUploading(false);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Profile Picture Upload</Text>

      {image && <Image source={{ uri: image }} style={styles.preview} />}

      <Pressable style={styles.button} onPress={pickImage}>
        <Text style={styles.buttonText}> Pick Image</Text>
      </Pressable>

      <Pressable 
        style={[styles.button, styles.uploadButton]} 
        onPress={uploadImage} 
        disabled={uploading || !image}
      >
        <Text style={styles.buttonText}>
          {uploading ? 'Uploading...' : 'Upload to Server'}
        </Text>
      </Pressable>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
    alignItems: 'center',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 30,
  },
  preview: {
    width: 200,
    height: 200,
    borderRadius: 100,
    marginBottom: 30,
  },
  button: {
    backgroundColor: '#007AFF',
    padding: 16,
    borderRadius: 12,
    width: '80%',
    alignItems: 'center',
    marginVertical: 8,
  },
  uploadButton: {
    backgroundColor: '#34C759',
  },
  buttonText: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },
});

export default FileUploadScreen;
```

---

### **Key Points**
- Use `new FormData()` to create the form
- Append files with proper `uri`, `name`, and `type`
- Set `Content-Type: 'multipart/form-data'`
- Handle large files carefully (compression)

## 3. File Download with `expo-file-system`

**Description**: `expo-file-system` allows you to download files from the internet and save them locally on the device for offline access.

---

### **Installation**
```bash
npx expo install expo-file-system
```

---

### **Basic Complete Example**
```jsx
import React, { useState } from 'react';
import { View, Pressable, Text, StyleSheet, Alert } from 'react-native';
import * as FileSystem from 'expo-file-system';

const FileDownloadScreen = () => {
  const [downloading, setDownloading] = useState(false);
  const [progress, setProgress] = useState(0);

  const downloadFile = async () => {
    const fileUrl = 'https://example.com/sample.pdf';
    const fileName = fileUrl.split('/').pop() || 'document.pdf';
    const downloadPath = FileSystem.documentDirectory + fileName;

    setDownloading(true);
    setProgress(0);

    try {
      const downloadResumable = FileSystem.createDownloadResumable(
        fileUrl,
        downloadPath,
        {},
        (downloadProgress) => {
          const progressPercent = Math.round(
            (downloadProgress.totalBytesWritten / downloadProgress.totalBytesExpectedToWrite) * 100
          );
          setProgress(progressPercent);
        }
      );

      const { uri } = await downloadResumable.downloadAsync();
      
      Alert.alert(' Download Complete', `File saved to: ${uri}`);
      console.log('Downloaded to:', uri);
    } catch (error) {
      Alert.alert('Download Failed', error.message);
      console.error(error);
    } finally {
      setDownloading(false);
      setProgress(0);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>File Downloader</Text>

      <Pressable style={styles.button} onPress={downloadFile} disabled={downloading}>
        <Text style={styles.buttonText}>
          {downloading ? `Downloading... ${progress}%` : ' Download PDF'}
        </Text>
      </Pressable>

      {downloading && (
        <Text style={styles.progress}>Progress: {progress}%</Text>
      )}
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
    justifyContent: 'center',
    alignItems: 'center',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 40,
  },
  button: {
    backgroundColor: '#007AFF',
    padding: 18,
    borderRadius: 12,
    width: '80%',
    alignItems: 'center',
  },
  buttonText: {
    color: '#fff',
    fontSize: 17,
    fontWeight: '600',
  },
  progress: {
    marginTop: 20,
    fontSize: 16,
    color: '#666',
  },
});

export default FileDownloadScreen;
```

---

### **Key Methods**

| Method                          | Description |
|--------------------------------|-----------|
| `downloadAsync(url, path)`     | Simple download |
| `createDownloadResumable()`    | Download with progress tracking |
| `documentDirectory`            | Path for app documents |

## 4. NetInfo – Network Status Detection

**Description**: `@react-native-community/netinfo` allows your app to detect network connectivity status and changes in real-time.

---

### **Installation**
```bash
npx expo install @react-native-community/netinfo
```

---

### **Basic Complete Example**
```jsx
import React, { useState, useEffect } from 'react';
import { View, Text, StyleSheet, Alert } from 'react-native';
import NetInfo from '@react-native-community/netinfo';

const NetworkStatusScreen = () => {
  const [isConnected, setIsConnected] = useState(true);
  const [connectionType, setConnectionType] = useState('unknown');

  useEffect(() => {
    // Initial check
    const checkConnection = async () => {
      const state = await NetInfo.fetch();
      setIsConnected(state.isConnected);
      setConnectionType(state.type);
    };

    checkConnection();

    // Listen for changes
    const unsubscribe = NetInfo.addEventListener(state => {
      setIsConnected(state.isConnected);
      setConnectionType(state.type);

      if (!state.isConnected) {
        Alert.alert('Offline', 'You are currently offline. Some features may not work.');
      } else {
        console.log('Back online! Connection type:', state.type);
      }
    });

    return () => unsubscribe();
  }, []);

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Network Status</Text>

      <View style={[styles.statusContainer, isConnected ? styles.online : styles.offline]}>
        <Text style={styles.statusText}>
          {isConnected ? ' Online' : ' Offline'}
        </Text>
        <Text style={styles.connectionType}>
          Connection: {connectionType.toUpperCase()}
        </Text>
      </View>

      <Text style={styles.info}>
        {isConnected 
          ? 'You have internet access.' 
          : 'Please check your connection.'}
      </Text>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
    justifyContent: 'center',
    alignItems: 'center',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 40,
  },
  statusContainer: {
    padding: 30,
    borderRadius: 20,
    alignItems: 'center',
    marginBottom: 30,
  },
  online: {
    backgroundColor: '#e6f4ea',
  },
  offline: {
    backgroundColor: '#fce8e6',
  },
  statusText: {
    fontSize: 28,
    fontWeight: 'bold',
  },
  connectionType: {
    marginTop: 12,
    fontSize: 16,
    color: '#666',
  },
  info: {
    fontSize: 16,
    textAlign: 'center',
    color: '#666',
  },
});

export default NetworkStatusScreen;
```

---

### **Key Methods**

| Method                     | Description |
|---------------------------|-----------|
| `NetInfo.fetch()`         | Get current network state |
| `NetInfo.addEventListener()` | Listen for network changes |

---

### **Useful Properties**
- `isConnected`
- `type` (wifi, cellular, none, etc.)
- `isInternetReachable`
- `details` (IP address, strength, etc.)

## 5. Offline-First Development

**Description**: Offline-First Development means your app works well even without internet. It caches data, queues failed requests, and syncs when connectivity returns.

---

### **Key Strategies**

1. **Cache API responses** (MMKV or AsyncStorage)
2. **Queue failed requests** for later retry
3. **Show cached data** when offline
4. **Retry automatically** when back online

---

### **Basic Complete Example**
```jsx
import React, { useState, useEffect } from 'react';
import { View, Text, FlatList, StyleSheet } from 'react-native';
import NetInfo from '@react-native-community/netinfo';
import AsyncStorage from '@react-native-async-storage/async-storage';

const OfflineFirstScreen = () => {
  const [isOnline, setIsOnline] = useState(true);
  const [data, setData] = useState([]);
  const [cachedData, setCachedData] = useState([]);

  // Monitor network status
  useEffect(() => {
    const unsubscribe = NetInfo.addEventListener(state => {
      const wasOnline = isOnline;
      setIsOnline(state.isConnected);

      if (!wasOnline && state.isConnected) {
        // Back online - sync data
        syncOfflineData();
      }
    });

    return unsubscribe;
  }, [isOnline]);

  // Load cached data on mount
  useEffect(() => {
    loadCachedData();
  }, []);

  const loadCachedData = async () => {
    try {
      const cached = await AsyncStorage.getItem('cachedUsers');
      if (cached) {
        setCachedData(JSON.parse(cached));
        setData(JSON.parse(cached)); // Show cached data
      }
    } catch (error) {
      console.error(error);
    }
  };

  const fetchAndCacheData = async () => {
    try {
      const response = await fetch('https://jsonplaceholder.typicode.com/users');
      const users = await response.json();
      
      setData(users);
      await AsyncStorage.setItem('cachedUsers', JSON.stringify(users));
    } catch (error) {
      console.error('Failed to fetch:', error);
      // Show cached data if available
    }
  };

  const syncOfflineData = async () => {
    console.log(' Syncing offline data...');
    await fetchAndCacheData();
  };

  return (
    <View style={styles.container}>
      <View style={[styles.statusBar, isOnline ? styles.online : styles.offline]}>
        <Text style={styles.statusText}>
          {isOnline ? ' Online' : ' Offline - Showing cached data'}
        </Text>
      </View>

      <FlatList
        data={data.length > 0 ? data : cachedData}
        keyExtractor={item => item.id.toString()}
        renderItem={({ item }) => (
          <View style={styles.item}>
            <Text style={styles.name}>{item.name}</Text>
            <Text style={styles.email}>{item.email}</Text>
          </View>
        )}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8f9fa',
  },
  statusBar: {
    padding: 12,
    alignItems: 'center',
  },
  online: {
    backgroundColor: '#e6f4ea',
  },
  offline: {
    backgroundColor: '#fce8e6',
  },
  statusText: {
    fontWeight: '600',
  },
  item: {
    padding: 16,
    backgroundColor: '#fff',
    marginVertical: 6,
    marginHorizontal: 16,
    borderRadius: 12,
  },
  name: {
    fontSize: 17,
    fontWeight: '600',
  },
  email: {
    fontSize: 15,
    color: '#666',
    marginTop: 4,
  },
});

export default OfflineFirstScreen;
```

---

### **Core Strategies for Offline-First**

1. **Cache aggressively**
2. **Queue failed mutations**
3. **Show cached data immediately**
4. **Sync when back online**
5. **Provide clear offline indicators**
