#  📱 React Hooks in React Native

---

#  Learning Objectives

After completing this module, you will be able to:

- Understand why Hooks are used
- Manage state with `useState`
- Handle side effects with `useEffect`
- Access mutable values using `useRef`
- Optimize performance using `useMemo`
- Prevent unnecessary re-renders using `useCallback`
- Share global data using `useContext`
- Build reusable Custom Hooks

---

# 1️⃣ What are Hooks?

Hooks are special React functions that let functional components use:

- State
- Lifecycle features
- References
- Context
- Performance optimizations

React Native uses the **same Hooks API** as React.js.

---

# 2️⃣ useState

`useState` is used to create and update component state.

```jsx
import { useState } from "react";
import { View, Text, Pressable } from "react-native";

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <View>
      <Text>{count}</Text>

      <Pressable onPress={() => setCount(count + 1)}>
        <Text>Increase</Text>
      </Pressable>
    </View>
  );
}
```

### Use Cases

- Counter
- Toggle
- Forms
- API Data
- Theme

---

# 3️⃣ useEffect

Used for side effects.

```jsx
useEffect(() => {
  console.log("Component Mounted");
}, []);
```

### Common Use Cases

- API calls
- Event listeners
- Timers
- Subscriptions

### Dependency Array

```jsx
useEffect(() => {}, []);
useEffect(() => {}, [user]);
useEffect(() => {});
```

---

# 4️⃣ useRef

Stores mutable values without causing re-renders.

```jsx
const inputRef = useRef(null);

<TextInput ref={inputRef} />

inputRef.current.focus();
```

### Use Cases

- Focus input
- Store timer IDs
- Access component instances

---

# 5️⃣ useMemo

Caches expensive calculations.

```jsx
const total = useMemo(() => {
  return items.reduce((sum, item) => sum + item.price, 0);
}, [items]);
```

### Use When

- Heavy calculations
- Filtered lists
- Sorted data

---

# 6️⃣ useCallback

Memoizes functions.

```jsx
const handlePress = useCallback(() => {
  console.log("Pressed");
}, []);
```

Useful when passing callbacks to memoized child components.

---

# 7️⃣ useContext

Used to share global data without prop drilling.

```jsx
const ThemeContext = createContext();

const theme = useContext(ThemeContext);
```

### Common Use Cases

- Authentication
- Theme
- Language
- User Profile

---

# 8️⃣ Custom Hooks

Custom Hooks allow reusable logic.

```jsx
import { useState } from "react";

export function useToggle(initial = false) {
  const [value, setValue] = useState(initial);

  const toggle = () => setValue(!value);

  return [value, toggle];
}
```

Usage:

```jsx
const [isOpen, toggle] = useToggle();
```

---

#  Hook Selection Guide

| Requirement | Hook |
|------------|------|
| State | useState |
| Side Effects | useEffect |
| Mutable Reference | useRef |
| Expensive Calculation | useMemo |
| Memoized Function | useCallback |
| Global State | useContext |
| Reusable Logic | Custom Hook |

---

#  Performance Tips

- Don't overuse `useMemo` and `useCallback`
- Keep dependency arrays correct
- Split large components
- Create reusable hooks
- Avoid unnecessary state

---

#  Common Mistakes

- Missing dependencies in `useEffect`
- Infinite loops
- Using `useRef` as state
- Overusing `useMemo`
- Creating Hooks inside conditions
- Calling Hooks inside loops

#  Key Takeaways

- Hooks are the foundation of modern React Native development.
- `useState` manages state.
- `useEffect` handles side effects.
