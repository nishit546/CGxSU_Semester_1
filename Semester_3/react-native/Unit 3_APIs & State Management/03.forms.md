
# React Native – Forms

**Description**: Forms are one of the most common and important parts of any mobile application. This module covers the best modern tools and patterns for building forms in React Native.

---

## 1. Why We Need Better Form Libraries?

Building forms with only native `TextInput` + `useState` works for very simple cases, but quickly becomes difficult as your form grows.

### **Common Problems**
- Managing many `useState` hooks
- Handling real-time validation
- Showing and clearing errors cleanly
- Resetting forms after submission
- Watching form values
- Handling complex validation logic
- Poor performance with many fields
- Difficult to maintain and test

---

### **Benefits of Modern Form Libraries**

Modern solutions like **React Hook Form** and **Formik** solve these problems elegantly by:
- Reducing boilerplate code
- Providing built-in validation support
- Offering excellent performance
- Making form state management predictable
- Supporting easy reset, watch, and submit patterns

---

### **Popular Form Solutions (2025)**

| Library            | Performance | Bundle Size | Recommendation |
|--------------------|-------------|-------------|----------------|
| **React Hook Form** | Excellent   | Small       | **Best choice** |
| **Formik**         | Good        | Medium      | Still popular |
| Native + useState  | Poor        | None        | Only for tiny forms |

---

## 2. React Hook Form (Recommended)

**Description**: React Hook Form is currently the most popular and performant form library in the React ecosystem. It uses uncontrolled components and minimizes re-renders for excellent performance.

---

### **Key Benefits**
- Minimal re-renders (best performance)
- Small bundle size
- Easy integration with Yup / Zod
- Great developer experience
- Built-in validation support
- Works excellently with React Native

---

### **Installation**
```bash
npm install react-hook-form
# For validation with Yup
npm install @hookform/resolvers yup
```

---

### **Basic Complete Example**
```jsx
import React from 'react';
import { View, TextInput, Pressable, Text, StyleSheet } from 'react-native';
import { useForm } from 'react-hook-form';

const LoginForm = () => {
  const {
    register,
    handleSubmit,
    setValue,
    formState: { errors },
  } = useForm();

  const onSubmit = (data) => {
    console.log('Form Data:', data);
    // Call API here
  };

  return (
    <View style={styles.container}>
      <TextInput
        style={styles.input}
        placeholder="Email Address"
        keyboardType="email-address"
        onChangeText={(text) => setValue('email', text)}
      />
      {errors.email && <Text style={styles.error}>{errors.email.message}</Text>}

      <TextInput
        style={styles.input}
        placeholder="Password"
        secureTextEntry
        onChangeText={(text) => setValue('password', text)}
      />
      {errors.password && <Text style={styles.error}>{errors.password.message}</Text>}

      <Pressable style={styles.button} onPress={handleSubmit(onSubmit)}>
        <Text style={styles.buttonText}>Login</Text>
      </Pressable>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    padding: 20,
  },
  input: {
    backgroundColor: '#fff',
    borderRadius: 12,
    padding: 16,
    fontSize: 16,
    marginBottom: 12,
    borderWidth: 1,
    borderColor: '#ddd',
  },
  button: {
    backgroundColor: '#007AFF',
    padding: 16,
    borderRadius: 12,
    alignItems: 'center',
    marginTop: 20,
  },
  buttonText: {
    color: '#fff',
    fontSize: 17,
    fontWeight: '600',
  },
  error: {
    color: '#ff3b30',
    fontSize: 13,
    marginBottom: 12,
    marginLeft: 4,
  },
});

export default LoginForm;
```

### **With Yup Validation (Recommended)**
```jsx
import { yupResolver } from '@hookform/resolvers/yup';
import * as yup from 'yup';

const schema = yup.object({
  email: yup.string()
    .email('Please enter a valid email')
    .required('Email is required'),

const { register, handleSubmit, formState: { errors } } = useForm({

  mode: 'onBlur',           // Validate on blur
});
```

### **Why React Hook Form is Recommended**
- Excellent performance
- Small bundle size
- Easy to use with validation libraries
- Great TypeScript support
- Active development and community

## 3. Formik (Still Popular)

**Description**: Formik is a well-established, battle-tested form library for React and React Native. It has excellent documentation and is still widely used in many production apps.

---

### **Installation**
```bash
npm install formik yup
# or
yarn add formik yup
```

---

### **Basic Complete Example**
```jsx
import React from 'react';
import { View, TextInput, Pressable, Text, StyleSheet } from 'react-native';
import { Formik } from 'formik';
import * as Yup from 'yup';

const validationSchema = Yup.object({
  email: Yup.string()
    .email('Please enter a valid email address')
    .required('Email is required'),
  password: Yup.string()
    .min(6, 'Password must be at least 6 characters')
    .required('Password is required'),
  rememberMe: Yup.boolean(),
});

const LoginForm = () => {
  return (
    <Formik
      initialValues={{ email: '', password: '', rememberMe: false }}
      validationSchema={validationSchema}
      onSubmit={(values, { resetForm }) => {
        console.log('Form submitted:', values);
        // Call API here
        resetForm(); // Optional: reset form after submit
      }}
    >
      {({ handleChange, handleSubmit, values, errors, touched, setFieldValue }) => (
        <View style={styles.container}>
          <TextInput
            style={styles.input}
            placeholder="Email Address"
            keyboardType="email-address"
            onChangeText={handleChange('email')}
            value={values.email}
          />
          {touched.email && errors.email && (
            <Text style={styles.errorText}>{errors.email}</Text>
          )}

          <TextInput
            style={styles.input}
            placeholder="Password"
            secureTextEntry
            onChangeText={handleChange('password')}
            value={values.password}
          />
          {touched.password && errors.password && (
            <Text style={styles.errorText}>{errors.password}</Text>
          )}

          <Pressable style={styles.button} onPress={handleSubmit}>
            <Text style={styles.buttonText}>Login</Text>
          </Pressable>
        </View>
      )}
    </Formik>
  );
};

const styles = StyleSheet.create({
  container: {
    padding: 20,
  },
  input: {
    backgroundColor: '#fff',
    borderRadius: 12,
    padding: 16,
    fontSize: 16,
    marginBottom: 8,
    borderWidth: 1,
    borderColor: '#ddd',
  },
  button: {
    backgroundColor: '#007AFF',
    padding: 16,
    borderRadius: 12,
    alignItems: 'center',
    marginTop: 20,
  },
  buttonText: {
    color: '#fff',
    fontSize: 17,
    fontWeight: '600',
  },
  errorText: {
    color: '#ff3b30',
    fontSize: 13,
    marginBottom: 12,
    marginLeft: 4,
  },
});

export default LoginForm;
```

---

### **Key Formik Props**

| Prop                | Description |
|---------------------|-----------|
| `initialValues`     | Initial form values |
| `validationSchema`  | Yup validation schema |
| `onSubmit`          | Function called on submit |
| `values`            | Current form values |
| `errors`            | Validation errors |
| `touched`           | Fields that have been touched |

---

**Recommendation**:  
While **React Hook Form** is currently more popular for new projects due to better performance, **Formik** remains a solid choice with excellent documentation and a large community.

## 4. Yup – Schema Validation

**Description**: Yup is the most popular schema validation library in the React ecosystem. It works excellently with both **React Hook Form** and **Formik**.

---

### **Basic Complete Example**
```jsx
import * as yup from 'yup';

export const signupSchema = yup.object().shape({
  fullName: yup
    .string()
    .required('Full name is required')
    .min(3, 'Name must be at least 3 characters')
    .max(50, 'Name is too long'),

  email: yup
    .string()
    .email('Please enter a valid email')
    .required('Email is required'),

  password: yup
    .string()
    .min(8, 'Password must be at least 8 characters')
    .matches(/[A-Z]/, 'Password must contain at least one uppercase letter')
    .matches(/[0-9]/, 'Password must contain at least one number')
    .matches(/[@$!%*?&]/, 'Password must contain at least one special character')
    .required('Password is required'),

  confirmPassword: yup
    .string()
    .oneOf([yup.ref('password')], 'Passwords must match')
    .required('Please confirm your password'),

  age: yup
    .number()
    .min(18, 'You must be at least 18 years old')
    .required('Age is required'),

  acceptTerms: yup
    .boolean()
    .oneOf([true], 'You must accept the terms and conditions'),
});
```

---

### **Usage with React Hook Form**
```jsx
import { useForm } from 'react-hook-form';
import { yupResolver } from '@hookform/resolvers/yup';
import { signupSchema } from '../validation/signupSchema';

const SignupForm = () => {
  const { register, handleSubmit, formState: { errors } } = useForm({
    resolver: yupResolver(signupSchema),
    mode: 'onBlur',           // Validate on blur
  });

  const onSubmit = (data) => {
    console.log('Form submitted:', data);
  };

  return (
    <View>
      <TextInput
        placeholder="Full Name"
        onChangeText={(text) => register('fullName').onChange(text)}
      />
      {errors.fullName && <Text style={styles.error}>{errors.fullName.message}</Text>}

      {/* Other fields... */}

      <Pressable onPress={handleSubmit(onSubmit)}>
        <Text>Sign Up</Text>
      </Pressable>
    </View>
  );
};
```

---

### **Common Validation Rules**

| Rule                        | Example |
|----------------------------|--------|
| Required                   | `.required('This field is required')` |
| Email                      | `.email('Invalid email')` |
| Min Length                 | `.min(8, 'Too short')` |
| Max Length                 | `.max(50, 'Too long')` |
| Matches Regex              | `.matches(/[A-Z]/, 'Must have uppercase')` |
| Confirm Password           | `.oneOf([yup.ref('password')], 'Passwords must match')` |
| Number Range               | `.min(18).max(100)` |
| Boolean                    | `.oneOf([true], 'Must accept')` |

## 5. React Hook Form vs Formik

**Description**: Both React Hook Form and Formik are popular libraries for handling forms in React Native. React Hook Form has become the preferred choice in recent years due to its superior performance.

---

### **Comparison Table**

| Feature                    | **React Hook Form**                          | **Formik**                              |
|---------------------------|---------------------------------------------|-----------------------------------------|
| Performance               | **Excellent** (Minimal re-renders)          | Good                                    |
| Bundle Size               | Smaller                                     | Larger                                  |
| Learning Curve            | Easy                                        | Easy                                    |
| Validation                | Excellent (with Yup/Zod)                    | Excellent                               |
| Controlled vs Uncontrolled| Uncontrolled (better performance)           | Controlled                              |
| Community (2025+)         | Very Strong & Growing                       | Mature but slower growth                |
| Native Support            | Excellent                                   | Good                                    |

**Recommendation**: Use **React Hook Form + Yup/Zod** for new projects.

---

### **Basic Example: React Hook Form**
```jsx
import { useForm } from 'react-hook-form';
import { yupResolver } from '@hookform/resolvers/yup';
import * as yup from 'yup';

const schema = yup.object({
  email: yup.string().email('Invalid email').required('Email is required'),
  password: yup.string().min(6).required(),
});

const LoginForm = () => {
  const { register, handleSubmit, formState: { errors }, setValue } = useForm({
    resolver: yupResolver(schema),
  });

  const onSubmit = (data) => {
    console.log(data);
  };

  return (
    <View>
      <TextInput
        placeholder="Email"
        onChangeText={(text) => setValue('email', text)}
      />
      {errors.email && <Text style={styles.error}>{errors.email.message}</Text>}

      <TextInput
        placeholder="Password"
        secureTextEntry
        onChangeText={(text) => setValue('password', text)}
      />
      {errors.password && <Text style={styles.error}>{errors.password.message}</Text>}

      <Pressable onPress={handleSubmit(onSubmit)}>
        <Text>Login</Text>
      </Pressable>
    </View>
  );
};
```

---

### **When to Choose Which?**

**Choose React Hook Form when**:
- You want maximum performance
- Building large forms
- Prefer minimal re-renders
- Using modern React patterns

**Choose Formik when**:
- Working on legacy projects already using Formik
- Need very advanced form features
- Team is more familiar with Formik

---

**Final Recommendation (2025+)**:  
**React Hook Form + Zod** is currently the best choice for most new React Native projects.
