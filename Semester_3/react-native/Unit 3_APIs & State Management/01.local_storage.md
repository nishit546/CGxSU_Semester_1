
# React Native – Local Storage

**Description**: Local storage allows your app to save data directly on the user’s device. This enables offline functionality, faster performance, and better user experience by remembering preferences and state.

---

## 1. What is Local Storage?

Local storage saves data on the device so it remains available even when the user is offline or the app is closed.

### **Common Use Cases**
- User authentication (access tokens)
- App theme preference (light/dark mode)
- Language selection
- Onboarding completion status
- Cached API responses
- User settings and preferences
- Recently viewed items
- Shopping cart (temporary)

---

### **Why Use Local Storage?**
- **Offline Support** – App works without internet
- **Performance** – Faster than fetching from server
- **Personalization** – Remembers user choices
- **Better UX** – No need to login every time

---

### **Available Storage Solutions in React Native**

| Storage            | Speed      | Security     | Recommended For |
|--------------------|------------|--------------|-----------------|
| **MMKV**           | Very Fast  | Basic        | General data (new projects) |
| **AsyncStorage**   | Moderate   | None         | Simple data, legacy |
| **SecureStore**    | Moderate   | High         | Tokens & credentials |
| **Redux Persist**  | Good       | Depends      | Global state |

---

## 2. Storage Options Comparison

**Description**: Choosing the right storage solution is crucial for performance, security, and maintainability in React Native apps. Here's a clear comparison of the most popular options.

---

### **Storage Options Comparison**

| Storage          | Speed          | Async/Sync     | Secure     | Encryption | Best For                              | Recommendation |
|------------------|----------------|----------------|------------|------------|---------------------------------------|----------------|
| **MMKV**         | **Very Fast**  | Synchronous    | No         | Optional   | General data, preferences, user settings | **Best for new projects** |
| **AsyncStorage** | Moderate       | Asynchronous   | No         | No         | Simple data, legacy code              | Still acceptable |
| **SecureStore**  | Moderate       | Asynchronous   | **Yes**    | Yes        | Tokens, API keys, credentials         | **Must use for sensitive data** |
| **Redux Persist**| Depends        | Async          | No         | No         | Global state persistence              | For Redux apps |

---

### **When to Use Which?**

**Use MMKV when:**
- You need maximum performance
- Storing non-sensitive data (theme, user preferences, settings, cache)
- Building new projects

**Use SecureStore when:**
- Storing authentication tokens (JWT)
- Saving API keys or sensitive credentials
- Handling any private user information

**Use AsyncStorage when:**
- Working on legacy projects
- Need simple key-value storage
- For small amounts of data

---

### **Practical Recommendation (Best Practice)**
```jsx
// For normal data
import { MMKV } from 'react-native-mmkv';
export const storage = new MMKV();

// For sensitive data (tokens)
import * as SecureStore from 'expo-secure-store';

// Example usage
const saveUserData = async (user) => {
  storage.setObject('userProfile', user);           // Fast + non-sensitive
};

const saveAuthToken = async (token) => {
  await SecureStore.setItemAsync('accessToken', token); // Secure
};
```

---

### **Performance & Security Summary**

| Requirement               | Recommended Solution      |
|---------------------------|---------------------------|
| Best Performance          | MMKV                      |
| Maximum Security          | SecureStore               |
| Legacy Code Compatibility | AsyncStorage              |
| Large Complex State       | Redux + MMKV / Redux Persist |

Most modern apps use **MMKV** for general storage + **SecureStore** for authentication tokens.

## 3. react-native-mmkv (Recommended for New Projects)

**Description**: MMKV is a high-performance, cross-platform key-value storage solution. It is currently one of the **fastest** storage options available in React Native and is recommended for new projects.

---

### **Installation**
```bash
npm install react-native-mmkv
# or
yarn add react-native-mmkv
```

---

### **Basic Complete Example**
```jsx
// storage.js
import { MMKV } from 'react-native-mmkv';

// Create a global storage instance
export const storage = new MMKV({
  id: 'app-storage',        // Optional: unique ID
  encryptionKey: 'your-secret-key', // Optional encryption
});

// storageService.js
export const StorageService = {
  // Save primitive values
  setString: (key, value) => storage.set(key, value),
  setBoolean: (key, value) => storage.set(key, value),
  setNumber: (key, value) => storage.set(key, value),

  // Get values
  getString: (key) => storage.getString(key),
  getBoolean: (key) => storage.getBoolean(key),
  getNumber: (key) => storage.getNumber(key),

  // Objects
  setObject: (key, value) => {
    storage.set(key, JSON.stringify(value));
  },

  getObject: (key) => {
    const value = storage.getString(key);
    return value ? JSON.parse(value) : null;
  },

  // Delete
  remove: (key) => storage.delete(key),
  clearAll: () => storage.clearAll(),
};
```

## 4. AsyncStorage (Still Important)

**Description**: `AsyncStorage` is a simple, unencrypted, asynchronous key-value storage system for React Native. It is still widely used for non-sensitive data like user preferences, settings, and app state.

> **Note**: For sensitive data (tokens, passwords), use `Expo SecureStore` instead.

---

### **Installation (Expo)**
```bash
npx expo install @react-native-async-storage/async-storage
```

---

## 5. Expo SecureStore (For Sensitive Data)

**Description**: `SecureStore` is an Expo module that provides secure, encrypted storage for sensitive data like JWT tokens, API keys, passwords, and user credentials.

> **Important**: Never store sensitive information in `AsyncStorage`, `MMKV`, or plain files.

---

### **Installation**
```bash
npx expo install expo-secure-store
```

## 6. Authentication Token Storage (Best Practice)

**Description**: Securely storing authentication tokens is critical for maintaining user sessions. The best practice is to use `Expo SecureStore` for tokens and a fast storage solution (like MMKV or AsyncStorage) for simple flags.

---

### **Recommended Implementation**
```jsx
// auth/storage.js
import * as SecureStore from 'expo-secure-store';
import AsyncStorage from '@react-native-async-storage/async-storage';

export const AuthStorage = {
  // Save token (Secure)
  saveToken: async (token) => {
    await SecureStore.setItemAsync('accessToken', token);
  },

  // Get token
  getToken: async () => {
    return await SecureStore.getItemAsync('accessToken');
  },

  // Save login flag (Fast)
  setLoggedIn: async (value = true) => {
    await AsyncStorage.setItem('isLoggedIn', JSON.stringify(value));
  },

  // Check login status
  isLoggedIn: async () => {
    try {
      const token = await SecureStore.getItemAsync('accessToken');
      const flag = await AsyncStorage.getItem('isLoggedIn');
      return !!token && flag === 'true';
    } catch (error) {
      return false;
    }
  },

  // Save user data
  saveUser: async (user) => {
    await AsyncStorage.setItem('userData', JSON.stringify(user));
  },

  // Get user data
  getUser: async () => {
    const user = await AsyncStorage.getItem('userData');
    return user ? JSON.parse(user) : null;
  },

  // Logout (Clear everything)
  logout: async () => {
    await SecureStore.deleteItemAsync('accessToken');
    await AsyncStorage.multiRemove(['isLoggedIn', 'userData']);
  },
};
```

---

### **Usage in Login Flow**
```jsx
const handleLogin = async (credentials) => {
  try {
    const response = await api.post('/auth/login', credentials);
    const { token, user } = response.data;

    // Save securely
    await AuthStorage.saveToken(token);
    await AuthStorage.saveUser(user);
    await AuthStorage.setLoggedIn(true);

    // Navigate to main app
    navigation.reset({
      index: 0,
      routes: [{ name: 'Main' }],
    });
  } catch (error) {
    Alert.alert('Login Failed', error.response?.data?.message || 'Invalid credentials');
  }
};
```

---

### **Usage in App Start (Auth Check)**
```jsx
// App.js
const [isAuthenticated, setIsAuthenticated] = useState(false);
const [loading, setLoading] = useState(true);

useEffect(() => {
  const checkAuth = async () => {
    const loggedIn = await AuthStorage.isLoggedIn();
    setIsAuthenticated(loggedIn);
    setLoading(false);
  };

  checkAuth();
}, []);

if (loading) return <SplashScreen />;

return (
  <NavigationContainer>
    {isAuthenticated ? <MainNavigator /> : <AuthNavigator />}
  </NavigationContainer>
);
```

---

### **Best Practices Summary**

| Item                | Storage Method       | Reason |
|---------------------|----------------------|--------|
| Access Token        | `SecureStore`        | Encrypted & secure |
| Refresh Token       | `SecureStore`        | Highly sensitive |
| User Profile        | `AsyncStorage`       | Non-sensitive |
| Login Flag          | `AsyncStorage`       | Fast check |
| Theme Preference    | `AsyncStorage`       | Simple value |

---

**Security Tips**:
- Never store plain passwords
- Implement token refresh logic
- Clear all storage on logout
- Use short-lived access tokens

## 7. Theme Persistence

**Description**: Theme Persistence saves the user's preferred theme (Light / Dark) so it remains consistent even after the app is closed and reopened.

---

### **Basic Complete Example**
```jsx
// context/ThemeContext.js
import React, { createContext, useContext, useState, useEffect } from 'react';
import AsyncStorage from '@react-native-async-storage/async-storage';

const ThemeContext = createContext();

export const ThemeProvider = ({ children }) => {
  const [theme, setTheme] = useState('light');

  // Load saved theme on app start
  useEffect(() => {
    loadSavedTheme();
  }, []);

  const loadSavedTheme = async () => {
    try {
      const savedTheme = await AsyncStorage.getItem('userTheme');
      if (savedTheme) {
        setTheme(savedTheme);
      }
    } catch (error) {
      console.error('Failed to load theme:', error);
    }
  };

  const toggleTheme = async () => {
    const newTheme = theme === 'light' ? 'dark' : 'light';
    setTheme(newTheme);
    
    try {
      await AsyncStorage.setItem('userTheme', newTheme);
    } catch (error) {
      console.error('Failed to save theme:', error);
    }
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
};

export const useTheme = () => useContext(ThemeContext);
```

### **Usage in App**
```jsx
// App.js
import { ThemeProvider } from './context/ThemeContext';

const App = () => {
  return (
    <ThemeProvider>
      <MainNavigator />
    </ThemeProvider>
  );
};
```

---

### **Usage in Components**
```jsx
const SettingsScreen = () => {
  const { theme, toggleTheme } = useTheme();

  return (
    <View style={[styles.container, theme === 'dark' && styles.darkContainer]}>
      <Text style={[styles.text, theme === 'dark' && styles.darkText]}>
        Current Theme: {theme.toUpperCase()}
      </Text>

      <Pressable style={styles.button} onPress={toggleTheme}>
        <Text style={styles.buttonText}>
          Switch to {theme === 'light' ? 'Dark' : 'Light'} Mode
        </Text>
      </Pressable>
    </View>
  );
};
```

---

### **Key Methods**

| Action                    | Code |
|--------------------------|------|
| Save Theme               | `AsyncStorage.setItem('userTheme', 'dark')` |
| Load Theme               | `AsyncStorage.getItem('userTheme')` |
| Default to 'light'       | `savedTheme || 'light'` |

---

### **Advanced: System Theme Sync**
```jsx

    const saved = await AsyncStorage.getItem('userTheme');
    if (saved) {
      setTheme(saved);
    } else {
      // Follow system theme

      setTheme(systemTheme);
    }
  };
  loadTheme();
}, []);
```

## 8. Choosing the Right Storage

**Description**: Selecting the correct storage solution based on the type of data is crucial for performance, security, and maintainability in React Native apps.

---

### **Storage Decision Guide**

| Type of Data                    | Recommended Storage       | Reason |
|--------------------------------|---------------------------|--------|
| **User preferences** (theme, language) | **MMKV**                  | Very fast, simple key-value |
| **App settings**               | **MMKV**                  | High performance |
| **Cached data** (API responses) | **MMKV**                  | Fast read/write |
| **Recently viewed items**      | **MMKV**                  | Good for small lists |
| **Access Token**               | **SecureStore**           | Encrypted & secure |
| **Refresh Token**              | **SecureStore**           | Highly sensitive |
| **User Profile** (non-sensitive) | **MMKV** or AsyncStorage | Balance of speed & simplicity |
| **Passwords**                  | **Never store locally**   | Use proper auth flow |

---

### **Decision Flowchart**

1. **Is the data sensitive?** (tokens, keys, credentials)
   - → **Use SecureStore**

2. **Is performance critical?** (frequent reads/writes)
   - → **Use MMKV**

3. **Working with legacy code?**
   - → **Use AsyncStorage**

4. **Storing complex global state?**
   - → Consider **Redux Persist** + MMKV

---

### **Practical Example**
```jsx
import { storage } from '../storage';           // MMKV
import * as SecureStore from 'expo-secure-store';

// Non-sensitive data
const savePreferences = (preferences) => {
  storage.setObject('userPreferences', preferences);
};

// Sensitive data
const saveAuthToken = async (token) => {
  await SecureStore.setItemAsync('accessToken', token);
};

// Reading on app start
const initializeApp = async () => {
  const preferences = storage.getObject('userPreferences');
  const token = await SecureStore.getItemAsync('accessToken');
  
  // ... initialize app state
};
```

---

### **Final Recommendations**

- **New Projects** → **MMKV** (general) + **SecureStore** (auth)
- **Legacy Projects** → **AsyncStorage** + **SecureStore**
- **Never store** plain passwords or highly sensitive data locally
