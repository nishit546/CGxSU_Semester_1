
# React Native – API Integration

**Description**: API Integration is the process of connecting your React Native app to a backend server to fetch, send, update, and delete data. It is one of the most essential skills for building real-world mobile applications.

---

## 1. What is an API?

**API (Application Programming Interface)** is a way for your React Native app to communicate with a backend server (or third-party services).

### **Typical Flow**
1. App sends an **HTTP request** (GET, POST, PUT, etc.)
2. Server processes the request and performs operations
3. Server returns a **JSON response**
4. App receives the data and updates the UI

---

### **Real-World Examples**
- Fetching user profile
- Loading product list from e-commerce backend
- Submitting a form (signup, feedback)
- Posting a new tweet / status
- Deleting a post
- Uploading photos

---

### **Common API Response Formats**
- JSON (most common)
- RESTful APIs
- GraphQL (alternative)

---

### **Basic Structure of an API Call**
```jsx
const fetchData = async () => {
  try {
    const response = await fetch('https://api.example.com/data');
    const json = await response.json();
    console.log(json);
  } catch (error) {
    console.error('API Error:', error);
  }
};
```

---

**Why API Integration Matters?**
- Makes your app dynamic instead of static
- Connects frontend with backend/database
- Enables real-time features
- Allows authentication & user management
- Powers almost all modern mobile apps

---

## 2. HTTP Methods (CRUD Operations)

**Description**: HTTP methods define the type of action you want to perform on a resource. They form the foundation of RESTful APIs and correspond to CRUD (Create, Read, Update, Delete) operations.

---

### **HTTP Methods Overview**

| Method   | Purpose                  | CRUD Operation | Example Use Case                  | Safe* | Idempotent* |
|----------|--------------------------|----------------|-----------------------------------|-------|-------------|
| **GET**  | Fetch data               | Read           | Get list of users / product details | Yes   | Yes         |
| **POST** | Create new data          | Create         | Register user, create post        | No    | No          |
| **PUT**  | Replace entire resource  | Update         | Update full user profile          | No    | Yes         |
| **PATCH**| Update partial data      | Update         | Change only email or status       | No    | No          |
| **DELETE**| Delete data             | Delete         | Remove a post or user             | No    | Yes         |

> *Safe = Doesn't change server state  
> *Idempotent = Repeating the call doesn't change the result

---

### **Basic Examples**

#### **1. GET Request**
```jsx
const getUsers = async () => {
  const response = await fetch('https://api.example.com/users');
  const data = await response.json();
  return data;
};
```

#### **2. POST Request**
```jsx
const createUser = async (newUser) => {
  const response = await fetch('https://api.example.com/users', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(newUser),
  });
  return await response.json();
};
```

#### **3. PUT Request**
```jsx
const updateUser = async (id, fullUserData) => {
  await fetch(`https://api.example.com/users/${id}`, {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(fullUserData),
  });
};
```

#### **4. PATCH Request**
```jsx
const updateEmail = async (id, newEmail) => {
  await fetch(`https://api.example.com/users/${id}`, {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ email: newEmail }),
  });
};
```

#### **5. DELETE Request**
```jsx
const deleteUser = async (id) => {
  await fetch(`https://api.example.com/users/${id}`, {
    method: 'DELETE',
  });
};
```

---

### **When to Use Which Method?**

- **GET** → Reading data (never changes data)
- **POST** → Creating new resources
- **PUT** → Full update (send entire object)
- **PATCH** → Partial update (send only changed fields) ← **Most common for edits**
- **DELETE** → Removing resources

---

## 3. Using `fetch()` (Built-in)

**Description**: `fetch()` is the built-in JavaScript API for making HTTP requests. It’s available in React Native without any additional installation and is based on Promises.

---

### **Basic Complete Example**
```jsx
import React, { useState, useEffect } from 'react';
import { View, Text, FlatList, StyleSheet, ActivityIndicator } from 'react-native';

const FetchExample = () => {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetchUsers();
  }, []);

  const fetchUsers = async () => {
    try {
      setLoading(true);
      setError(null);

      const response = await fetch('https://jsonplaceholder.typicode.com/users');

      if (!response.ok) {
        throw new Error(`HTTP error! Status: ${response.status}`);
      }

      const data = await response.json();
      setUsers(data);
    } catch (err) {
      setError(err.message);
      console.error('Error fetching users:', err);
    } finally {
      setLoading(false);
    }
  };

  return (
    <View style={styles.container}>
      {loading && <ActivityIndicator size="large" color="#007AFF" />}

      {error && <Text style={styles.error}>Error: {error}</Text>}

      <FlatList
        data={users}
        keyExtractor={item => item.id.toString()}
        renderItem={({ item }) => (
          <View style={styles.item}>
            <Text style={styles.name}>{item.name}</Text>
            <Text style={styles.email}>{item.email}</Text>
          </View>
        )}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 16,
    backgroundColor: '#f8f9fa',
  },
  item: {
    padding: 16,
    backgroundColor: '#fff',
    marginVertical: 8,
    borderRadius: 12,
  },
  name: {
    fontSize: 17,
    fontWeight: '600',
  },
  email: {
    fontSize: 15,
    color: '#666',
    marginTop: 4,
  },
  error: {
    color: 'red',
    padding: 16,
    textAlign: 'center',
  },
});

export default FetchExample;
```

---

### **POST Request Example**
```jsx
const createUser = async (userData) => {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/users', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(userData),
    });

    if (!response.ok) throw new Error('Failed to create user');

    const result = await response.json();
    return result;
  } catch (error) {
    console.error('POST Error:', error);
    throw error;
  }
};
```

---

### **Common HTTP Methods with `fetch()`**

| Method   | Usage | Example |
|----------|-------|---------|
| GET      | Fetch data | `fetch(url)` |
| POST     | Create resource | `fetch(url, { method: 'POST', body })` |
| PUT      | Replace resource | `fetch(url, { method: 'PUT', body })` |
| PATCH    | Partial update | `fetch(url, { method: 'PATCH', body })` |
| DELETE   | Delete resource | `fetch(url, { method: 'DELETE' })` |

---

**Note**:  
While `fetch()` is built-in and good for simple use cases, **Axios** is generally recommended for larger apps due to better error handling, interceptors, and cleaner syntax.

## 4. Axios (Recommended for Production)

**Description**: Axios is a popular, promise-based HTTP client that works great in both browser and React Native. It is widely recommended for production apps due to its clean syntax, powerful features, and excellent error handling.

---

### **Installation**
```bash
npm install axios
# or
yarn add axios
```

---

### **Basic Complete Example**
```jsx
// services/apiService.js
import axios from 'axios';

const BASE_URL = 'https://jsonplaceholder.typicode.com';

const api = axios.create({
  baseURL: BASE_URL,
  timeout: 10000,
});

// GET Request
export const getUsers = async () => {
  try {
    const response = await api.get('/users');
    return response.data;
  } catch (error) {
    console.error('Error fetching users:', error.message);
    throw error;
  }
};

// POST Request
export const createUser = async (userData) => {
  try {
    const response = await api.post('/users', userData);
    return response.data;
  } catch (error) {
    console.error('Error creating user:', error.message);
    throw error;
  }
};

// Example usage in a screen
const UsersScreen = () => {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    const loadUsers = async () => {
      const data = await getUsers();
      setUsers(data);
    };
    loadUsers();
  }, []);
};
```

---

### **Advantages of Axios over `fetch`**

| Feature                    | Axios                                   | fetch                              |
|---------------------------|-----------------------------------------|------------------------------------|
| Syntax                    | Cleaner with async/await                | More verbose                       |
| JSON Parsing              | Automatic                               | Manual (`response.json()`)         |
| Error Handling            | Better (throws for HTTP errors)         | Only rejects on network failure    |
| Interceptors              | Yes (Request & Response)                | No                                 |
| Timeout                   | Built-in                                | Manual implementation              |
| Progress Tracking         | Supported                               | Limited                            |
| Cancellation              | Easy                                    | More complex                       |

---

### **More Examples**

#### **1. PUT Request**
```jsx
await api.put(`/users/${id}`, updatedUser);
```

#### **2. DELETE Request**
```jsx
await api.delete(`/users/${id}`);
```

#### **3. With Query Parameters**
```jsx
await api.get('/users', {
  params: {
    page: 1,
    limit: 20,
    search: 'john'
  }
});
```

#### **4. File Upload**
```jsx
const formData = new FormData();
formData.append('avatar', {
  uri: imageUri,
  name: 'avatar.jpg',
  type: 'image/jpeg',
});

await api.post('/upload', formData, {
  headers: { 'Content-Type': 'multipart/form-data' }
});
```

---

**Why Use Axios in Production?**
- Much cleaner code
- Centralized configuration
- Powerful interceptors (great for auth)
- Consistent error format
- Great TypeScript support

## 5. PUT & PATCH Requests

**Description**: 
- **PUT** → Used to **replace** an entire resource.
- **PATCH** → Used to **update** only specific fields of a resource.

---

### **Basic Complete Example**
```jsx
import React, { useState } from 'react';
import {
  View,
  TextInput,
  Pressable,
  Text,
  StyleSheet,
  Alert,
  ActivityIndicator,
} from 'react-native';
import api from '../api';

const EditProfileScreen = ({ route, navigation }) => {
  const { user } = route.params;

  const [formData, setFormData] = useState({
    name: user.name,
    email: user.email,
    bio: user.bio || '',
  });

  const [loading, setLoading] = useState(false);

  const handleUpdate = async () => {
    if (!formData.name || !formData.email) {
      Alert.alert('Error', 'Name and Email are required');
      return;
    }

    setLoading(true);

    try {
      // Option 1: PUT - Replace entire resource
      // await api.put(`/users/${user.id}`, formData);

      // Option 2: PATCH - Update only changed fields (Recommended)
      await api.patch(`/users/${user.id}`, {
        name: formData.name,
        email: formData.email,
        bio: formData.bio,
      });

      Alert.alert('Success', 'Profile updated successfully!');
      navigation.goBack();
    } catch (error) {
      Alert.alert(
        'Error',
        error.response?.data?.message || 'Failed to update profile'
      );
    } finally {
      setLoading(false);
    }
  };

  return (
    <View style={styles.container}>
      <TextInput
        style={styles.input}
        placeholder="Name"
        value={formData.name}
        onChangeText={(text) => setFormData({ ...formData, name: text })}
      />

      <TextInput
        style={styles.input}
        placeholder="Email"
        keyboardType="email-address"
        value={formData.email}
        onChangeText={(text) => setFormData({ ...formData, email: text })}
      />

      <TextInput
        style={[styles.input, styles.textArea]}
        placeholder="Bio"
        multiline
        value={formData.bio}
        onChangeText={(text) => setFormData({ ...formData, bio: text })}
      />

      <Pressable style={styles.button} onPress={handleUpdate} disabled={loading}>
        {loading ? (
          <ActivityIndicator color="#fff" />
        ) : (
          <Text style={styles.buttonText}>Update Profile</Text>
        )}
      </Pressable>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
  },
  input: {
    backgroundColor: '#fff',
    borderRadius: 12,
    padding: 16,
    fontSize: 16,
    marginBottom: 16,
    borderWidth: 1,
    borderColor: '#ddd',
  },
  textArea: {
    height: 120,
    textAlignVertical: 'top',
  },
  button: {
    backgroundColor: '#007AFF',
    padding: 16,
    borderRadius: 12,
    alignItems: 'center',
    marginTop: 20,
  },
  buttonText: {
    color: '#fff',
    fontSize: 17,
    fontWeight: '600',
  },
});

export default EditProfileScreen;
```

---

### **PUT vs PATCH Comparison**

| Feature           | **PUT**                                      | **PATCH**                                      |
|-------------------|----------------------------------------------|------------------------------------------------|
| Purpose           | Replace entire resource                      | Update specific fields                         |
| Data Sent         | Full object                                  | Only changed fields                            |
| Idempotent        | Yes                                          | Yes                                            |
| Common Use Case   | Update complete profile                      | Change email, status, or single field          |

---

### **More Examples**

#### **1. PUT Request**
```jsx
await api.put(`/users/${id}`, {
  name: 'John Doe',
  email: 'john@example.com',
  age: 28,
  // All fields must be sent
});
```

#### **2. PATCH Request**
```jsx
await api.patch(`/users/${id}`, {
  status: 'active',
  lastLogin: new Date().toISOString(),
});
```

#### **3. With FormData (for file upload)**
```jsx

formData.append('avatar', file);

```

## 6. DELETE Request

**Description**: The DELETE method is used to remove a resource from the server (e.g., delete a user, post, product, or order).

---

### **Basic Complete Example**
```jsx
import React, { useState } from 'react';
import {
  View,
  Text,
  FlatList,
  Pressable,
  Alert,
  StyleSheet,
  ActivityIndicator,
} from 'react-native';
import api from '../api';

const UsersScreen = () => {
  const [users, setUsers] = useState([]);
  const [deletingId, setDeletingId] = useState(null);

  const deleteUser = async (id) => {
    Alert.alert(
      "Delete User",
      "Are you sure you want to delete this user?",
      [
        { text: "Cancel", style: "cancel" },
        {
          text: "Delete",
          style: "destructive",
          onPress: async () => {
            try {
              setDeletingId(id);
              await api.delete(`/users/${id}`);
              
              // Remove from local state
              setUsers(prev => prev.filter(user => user.id !== id));
            } catch (error) {
              Alert.alert("Error", "Failed to delete user");
            } finally {
              setDeletingId(null);
            }
          },
        },
      ]
    );
  };

  const renderItem = ({ item }) => (
    <View style={styles.item}>
      <View>
        <Text style={styles.name}>{item.name}</Text>
        <Text style={styles.email}>{item.email}</Text>
      </View>

      <Pressable
        style={styles.deleteButton}
        onPress={() => deleteUser(item.id)}
        disabled={deletingId === item.id}
      >
        {deletingId === item.id ? (
          <ActivityIndicator color="#fff" size="small" />
        ) : (
          <Text style={styles.deleteText}>Delete</Text>
        )}
      </Pressable>
    </View>
  );

  return (
    <View style={styles.container}>
      <FlatList
        data={users}
        keyExtractor={item => item.id.toString()}
        renderItem={renderItem}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8f9fa',
  },
  item: {
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
    padding: 16,
    backgroundColor: '#fff',
    marginHorizontal: 16,
    marginVertical: 6,
    borderRadius: 12,
  },
  name: {
    fontSize: 17,
    fontWeight: '600',
  },
  email: {
    fontSize: 15,
    color: '#666',
    marginTop: 4,
  },
  deleteButton: {
    backgroundColor: '#FF3B30',
    paddingHorizontal: 16,
    paddingVertical: 8,
    borderRadius: 8,
    width: 80,
    alignItems: 'center',
  },
  deleteText: {
    color: '#fff',
    fontWeight: '600',
  },
});

export default UsersScreen;
```

---

### **DELETE Request Patterns**

| Method                        | Usage |
|-------------------------------|-------|
| `api.delete('/users/123')`    | Simple delete by ID |
| `api.delete('/posts/45', { data: { reason: 'spam' } })` | With body |
| `await api.delete(url)`       | Most common |

---

### **Advanced: Optimistic Delete**
```jsx
const deleteUserOptimistic = async (id) => {
  const previousUsers = [...users];
  setUsers(prev => prev.filter(u => u.id !== id)); // Optimistic update

  try {
    await api.delete(`/users/${id}`);
  } catch (error) {
    setUsers(previousUsers); // Rollback
    Alert.alert("Failed", "Could not delete user");
  }
};
```

## 7. Loading & Error Handling

**Description**: Proper loading and error handling is essential for good user experience. It prevents users from seeing blank screens or crashing when something goes wrong with API calls.

---

### **Basic Complete Example**
```jsx
import React, { useState, useEffect } from 'react';
import {
  View,
  Text,
  FlatList,
  ActivityIndicator,
  StyleSheet,
  Pressable,
} from 'react-native';
import api from '../api';

const UsersScreen = () => {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  const [refreshing, setRefreshing] = useState(false);

  const fetchUsers = async () => {
    try {
      setError(null);
      const response = await api.get('/users');
      setUsers(response.data);
    } catch (err) {
      setError(err.response?.data?.message || err.message || 'Failed to load users');
    } finally {
      setLoading(false);
      setRefreshing(false);
    }
  };

  useEffect(() => {
    fetchUsers();
  }, []);

  const onRefresh = () => {
    setRefreshing(true);
    fetchUsers();
  };

  const retry = () => {
    setLoading(true);
    fetchUsers();
  };

  if (loading && users.length === 0) {
    return (
      <View style={styles.center}>
        <ActivityIndicator size="large" color="#007AFF" />
        <Text style={styles.loadingText}>Loading users...</Text>
      </View>
    );
  }

  return (
    <View style={styles.container}>
      {error && (
        <View style={styles.errorContainer}>
          <Text style={styles.errorText}>Error: {error}</Text>
          <Pressable style={styles.retryButton} onPress={retry}>
            <Text style={styles.retryText}>Retry</Text>
          </Pressable>
        </View>
      )}

      <FlatList
        data={users}
        keyExtractor={item => item.id.toString()}
        renderItem={({ item }) => (
          <View style={styles.item}>
            <Text style={styles.name}>{item.name}</Text>
            <Text style={styles.email}>{item.email}</Text>
          </View>
        )}
        refreshing={refreshing}
        onRefresh={onRefresh}
        ListEmptyComponent={
          !loading && !error && <Text style={styles.empty}>No users found</Text>
        }
      />
    </View>
  );
};

const styles = StyleSheet.create({

    flex: 1,

  },
  retryButton: {
    backgroundColor: '#007AFF',
    padding: 10,
    borderRadius: 8,
    alignItems: 'center',
  },
  retryText: {
    color: '#fff',
    fontWeight: '600',
  },
  item: {
    padding: 16,
    backgroundColor: '#fff',
    marginHorizontal: 16,
    marginVertical: 6,
    borderRadius: 12,
  },
  name: {
    fontSize: 17,
    fontWeight: '600',
  },
  email: {
    fontSize: 15,
    color: '#666',
    marginTop: 4,
  },
  empty: {
    textAlign: 'center',
    marginTop: 50,
    color: '#666',
    fontSize: 16,
  },
});
```

### **Key Principles**

| State       | When to Show                              | Best Practice |
|-------------|-------------------------------------------|---------------|
| `loading`   | Initial load or full screen refresh       | Show full screen loader |
| `refreshing`| Pull-to-refresh                           | Use `RefreshControl` |
| `error`     | API failed                                | Show retry button |
| `empty`     | No data returned                          | Friendly empty state |

## 8. Authorization Headers

**Description**: Authorization headers are used to send authentication tokens (usually JWT) with API requests so the server can verify the user’s identity and permissions.

---

### **Basic Complete Example**

#### **1. Manual Way (Not Recommended for Large Apps)**
```jsx
import axios from 'axios';

const fetchProfile = async () => {
  const token = await AsyncStorage.getItem('userToken');
  
  const config = {
    headers: {
      Authorization: `Bearer ${token}`,
    },
  };

  const response = await axios.get('https://api.yourapp.com/profile', config);
  return response.data;
};
```

#### **2. Best Practice – Using Axios Instance (Recommended)**
```jsx
// api.js
import axios from 'axios';
import AsyncStorage from '@react-native-async-storage/async-storage';

const api = axios.create({
  baseURL: 'https://api.yourapp.com',
  timeout: 10000,
});

// Automatically add token to every request
api.interceptors.request.use(
  async (config) => {
    const token = await AsyncStorage.getItem('userToken');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);

export default api;
```

#### **3. Usage (Clean & Simple)**
```jsx
import api from '../api';

const HomeScreen = () => {
  useEffect(() => {
    const loadUserData = async () => {
      try {
        const response = await api.get('/profile');     // No need to pass token!
        const response2 = await api.get('/orders');
        console.log(response.data);
      } catch (error) {
        console.error(error.response?.data || error.message);
      }
    };

    loadUserData();
  }, []);
};
```

---

### **Common Authorization Header Formats**

| Type              | Header Format                              | Use Case |
|-------------------|--------------------------------------------|----------|
| **Bearer Token**  | `Authorization: Bearer <token>`            | JWT (Most Common) |
| **Basic Auth**    | `Authorization: Basic <base64>`            | Username + Password |
| **API Key**       | `Authorization: Api-Key <key>` or custom header | Third-party APIs |

---

### **More Examples**

#### **1. Refresh Token Logic (Advanced)**
```jsx
api.interceptors.response.use(
  response => response,
  async (error) => {
    if (error.response.status === 401) {
      // Try to refresh token
      try {
        const refreshToken = await AsyncStorage.getItem('refreshToken');
        const res = await axios.post('/auth/refresh', { refreshToken });
        await AsyncStorage.setItem('userToken', res.data.token);
        
        // Retry original request
        return api(error.config);
      } catch (refreshError) {
        // Logout user
      }
    }
    return Promise.reject(error);
  }
);
```

#### **2. Multiple Authorization Types**
```jsx
config.headers = {
  Authorization: `Bearer ${token}`,
  'X-API-Key': apiKey,
};
```

## 9. Axios Instance (Best Practice)

**Description**: Creating a centralized Axios instance is a best practice in React Native apps. It helps manage base URL, headers, timeouts, interceptors, and error handling in one place.

---

### **Basic Complete Example**

#### **1. Create Axios Instance (`api.js`)**
```jsx
// api.js
import axios from 'axios';
import AsyncStorage from '@react-native-async-storage/async-storage';

const api = axios.create({
  baseURL: 'https://api.yourapp.com/v1',
  timeout: 15000,
  headers: {
    'Content-Type': 'application/json',
  },
});

// Request Interceptor - Add Token
api.interceptors.request.use(
  async (config) => {
    try {
      const token = await AsyncStorage.getItem('userToken');
      if (token) {
        config.headers.Authorization = `Bearer ${token}`;
      }
    } catch (error) {
      console.log('Error getting token:', error);
    }
    return config;
  },
  (error) => Promise.reject(error)
);

// Response Interceptor - Global Error Handling
api.interceptors.response.use(
  (response) => response,
  async (error) => {
    const originalRequest = error.config;

    // Handle token expiration (401)
    if (error.response?.status === 401 && !originalRequest._retry) {
      originalRequest._retry = true;
      try {
        // Refresh token logic here (if you have refresh token)
        // const newToken = await refreshToken();
        // await AsyncStorage.setItem('userToken', newToken);
        return api(originalRequest);
      } catch (refreshError) {
        // Logout user if refresh fails
        await AsyncStorage.removeItem('userToken');
        // Redirect to login (use navigation or context)
      }
    }

    return Promise.reject(error);
  }
);

export default api;
```

#### **2. Usage in Components**
```jsx
// screens/HomeScreen.js
import React, { useState, useEffect } from 'react';
import { View, Text, FlatList } from 'react-native';
import api from '../api';

const HomeScreen = () => {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const fetchUsers = async () => {
      try {
        const response = await api.get('/users');
        setUsers(response.data);
      } catch (error) {
        console.error('Failed to fetch users:', error.response?.data || error.message);
      } finally {
        setLoading(false);
      }
    };

    fetchUsers();
  }, []);

  return (
    <View>
      <FlatList
        data={users}
        keyExtractor={item => item.id.toString()}
        renderItem={({ item }) => <Text>{item.name}</Text>}
      />
    </View>
  );
};

export default HomeScreen;
```

### **Key Benefits of Axios Instance**

| Benefit                    | Description |
|---------------------------|-----------|
| Centralized Configuration | Base URL, timeout, headers in one place |
| Automatic Token Handling  | No need to add token in every request |
| Global Error Handling     | Handle 401, network errors centrally |
| Easy Maintenance          | Change API base URL in one file |
| Interceptors              | Request & Response middleware |

### **More Useful Configurations**

```jsx
api.post('/upload', formData, {

});
```
