
# Performance Optimization in React Native

In React Native, **performance optimization** is the process of making your app faster, smoother, and more efficient. A well-optimized app feels responsive, uses less battery, consumes less memory, loads quickly, and provides a native-like experience — even when handling large datasets, complex UIs, or heavy animations.

Good performance is not just about speed — it’s about **perceived smoothness**, battery efficiency, and overall user satisfaction.

---

## What are Performance Issues?

Slow or laggy apps frustrate users and often lead to bad reviews or uninstalls. Even if your app works correctly, poor performance can make it feel broken.

**Real-life Analogy**:  
Just like a car needs regular tuning, oil changes, and clean filters to run smoothly on the highway, your React Native app needs continuous optimization to avoid "traffic jams" in rendering, memory bloat, and CPU overload.

### Common Performance Problems

- **Unnecessary component re-renders** — The most common issue in React Native
- **Laggy lists** — Especially `FlatList` with 100+ items without optimization
- **Large unoptimized images** — Slow loading and high memory usage
- **Expensive calculations** running on every render (filtering, sorting, mapping)
- **Memory leaks** — Components not cleaning up listeners, timers, or subscriptions
- **Janky animations** — Caused by heavy JavaScript work on the main thread
- **Unstable function references** — Passing new functions on every render to memoized children
- **Over-fetching data** and unnecessary network calls
- **Large bundle size** and slow startup time

**Why it matters**: React Native is single-threaded (JavaScript thread). When this thread is blocked, the entire UI becomes unresponsive.

---

### **Simple Example: Preventing Unnecessary Re-renders**

This example demonstrates the most common performance issue — **parent re-renders causing child components to re-render unnecessarily**.

```jsx
import React, { useState, useCallback } from 'react';
import { View, Text, Button, StyleSheet } from 'react-native';

//  Without optimization - re-renders on every parent update
const ExpensiveListItem = ({ item }) => {
  console.log(' Rendering ExpensiveListItem:', item.name);
  
  // Imagine this component does heavy work (image processing, calculations, etc.)
  return (
    <View style={styles.item}>
      <Text style={styles.name}>{item.name}</Text>
      <Text style={styles.info}>{item.email}</Text>
    </View>
  );
};

const App = () => {
  const [count, setCount] = useState(0);
  const [users] = useState([
    { id: 1, name: 'Alice', email: 'alice@example.com' },
    { id: 2, name: 'Bob', email: 'bob@example.com' },
    { id: 3, name: 'Carol', email: 'carol@example.com' },
  ]);

  const handlePress = useCallback(() => setCount(c => c + 1), []);

  return (
    <View style={styles.container}>
      <Text style={styles.counter}>Parent Counter: {count}</Text>
      <Button title="Increment Counter (Causes Re-renders)" onPress={handlePress} />
      
      {/* These items re-render even though their data didn't change */}
      {users.map(user => (
        <ExpensiveListItem key={user.id} item={user} />
      ))}
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20, backgroundColor: '#f8f9fa' },
  counter: { 
    fontSize: 18, 
    fontWeight: 'bold', 
    marginBottom: 15,
    textAlign: 'center' 
  },
  item: { 
    padding: 16, 
    backgroundColor: 'white', 
    marginVertical: 6, 
    borderRadius: 12,
    shadowColor: '#000',
    shadowOpacity: 0.08,
    elevation: 3,
  },
  name: { fontSize: 17, fontWeight: '600' },
  info: { color: '#666', marginTop: 4 },
});

export default App;
```

### Explanation of the Problem

- Every time you press the button, `App` re-renders.
- Even though `users` data never changed, all `ExpensiveListItem` components re-render.
- In real apps, this can cause major lag when you have 50+ list items or complex components.

### How to Fix It (Teaser)

We will solve this using:
- `React.memo()` → Prevent re-renders
- `useCallback()` → Stable function references

## 1. React.memo

**Description**: `React.memo` is a higher-order component that memoizes functional components. It prevents re-rendering when props haven't changed (using shallow comparison).

### **What is React.memo?**

`React.memo` is React’s built-in way to optimize functional components. It works similarly to `PureComponent` in class components. 

Before rendering, it performs a **shallow comparison** of the current props with the previous props. If they are the same, React skips the render entirely. This is extremely useful for components that:
- Appear repeatedly (list items, cards, grid tiles)
- Are expensive to render (contain images, complex layouts, calculations)
- Receive the same data frequently while the parent component re-renders

**Note**: It only does **shallow comparison**. So if you pass objects or arrays, make sure you’re not creating new references unnecessarily (this is where `useCallback` and `useMemo` help).

---

### **Basic Complete Example**

```jsx
import React, { useState, useCallback } from 'react';
import { View, Text, Button, StyleSheet } from 'react-native';

const UserCard = React.memo(({ user, onFollow }) => {
  console.log(` Rendering UserCard for: ${user.name}`); 
  // This should only log when user data actually changes

  return (
    <View style={styles.card}>
      <Text style={styles.name}>{user.name}</Text>
      <Text style={styles.bio}>{user.bio}</Text>
      <Button title="Follow" onPress={() => onFollow(user.id)} />
    </View>
  );
});

const App = () => {
  const [users] = useState([
    { id: 1, name: 'Alice', bio: 'Mobile Developer' },
    { id: 2, name: 'Bob', bio: 'UI/UX Designer' },
    { id: 3, name: 'Carol', bio: 'Product Manager' },
  ]);
  
  const [counter, setCounter] = useState(0);

  const handleFollow = useCallback((userId) => {
    console.log('Following user:', userId);
    // You can put API call or state update here
  }, []);

  return (
    <View style={styles.container}>
      <Text style={styles.counter}>Parent Component Renders: {counter}</Text>
      <Button 
        title="Re-render Parent (Without Breaking Children)" 
        onPress={() => setCounter(c => c + 1)} 
      />
      
      {users.map(user => (
        <UserCard 
          key={user.id} 
          user={user} 
          onFollow={handleFollow} 
        />
      ))}
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20, backgroundColor: '#f8f9fa' },
  card: { 
    backgroundColor: 'white', 
    padding: 18, 
    marginVertical: 8, 
    borderRadius: 12,
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 2 },
    shadowOpacity: 0.1,
    shadowRadius: 8,
    elevation: 4,
  },
  name: { fontSize: 18, fontWeight: 'bold', marginBottom: 4 },
  bio: { color: '#555', fontSize: 15 },
  counter: { 
    fontSize: 17, 
    fontWeight: '600', 
    marginBottom: 15,
    textAlign: 'center' 
  },
});

export default App;
```

---

### **Important Concepts & Options**

| Option                    | What it does                                           | When to Use |
|---------------------------|--------------------------------------------------------|-------------|
| Default (Shallow Compare) | Compares props using `Object.is` (reference equality) | Most common cases |
| Custom `areEqual`         | Lets you define your own comparison logic              | When props are complex objects/arrays |
| `React.memo(Component)`   | Wraps and memoizes the component                       | List items, cards, modals, headers |

---

### **More Examples**

#### **1. Memoized Header Component**
```jsx

```

#### **2. Memoized List Item with Image**


#### **3. With Custom Comparison Function**
```jsx
const areEqual = (prevProps, nextProps) => {
  return (

```

#### **4. Memoized Expensive Component (Chart / Graph)**

---

### **When NOT to use React.memo**

- Very small and simple components (overhead might hurt performance)
- Components whose props change very frequently
- Components that are rendered only once

**Pro Tip**: Always use `React.memo` together with `useCallback` (for functions) and `useMemo` (for objects/arrays) for maximum benefit.

## 2. useMemo

**Description**: `useMemo` caches the result of expensive calculations and recomputes the value only when its dependencies change.

### **What is useMemo?**

`useMemo` is a React Hook that helps you optimize performance by **remembering** (memoizing) the result of a computation. 

Without `useMemo`, expensive operations like filtering large arrays, sorting data, complex calculations, or data transformations would run on **every render**. This can cause lag, especially when the component re-renders due to unrelated state changes (like a counter or theme toggle).

`useMemo` tells React:  
> “Only recalculate this when these specific values change.”

It returns a **cached value** and is especially powerful when combined with `React.memo` on child components.

---

### **Basic Complete Example**

```jsx
import React, { useState, useMemo } from 'react';
import { View, Text, TextInput, StyleSheet, FlatList } from 'react-native';

const App = () => {
  const [searchTerm, setSearchTerm] = useState('');
  
  const [products] = useState([
    { id: 1, name: 'Laptop', price: 999, category: 'electronics' },
    { id: 2, name: 'Phone', price: 699, category: 'electronics' },
    { id: 3, name: 'Headphones', price: 149, category: 'accessories' },
    // Imagine 1000+ products here...
  ]);

  // Expensive filtering + sorting
  const filteredProducts = useMemo(() => {
    console.log(' Expensive filtering & sorting running...');
    
    return products
      .filter(product => 
        product.name.toLowerCase().includes(searchTerm.toLowerCase()) ||
        product.category.toLowerCase().includes(searchTerm.toLowerCase())
      )
      .sort((a, b) => b.price - a.price); // Sort by price (descending)
  }, [searchTerm, products]);

  return (
    <View style={styles.container}>
      <TextInput
        style={styles.input}
        placeholder="Search products..."
        value={searchTerm}
        onChangeText={setSearchTerm}
      />
      
      <Text style={styles.result}>
        Found {filteredProducts.length} products
      </Text>

      <FlatList
        data={filteredProducts}
        keyExtractor={item => item.id.toString()}
        renderItem={({ item }) => (
          <View style={styles.item}>
            <Text style={styles.name}>{item.name}</Text>
            <Text style={styles.price}>${item.price}</Text>
          </View>
        )}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20, backgroundColor: '#f8f9fa' },
  input: { 
    borderWidth: 1, 
    borderColor: '#ddd', 
    padding: 14, 
    borderRadius: 10, 
    marginBottom: 15,
    fontSize: 16 
  },
  result: { 
    fontSize: 16, 
    fontWeight: '600', 
    marginBottom: 12 
  },
  item: {
    backgroundColor: 'white',
    padding: 16,
    marginVertical: 6,
    borderRadius: 10,
    shadowColor: '#000',
    shadowOpacity: 0.07,
    elevation: 2,
  },
  name: { fontSize: 17, fontWeight: '500' },
  price: { color: '#007AFF', fontWeight: 'bold', marginTop: 4 },
});

export default App;
```

---

### **Important Concepts**

| Concept                    | What it does                                      | Best Practice |
|---------------------------|---------------------------------------------------|---------------|
| Dependencies Array        | Controls when to recalculate                      | Include all used values |
| Empty Dependencies `[]`   | Runs only once (like `useEffect`)                 | For static computations |
| Expensive Operations       | Filtering, sorting, mapping, calculations         | Wrap with `useMemo` |

---

### **More Examples**

#### **1. Expensive Calculation (Total Price)**
```jsx
const totalPrice = useMemo(() => {
  return cartItems.reduce((sum, item) => sum + item.price * item.quantity, 0);
}, [cartItems]);
```

#### **2. Filtered & Grouped Data**
```jsx
const groupedByCategory = useMemo(() => {
  const groups = {};
  products.forEach(product => {
    if (!groups[product.category]) groups[product.category] = [];
    groups[product.category].push(product);
  });
  return groups;
}, [products]);
```

#### **3. Derived Data for Charts**
```jsx
const chartData = useMemo(() => {
  return salesData.map(month => ({
    month: month.name,
    revenue: month.sales * 0.85, // After tax calculation
    growth: calculateGrowth(month.sales, previousMonth.sales)
  }));
}, [salesData]);
```

#### **4. Memoized Object (for passing to children)**
```jsx
const userConfig = useMemo(() => ({
  theme: currentTheme,
  language: 'en',
  permissions: userPermissions,
}), [currentTheme, userPermissions]);
```

---

### **When to Use useMemo**

- Heavy computations (filtering, sorting, mapping large arrays)
- Creating complex objects or arrays that are passed as props
- Expensive string manipulations or date formatting
- Data transformations for charts or analytics

### **When NOT to Use useMemo**

- Trivial calculations (simple math)
- Small components or values that are cheap to compute
- When the computation runs very rarely

**Pro Tip**: Use the React DevTools Profiler to identify expensive renders before applying `useMemo`.

## 3. useCallback

**Description**: `useCallback` memoizes callback functions so they are not recreated on every render.

### **What is useCallback?**

`useCallback` is a React Hook that returns a **memoized version** of a callback function. It only changes the function reference when one of its dependencies changes.

This is extremely important because:

- In JavaScript, every time a component renders, a new function is created in memory.
- If you pass that new function to a `React.memo`-wrapped child component, it will cause the child to re-render even if props appear the same.
- `useCallback` helps maintain **stable function references**, which is key for performance when working with memoized components.

---

### **Basic Complete Example**

```jsx
import React, { useState, useCallback } from 'react';
import { View, Text, Button, StyleSheet } from 'react-native';

// Memoized Child Component
const UserCard = React.memo(({ user, onFollow }) => {
  console.log(`Rendering UserCard: ${user.name}`);
  return (
    <View style={styles.card}>
      <Text style={styles.name}>{user.name}</Text>
      <Text>{user.bio}</Text>
      <Button title="Follow" onPress={() => onFollow(user.id)} />
    </View>
  );
});

const App = () => {
  const [users] = useState([
    { id: 1, name: 'Alice', bio: 'Developer' },
    { id: 2, name: 'Bob', bio: 'Designer' },
    { id: 3, name: 'Carol', bio: 'Manager' },
  ]);
  
  const [counter, setCounter] = useState(0);

  //  Stable function reference using useCallback
  const handleFollow = useCallback((userId) => {
    console.log('Following user ID:', userId);
    // You can call API, update state, etc. here
  }, []); // Empty dependency array = function never changes

  return (
    <View style={styles.container}>
      <Text style={styles.counter}>Parent Renders: {counter}</Text>
      <Button 
        title="Re-render Parent" 
        onPress={() => setCounter(c => c + 1)} 
      />
      
      {users.map(user => (
        <UserCard 
          key={user.id} 
          user={user} 
          onFollow={handleFollow} 
        />
      ))}
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20, backgroundColor: '#f8f9fa' },
  card: { 
    backgroundColor: 'white', 
    padding: 16, 
    marginVertical: 8, 
    borderRadius: 12,
    elevation: 3,
  },
  name: { fontSize: 18, fontWeight: 'bold' },
  counter: { fontSize: 17, fontWeight: '600', marginBottom: 15 },
});

export default App;
```

---

### **Important Concepts**

| Concept                    | What it does                                      | Recommendation |
|---------------------------|---------------------------------------------------|----------------|
| Dependencies Array        | Controls when the function is recreated           | Include all values used inside |
| Empty Array `[]`          | Creates a function that never changes             | For functions that don’t depend on props/state |
| Missing Dependencies      | Can cause stale closures (bugs)                   | Use ESLint `react-hooks/exhaustive-deps` |

---

### **More Examples**

#### **1. With Dependencies (Using State)**
```jsx
const handleLike = useCallback((postId) => {
  setLikes(prev => ({ ...prev, [postId]: true }));
  sendLikeToServer(postId); // API call
}, []); // No dependency on likes because we use functional update
```

#### **2. Passing Multiple Callbacks**
```jsx
const handleEdit = useCallback((userId) => { /* edit logic */ }, []);
const handleDelete = useCallback((userId) => { /* delete logic */ }, []);
const handleShare = useCallback((userId) => { /* share logic */ }, []);

<UserCard 
  user={user}
  onEdit={handleEdit}
  onDelete={handleDelete}
  onShare={handleShare}
/>
```

#### **3. Debounced Search Callback**
```jsx
const debouncedSearch = useCallback(
  debounce((term) => {
    fetchSearchResults(term);
  }, 300),
  []
);
```

#### **4. Event Handler with Dependencies**
```jsx
const handleItemPress = useCallback((item) => {
  navigation.navigate('Details', { itemId: item.id });
  trackEvent('item_viewed', { itemId: item.id });
}, [navigation]); // navigation from useNavigation()
```

---

### **When to Use useCallback**

- Passing functions to `React.memo` child components
- Passing functions as dependencies to `useEffect` or `useMemo`
- Creating stable event handlers
- Any callback that would otherwise cause unnecessary re-renders

### **When NOT to Use useCallback**

- Simple functions used only inside the component
- Functions that are not passed down or used in dependencies
- Very small components where the optimization cost outweighs the benefit

**Pro Tip**: Use `useCallback` + `React.memo` together. One without the other often gives limited benefit.

## 4. React.memo vs useMemo vs useCallback

**Description**: Understanding when to use each optimization tool is crucial for effective performance tuning in React Native.

### **What is the Difference?**

These three tools solve different performance problems but often work together. Using the wrong one (or none) can lead to unnecessary re-renders and laggy apps.

### **Detailed Comparison**

| Feature                    | **React.memo**                          | **useMemo**                                | **useCallback**                          |
|---------------------------|-----------------------------------------|--------------------------------------------|------------------------------------------|
| **Optimizes**             | Component rendering                     | Expensive computed values                  | Function references                      |
| **Purpose**               | Skip re-renders when props are same     | Cache heavy calculations                   | Prevent new function creation            |
| **Returns**               | Memoized component                      | Cached value                               | Memoized function                        |
| **Prevents Re-renders**   | Yes (directly)                          | Indirectly (by stabilizing props)          | Indirectly (when passed to memoized children) |
| **Use Case**              | List items, cards, repeated UI          | Filtering, sorting, data transformation    | Event handlers, callbacks to children    |
| **Dependencies**          | Props (shallow compare)                 | Dependency array                           | Dependency array                         |
| **When to Use**           | Child components that receive props     | Expensive computations                     | Functions passed down as props           |

---

### **Visual Explanation with Code**

#### **Without Optimization (Bad Performance)**
```jsx
const App = () => {
  const [count, setCount] = useState(0);

  const data = expensiveCalculation();           // Runs every render
  const handlePress = () => console.log('pressed'); // New function every render

  return <ExpensiveList data={data} onPress={handlePress} />;
};
```

#### **With Proper Optimization (Good Performance)**
```jsx
const App = () => {
  const [count, setCount] = useState(0);

  // 1. Memoize expensive value
  const data = useMemo(() => expensiveCalculation(), []);

  // 2. Memoize callback
  const handlePress = useCallback(() => console.log('pressed'), []);

  // 3. Memoize child component
  return <MemoizedList data={data} onPress={handlePress} />;
};
```

### **Real-World Usage Patterns**

#### **Example 1: Complete Optimized List Item**
```jsx
// Child Component
const PostCard = React.memo(({ post, onLike, onComment }) => {
  return (
    <View>
      <Text>{post.title}</Text>
      <Button title="Like" onPress={() => onLike(post.id)} />
      <Button title="Comment" onPress={() => onComment(post.id)} />
    </View>
  );
});

// Parent Component
const Feed = () => {
  const [posts] = useState([...]);

  const handleLike = useCallback((postId) => {
    // like logic
  }, []);

  const handleComment = useCallback((postId) => {
    // comment logic
  }, []);

  const sortedPosts = useMemo(() => {
    return [...posts].sort((a, b) => b.likes - a.likes);
  }, [posts]);

  return sortedPosts.map(post => (
    <PostCard 
      key={post.id} 
      post={post} 
      onLike={handleLike} 
      onComment={handleComment} 
    />
  ));
};
```

#### **Example 2: Form with Multiple Callbacks**
```jsx
const UserForm = () => {
  const [formData, setFormData] = useState({});

  const updateField = useCallback((field, value) => {
    setFormData(prev => ({ ...prev, [field]: value }));
  }, []);

  const handleSubmit = useCallback(() => {
    saveUser(formData);
  }, [formData]);

  return (
    <FormFields 
      data={formData} 
      onChange={updateField} 
      onSubmit={handleSubmit} 
    />
  );
};
```

---

### **Best Practices Summary**

1. **Use `React.memo`** for components that re-render often with same props.
2. **Use `useMemo`** for expensive calculations or derived data.
3. **Use `useCallback`** for functions passed to memoized components or used in dependency arrays.
4. **Combine all three** for maximum performance in lists and complex UIs.
5. **Profile first** — Use React DevTools Profiler to identify actual bottlenecks before optimizing.

**Common Mistake**:  
Applying these hooks everywhere "just in case". This can actually hurt readability and sometimes performance due to overhead.

## 5. FlashList – High Performance List Rendering

**Description**: `FlashList` is a blazing-fast list component from Shopify that serves as a high-performance drop-in replacement for `FlatList`.

### **What is FlashList?**

`FlashList` is built on top of React Native’s `RecyclerListView` and offers significantly better performance than the default `FlatList`. 

It excels at rendering large lists (hundreds or thousands of items) with smooth scrolling, lower memory usage, and faster rendering times.

**Why FlashList is better than FlatList**:
- Much faster item rendering
- Better memory recycling
- Consistent 60 FPS scrolling
- Easier configuration with `estimatedItemSize`
- Reduced CPU and memory usage

---

### **Installation**

```bash
npm install @shopify/flash-list
# or
yarn add @shopify/flash-list
```

> **Note**: After installation, restart your development server.

---

### **Basic Complete Example**

```jsx
import React, { useState, useCallback } from 'react';
import { View, Text, StyleSheet } from 'react-native';
import { FlashList } from '@shopify/flash-list';

const MemoizedUserCard = React.memo(({ user, onPress }) => (
  <View style={styles.card}>
    <Text style={styles.name}>{user.name}</Text>
    <Text style={styles.email}>{user.email}</Text>
    <Text onPress={() => onPress(user.id)} style={styles.follow}>Follow</Text>
  </View>
));

const App = () => {
  const [users] = useState(Array.from({ length: 1000 }, (_, index) => ({
    id: index + 1,
    name: `User ${index + 1}`,
    email: `user${index + 1}@example.com`,
  })));

  const handlePress = useCallback((id) => {
    console.log('Pressed user:', id);
  }, []);

  return (
    <View style={styles.container}>
      <Text style={styles.header}>User List ({users.length} items)</Text>
      
      <FlashList
        data={users}
        estimatedItemSize={85}           // Very important for performance!
        renderItem={({ item }) => (
          <MemoizedUserCard user={item} onPress={handlePress} />
        )}
        keyExtractor={item => item.id.toString()}
        showsVerticalScrollIndicator={false}
        contentContainerStyle={styles.list}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#f8f9fa' },
  header: { 
    fontSize: 20, 
    fontWeight: 'bold', 
    padding: 20, 
    textAlign: 'center' 
  },
  list: { paddingHorizontal: 16 },
  card: {
    backgroundColor: 'white',
    padding: 16,
    marginVertical: 6,
    borderRadius: 12,
    shadowColor: '#000',
    shadowOpacity: 0.08,
    elevation: 3,
  },
  name: { fontSize: 17, fontWeight: '600' },
  email: { color: '#666', marginVertical: 4 },
  follow: { color: '#007AFF', marginTop: 8, fontWeight: 'bold' },
});

export default App;
```

---

### **Important Props**

| Prop                    | What it does                                      | Recommendation |
|-------------------------|---------------------------------------------------|----------------|
| `estimatedItemSize`     | Tells FlashList approximate height of each item   | **Must provide** for best performance |
| `data`                  | Array of data to render                           | Same as FlatList |
| `renderItem`            | Function to render each item                      | Use memoized component |
| `keyExtractor`          | Unique key for each item                          | Required for good recycling |
| `onEndReached`          | Load more data when scrolling near the end        | Great for pagination |

---

### **More Examples**

## 6. Image Optimization

**Description**: Images are one of the biggest performance bottlenecks in mobile apps. Optimizing them is essential for fast loading and smooth scrolling.

### **What is Image Optimization?**

Large, unoptimized images consume high memory, slow down rendering, and increase app size. Poor image handling can cause laggy lists and high battery drain.

Good image optimization includes proper sizing, caching, format selection, and lazy loading.

---

### **Basic Complete Example (Using expo-image)**

```jsx
import React from 'react';
import { View, Text, StyleSheet, FlatList } from 'react-native';
import { Image } from 'expo-image';

const OptimizedImageCard = React.memo(({ item }) => (
  <View style={styles.card}>
    <Image
      source={{ uri: item.imageUrl }}
      style={styles.image}
      contentFit="cover"
      transition={200}                    // Smooth fade-in
      cachePolicy="memory-disk"           // Smart caching
      placeholder={require('../../assets/placeholder.png')}
    />
    <Text style={styles.title}>{item.title}</Text>
  </View>
));

const App = () => {
  const [images] = useState([/* array of 200+ image items */]);

  return (
    <View style={styles.container}>
      <FlatList
        data={images}
        renderItem={({ item }) => <OptimizedImageCard item={item} />}
        keyExtractor={item => item.id}
        estimatedItemSize={280}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#f8f9fa' },
  card: { margin: 12, backgroundColor: 'white', borderRadius: 12, overflow: 'hidden' },
  image: { 
    width: '100%', 
    height: 220,
    backgroundColor: '#e0e0e0' 
  },
  title: { padding: 12, fontSize: 16, fontWeight: '600' },
});

export default App;
```

---

### **Important Best Practices**

| Technique                    | What it does                                      | Recommendation |
|-----------------------------|---------------------------------------------------|----------------|
| Use `expo-image`            | Modern image component with caching & transitions | **Highly Recommended** |
| `contentFit`                | Controls how image fits in container              | `"cover"`, `"contain"` |
| `cachePolicy`               | Controls caching strategy                         | `"memory-disk"` |
| `placeholder`               | Shows blur/placeholder while loading              | Local asset |
| Proper Dimensions           | Set exact width & height                          | Avoid layout shifts |

### **Additional Tips**

- **Compress images** before uploading (use tools like TinyPNG)
- Prefer **WebP** format when possible (smaller size)
- Resize images on the server (don’t send 4000px images for 300px display)
- Use `react-native-fast-image` for bare React Native projects
- Lazy load images in long lists (FlashList + expo-image works great)
- Avoid using very large GIFs

**Pro Tip**: Always test image performance on low-end devices.
