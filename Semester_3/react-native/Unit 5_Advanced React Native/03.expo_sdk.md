
#  Expo SDK

In React Native, the **Expo SDK** is one of the most powerful tools for building mobile apps quickly. It provides easy access to native device features without writing platform-specific code (Kotlin/Java or Swift/Objective-C).

---

## What is the Expo SDK?

The Expo SDK is a **collection of officially maintained libraries** that give your React Native app access to native device capabilities in a simple, cross-platform way.

### **Benefits of Using Expo SDK**

- No need to write native code for most common features
- Consistent API across Android and iOS
- Excellent documentation and community support
- Automatic compatibility management
- Works great with both Expo Go and custom development builds

**Installation Command (Recommended)**:
```bash
npx expo install <package-name>
```

This command ensures version compatibility with your current Expo SDK version.

---

### **Simple Example: Using Multiple Expo Modules**

```jsx
import React, { useState } from 'react';
import { View, Text, Button, StyleSheet, Alert } from 'react-native';
import * as Location from 'expo-location';
import * as ImagePicker from 'expo-image-picker';

const App = () => {
  const [location, setLocation] = useState(null);

  const getLocation = async () => {
    const { status } = await Location.requestForegroundPermissionsAsync();
    if (status !== 'granted') {
      Alert.alert('Permission Denied', 'Location access is required');
      return;
    }

    const currentLocation = await Location.getCurrentPositionAsync({});
    setLocation(currentLocation.coords);
    Alert.alert('Location', `Lat: ${currentLocation.coords.latitude}\nLng: ${currentLocation.coords.longitude}`);
  };

  const pickImage = async () => {
    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      allowsEditing: true,
      quality: 0.8,
    });

    if (!result.canceled) {
      Alert.alert('Image Selected', result.assets[0].uri);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Expo SDK Demo</Text>
      
      <Button title="Get Current Location" onPress={getLocation} />
      <Button title="Pick Image from Gallery" onPress={pickImage} />
      
      {location && (
        <Text style={styles.info}>
           {location.latitude.toFixed(4)}, {location.longitude.toFixed(4)}
        </Text>
      )}
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center', padding: 20, gap: 15 },
  title: { fontSize: 26, fontWeight: 'bold', marginBottom: 20 },
  info: { marginTop: 20, fontSize: 16, textAlign: 'center' },
});

export default App;
```

## 1. expo-camera

**Description**: `expo-camera` provides access to the device’s camera hardware with a simple React Native API.

### **What is expo-camera?**

`expo-camera` is one of the most popular Expo SDK modules. It gives you full control over the camera — including preview, photo capture, video recording, barcode scanning, and more — without writing any native code.

---

### **Installation**

```bash
npx expo install expo-camera
```

---

### **Basic Complete Example**

```jsx
import React, { useState, useRef } from 'react';
import { View, Text, StyleSheet, Button, Alert } from 'react-native';
import { Camera } from 'expo-camera';

const App = () => {
  const [permission, setPermission] = useState(null);
  const [cameraType, setCameraType] = useState(Camera.Constants.Type.back);
  const cameraRef = useRef(null);

  const requestPermissions = async () => {
    const { status } = await Camera.requestCameraPermissionsAsync();
    setPermission(status);
  };

  const takePicture = async () => {
    if (cameraRef.current) {
      const photo = await cameraRef.current.takePictureAsync({
        quality: 0.9,
        skipProcessing: true,
      });
      Alert.alert('Photo Captured!', `Saved to: ${photo.uri}`);
      console.log('Photo URI:', photo.uri);
    }
  };

  const toggleCameraType = () => {
    setCameraType(cameraType === Camera.Constants.Type.back 
      ? Camera.Constants.Type.front 
      : Camera.Constants.Type.back);
  };

  if (permission === null) {
    return (
      <View style={styles.container}>
        <Button title="Grant Camera Permission" onPress={requestPermissions} />
      </View>
    );
  }

  if (permission === 'denied') {
    return <Text>No access to camera</Text>;
  }

  return (
    <View style={styles.container}>
      <Camera 
        ref={cameraRef}
        style={styles.camera} 
        type={cameraType}
        ratio="16:9"
      >
        <View style={styles.controls}>
          <Button title="Flip Camera" onPress={toggleCameraType} />
          <Button title=" Take Photo" onPress={takePicture} />
        </View>
      </Camera>
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1 },
  camera: { flex: 1 },
  controls: { 
    position: 'absolute', 
    bottom: 40, 
    left: 0, 
    right: 0, 
    flexDirection: 'row', 
    justifyContent: 'space-around' 
  },
});

export default App;
```

## 2. expo-location

**Description**: `expo-location` provides access to device location services (GPS, network, etc.) with a clean cross-platform API.

### **What is expo-location?**

`expo-location` allows you to get the device’s current location, track movement in real-time, perform reverse geocoding (address from coordinates), and more — all without writing native code.

---

### **Installation**

```bash
npx expo install expo-location
```

---

### **Basic Complete Example**

```jsx
import React, { useState } from 'react';
import { View, Text, Button, StyleSheet, Alert } from 'react-native';
import * as Location from 'expo-location';

const App = () => {
  const [location, setLocation] = useState(null);
  const [address, setAddress] = useState(null);
  const [loading, setLoading] = useState(false);

  const getCurrentLocation = async () => {
    setLoading(true);
    try {
      const { status } = await Location.requestForegroundPermissionsAsync();
      
      if (status !== 'granted') {
        Alert.alert('Permission Denied', 'Location permission is required');
        return;
      }

      const currentLocation = await Location.getCurrentPositionAsync({
        accuracy: Location.Accuracy.High,
      });
      
      setLocation(currentLocation.coords);

      // Reverse Geocoding
      const reverseGeocode = await Location.reverseGeocodeAsync({
        latitude: currentLocation.coords.latitude,
        longitude: currentLocation.coords.longitude,
      });

      setAddress(reverseGeocode[0]);
    } catch (error) {
      Alert.alert('Error', error.message);
    } finally {
      setLoading(false);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>expo-location Demo</Text>

      <Button 
        title={loading ? "Getting Location..." : "Get Current Location"} 
        onPress={getCurrentLocation} 
        disabled={loading}
      />

      {location && (
        <View style={styles.result}>
          <Text style={styles.label}> Current Coordinates:</Text>
          <Text>Latitude: {location.latitude.toFixed(6)}</Text>
          <Text>Longitude: {location.longitude.toFixed(6)}</Text>
          <Text>Accuracy: {location.accuracy} meters</Text>
        </View>
      )}

      {address && (
        <View style={styles.result}>
          <Text style={styles.label}> Address:</Text>
          <Text>{address.city}, {address.region}</Text>
          <Text>{address.country}</Text>
          <Text>{address.street}</Text>
        </View>
      )}
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center', padding: 20 },
  title: { fontSize: 26, fontWeight: 'bold', marginBottom: 30 },
  result: { 
    marginTop: 20, 
    backgroundColor: '#f0f0f0', 
    padding: 18, 
    borderRadius: 12,
    width: '100%',
    alignItems: 'center'
  },
  label: { fontSize: 16, fontWeight: '600', marginBottom: 8 },
});

export default App;
```

---

### **Important Features**

| Feature                    | Description                                      | Common Use |
|---------------------------|--------------------------------------------------|----------|

| `watchPositionAsync`      | Real-time location tracking                      | Ride sharing, fitness |

## 3. expo-image-picker

**Description**: `expo-image-picker` allows users to select images or videos from the device’s gallery or directly launch the camera.

### **What is expo-image-picker?**

`expo-image-picker` is a powerful and easy-to-use module that provides access to the native image and video picker. It handles permissions, returns optimized assets, and supports editing — all with a consistent API across iOS and Android.

---

### **Installation**

```bash
npx expo install expo-image-picker
```

---

### **Basic Complete Example**

```jsx
import React, { useState } from 'react';
import { View, Text, Image, Button, StyleSheet, Alert } from 'react-native';
import * as ImagePicker from 'expo-image-picker';

const App = () => {
  const [selectedImage, setSelectedImage] = useState(null);

  const pickImage = async () => {
    // Request permission
    const { status } = await ImagePicker.requestMediaLibraryPermissionsAsync();
    
    if (status !== 'granted') {
      Alert.alert('Permission Denied', 'Sorry, we need media library access.');
      return;
    }

    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      allowsEditing: true,
      aspect: [4, 3],
      quality: 0.8,
    });

    if (!result.canceled) {
      setSelectedImage(result.assets[0].uri);
    }
  };

  const takePhoto = async () => {
    const { status } = await ImagePicker.requestCameraPermissionsAsync();
    
    if (status !== 'granted') {
      Alert.alert('Permission Denied', 'Camera access is required');
      return;
    }

    const result = await ImagePicker.launchCameraAsync({
      allowsEditing: true,
      aspect: [1, 1],
      quality: 0.9,
    });

    if (!result.canceled) {
      setSelectedImage(result.assets[0].uri);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>expo-image-picker</Text>

      {selectedImage && (
        <Image source={{ uri: selectedImage }} style={styles.image} />
      )}

      <View style={styles.buttonGroup}>
        <Button title=" Pick from Gallery" onPress={pickImage} />
        <Button title=" Take Photo" onPress={takePhoto} />
      </View>

      {selectedImage && (
        <Text style={styles.uri} numberOfLines={1}>
          {selectedImage}
        </Text>
      )}
    </View>
  );
};

const styles = StyleSheet.create({
  container: { 
    flex: 1, 
    justifyContent: 'center', 
    alignItems: 'center', 
    padding: 20 
  },
  title: { fontSize: 26, fontWeight: 'bold', marginBottom: 30 },
  image: { 
    width: 280, 
    height: 280, 
    borderRadius: 20, 
    marginVertical: 20,
    backgroundColor: '#eee' 
  },

    textAlign: 'center'

});
```

## 4. expo-av

**Description**: `expo-av` is Expo’s powerful module for playing and recording audio and video.

### **What is expo-av?**

`expo-av` provides a unified API to handle audio playback, video playback, and audio recording. It uses native players under the hood, giving excellent performance and features like background playback, controls, and more.

---

### **Installation**

```bash
npx expo install expo-av
```

---

### **Basic Complete Example (Audio Player + Recorder)**

```jsx
import React, { useState } from 'react';
import { View, Text, Button, StyleSheet } from 'react-native';
import { Audio } from 'expo-av';

const App = () => {
  const [sound, setSound] = useState(null);
  const [isPlaying, setIsPlaying] = useState(false);
  const [recording, setRecording] = useState(null);

  const playSound = async () => {
    const { sound } = await Audio.Sound.createAsync(
      { uri: 'https://www.soundhelix.com/examples/mp3/SoundHelix-Song-1.mp3' },
      { shouldPlay: true }
    );
    setSound(sound);
    setIsPlaying(true);

    sound.setOnPlaybackStatusUpdate((status) => {
      setIsPlaying(status.isPlaying);
    });
  };

  const stopSound = async () => {
    if (sound) {
      await sound.stopAsync();
      setIsPlaying(false);
    }
  };

  const startRecording = async () => {
    try {
      await Audio.requestPermissionsAsync();
      await Audio.setAudioModeAsync({ allowsRecordingIOS: true });

      const { recording } = await Audio.Recording.createAsync(
        Audio.RecordingOptionsPresets.HIGH_QUALITY
      );
      setRecording(recording);
    } catch (err) {
      console.error('Failed to start recording', err);
    }
  };

  const stopRecording = async () => {
    if (recording) {
      await recording.stopAndUnloadAsync();
      const uri = recording.getURI();
      console.log('Recording saved at:', uri);
      setRecording(null);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>expo-av Demo</Text>

      <View style={styles.section}>
        <Text style={styles.sectionTitle}> Audio Player</Text>
        <Button title={isPlaying ? "Stop" : "Play Sample Audio"} onPress={isPlaying ? stopSound : playSound} />
      </View>

      <View style={styles.section}>
        <Text style={styles.sectionTitle}> Voice Recorder</Text>
        <Button 
          title={recording ? "Stop Recording" : "Start Recording"} 
          onPress={recording ? stopRecording : startRecording} 
        />
      </View>
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center', padding: 20 },
  title: { fontSize: 26, fontWeight: 'bold', marginBottom: 40 },
  section: { marginVertical: 20, alignItems: 'center' },
  sectionTitle: { fontSize: 18, fontWeight: '600', marginBottom: 12 },
});

export default App;
```

## 5. expo-file-system

**Description**: `expo-file-system` provides a powerful API to read, write, download, and manage files on the device’s file system.

### **What is expo-file-system?**

`expo-file-system` gives you direct access to the device’s file system in a cross-platform way. It’s essential for downloading files, saving user data, caching content, and working with offline functionality.

---

### **Installation**

```bash
npx expo install expo-file-system
```

---

### **Basic Complete Example**

```jsx
import React, { useState } from 'react';
import { View, Text, Button, StyleSheet, Alert } from 'react-native';
import * as FileSystem from 'expo-file-system';

const App = () => {
  const [downloadProgress, setDownloadProgress] = useState(0);
  const [fileUri, setFileUri] = useState(null);

  const downloadFile = async () => {
    const fileUri = FileSystem.documentDirectory + 'sample.pdf';
    
    const downloadResumable = FileSystem.createDownloadResumable(
      'https://www.w3.org/WAI/ER/tests/xhtml/testfiles/resources/pdf/dummy.pdf',
      fileUri,
      {},
      (downloadProgress) => {
        const progress = downloadProgress.totalBytesWritten / downloadProgress.totalBytesExpectedToWrite;
        setDownloadProgress(Math.round(progress * 100));
      }
    );

    try {
      const { uri } = await downloadResumable.downloadAsync();
      setFileUri(uri);
      Alert.alert('Success', `File downloaded to:\n${uri}`);
    } catch (error) {
      Alert.alert('Download Failed', error.message);
    }
  };

  const readFile = async () => {
    if (fileUri) {
      const content = await FileSystem.readAsStringAsync(fileUri);
      Alert.alert('File Content', content.substring(0, 200) + '...');
    } else {
      Alert.alert('No File', 'Please download a file first');
    }
  };

  const deleteFile = async () => {
    if (fileUri) {
      await FileSystem.deleteAsync(fileUri);
      setFileUri(null);
      setDownloadProgress(0);
      Alert.alert('Deleted', 'File has been deleted');
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>expo-file-system</Text>

      <Button title="Download Sample PDF" onPress={downloadFile} />
      
      {downloadProgress > 0 && (
        <Text style={styles.progress}>Download Progress: {downloadProgress}%</Text>
      )}

      {fileUri && (
        <View style={styles.actions}>
          <Button title="Read File" onPress={readFile} />
          <Button title="Delete File" onPress={deleteFile} color="red" />
        </View>
      )}
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center', padding: 20 },
  title: { fontSize: 26, fontWeight: 'bold', marginBottom: 30 },
  progress: { marginVertical: 15, fontSize: 16, fontWeight: '600' },
  actions: { marginTop: 20, gap: 12, width: '80%' },
});

export default App;
```

## 6. expo-sharing

**Description**: `expo-sharing` allows you to share files, images, PDFs, and other content using the native share sheet.

### **What is expo-sharing?**

`expo-sharing` provides access to the system’s native sharing capabilities (iOS Share Sheet and Android Intent chooser). It’s perfect for letting users share content from your app to other installed apps (Messages, WhatsApp, Email, Social Media, etc.).

---

### **Installation**

```bash
npx expo install expo-sharing
```

---

### **Basic Complete Example**

```jsx
import React, { useState } from 'react';
import { View, Text, Button, StyleSheet, Alert } from 'react-native';
import * as Sharing from 'expo-sharing';
import * as FileSystem from 'expo-file-system';

const App = () => {
  const [isSharing, setIsSharing] = useState(false);

  const shareImage = async () => {
    setIsSharing(true);
    try {
      // You can share a local URI or remote URL (with limitations)
      await Sharing.shareAsync('https://picsum.photos/id/1015/800/600', {
        dialogTitle: 'Share this beautiful image',
        mimeType: 'image/jpeg',
      });
    } catch (error) {
      Alert.alert('Share Failed', error.message);
    } finally {
      setIsSharing(false);
    }
  };

  const sharePDF = async () => {
    setIsSharing(true);
    try {
      const fileUri = FileSystem.documentDirectory + 'report.pdf';
      
      // Download a sample file first
      await FileSystem.downloadAsync(
        'https://www.w3.org/WAI/ER/tests/xhtml/testfiles/resources/pdf/dummy.pdf',
        fileUri
      );

      const isAvailable = await Sharing.isAvailableAsync();
      
      if (isAvailable) {
        await Sharing.shareAsync(fileUri, {
          mimeType: 'application/pdf',
          dialogTitle: 'Share Trip Report',
          UTI: 'com.adobe.pdf', // iOS specific
        });
      } else {
        Alert.alert('Sharing not available on this device');
      }
    } catch (error) {
      Alert.alert('Error', error.message);
    } finally {
      setIsSharing(false);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>expo-sharing</Text>
      <Text style={styles.subtitle}>Share content using native share sheet</Text>

      <View style={styles.buttonContainer}>
        <Button 
          title="Share Sample Image" 
          onPress={shareImage} 
          disabled={isSharing}
        />
        <Button 
          title="Download & Share PDF" 
          onPress={sharePDF} 
          disabled={isSharing}
        />
      </View>
    </View>
  );
};

const styles = StyleSheet.create({
  container: { 
    flex: 1, 
    justifyContent: 'center',
```

## 7. expo-router

**Description**: `expo-router` is Expo’s file-based routing solution for React Native apps.

### **What is expo-router?**

`expo-router` is a powerful navigation library built on top of React Navigation. It uses a **file-system based router**, meaning the folder structure in your `app/` directory automatically becomes your app’s navigation.

It’s the **recommended** way to handle navigation in modern Expo apps.

---

### **Installation**

```bash
npx expo install expo-router
```

Then update your `app.json` (or `app.config.js`):

```json
{
  "expo": {
    "scheme": "myapp",
    "newArchEnabled": true
  }
}
```

---

### **Basic Folder Structure Example**

```text
app/
├── index.tsx                 → Home Screen (/)
├── profile.tsx               → Profile Screen (/profile)
├── settings.tsx              → Settings Screen (/settings)
├── _layout.tsx               → Root Layout
├── product/
│   ├── [id].tsx              → Dynamic Route (/product/123)
│   └── list.tsx              → (/product/list)
└── (tabs)/                   → Tab Navigation Group
    ├── home.tsx
    ├── explore.tsx
    └── profile.tsx
```

---

### **Complete Basic Example**

**`app/_layout.tsx`** (Root Layout)
```tsx
import { Stack } from 'expo-router';

export default function RootLayout() {
  return (
    <Stack>
      <Stack.Screen name="index" options={{ title: 'Home' }} />
      <Stack.Screen name="profile" options={{ title: 'My Profile' }} />
    </Stack>
  );
}
```

**`app/index.tsx`** (Home Screen)
```tsx
import { View, Text, Button, StyleSheet } from 'react-native';
import { Link, router } from 'expo-router';

export default function Home() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Welcome to Expo Router</Text>
      
      <Link href="/profile" asChild>
        <Button title="Go to Profile" />
      </Link>

      <Button 
        title="Go to Product 42" 
        onPress={() => router.push('/product/42')} 
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center' },
  title: { fontSize: 28, fontWeight: 'bold', marginBottom: 30 },
});
```

**`app/profile.tsx`**
```tsx
import { View, Text, Button } from 'react-native';
import { router } from 'expo-router';

export default function Profile() {
  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
      <Text style={{ fontSize: 24 }}>Profile Screen</Text>
```

### **Key Features**

| Feature                    | Description                                      |
|---------------------------|--------------------------------------------------|
| File-based Routing        | Folders = Routes                                 |
| Dynamic Routes            | `[id].tsx` for dynamic segments                  |
| Nested Layouts            | Group routes with `(group)` folders              |

## 8. expo-updates (Over-the-Air Updates)

**Description**: `expo-updates` enables you to push updates to your app without submitting a new version to the App Store or Play Store.

### **What is expo-updates?**

`expo-updates` allows **Over-The-Air (OTA)** updates. You can fix bugs, update UI, or even ship small features instantly to users who already have your app installed.

---

### **Installation**

```bash
npx expo install expo-updates
```

---

### **Basic Setup Example**

**Configure in `app.json`**:
```json
{
  "expo": {
    "updates": {
      "url": "https://u.expo.dev/your-project-id"
    },
    "runtimeVersion": {
      "policy": "appVersion"
    }
  }
}
```

**Check for updates in code**:
```jsx
import * as Updates from 'expo-updates';
import { useEffect } from 'react';
import { Alert } from 'react-native';

export default function App() {
  useEffect(() => {
    async function checkForUpdates() {
      try {
        const update = await Updates.checkForUpdateAsync();
        
        if (update.isAvailable) {
          await Updates.fetchUpdateAsync();
          Alert.alert(
            'Update Available',
            'A new version is available. Restarting app...',
            [{ text: 'OK', onPress: () => Updates.reloadAsync() }]
          );
        }
      } catch (error) {
        console.log('Update check failed', error);
      }
    }

    checkForUpdates();
  }, []);

  return <YourAppContent />;
}
```

---

### **Key Features**

- Automatic background updates
- Manual update checks
- Rollback support
- Works with both development and production builds

---

### **Common Use Cases**

- Hotfixes and bug fixes
- UI/UX improvements
- Small feature additions
- A/B testing
- Emergency security patches
