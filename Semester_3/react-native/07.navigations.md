# 1️⃣ What is Navigation?

Navigation allows users to move between screens.

Examples:

- Login → Home
- Home → Profile
- Product → Details
- Cart → Checkout

React Native uses **React Navigation** for navigation.

---

# 2️⃣ Installing React Navigation

```bash
npm install @react-navigation/native
```

Expo dependencies:

```bash
npx expo install react-native-screens react-native-safe-area-context
```

Wrap your app:

```jsx
import { NavigationContainer } from "@react-navigation/native";

export default function App() {
  return (
    <NavigationContainer>
      {/* Navigators */}
    </NavigationContainer>
  );
}
```

---

# 3️⃣ Stack Navigation

Stack works like a pile of pages.

```
Login
  ↓
Home
  ↓
Details
```

Install:

```bash
npm install @react-navigation/native-stack
```

Example:

```jsx
const Stack = createNativeStackNavigator();

<Stack.Navigator>
  <Stack.Screen name="Home" component={HomeScreen}/>
  <Stack.Screen name="Details" component={DetailsScreen}/>
</Stack.Navigator>
```

### Common Methods

```jsx
navigation.navigate("Home");
navigation.goBack();
navigation.replace("Login");
navigation.push("Details");
navigation.pop();
```

# 4️⃣ Bottom Tab Navigation

Useful for major sections.

```
 Home
 Search
 Favorites
 Profile
```

Install:

```bash
npm install @react-navigation/bottom-tabs
```

Example:

```jsx
<Tab.Navigator>
  <Tab.Screen name="Home" component={Home}/>
  <Tab.Screen name="Profile" component={Profile}/>
</Tab.Navigator>
```

Best for:

- Social apps
- Shopping apps
- Banking apps

---

# 5️⃣ Drawer Navigation

Provides a side menu.

Install:

```bash
npm install @react-navigation/drawer
```

Example:

```jsx
<Drawer.Navigator>
  <Drawer.Screen name="Dashboard" component={Dashboard}/>
  <Drawer.Screen name="Settings" component={Settings}/>
</Drawer.Navigator>
```

Use for:

- Settings
- Help
- About
- Logout

---

# 6️⃣ Nested Navigation

Real apps combine multiple navigators.

Example:

```
NavigationContainer
        │
 Stack Navigator
        │
 Bottom Tabs
        │
 Home
 Search
 Profile
```

Benefits:

- Better app structure
- Easier scalability

---

# 7️⃣ Passing Parameters

Navigate with data:

```jsx
navigation.navigate("Details", {
  id: 10,
  name: "Laptop",
});
```

Receive data:

```jsx
function Details({ route }) {
  const { id, name } = route.params;
}
```

---

# 8️⃣ Authentication Flow

Typical flow:

```
Splash
   │
Login
   │
OTP
   │
Home
```

Use a token to determine whether the user should see Auth screens or the Main App.

---

# 9️⃣ Protected Screens

Only authenticated users should access private screens.

Example logic:

```jsx
return token ? <AppStack /> : <AuthStack />;
```

Use:

- AsyncStorage
- SecureStore
- JWT Tokens

---

# 🔟 Deep Linking Basics

Deep Linking opens a specific screen using a URL.

Example:

```
myapp://profile
```

Useful for:

- Email verification
- Password reset
- Promotions
- Notifications

Basic configuration:

```jsx
const linking = {
  prefixes: ["myapp://"],
};
```

#  Navigation Comparison

| Navigator | Best Use |
|-----------|----------|
| Stack | Page-to-page flow |
| Bottom Tabs | Main app sections |
| Drawer | Global menu |
| Nested | Large applications |

---

#  Best Practices

- Keep navigation simple
- Use Stack for screen flow
- Use Tabs for primary sections
- Avoid deeply nested navigators
- Use meaningful screen names
- Pass only required parameters

---

#  Common Mistakes

- Forgetting NavigationContainer
- Misspelled screen names
- Navigating to unregistered screens
- Passing large objects as params
- Too many nested navigators
