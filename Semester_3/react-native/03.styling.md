# 1️⃣ What is Styling?

Unlike React.js, **React Native does not use CSS files**.

Instead, styles are written as **JavaScript objects** and passed through the `style` prop.

Example:

```jsx
import { Text } from "react-native";

export default function App() {
  return (
    <Text style={{ color: "blue", fontSize: 20 }}>
      Hello React Native
    </Text>
  );
}
```

### Why doesn't React Native use CSS?

React Native renders **native Android and iOS components**, not HTML elements.

Since there is no browser, CSS files cannot be interpreted.

Instead, React Native converts JavaScript style objects into native platform styles.

---

# 2️⃣ StyleSheet (Recommended)

The recommended way to write styles is using `StyleSheet.create()`.

```jsx
import { StyleSheet, Text } from "react-native";

const styles = StyleSheet.create({
  title: {
    fontSize: 24,
    fontWeight: "bold",
    color: "#2563EB",
  },
});

export default function App() {
  return (
    <Text style={styles.title}>
      Welcome
    </Text>
  );
}
```

## Advantages

* Better readability
* Reusable styles
* Cleaner code
* Easier maintenance
* Small performance optimization
* Better IDE auto-completion

### Best Practice

Create styles at the bottom of every component.

```jsx
const styles = StyleSheet.create({
  container: {},
  title: {},
  button: {},
});
```

# 3️⃣ Inline Styles

Inline styles are written directly inside the component.

```jsx
<Text style={{ color: "red", fontSize: 18 }}>
  Inline Style
</Text>
```

### Good For

* Quick testing
* Dynamic values
* Small examples

### Avoid For

* Large applications
* Repeated UI
* Production screens

Example of dynamic styling:

```jsx
<Text
  style={{
    color: isDark ? "#FFF" : "#000",
  }}
>
  Hello
</Text>
```

# 4️⃣ Flexbox

React Native uses **Flexbox** for layout.

Unlike CSS, the default direction is:

```text
flexDirection : column
justifyContent : flex-start
alignItems : stretch
```

---

## flexDirection

Controls the main axis.

```jsx
<View style={{ flexDirection: "row" }}>
  <Text>A</Text>
  <Text>B</Text>
</View>
```

Values:

* column
* row
* column-reverse
* row-reverse

---

## justifyContent

Controls alignment on the **main axis**.

Values:

* flex-start
* center
* flex-end
* space-between
* space-around
* space-evenly

```jsx
<View
  style={{
    flex: 1,
    justifyContent: "center",
  }}
>
  <Text>Centered</Text>
</View>
```

---

## alignItems

Controls alignment on the **cross axis**.

```jsx
<View
  style={{
    alignItems: "center",
  }}
>
  <Text>Hello</Text>
</View>
```

Values:

* stretch
* center
* flex-start
* flex-end
* baseline

---

## flex

Controls how much space a component occupies.

```jsx
<View style={{ flex: 1 }} />
```

Example:

```jsx
<View style={{ flex: 1 }} />
<View style={{ flex: 2 }} />
```

The second view occupies twice as much space.

# 6️⃣ zIndex

`zIndex` controls which component appears on top.

```jsx
<View style={{ zIndex: 2 }} />
```

Higher value = appears above lower values.

Common use cases:

* Floating Action Button
* Modal overlays
* Badges
* Cards

# 7️⃣ Responsive Design

A good mobile application should work on:

* Small phones
* Large phones
* Tablets

---

## Using Flexbox

```jsx
<View style={{ flex: 1 }} />
```

---

## Using Dimensions

```jsx
import { Dimensions } from "react-native";

const width = Dimensions.get("window").width;
const height = Dimensions.get("window").height;
```

---

## Using Percentages

```jsx
<View
  style={{
    width: "90%",
  }}
/>
```

---

## Using useWindowDimensions

Preferred when the screen can change size.

```jsx
import { useWindowDimensions } from "react-native";

const { width } = useWindowDimensions();
```

---

## Responsive Best Practices

* Prefer Flexbox
* Avoid fixed widths
* Avoid hardcoded heights
* Use percentages where appropriate
* Test on Android and iOS
* Test different screen sizes

# 8️⃣ Platform Specific Styling

Sometimes Android and iOS require different styles.

```jsx
import { Platform } from "react-native";

const styles = {
  text: {
    fontSize: Platform.OS === "ios" ? 20 : 18,
  },
};
```

You can also write:

```jsx
paddingTop: Platform.select({
  ios: 40,
  android: 20,
})
```

# 1️⃣1️⃣ Themes

Instead of repeating colors everywhere, create a reusable theme.

```jsx
export const theme = {
  colors: {
    primary: "#2563EB",
    secondary: "#10B981",
    background: "#FFFFFF",
    text: "#111827",
  },

  spacing: {
    sm: 8,
    md: 16,
    lg: 24,
  },
};
```

Usage:

```jsx
<Text
  style={{
    color: theme.colors.primary,
  }}
>
  Hello
</Text>
```

Benefits:

* Consistency
* Easy maintenance
* Faster redesigns

# 1️⃣2️⃣ Dark Mode

React Native provides `useColorScheme()`.

```jsx
import { useColorScheme } from "react-native";

const scheme = useColorScheme();

const background =
  scheme === "dark"
    ? "#000"
    : "#FFF";
```

Benefits:

* Better battery life on OLED devices
* Better night-time experience
* Matches device settings
* Modern user experience

# 1️⃣3️⃣ NativeWind (Tailwind CSS for React Native)

NativeWind brings the **utility-first Tailwind CSS workflow** to React Native.

Instead of writing large StyleSheet objects, you style components using the `className` prop.

NativeWind has become **one of the fastest-growing styling solutions** in the React Native ecosystem and is widely used in modern Expo and React Native projects.

---

## Why NativeWind?

* Utility-first styling
* Faster UI development
* Less boilerplate
* Easy to maintain
* Familiar for Tailwind CSS developers
* Excellent developer experience
* Supports Dark Mode
* Supports conditional styling
* Works with Expo and React Native CLI

---

## Installation

```bash
npm install nativewind
```

For Expo projects:

```bash
npx expo install nativewind react-native-safe-area-context
```

> Additional Tailwind configuration is required during project setup.

---

## Example

```jsx
import { View, Text } from "react-native";

export default function App() {
  return (
    <View className="flex-1 justify-center items-center bg-blue-500">
      <Text className="text-white text-xl font-bold">
        Hello NativeWind
      </Text>
    </View>
  );
}
```

---

## NativeWind vs StyleSheet

| StyleSheet               | NativeWind                     |
| ------------------------ | ------------------------------ |
| Built into React Native  | Third-party library            |
| JavaScript style objects | Tailwind utility classes       |
| No setup required        | Requires initial setup         |
| Great for dynamic styles | Great for rapid UI development |
| More verbose             | More concise                   |

### Which One Should You Learn?

Learn **StyleSheet first** because it is the core React Native styling API and is frequently asked in interviews.

Then learn **NativeWind**, as many modern React Native teams use it to speed up development and maintain consistent UI styling.

Knowing both makes you more versatile.

#  Key Takeaways

* React Native uses JavaScript objects instead of CSS files.
* `StyleSheet.create()` is the recommended foundation for styling.
* Flexbox is the primary layout system.
* Responsive layouts should use Flexbox, percentages, and `useWindowDimensions()`.
* Themes improve consistency and maintainability.
* Dark Mode enhances user experience.
* NativeWind brings Tailwind CSS to React Native and is becoming a popular choice for modern React Native development.
* Every React Native developer should understand both **StyleSheet** and **NativeWind**.
