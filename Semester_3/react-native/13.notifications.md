# 1️⃣ What are Notifications?

Notifications are messages delivered to users even when the app is in the background or closed.

### Types

- Local Notifications
- Push Notifications
- Scheduled Notifications
- Foreground Notifications

### Real-world Examples

- Food delivery updates
- OTP reminders
- Calendar reminders
- Chat messages
- Promotional offers

# 2️⃣ Expo Notifications

## Installation

```bash
npx expo install expo-notifications
```

Configure notification handler:

```jsx
import * as Notifications from "expo-notifications";

Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowBanner: true,
    shouldShowList: true,
    shouldPlaySound: true,
    shouldSetBadge: true,
  }),
});
```

# 3️⃣ Request Notification Permission

```jsx
const { status } =
await Notifications.requestPermissionsAsync();

if (status !== "granted") {
  return;
}
```

# 4️⃣ Local Notifications

Local notifications are created by the app itself.

```jsx
await Notifications.scheduleNotificationAsync({
  content: {
    title: "Reminder",
    body: "Time to study React Native!",
  },
  trigger: null,
});
```

### Use Cases

- Reminder apps
- Habit trackers
- Alarm apps
- Offline notifications

---

# 5️⃣ Push Notifications

Push notifications are sent from a server.

```text
Backend Server
      │
Expo Push Service / FCM
      │
User Device
      │
Notification
```

### Flow

1. Get push token
2. Save token on backend
3. Send notification from server
4. User receives notification

# 6️⃣ Expo Push Token

```jsx
const token =
(await Notifications.getExpoPushTokenAsync())
.data;

console.log(token);
```

Store this token in your backend.

# 7️⃣ Firebase Cloud Messaging (FCM)

FCM is Google's notification service.

### Install

```bash
npm install @react-native-firebase/app
npm install @react-native-firebase/messaging
```

### Why FCM?

- Reliable delivery
- Topic messaging
- Device targeting
- High scalability

# 8️⃣ Scheduling Notifications

Schedule a notification after a delay.

```jsx
await Notifications.scheduleNotificationAsync({
  content: {
    title: "Drink Water ",
    body: "Stay hydrated!",
  },
  trigger: {
    seconds: 10,
    repeats: false,
  },
});
```

Daily reminder:

```jsx
trigger: {
  hour: 9,
  minute: 0,
  repeats: true,
}
```

---

# 9️⃣ Cancel Notifications

Cancel all notifications:

```jsx
await Notifications.cancelAllScheduledNotificationsAsync();
```

Cancel a specific notification:

```jsx
await Notifications.cancelScheduledNotificationAsync(id);
```

# 🔟 Foreground vs Background Notifications

| Foreground | Background |
|------------|------------|
| App is open | App is minimized/closed |
| Can update UI instantly | Shown in system notification tray |

```text
User Grants Permission
        │
Generate Push Token
        │
Save Token to Backend
        │
Server Sends Notification
        │
Expo / FCM
        │
Device Receives Notification
```

#  Production Best Practices

- Request permission only when necessary
- Save push tokens securely
- Handle notification taps
- Avoid notification spam
- Group similar notifications
- Schedule reminders responsibly
- Test on real devices

---

#  Common Mistakes

- Forgetting permission request
- Not saving push token
- Testing push notifications only on simulators
- Sending duplicate notifications
- Ignoring notification click events
