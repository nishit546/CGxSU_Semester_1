# 1️⃣ What is Local Storage?

Local storage allows an application to save data directly on the user's device.

Unlike API data, local storage remains available even when there is no internet connection.

Common examples include:

* Login session
* User preferences
* App theme
* Language selection
* Cached API data
* Onboarding status

Benefits:

* Faster app startup
* Offline support
* Better user experience
* Reduced API calls

---

# 2️⃣ react-native-mmkv (Recommended)

**react-native-mmkv** is one of the most popular local storage libraries in modern React Native applications.

It is built on **MMKV**, a high-performance key-value storage library originally developed by Tencent.

Unlike AsyncStorage, MMKV uses **JSI (JavaScript Interface)**, making storage operations **synchronous** and significantly faster.

### Why MMKV?

* Extremely fast
* Synchronous API
* JSI-based architecture
* Very low latency
* Excellent for frequently accessed data
* Preferred in many modern React Native applications

---

## Installation

```bash
npm install react-native-mmkv
```

> **Note:** MMKV requires the React Native New Architecture (JSI). It works well in React Native CLI projects. Expo support depends on your workflow (for example, prebuild or a custom development client), so check the latest compatibility before using it in an Expo project.

---

## Create Storage

```jsx
import { MMKV } from "react-native-mmkv";

export const storage = new MMKV();
```

---

## Store String

```jsx
storage.set("username", "Rajesh");
```

---

## Read String

```jsx
const username = storage.getString("username");

console.log(username);
```

---

## Store Number

```jsx
storage.set("age", 25);
```

---

## Read Number

```jsx
const age = storage.getNumber("age");
```

---

## Store Boolean

```jsx
storage.set("isLoggedIn", true);
```

---

## Read Boolean

```jsx
const isLoggedIn = storage.getBoolean("isLoggedIn");
```

---

## Delete Data

```jsx
storage.delete("username");
```

---

## Clear Storage

```jsx
storage.clearAll();
```

# 3️⃣ AsyncStorage (Still Widely Used)

Before MMKV became popular, **AsyncStorage** was the standard local storage solution in React Native.

It is still:

* Maintained
* Reliable
* Frequently found in existing codebases
* Commonly discussed in interviews

Although newer projects often choose MMKV for performance, you should still understand AsyncStorage because you'll likely encounter it while maintaining older applications.

---

## Installation

```bash
npx expo install @react-native-async-storage/async-storage
```

---

## Store Data

```jsx
import AsyncStorage from "@react-native-async-storage/async-storage";

await AsyncStorage.setItem(
  "username",
  "Rajesh"
);
```

---

## Read Data

```jsx
const username =
  await AsyncStorage.getItem("username");
```

---

## Update Data

```jsx
await AsyncStorage.setItem(
  "username",
  "Amit"
);
```

---

## Delete Data

```jsx
await AsyncStorage.removeItem("username");
```

---

## Clear Storage

```jsx
await AsyncStorage.clear();
```

# 4️⃣ Working with Objects

Local storage stores **strings**, so JavaScript objects must be converted using JSON.

---

## MMKV Example

### Save Object

```jsx
const user = {
  id: 1,
  name: "Rajesh",
  role: "Developer",
};

storage.set(
  "user",
  JSON.stringify(user)
);
```

---

### Read Object

```jsx
const data = storage.getString("user");

const user = JSON.parse(data);
```

---

## AsyncStorage Example

### Save Object

```jsx
await AsyncStorage.setItem(
  "user",
  JSON.stringify(user)
);
```

---

### Read Object

```jsx
const data =
  await AsyncStorage.getItem("user");

const user = JSON.parse(data);
```

# 5️⃣ Expo SecureStore

Sensitive information should **never** be stored in MMKV or AsyncStorage.

For Expo applications, use **Expo SecureStore**.

SecureStore encrypts data using the platform's secure storage mechanisms.

Use it for:

* JWT tokens
* Refresh tokens
* API credentials
* Sensitive authentication data

---

## Installation

```bash
npx expo install expo-secure-store
```

---

## Save Secure Data

```jsx
import * as SecureStore from "expo-secure-store";

await SecureStore.setItemAsync(
  "token",
  "abc123"
);
```

---

## Read Secure Data

```jsx
const token =
  await SecureStore.getItemAsync("token");
```

---

## Delete Secure Data

```jsx
await SecureStore.deleteItemAsync("token");
```

# 6️⃣ Authentication Token Storage

A typical authentication flow looks like this:

```text
Login
   │
Receive Access Token
   │
Store Token in SecureStore
   │
Attach Token to API Requests
   │
Logout
   │
Delete Token
```

Example:

```jsx
await SecureStore.setItemAsync(
  "accessToken",
  token
);
```

Never:

* Hardcode tokens
* Store passwords
* Expose secrets inside your application

# 7️⃣ Theme Persistence

Save the user's preferred theme.

Using MMKV:

```jsx
storage.set(
  "theme",
  "dark"
);
```

Read the theme:

```jsx
const theme =
  storage.getString("theme");
```

Using AsyncStorage:

```jsx
await AsyncStorage.setItem(
  "theme",
  "dark"
);

const theme =
  await AsyncStorage.getItem("theme");
```

This allows your application to automatically restore the selected theme after restarting.

# 8️⃣ Choosing the Right Storage

| Data                 | Recommended Storage |
| -------------------- | ------------------- |
| Theme                | MMKV                |
| Language             | MMKV                |
| User Preferences     | MMKV                |
| Cached API Data      | MMKV                |
| Authentication Token | SecureStore         |
| Refresh Token        | SecureStore         |
| Password             | Never Store         |

---

# 9️⃣ MMKV vs AsyncStorage vs SecureStore

| Feature          | MMKV                      | AsyncStorage      | SecureStore              |
| ---------------- | ------------------------- | ----------------- | ------------------------ |
| Speed            | ⭐⭐⭐⭐⭐                     | ⭐⭐                | ⭐⭐⭐                      |
| API              | Synchronous               | Asynchronous      | Asynchronous             |
| Sensitive Data   | ❌                         | ❌                 | ✅                        |
| Encryption       | ❌                         | ❌                 | ✅                        |
| JSON Support     | Manual                    | Manual            | Manual                   |
| Production Ready | ✅                         | ✅                 | ✅                        |
| Best Use Case    | Preferences & cached data | Existing projects | Authentication & secrets |

# 🔟 Best Practices

* Prefer MMKV for modern applications when your project setup supports it.
* Learn AsyncStorage because many existing applications still use it.
* Use SecureStore for sensitive information.
* Convert objects using `JSON.stringify()`.
* Parse stored objects using `JSON.parse()`.
* Remove authentication tokens during logout.
* Avoid storing large files in local storage.
* Handle missing values safely.
* Keep storage keys consistent across the project.

---

#  Common Mistakes

* Saving passwords in AsyncStorage or MMKV
* Forgetting `JSON.stringify()`
* Forgetting `JSON.parse()`
* Clearing all storage unintentionally
* Not removing tokens during logout
* Storing huge datasets locally
* Using local storage as a database

#  Key Takeaways

* Local storage enables offline functionality and improves the user experience.
* **react-native-mmkv** is the preferred storage solution for many modern React Native applications because of its speed and synchronous API.
* **AsyncStorage** remains important to learn because it is widely used in existing applications and interviews.
* **Expo SecureStore** should always be used for sensitive information such as authentication tokens.
* Store JavaScript objects using `JSON.stringify()` and retrieve them using `JSON.parse()`.
* Choose the storage solution based on the type and sensitivity of the data rather than using a single solution for everything.
