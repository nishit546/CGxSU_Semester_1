#  📱 Images & Assets in React Native

---

#  Learning Objectives

- Display local and remote images
- Use ImageBackground
- Work with SVG images
- Understand resize modes
- Optimize images
- Follow best practices

---

# 1️⃣ What are Assets?
Assets include images, icons, fonts, videos, audio, SVGs, and PDFs.

```text
assets/
├── images/
├── icons/
├── fonts/
├── svg/
└── videos/
```

---
# 2️⃣ Local Images
```jsx
<Image
 source={require('./assets/images/profile.png')}
 style={{width:120,height:120}}
/>
```
Advantages:
- Offline support
- Fast loading
- Bundled with app

---
# 3️⃣ Remote Images
```jsx
<Image
 source={{uri:'https://picsum.photos/300'}}
 style={{width:300,height:200}}
/>
```
Remember: Image always needs width and height.

---
# 4️⃣ ImageBackground
```jsx
<ImageBackground
 source={require('./assets/images/banner.jpg')}
 style={{width:'100%',height:220}}
>
 <Text>Welcome</Text>
</ImageBackground>
```
Use for banners, login screens and headers.

---
# 5️⃣ SVG
Install:
```bash
npx expo install react-native-svg
```
Example:
```jsx
import Svg,{Circle} from 'react-native-svg';
<Svg width='100' height='100'><Circle cx='50' cy='50' r='40' fill='blue'/></Svg>
```
Benefits:
- Scalable
- Lightweight
- Great for icons

---
# 6️⃣ Resize Modes
Supported modes:
- cover
- contain
- stretch
- repeat
- center

```jsx
<Image resizeMode='cover' />
```

---
# 7️⃣ Image Optimization
- Compress images
- Prefer WebP where possible
- Cache remote images
- Resize before upload
- Avoid oversized assets

For Expo image caching:
```bash
npx expo install expo-image
```

---
# 8️⃣ Loading & Error Handling
```jsx
<ActivityIndicator size='large'/>
```
```jsx
<Image onError={()=>console.log('Failed to load image')} />
```

---
# 9️⃣ Performance Tips
- Use FlatList for image galleries
- Lazy load images
- Cache remote images
- Optimize asset sizes

---
#  Common Mistakes
- Missing width/height
- Huge image files
- Invalid URLs
- Too many remote requests
