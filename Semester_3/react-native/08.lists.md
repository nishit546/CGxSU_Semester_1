# 1️⃣ What are Lists?

Lists are used to display collections of data.

Examples:

- Products
- Contacts
- Chat Messages
- Notifications
- Movies
- Users
- Orders

React Native provides optimized components for rendering lists.

# 2️⃣ FlatList

`FlatList` is the recommended component for rendering large datasets.

### Basic Example

```jsx
import { FlatList, Text } from "react-native";

const DATA = [
  { id: "1", name: "Rajesh" },
  { id: "2", name: "Amit" },
];

<FlatList
  data={DATA}
  keyExtractor={(item) => item.id}
  renderItem={({ item }) => (
    <Text>{item.name}</Text>
  )}
/>
```

### Important Props

- data
- renderItem
- keyExtractor
- horizontal
- numColumns
- ListHeaderComponent
- ListFooterComponent
- ListEmptyComponent

### Advantages

- Lazy rendering
- Better performance
- Memory efficient

# 3️⃣ SectionList

Used for grouped data.

Example:

```jsx
<SectionList
  sections={[
    {
      title: "A",
      data: ["Aman", "Ankit"],
    },
    {
      title: "B",
      data: ["Bharat"],
    },
  ]}
  renderItem={({ item }) => (
    <Text>{item}</Text>
  )}
  renderSectionHeader={({ section }) => (
    <Text>{section.title}</Text>
  )}
/>
```

### Common Use Cases

- Contacts
- Countries
- Categories
- Chat History

# 4️⃣ Infinite Scroll

Infinite Scroll automatically loads more data when the user reaches the end of the list.

### FlatList Props

```jsx
onEndReached={() => loadMore()}
onEndReachedThreshold={0.5}
```

### Example Flow

```text
Load Page 1
      ↓
Scroll Down
      ↓
Load Page 2
      ↓
Load Page 3
```

### Best Practices

- Show loading indicator
- Prevent duplicate API calls
- Stop loading when data ends

---

# 5️⃣ Pagination

Pagination loads data page by page.

Example:

```text
Page 1
↓

Page 2
↓

Page 3
```

Example API:

```text
GET /users?page=2&limit=10
```

Benefits:

- Faster loading
- Lower memory usage
- Better user experience

---

# 6️⃣ Pull To Refresh

Refresh data by pulling the list downward.

Example:

```jsx
<FlatList
  refreshing={loading}
  onRefresh={fetchUsers}
/>
```

or

```jsx
<RefreshControl
  refreshing={loading}
  onRefresh={fetchUsers}
/>
```

Common Uses:

- Social Media Feeds
- News Apps
- E-commerce Apps

# 7️⃣ Performance Optimization

Large lists require optimization.

### Use keyExtractor

```jsx
keyExtractor={(item) => item.id}
```

### Memoize List Items

```jsx
export default React.memo(ItemCard);
```

### Use getItemLayout

Useful when item height is fixed.

```jsx
getItemLayout={(data, index) => ({
  length: 80,
  offset: 80 * index,
  index,
})}
```

### Avoid Inline Functions

Instead of:

```jsx
renderItem={({ item }) => <Card item={item} />}
```

Use memoized callbacks when appropriate.

# 8️⃣ FlatList vs ScrollView

| FlatList | ScrollView |
|----------|------------|
| Large Data | Small Data |
| Lazy Rendering | Renders All Items |
| Better Performance | Slower with Large Lists |

---

# 9️⃣ Virtualization

FlatList renders only visible items.

Benefits:

- Better memory usage
- Smooth scrolling
- Faster rendering

# 🔟 Common FlatList Props

| Prop | Purpose |
|------|---------|
| data | List Data |
| renderItem | Render Each Item |
| keyExtractor | Unique Key |
| horizontal | Horizontal List |
| numColumns | Grid Layout |
| refreshing | Pull to Refresh |
| onRefresh | Refresh Callback |
| onEndReached | Infinite Scroll |
| ListHeaderComponent | Header |
| ListFooterComponent | Footer |

---

#  Best Practices

- Use FlatList instead of ScrollView for large data
- Use unique keys
- Paginate APIs
- Cache responses
- Memoize components
- Avoid unnecessary re-renders
- Optimize images inside lists

---

#  Common Mistakes

- Using ScrollView for thousands of items
- Missing keyExtractor
- Duplicate API calls in Infinite Scroll
- Loading all records at once
- Heavy components inside renderItem
