# 2️⃣ React.memo

`React.memo` prevents a component from re-rendering when its props haven't changed.

```jsx
const UserCard = React.memo(({ name }) => {
  return <Text>{name}</Text>;
});
```

### Use Cases

- Cards
- List Items
- Headers
- Static Components

### Don't Use When

- Props change frequently
- Component is extremely small

---

# 3️⃣ useMemo

`useMemo` caches expensive calculations.

```jsx
const total = useMemo(() => {
  return products.reduce(
    (sum, item) => sum + item.price,
    0
  );
}, [products]);
```

### Best Use Cases

- Sorting
- Filtering
- Searching
- Calculations

---

# 4️⃣ useCallback

`useCallback` memoizes functions.

```jsx
const handlePress = useCallback(() => {
  console.log("Pressed");
}, []);
```

Useful when passing callbacks to memoized child components.

---

# 5️⃣ React.memo vs useMemo vs useCallback

| Feature | React.memo | useMemo | useCallback |
|---------|------------|----------|-------------|
| Optimizes | Component | Value | Function |
| Prevents Re-render | ✅ | ❌ | Indirectly |
| Stores Result | ❌ | ✅ | Function |

# 6️⃣ FlashList

FlashList is a high-performance replacement for FlatList.

## Installation

```bash
npm install @shopify/flash-list
```

Example:

```jsx
import { FlashList } from "@shopify/flash-list";

<FlashList
  data={users}
  estimatedItemSize={80}
  renderItem={({ item }) => (
    <UserCard user={item} />
  )}
/>
```

### Advantages

- Faster rendering
- Lower memory usage
- Better scrolling performance

# 7️⃣ Image Optimization

Large images slow down applications.

### Best Practices

- Compress images
- Use WebP when possible
- Cache remote images
- Resize before upload
- Lazy load image lists

Useful libraries:

- expo-image
- react-native-fast-image (Bare)

---

# 8️⃣ Re-render Optimization

Avoid unnecessary renders.

### Good Practices

- Split large components
- Use React.memo
- Memoize callbacks
- Memoize expensive values
- Avoid inline objects/functions when unnecessary

Bad:

```jsx
<Button onPress={() => save()} />
```

Better:

```jsx
const handleSave = useCallback(() => {
  save();
}, []);

<Button onPress={handleSave} />
```

#  Production Best Practices

- Render only what is needed
- Paginate APIs
- Virtualize long lists
- Keep components small
- Minimize network requests
- Clean up timers and listeners
- Avoid memory leaks

---

#  Common Mistakes

- Overusing `useMemo`
- Overusing `useCallback`
- Rendering thousands of items with ScrollView
- Huge image files
- Unnecessary global state
