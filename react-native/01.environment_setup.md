# 1️⃣ What is Node.js?

Node.js is a JavaScript runtime that allows JavaScript to run outside the browser.

React Native uses Node.js to:

- Install packages
- Run development servers
- Execute CLI commands
- Build applications

## Check Installation

```bash
node -v
```

Check npm version:

```bash
npm -v
```

If both commands return versions, Node.js is installed correctly.

---

# 2️⃣ Android Studio

Android Studio is Google's official IDE for Android development.

Even when using Expo, Android Studio is useful for:

- Android Emulator
- SDK Manager
- Debugging
- Production builds

## Install Components

- Android SDK
- Android SDK Platform
- Android SDK Command-line Tools
- Android Emulator

---

# 3️⃣ Java Development Kit (JDK)

JDK is required for Android development.

Verify installation:

```bash
java -version
```

If Java version is displayed, JDK is installed successfully.

---

# 4️⃣ Environment Variables

Environment variables allow your system to locate Android SDK and Java.

Common variables:

```text
JAVA_HOME
ANDROID_HOME
```

Add Android SDK tools to your system PATH.

Verify with:

```bash
adb --version
```

---

# 5️⃣ Expo CLI

Expo simplifies React Native development.

Create a new project:

```bash
npx create-expo-app@latest myApp
```

Go into the project:

```bash
cd myApp
```

Start the development server:

```bash
npx expo start
```

---

# 6️⃣ Metro Bundler

Metro is the JavaScript bundler used by React Native.

Responsibilities:

- Bundles JavaScript
- Watches file changes
- Serves the app during development
- Supports Fast Refresh

Typical terminal output:

```bash
Starting Metro Bundler...
```

---

# 7️⃣ Expo Go

Expo Go is the mobile app used to run Expo projects without creating APK files.

## Steps

1. Install Expo Go from Play Store or App Store.
2. Connect phone and computer to the same Wi-Fi.
3. Run:

```bash
npx expo start
```

4. Scan the QR code.

Your app launches instantly.

---

# 8️⃣ Fast Refresh

Fast Refresh updates only the modified code while preserving component state whenever possible.

### Benefits

- Faster development
- Keeps app state
- Automatic refresh

---

# 9️⃣ Hot Reload vs Fast Refresh

| Hot Reload | Fast Refresh |
|------------|--------------|
| Older feature | Modern feature |
| Sometimes loses state | Preserves state when possible |
| Less reliable | Recommended |

> **Recommendation:** Use **Fast Refresh** for all modern React Native projects.

# 🔟 Clearing Cache

Sometimes the Metro cache becomes corrupted.

Clear cache using:

```bash
npx expo start --clear
```

For npm:

```bash
npm cache clean --force
```

Delete dependencies if required:

```bash
rm -rf node_modules
npm install
```

#  Common Setup Errors

| Error | Solution |
|--------|----------|
| node is not recognized | Install Node.js and restart terminal |
| java not found | Install JDK and set JAVA_HOME |
| adb not recognized | Configure Android SDK PATH |
| QR code not working | Connect to same Wi-Fi or use Tunnel mode |
| Metro stuck | Run `npx expo start --clear` |
