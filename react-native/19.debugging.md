# 1️⃣ What is Debugging?

Debugging is the process of finding, analyzing, and fixing bugs in an application.

Common issues include:

- App crashes
- API failures
- UI rendering issues
- State management bugs
- Performance issues
- Memory leaks

# 2️⃣ Console Debugging

The simplest debugging method.

```jsx
console.log("User Loaded");

console.warn("Invalid Email");

console.error("API Failed");
```

### Console Methods

| Method | Purpose |
|---------|----------|
| console.log() | General Logs |
| console.warn() | Warning Messages |
| console.error() | Error Messages |
| console.table() | Display Arrays & Objects |
| console.time() | Performance Measurement |
| console.timeEnd() | Stop Timer |

Example:

```jsx
console.time("API");

await fetchUsers();

console.timeEnd("API");
```

# 3️⃣ LogBox

React Native provides LogBox to display warnings and errors.

Example Warning

```
VirtualizedLists should never be nested...
```

Hide specific warnings (Development Only):

```jsx
import { LogBox } from "react-native";

LogBox.ignoreLogs([
  "VirtualizedLists should never be nested",
]);
```

Hide all logs (Not Recommended):

```jsx
LogBox.ignoreAllLogs();
```

# 6️⃣ VS Code Debugging

VS Code supports debugging directly from the editor.

### Features

- Breakpoints
- Step Over
- Step Into
- Call Stack
- Variables
- Watch Expressions

Example

```
Click Left Margin

↓

Breakpoint Added

↓

Run Debugger

↓

Execution Stops
```

Benefits

- Easy debugging
- Variable inspection
- Faster issue detection

# 8️⃣ Debugging API Calls

```jsx

  const res = await axios.get("/users");

  console.log(res.data);

catch(error){
  console.error(error);
}
```

Always log:

- Status Code
- Response Data
- Error Message

#  Best Practices

- Write meaningful logs
- Remove unnecessary logs before production
- Never ignore important warnings
- Use breakpoints instead of excessive console logs
- Test edge cases
- Handle errors gracefully

---

#  Common Mistakes

- Ignoring warnings
- Using console.log everywhere
- Not checking API responses
- Forgetting try/catch
- Debugging only on emulator
- Not testing release builds
