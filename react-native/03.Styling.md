# 1️⃣ What is Styling?

Unlike React.js, React Native **does not use CSS files**.

Styles are written as **JavaScript objects** and applied using the `style` prop.

```jsx
<Text style={{ color: "blue", fontSize: 20 }}>
  Hello React Native
</Text>
```

---

# 2️⃣ StyleSheet (Recommended)

`StyleSheet.create()` is the recommended way to write styles.

### Advantages

- Better readability
- Reusable styles
- Slight performance optimization
- Easy maintenance

```jsx
import { StyleSheet, Text } from "react-native";

const styles = StyleSheet.create({
  title: {
    fontSize: 24,
    fontWeight: "bold",
    color: "#2563EB",
  },
});

export default function App() {
  return <Text style={styles.title}>Welcome</Text>;
}
```

---

# 3️⃣ Inline Styles

Inline styles are written directly inside components.

```jsx
<Text style={{ color: "red", fontSize: 18 }}>
  Inline Style
</Text>
```

### Use When

- Quick testing
- Small demos

### Avoid When

- Large applications
- Reusable UI

---

# 4️⃣ Flexbox

React Native uses **Flexbox by default**.

### Default Values

```text
flexDirection : column
justifyContent : flex-start
alignItems : stretch
```

## flexDirection

```jsx
<View style={{ flexDirection: "row" }}>
  <Text>A</Text>
  <Text>B</Text>
</View>
```

## justifyContent

- flex-start
- center
- flex-end
- space-between
- space-around
- space-evenly

```jsx
<View style={{ justifyContent: "center", flex: 1 }}>
  <Text>Centered</Text>
</View>
```

## alignItems

```jsx
<View style={{ alignItems: "center" }}>
  <Text>Hello</Text>
</View>
```

## flex

```jsx
<View style={{ flex: 1 }} />
```

---

# 5️⃣ Position

Position controls where an element appears.

## Relative (Default)

```jsx
<View style={{ position: "relative" }} />
```

## Absolute

```jsx
<View
  style={{
    position: "absolute",
    top: 20,
    right: 20,
  }}
/>
```

### Common Properties

- top
- bottom
- left
- right

---

# 6️⃣ Z-Index

`zIndex` controls which component appears on top.

```jsx
<View style={{ zIndex: 2 }} />
```

Higher value = displayed above lower values.

---

# 7️⃣ Responsive Design

A responsive UI adapts to different screen sizes.

## Using Flex

```jsx
<View style={{ flex: 1 }} />
```

## Using Dimensions

```jsx
import { Dimensions } from "react-native";

const width = Dimensions.get("window").width;
```

## Using Percentages

```jsx
<View style={{ width: "90%" }} />
```

### Best Practices

- Prefer Flexbox
- Avoid fixed widths
- Test on multiple screen sizes

---

# 8️⃣ Platform Specific Styling

Android and iOS sometimes require different styles.

```jsx
import { Platform } from "react-native";

const styles = {
  text: {
    fontSize: Platform.OS === "ios" ? 20 : 18,
  },
};
```

# 1️⃣1️⃣ Themes

A theme stores reusable colors, spacing, and typography.

Example:

```js
export const theme = {
  colors: {
    primary: "#2563EB",
    background: "#FFFFFF",
    text: "#111827",
  },
};
```

Usage:

```jsx
<Text style={{ color: theme.colors.primary }}>
  Hello
</Text>
```

---

# 1️⃣2️⃣ Dark Mode

Dark mode changes the app appearance based on the user's device theme.

```jsx
import { useColorScheme } from "react-native";

const scheme = useColorScheme();

const background =
  scheme === "dark" ? "#000" : "#FFF";
```

### Benefits

- Better battery life (OLED)
- Comfortable at night
- Modern user experience

# Common Mistakes

- Writing CSS files
- Overusing inline styles
- Forgetting `flex: 1`
- Using fixed widths everywhere
- Ignoring different screen sizes
- Not testing on Android and iOS

#  Key Takeaways

- React Native uses JavaScript objects instead of CSS.
- Prefer `StyleSheet.create()` for production apps.
- Flexbox is the primary layout system.
- Use `Dimensions` and Flexbox for responsive design.
- Themes improve code reuse.
- Dark Mode enhances user experience.
