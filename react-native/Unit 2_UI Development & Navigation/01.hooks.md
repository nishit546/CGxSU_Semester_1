
# React Native – React Hooks

**Description**: Hooks are the **heart of modern React Native development**. They allow you to use state, lifecycle methods, context, and other React features directly in functional components — without writing class components.

### **What are Hooks?**
Hooks were introduced in React 16.8. They let you “hook into” React features like state management, side effects, performance optimization, and more.

**React Native uses the exact same Hooks as React.js.**

Hooks make your code:
- Cleaner and more readable
- Easier to reuse (via Custom Hooks)
- Simpler to test
- More performant

---

### **Basic Complete Example**
```jsx
import React, { useState, useEffect } from 'react';
import { View, Text, Pressable, StyleSheet } from 'react-native';

const HooksDemo = () => {
  const [count, setCount] = useState(0);
  const [isActive, setIsActive] = useState(false);

  useEffect(() => {
    let timer;
    if (isActive) {
      timer = setInterval(() => {
        setCount(prev => prev + 1);
      }, 1000);
    }
    return () => clearInterval(timer);
  }, [isActive]);

  return (
    <View style={styles.container}>
      <Text style={styles.title}>React Hooks in Action</Text>
      
      <Text style={styles.count}>{count}</Text>
      
      <Pressable
        style={[styles.button, isActive && styles.activeButton]}
        onPress={() => setIsActive(!isActive)}
      >
        <Text style={styles.buttonText}>
          {isActive ? 'Stop Timer' : 'Start Timer'}
        </Text>
      </Pressable>

      <Pressable style={styles.resetButton} onPress={() => setCount(0)}>
        <Text style={styles.buttonText}>Reset Counter</Text>
      </Pressable>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#f8f9fa',
  },
  title: {
    fontSize: 28,
    fontWeight: 'bold',
    marginBottom: 40,
    color: '#007AFF',
  },
  count: {
    fontSize: 72,
    fontWeight: 'bold',
    marginBottom: 40,
  },
  button: {
    backgroundColor: '#007AFF',
    paddingVertical: 16,
    paddingHorizontal: 40,
    borderRadius: 12,
    marginBottom: 16,
  },
  activeButton: {
    backgroundColor: '#FF3B30',
  },
  resetButton: {
    backgroundColor: '#8E8E93',
    paddingVertical: 16,
    paddingHorizontal: 40,
    borderRadius: 12,
  },
  buttonText: {
    color: '#fff',
    fontSize: 17,
    fontWeight: '600',
  },
});

export default HooksDemo;
```

---

### **Rules of Hooks (Must Follow)**

| Rule | Explanation |
|------|-------------|
| **Top Level** | Only call hooks at the top level (never inside loops, conditions, or nested functions) |
| **React Functions** | Only call hooks from React functional components or other custom hooks |
| **Order Matters** | Hooks must be called in the same order on every render |

---

### **Most Important Hooks Overview**

| Hook            | Purpose                              | When to Use |
|-----------------|--------------------------------------|-----------|
| `useState`      | Local state management               | Counters, forms, toggles |
| `useEffect`     | Side effects                         | Data fetching, subscriptions, timers |
| `useContext`    | Global/shared state                  | Theme, Auth, Language |
| `useRef`        | Mutable reference                    | Focus, timers, DOM access |
| `useCallback`   | Memoize functions                    | Performance with child components |
| `useMemo`       | Memoize expensive values             | Filtering, heavy calculations |
| `useReducer`    | Complex state logic                  | Large forms, state machines |

---

### **Before vs After Hooks**

**Class Component (Old Way)**
```jsx
class Counter extends React.Component {
  state = { count: 0 };
  componentDidMount() { ... }
}
```

**Functional Component with Hooks (Modern Way)**
```jsx
const Counter = () => {
  const [count, setCount] = useState(0);
  useEffect(() => { ... }, []);
};
```

**Advantages of Hooks:**
- Cleaner and shorter code
- Better reusability through Custom Hooks
- Easier to test
- No `this` binding issues
- Better performance and tree shaking

## 1. useState – Managing State

**Description**: `useState` is the most fundamental React Hook. It allows functional components to have and manage their own state.

### **What is useState?**
`useState` returns an array with two values: the current state and a function to update it. Whenever you call the setter function, React re-renders the component with the new state.

---

| Concept                    | Description                                      | Example |
|---------------------------|--------------------------------------------------|--------|
| Initial Value             | Default state value                              | `useState(0)` |
| Functional Update         | Safe update based on previous state              | `setCount(prev => prev + 1)` |
| Multiple State Variables  | Separate concerns                                | `useState` for each value |
| Object State              | For related data                                 | `useState({ name: '', email: '' })` |

---

### **Common Use Cases**

- Form inputs (`email`, `password`, etc.)
- Toggle switches (`isVisible`, `isDarkMode`)
- Loading states (`loading`, `error`)
- Counters and quantities
- Lists and arrays
- Theme switching
- Modal visibility

---

**Best Practices:**
- Use functional updates when new state depends on old state
- Split state into multiple `useState` calls instead of one large object (unless using `useReducer`)
- Keep state minimal
- Avoid putting derived values in state

## 2. useEffect – Side Effects

**Description**: `useEffect` is a React Hook that lets you perform side effects in functional components. It runs after the render is committed to the screen.

### **What is useEffect?**
Side effects include data fetching, subscriptions, timers, manual DOM manipulations, and cleanup operations. `useEffect` helps you manage these without blocking the UI.

---

### **Basic Complete Example**
```jsx
import React, { useState, useEffect } from 'react';
import {
  View,
  Text,
  StyleSheet,
  ActivityIndicator,
  Pressable,
  Alert,
} from 'react-native';

const UseEffectScreen = () => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  const [count, setCount] = useState(0);

  // 1. Fetch data on mount (runs once)
  useEffect(() => {
    const fetchData = async () => {
      try {
        setLoading(true);
        // Simulate API call
        await new Promise(resolve => setTimeout(resolve, 1500));

        setData({
          title: 'Welcome to React Native',
          users: 12480,
          status: 'success',
        });
      } catch (err) {
        setError('Failed to load data');
      } finally {
        setLoading(false);
      }
    };

    fetchData();

    // Cleanup function (optional)
    return () => {
      console.log('Cleanup on unmount');
    };
  }, []); // Empty dependency array = run once on mount

  // 2. Runs when count changes
  useEffect(() => {
    if (count > 0) {
      console.log(`Count changed to: ${count}`);
    }
  }, [count]);

  const refetch = () => {
    setData(null);
    setError(null);
    setLoading(true);

    // Trigger the effect again (by changing a dependency or using a key)
    setTimeout(() => setCount(prev => prev + 1), 100);
  };

  if (loading) {
    return (
      <View style={styles.center}>
        <ActivityIndicator size="large" color="#007AFF" />
        <Text style={styles.loadingText}>Loading data...</Text>
      </View>
    );
  }

  return (
    <View style={styles.container}>
      <Text style={styles.title}>useEffect Examples</Text>

      {error ? (
        <Text style={styles.error}>{error}</Text>
      ) : (
        data && (
          <View style={styles.card}>
            <Text style={styles.cardTitle}>{data.title}</Text>
            <Text style={styles.cardText}>Total Users: {data.users}</Text>
            <Text style={styles.cardText}>Status: {data.status}</Text>
          </View>
        )
      )}

      <Pressable style={styles.button} onPress={refetch}>
        <Text style={styles.buttonText}>Refetch Data</Text>
      </Pressable>

      <Pressable style={styles.countButton} onPress={() => setCount(c => c + 1)}>
        <Text style={styles.buttonText}>Count: {count}</Text>
      </Pressable>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
  },
  center: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    textAlign: 'center',
    marginBottom: 30,
  },
  card: {
    backgroundColor: '#fff',
    padding: 20,
    borderRadius: 12,
    marginBottom: 20,
  },
  cardTitle: {
    fontSize: 18,
    fontWeight: '600',
    marginBottom: 8,
  },
  cardText: {
    fontSize: 16,
    marginBottom: 6,
  },
  button: {
    backgroundColor: '#007AFF',
    padding: 16,
    borderRadius: 12,
    alignItems: 'center',
    marginBottom: 12,
  },
  countButton: {
    backgroundColor: '#34C759',
    padding: 16,
    borderRadius: 12,
    alignItems: 'center',
  },
  buttonText: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },
  loadingText: {
    marginTop: 12,
    fontSize: 16,
    color: '#666',
  },
  error: {
    color: 'red',
    fontSize: 16,
    textAlign: 'center',
  },
});

export default UseEffectScreen;
```

---

### **Dependency Array Guide**

| Dependency Array       | When it Runs                              | Common Use Case |
|------------------------|-------------------------------------------|-----------------|
| `[]` (empty)           | Only on mount                             | Fetch data, setup subscriptions |
| `[dep1, dep2]`         | On mount + when dependencies change       | Data fetching based on ID |
| No array               | After **every** render                    | Usually avoided |
| Cleanup function       | Before next effect / on unmount           | Clear timers, subscriptions |

---

### **More Examples**

#### **1. Timer / Interval**
```jsx
useEffect(() => {
  const id = setInterval(() => {}, 1000);
  return () => clearInterval(id);
}, []);
```

#### **2. Event Listener**
```jsx
useEffect(() => {
  const subscription = someAPI.subscribe();
  return () => subscription.unsubscribe();
}, []);
```

#### **3. Document Title Update**
```jsx
useEffect(() => {
  document.title = `You clicked ${count} times`;
}, [count]);
```

#### **4. Conditional Effect**
```jsx
useEffect(() => {
  if (userId) fetchUser(userId);
}, [userId]);
```

---

**Best Practices:**
- Always include all dependencies in the array
- Use cleanup functions to prevent memory leaks
- Separate different side effects into multiple `useEffect` hooks
- Avoid putting objects/arrays directly in dependency array without memoization

## 3. useRef – Mutable Reference

**Description**: `useRef` returns a mutable object that persists across renders without triggering a re-render when its value changes. It’s commonly used to access DOM elements (or components) and store values that should survive re-renders.

### **What is useRef?**
`useRef` creates a box that can hold a mutable value. The `.current` property can be updated freely. Unlike `useState`, changing a ref does **not** cause the component to re-render.

---

### **Basic Complete Example**
```jsx
import React, { useRef, useState, useEffect } from 'react';
import {
  View,
  TextInput,
  Text,
  Pressable,
  StyleSheet,
  Alert,
} from 'react-native';

const UseRefScreen = () => {
  const inputRef = useRef(null);
  const timerRef = useRef(null);
  const [seconds, setSeconds] = useState(0);
  const [isRunning, setIsRunning] = useState(false);

  // Focus input on screen load
  useEffect(() => {
    inputRef.current?.focus();
  }, []);

  const focusInput = () => {
    inputRef.current?.focus();
  };

  const startTimer = () => {
    if (timerRef.current) return;

    setIsRunning(true);
    timerRef.current = setInterval(() => {
      setSeconds(prev => prev + 1);
    }, 1000);
  };

  const stopTimer = () => {
    if (timerRef.current) {
      clearInterval(timerRef.current);
      timerRef.current = null;
    }
    setIsRunning(false);
  };

  const resetTimer = () => {
    stopTimer();
    setSeconds(0);
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>useRef Demo</Text>

      {/* Input Focus Example */}
      <TextInput
        ref={inputRef}
        style={styles.input}
        placeholder="Click button to focus me..."
      />

      <Pressable style={styles.button} onPress={focusInput}>
        <Text style={styles.buttonText}>Focus Input</Text>
      </Pressable>

      {/* Timer Example */}
      <View style={styles.timerContainer}>
        <Text style={styles.timer}>
          {seconds} seconds
        </Text>

        <View style={styles.timerButtons}>
          <Pressable style={styles.timerButton} onPress={startTimer} disabled={isRunning}>
            <Text style={styles.timerButtonText}>Start</Text>
          </Pressable>

          <Pressable style={styles.timerButton} onPress={stopTimer} disabled={!isRunning}>
            <Text style={styles.timerButtonText}>Stop</Text>
          </Pressable>

          <Pressable style={styles.timerButton} onPress={resetTimer}>
            <Text style={styles.timerButtonText}>Reset</Text>
          </Pressable>
        </View>
      </View>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    textAlign: 'center',
    marginBottom: 30,
  },
  input: {
    height: 52,
    borderWidth: 1,
    borderColor: '#ddd',
    borderRadius: 12,
    paddingHorizontal: 16,
    fontSize: 16,
    backgroundColor: '#fff',
    marginBottom: 16,
  },
  button: {
    backgroundColor: '#007AFF',
    paddingVertical: 14,
    borderRadius: 12,
    alignItems: 'center',
    marginBottom: 40,
  },
  buttonText: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },
  timerContainer: {
    alignItems: 'center',
  },
  timer: {
    fontSize: 48,
    fontWeight: 'bold',
    marginBottom: 20,
  },
  timerButtons: {
    flexDirection: 'row',
    gap: 12,
  },
  timerButton: {
    backgroundColor: '#34C759',
    paddingVertical: 12,
    paddingHorizontal: 24,
    borderRadius: 10,
  },
  timerButtonText: {
    color: '#fff',
    fontWeight: '600',
  },
});

export default UseRefScreen;
```

---

### **Important Use Cases**

| Use Case                    | How to Use `useRef`                              |
|----------------------------|--------------------------------------------------|
| Access component (focus)   | `inputRef.current.focus()`                       |
| Store timer IDs            | `timerRef.current = setInterval(...)`            |
| Track previous values      | `prevRef.current = value`                        |
| Scroll to position         | `flatListRef.current.scrollToOffset(...)`        |
| Store values without re-render | Any mutable data                               |

---

### **More Examples**

#### **1. Store Previous Value**
```jsx
const prevCountRef = useRef();
useEffect(() => {
  prevCountRef.current = count;
}, [count]);
```

#### **2. ScrollView / FlatList Reference**
```jsx
const listRef = useRef(null);
listRef.current.scrollToEnd({ animated: true });
```

#### **3. Animation Values**
```jsx
const animationRef = useRef(new Animated.Value(0));
```

#### **4. Cleanup on Unmount**
```jsx
useEffect(() => {
  return () => {
    if (timerRef.current) clearInterval(timerRef.current);
  };
}, []);
```

---

**Key Points:**
- `useRef` does **not** trigger re-renders
- The object is mutable via `.current`
- Very useful for imperative operations (focus, scroll, timers, etc.)
- Persists for the entire lifetime of the component

## 4. useMemo – Performance Optimization

**Description**: `useMemo` is a React Hook that memoizes (caches) the result of an expensive calculation. It only recalculates the value when its dependencies change.

### **What is useMemo?**
`useMemo` helps improve performance by preventing unnecessary recalculations on every render. It is especially useful for heavy computations, filtering or sorting large lists, and complex object creation.

---

### **Basic Complete Example**
```jsx
import React, { useState, useMemo } from 'react';
import { View, TextInput, Text, FlatList, StyleSheet } from 'react-native';

const UseMemoScreen = () => {
  const [searchTerm, setSearchTerm] = useState('');
  const [users] = useState([
    { id: 1, name: 'Alice Johnson', age: 28 },
    { id: 2, name: 'Bob Smith', age: 34 },
    { id: 3, name: 'Charlie Brown', age: 22 },
    { id: 4, name: 'Diana Prince', age: 31 },
    { id: 5, name: 'Eve Adams', age: 25 },
    // ... imagine 1000+ users
  ]);

  // Expensive operation: filtering + sorting
  const filteredAndSortedUsers = useMemo(() => {
    console.log(' Filtering and sorting users...'); // For demo

    return users
      .filter(user =>
        user.name.toLowerCase().includes(searchTerm.toLowerCase())
      )
      .sort((a, b) => a.name.localeCompare(b.name));
  }, [users, searchTerm]); // Only recompute when these change

  return (
    <View style={styles.container}>
      <Text style={styles.title}>useMemo Demo</Text>

      <TextInput
        style={styles.searchInput}
        placeholder="Search users..."
        value={searchTerm}
        onChangeText={setSearchTerm}
      />

      <Text style={styles.resultCount}>
        Results: {filteredAndSortedUsers.length}
      </Text>

      <FlatList
        data={filteredAndSortedUsers}
        keyExtractor={item => item.id.toString()}
        renderItem={({ item }) => (
          <View style={styles.userCard}>
            <Text style={styles.name}>{item.name}</Text>
            <Text style={styles.age}>Age: {item.age}</Text>
          </View>
        )}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    textAlign: 'center',
    marginBottom: 20,
  },
  searchInput: {
    height: 50,
    borderWidth: 1,
    borderColor: '#ddd',
    borderRadius: 12,
    paddingHorizontal: 16,
    fontSize: 16,
    marginBottom: 16,
    backgroundColor: '#fff',
  },
  resultCount: {
    fontSize: 16,
    marginBottom: 12,
    color: '#666',
  },
  userCard: {
    backgroundColor: '#fff',
    padding: 16,
    borderRadius: 10,
    marginVertical: 6,
  },
  name: {
    fontSize: 17,
    fontWeight: '600',
  },
  age: {
    fontSize: 15,
    color: '#666',
    marginTop: 4,
  },
});

export default UseMemoScreen;
```

---

### **Important Parameters**

| Parameter            | Description                                      | Example |
|----------------------|--------------------------------------------------|--------|
| Computation function | The expensive function to memoize                | `() => heavyCalculation()` |
| Dependency array     | When to recalculate the value                    | `[users, searchTerm]` |

---

### **More Examples**

#### **1. Expensive Calculation**
```jsx
const totalPrice = useMemo(() => {
  return cart.reduce((sum, item) => sum + item.price * item.quantity, 0);
}, [cart]);
```

#### **2. Complex Object Creation**
```jsx
const userInfo = useMemo(() => ({
  fullName: `${firstName} ${lastName}`,
  isAdult: age >= 18,
  greeting: `Hello, ${firstName}!`,
}), [firstName, lastName, age]);
```

#### **3. Memoized Styles (Advanced)**
```jsx
const containerStyle = useMemo(() => ({
  backgroundColor: isDark ? '#121212' : '#ffffff',
  padding: 16,
}), [isDark]);
```

#### **4. Filtering Large Datasets**
```jsx
const visibleItems = useMemo(() => {
  return data.filter(item => item.status === 'active');
}, [data]);
```

---

**When to use `useMemo`?**

- Heavy computations (filtering, mapping, reducing large arrays)
- Creating expensive objects or arrays
- Preventing child components from re-rendering unnecessarily
- When the computation is inside a component that re-renders frequently

**Note**: Don’t overuse `useMemo`. It has its own overhead. Use it only when you have actual performance issues.

---

## 5. useCallback – Memoize Functions

**Description**: `useCallback` is a React Hook that returns a memoized version of a callback function. It helps prevent unnecessary re-creations of functions on every render, improving performance.

### **What is useCallback?**
By default, functions defined inside a component are recreated on every render. `useCallback` caches the function and only returns a new one when its dependencies change. It is especially useful when passing callbacks to child components that are wrapped with `React.memo`.

---

### **Basic Complete Example**
```jsx
import React, { useState, useCallback } from 'react';
import { View, Text, Pressable, StyleSheet, FlatList } from 'react-native';

const ChildItem = React.memo(({ title, onPress }) => {
  console.log(`Rendered: ${title}`); // For performance demo
  return (
    <Pressable style={styles.item} onPress={onPress}>
      <Text style={styles.itemText}>{title}</Text>
    </Pressable>
  );
});

const UseCallbackScreen = () => {
  const [count, setCount] = useState(0);
  const [items] = useState([
    'Apple', 'Banana', 'Cherry', 'Date', 'Elderberry'
  ]);

  // Without useCallback: This function would be recreated on every render
  // With useCallback: It only changes when items change
  const handleItemPress = useCallback((itemName) => {
    console.log(`You pressed: ${itemName}`);
    Alert.alert('Item Selected', `You chose ${itemName}`);
  }, []); // Empty dependency array = stable function

  const incrementCount = useCallback(() => {
    setCount(prev => prev + 1);
  }, []);

  return (
    <View style={styles.container}>
      <Text style={styles.title}>useCallback Demo</Text>
      
      <Text style={styles.counter}>Count: {count}</Text>

      <Pressable style={styles.button} onPress={incrementCount}>
        <Text style={styles.buttonText}>Increment Count</Text>
      </Pressable>

      <FlatList
        data={items}
        keyExtractor={(item, index) => index.toString()}
        renderItem={({ item }) => (
          <ChildItem 
            title={item} 
            onPress={() => handleItemPress(item)} 
          />
        )}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    textAlign: 'center',
    marginBottom: 20,
  },
  counter: {
    fontSize: 18,
    textAlign: 'center',
    marginBottom: 16,
  },
  button: {
    backgroundColor: '#007AFF',
    padding: 14,
    borderRadius: 10,
    alignItems: 'center',
    marginBottom: 20,
  },
  buttonText: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },
  item: {
    padding: 16,
    backgroundColor: '#fff',
    borderRadius: 8,
    marginVertical: 6,
  },
  itemText: {
    fontSize: 16,
  },
});

export default UseCallbackScreen;
```

---

### **Important Props / Parameters**

| Parameter            | Description                                      | Common Usage |
|----------------------|--------------------------------------------------|--------------|
| Callback function    | The function to memoize                          | Required |
| Dependency array     | When to recreate the function                    | `[dep1, dep2]` |
| Empty array `[]`     | Function created only once                       | Most stable |
| No array             | Behaves like normal function (not recommended)   | Avoid |

---

### **More Examples**

#### **1. Basic useCallback**
```jsx
const handleSubmit = useCallback((data) => {
  console.log('Submitted:', data);
}, [data]); // Include dependencies
```

#### **2. With Event Handlers**
```jsx
const onItemDelete = useCallback((id) => {
  setItems(prev => prev.filter(item => item.id !== id));
}, []);
```

#### **3. Passing to Memoized Child**
```jsx
<MemoizedChild onPress={handlePress} />
```

#### **4. Combined with useMemo**
```jsx
const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);
```

#### **5. useCallback for API Calls**
```jsx
const fetchData = useCallback(async () => {
  const res = await api.get('/data');
  setData(res.data);
}, []);
```

---

**When to use `useCallback`?**

- Passing functions to `React.memo` children
- Passing callbacks to `useEffect`
- When the function is expensive to recreate
- Stabilizing functions in custom hooks

**Performance Note**:  
Don’t overuse `useCallback`. Only apply it when you actually have performance issues or when passing callbacks to memoized components.

## 6. useContext – Global State

**Description**: `useContext` is a React Hook that allows you to share data across your component tree without passing props down manually (prop drilling). It’s perfect for global data like theme, user info, authentication, language, etc.

### **What is useContext?**
`Context` provides a way to pass data through the component tree without having to pass props at every level. You create a Context, provide the value at a high level, and consume it anywhere below in the tree.

---

### **Basic Complete Example**

#### **1. Creating the Context**
```jsx
// context/ThemeContext.js
import React, { createContext, useState, useContext } from 'react';

const ThemeContext = createContext();

export const ThemeProvider = ({ children }) => {
  const [theme, setTheme] = useState('light');

  const toggleTheme = () => {
    setTheme(prev => prev === 'light' ? 'dark' : 'light');
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
};

export const useTheme = () => {
  const context = useContext(ThemeContext);
  if (!context) {
    throw new Error('useTheme must be used within a ThemeProvider');
  }
  return context;
};
```

#### **2. Usage in App**
```jsx
// App.js
import React from 'react';
import { ThemeProvider } from './context/ThemeContext';
import HomeScreen from './screens/HomeScreen';

const App = () => {
  return (
    <ThemeProvider>
      <HomeScreen />
    </ThemeProvider>
  );
};

export default App;
```

#### **3. Consuming Context**
```jsx
// screens/HomeScreen.js
import React from 'react';
import { View, Text, Pressable, StyleSheet } from 'react-native';
import { useTheme } from '../context/ThemeContext';

  const isDark = theme === 'dark';

  return (
    <View style={[styles.container, isDark && styles.darkContainer]}>
      <Text style={[styles.text, isDark && styles.darkText]}>
        Current Theme: {theme.toUpperCase()}
      </Text>

      <Pressable style={styles.button} onPress={toggleTheme}>
        <Text style={styles.buttonText}>
          Switch to {isDark ? 'Light' : 'Dark'} Mode
        </Text>
      </Pressable>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#ffffff',
  },
  darkContainer: {
    backgroundColor: '#121212',
  },
  text: {
    fontSize: 24,
    marginBottom: 30,
  },
  darkText: {
    color: '#ffffff',
  },
  button: {
    backgroundColor: '#007AFF',
    paddingVertical: 14,
    paddingHorizontal: 32,
    borderRadius: 12,
  },
  buttonText: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },
});

export default HomeScreen;
```

---

### **Important Concepts**

| Concept                | Description                                      | Usage |
|------------------------|--------------------------------------------------|-------|
| `createContext()`      | Creates a new Context object                     | Top level |
| `Provider`             | Provides the value to all children               | Wrap your app / section |
| `useContext()`         | Consumes the context value                       | Inside components |
| Custom Hook (`useTheme`) | Makes usage easier and adds safety             | Recommended |

---

### **More Examples**

#### **1. Auth Context**
```jsx

```

#### **2. Language / i18n Context**
```jsx

```

```jsx

```

#### **4. Multiple Contexts**
You can use multiple contexts in the same component.

#### **5. Default Values**
```jsx

```

---

**Best Practices for Context:**
- Create a custom hook for each context
- Always check if context exists (throw error if not)
- Keep context values minimal (avoid putting everything in one context)
- Split large contexts into smaller ones when needed

**When to use Context:**
- Theme
- Authentication
- User data
- Global settings
- Shopping cart
- Language preference

## 7. Custom Hooks

**Description**: Custom Hooks allow you to extract and reuse stateful logic between components. They are JavaScript functions that start with `use` and can call other React hooks.

### **What are Custom Hooks?**
Custom Hooks help you follow the **DRY** (Don’t Repeat Yourself) principle. Instead of duplicating logic (like form handling, API calls, toggles, timers, etc.) across multiple components, you can create reusable hooks.

They make your code cleaner, more testable, and easier to maintain.

---

### **Basic Complete Example**

#### **1. `useToggle` Hook**
```jsx
// hooks/useToggle.js
import { useState } from 'react';

export const useToggle = (initialValue = false) => {
  const [value, setValue] = useState(initialValue);

  const toggle = () => setValue(prev => !prev);
  const setTrue = () => setValue(true);
  const setFalse = () => setValue(false);

  return [value, toggle, setTrue, setFalse];
};
```

#### **2. Usage Example**
```jsx
import React from 'react';
import { View, Text, Pressable, StyleSheet } from 'react-native';
import { useToggle } from './hooks/useToggle';

const CustomHookScreen = () => {
  const [isVisible, toggleVisibility] = useToggle(false);
  const [isDarkMode, toggleDarkMode] = useToggle(false);

  return (
    <View style={[styles.container, isDarkMode && styles.darkMode]}>
      <Text style={[styles.text, isDarkMode && styles.darkText]}>
        {isVisible ? 'Hello, I am Visible!' : 'I am Hidden'}
      </Text>

      <Pressable style={styles.button} onPress={toggleVisibility}>
        <Text style={styles.buttonText}>
          {isVisible ? 'Hide Text' : 'Show Text'}
        </Text>
      </Pressable>

      <Pressable style={styles.button} onPress={toggleDarkMode}>
        <Text style={styles.buttonText}>
          Toggle {isDarkMode ? 'Light' : 'Dark'} Mode
        </Text>
      </Pressable>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#fff',
  },
  darkMode: {
    backgroundColor: '#121212',
  },
  text: {
    fontSize: 20,
    marginBottom: 30,
  },
  darkText: {
    color: '#fff',
  },
  button: {
    backgroundColor: '#007AFF',
    paddingVertical: 12,
    paddingHorizontal: 24,
    borderRadius: 10,
    marginVertical: 10,
  },
  buttonText: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },
});

export default CustomHookScreen;
```

### **Important Built-in Hooks (Quick Reference)**

| Need                                | Recommended Hook       |
|-------------------------------------|------------------------|
| Local State                         | `useState`             |
| Side Effects (API, timers, subscriptions) | `useEffect`       |
| Mutable value without re-render     | `useRef`               |
| Expensive calculations              | `useMemo`              |
| Memoized callback functions         | `useCallback`          |
| Global / Shared state               | `useContext`           |
| Complex state logic                 | `useReducer`           |
| Reusable logic across components    | **Custom Hook**        |

### **More Useful Custom Hooks Examples**

#### **1. `useForm` Hook (Very Popular)**
```jsx
export const useForm = (initialValues = {}) => {
  const [values, setValues] = useState(initialValues);
  const [errors, setErrors] = useState({});

  const handleChange = (name, value) => {
    setValues(prev => ({ ...prev, [name]: value }));
    if (errors[name]) setErrors(prev => ({ ...prev, [name]: '' }));
  };

  return { values, errors, handleChange, setErrors };
};
```

#### **2. `useDebounce` Hook**
```jsx
export const useDebounce = (value, delay = 500) => {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => clearTimeout(handler);
  }, [value, delay]);

  return debouncedValue;
};
```

#### **3. `useFetch` Hook**
```jsx
export const useFetch = (url) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  // Fetch logic...
};
```

#### **4. `useLocalStorage` (with AsyncStorage)**
Useful for persisting data in React Native apps.

#### **5. `useKeyboard`**
Detect when keyboard is shown/hidden.

---

**Benefits of Custom Hooks:**
- Reusability
- Better code organization
- Easier testing
- Separation of concerns
- Cleaner components

**Rule**: Always start your custom hook name with `use` (e.g., `useAuth`, `useTimer`, `useLocation`).
