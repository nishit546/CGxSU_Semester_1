# üß© React Native Components

Components are the **building blocks of a React Native application**.  
Every screen, button, text, or image you see in a React Native app is made using components.

---

## What is a Component?

A **component** is a reusable piece of UI that controls how a part of the app looks and behaves.

In React Native:
- Everything is a component
- Components can be reused
- Components can accept data using **props**
- Components can manage data using **state**

---

## üß© Core Built-in Components

React Native provides many **pre-built components** to build UI.

---

### View
- Works like a `div` in web
- Used for layout and grouping

```jsx
<View>
  <Text>Inside View</Text>
</View>
```
---

### Text
- Used to display text
- All text must be inside a `Text` component

```jsx
<Text>Hello World</Text>
```
> ‚ö†Ô∏è In React Native, you **cannot** place plain text directly inside `View`.  

---

### Image
- Used to display images

```jsx
  <Image
  source={{ uri: "https://example.com/image.png" }}
  style={{ width: 100, height: 100 }}
/>
```
> ‚ÑπÔ∏è In React Native, images **must** have width and height.

---

### ScrollView
- Enables scrolling when content overflows

```jsx
  <ScrollView>
  <Text>Scrollable content</Text>
</ScrollView>
```
---

### Pressable
- Used for touch interactions (buttons)

```jsx
  <Pressable onPress={() => alert("Pressed!")}>
  <Text>Click Me</Text>
</Pressable>
```
> ‚ÑπÔ∏è `Pressable` is preferred over `Button` for custom styling.

## üì¶ Props (Passing Data to Components)

Props allow you to pass data from one component to another.

```jsx
function Greeting(props) {
  return <Text>Hello {props.name}</Text>;
}

<Greeting name="Rajesh" />
```
üëâ **Props are read-only.**

---

## üîÅ State (Managing Data Inside a Component)

State is used to store and update data inside a component.

```jsx
import { useState } from "react";
import { View, Text, Pressable } from "react-native";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <View>
      <Text>Count: {count}</Text>
      <Pressable onPress={() => setCount(count + 1)}>
        <Text>Increase</Text>
      </Pressable>
    </View>
  );
}
```
üëâ When state changes, the UI automatically updates.

---

## üß† Stateless vs Stateful Components

- **Stateless Component**
  - Uses only props
  - Does not manage its own data

- **Stateful Component**
  - Uses state
  - Manages dynamic data and UI updates

## ‚ùì When Should You Create a New Component?

Create a new component when:
- UI is reused multiple times
- Code becomes too long
- Logic can be separated
- UI represents a single responsibility

---

## ‚úÖ Best Practices for Components

- Keep components **small and focused**
- Reuse components whenever possible
- Separate UI and logic when possible
- Use meaningful component names
- Prefer functional components

---

## üìù Summary

- Components are the foundation of React Native apps
- Everything you see is a component
- Functional components are recommended
- Props pass data, state manages data
- Reusable components make apps scalable
