---

# 1️⃣ What are Permissions?

Permissions allow your app to access protected device resources.

Examples:

- Camera
- Photos
- Microphone
- Location
- Contacts
- Notifications

Without permission, the operating system blocks access.

---

# 2️⃣ Permission Flow

```text
User Opens Feature
        │
Request Permission
        │
Granted?
 ├── Yes → Access Feature
 └── No  → Explain & Handle Gracefully
```

---

# 3️⃣ Android Permissions

Android requests many permissions at runtime.

Common permissions:

- Camera
- Storage / Photos
- Microphone
- Fine Location
- Coarse Location

### Best Practices

- Ask only when required.
- Explain why the permission is needed.
- Don't request every permission on app launch.

---

# 4️⃣ iOS Permissions

iOS also requires runtime permission and displays a system dialog.

Important points:

- Permission prompts are shown only once.
- If denied, users must enable permissions from **Settings**.
- Always provide a clear reason before requesting access.

# 5️⃣ Camera Permission

Using Expo Camera:

```bash
npx expo install expo-camera
```

```jsx
import { useCameraPermissions } from "expo-camera";

const [permission, requestPermission] =
  useCameraPermissions();

if (!permission?.granted) {
  await requestPermission();
}
```

Use Cases:

- QR Scanner
- Profile Photo
- Document Scanner

---

# 6️⃣ Gallery Permission

Using Expo Image Picker:

```bash
npx expo install expo-image-picker
```

```jsx
import * as ImagePicker from "expo-image-picker";

await ImagePicker.requestMediaLibraryPermissionsAsync();
```

Use Cases:

- Profile Image
- Product Images
- Photo Upload

---

# 7️⃣ Microphone Permission

Required for:

- Voice Notes
- Audio Recording
- Voice Search

```bash
npx expo install expo-av
```

```jsx
import { Audio } from "expo-av";

await Audio.requestPermissionsAsync();
```

---

# 8️⃣ Location Permission

Install:

```bash
npx expo install expo-location
```

Foreground permission:

```jsx
import * as Location from "expo-location";

await Location.requestForegroundPermissionsAsync();
```

Background location (advanced apps):

```jsx
await Location.requestBackgroundPermissionsAsync();
```

Use Cases:

- Ride Sharing
- Food Delivery
- Maps
- Attendance

---

# 9️⃣ Permission Status

Typical states:

| Status | Meaning |
|--------|---------|
| granted | Access allowed |
| denied | User denied |
| undetermined | Not requested yet |

Always check the status before using a device feature.

---

# 🔟 Handling Denied Permissions

```text
Permission Denied
        │
Explain Why Access Is Needed
        │
Ask Again (if appropriate)
        │
Open Device Settings
```

Never crash the app if permission is denied.

---

#  Production Best Practices

- Request permissions only when needed.
- Show an explanation before the system dialog.
- Handle denial gracefully.
- Disable unavailable features instead of crashing.
- Test on both Android and iOS.
- Keep permission requests contextual.

---

#  Common Mistakes

- Asking all permissions at startup
- Ignoring denied permissions
- Assuming permission is always granted
- Not testing on real devices
- Re-requesting permissions repeatedly
