
# React Native - Images & Assets

Images and assets play a vital role in creating beautiful and engaging mobile applications. This module covers everything you need to know about handling images, icons, SVGs, and other assets in React Native.

---

## What are Assets?

**Description**: Assets are files used in your app such as images, icons, fonts, videos, audio files, SVGs, and documents.

### **What are Assets in React Native?**

In React Native, **assets** refer to any static files that are part of your application. These files are either bundled with your app or loaded from the internet.

Proper management of assets is essential for:
- Good performance
- Professional UI/UX
- Smaller app size
- Better maintainability

---

### **Recommended Project Structure**

```bash
assets/
├── images/           # PNG, JPG, WebP photos
├── icons/            # Small icons (PNG/SVG)
├── fonts/            # Custom font files (.ttf, .otf)
├── svg/              # SVG vector files
├── videos/           # Video files
├── illustrations/    # App illustrations and onboarding images
└── animations/       # Lottie JSON files (optional)
```

---

### **Basic Example - Using a Local Asset**

```jsx
import React from 'react';
import { Image, View, StyleSheet } from 'react-native';

const App = () => {
  return (
    <View style={styles.container}>
      <Image
        source={require('./assets/images/welcome.png')}
        style={styles.image}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
  },
  image: {
    width: 280,
    height: 280,
  },
});

export default App;
```

---

### **Types of Assets**

| Type            | Examples                        | How to Use |
|-----------------|---------------------------------|------------|
| Local Images    | PNG, JPG, WebP                  | `require('./assets/...')` |
| Remote Images   | URLs from internet              | `{ uri: 'https://...' }` |
| SVGs            | Vector icons & illustrations    | `react-native-svg` |
| Fonts           | Custom fonts                    | `expo-font` |
| Videos          | MP4 files                       | `expo-av` |

## 1. Local Images

**Description**: Local images are bundled with your app during build. They load instantly and work offline.

### **What are Local Images?**

Local images are stored inside your project folder (usually in the `assets` directory) and are bundled with your app when you build it. 

They are ideal for logos, icons, onboarding illustrations, and any image that doesn't need to change frequently.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { Image, View, StyleSheet } from 'react-native';

const App = () => {
  return (
    <View style={styles.container}>
      <Image
        source={require('./assets/images/profile.png')}
        style={styles.profileImage}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#f8fafc',
  },
  profileImage: {
    width: 150,
    height: 150,
    borderRadius: 75,
    borderWidth: 3,
    borderColor: '#fff',
    shadowColor: '#000',
    shadowOpacity: 0.1,
    shadowRadius: 10,
  },
});

export default App;
```

---

### **How to Add Local Images**

1. Create an `assets/images` folder in your project root.
2. Place your image files inside it (`.png`, `.jpg`, `.jpeg`).
3. Use `require('./path/to/image.png')` to import them.

---

### **Advantages of Local Images**

| Advantage               | Description |
|-------------------------|-----------|
| Fast Loading            | No network request needed |
| Works Offline           | Available even without internet |
| No Network Dependency   | More reliable |
| Build Time Optimization | Images are optimized during build |

## 2. Remote Images

**Description**: Images loaded from the internet using a URL.

### **What are Remote Images?**

Remote images are loaded from the internet (via a URL) instead of being bundled with your app. They are commonly used for user avatars, product images, news thumbnails, etc.

**Important Rule**: For remote images, you **must** specify `width` and `height`.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { Image, StyleSheet } from 'react-native';

const App = () => {
  return (
    <Image
      source={{ uri: 'https://picsum.photos/id/1015/400/300' }}
      style={styles.image}
    />
  );
};

const styles = StyleSheet.create({
  image: {
    width: 400,
    height: 300,
    borderRadius: 15,
  },
});

export default App;
```

---

### **Complete Example with Loading & Error Handling**

```jsx
import React, { useState } from 'react';
import { Image, View, Text, ActivityIndicator, StyleSheet } from 'react-native';

const RemoteImage = ({ url }) => {
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(false);

  return (
    <View style={styles.container}>
      {loading && (
        <ActivityIndicator size="large" color="#007AFF" style={styles.loader} />
      )}

      <Image
        source={{ uri: url }}
        style={styles.image}
        onLoadStart={() => setLoading(true)}
        onLoadEnd={() => setLoading(false)}
        onError={() => {
          setError(true);
          setLoading(false);
        }}
      />

      {error && <Text style={styles.errorText}>Failed to load image</Text>}
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    alignItems: 'center',
    justifyContent: 'center',
  },
  image: {
    width: 300,
    height: 200,
    borderRadius: 16,
  },
  loader: {
    position: 'absolute',
  },
  errorText: {
    color: '#ef4444',
    marginTop: 8,
  },
});

export default RemoteImage;
```

---

### **Important Rules & Tips**

| Rule / Tip                    | Description |
|-------------------------------|-----------|
| Width & Height                | Must be provided for remote images |
| Loading State                 | Always show `ActivityIndicator` |
| Error Handling                | Handle failed loads gracefully |
| Caching                       | Use `expo-image` for better caching |
| Placeholder                   | Show a default image while loading |

## 3. ImageBackground

**Description**: Used when you want an image as a background with content on top of it.

### **What is ImageBackground?**

`ImageBackground` is a built-in React Native component that allows you to set an image as the background of a container while placing other components (Text, Buttons, etc.) on top of it.

It is especially useful for hero sections, login screens, and onboarding flows.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { ImageBackground, Text, StyleSheet, View } from 'react-native';

const App = () => {
  return (
    <ImageBackground
      source={require('./assets/images/banner.jpg')}
      style={styles.banner}
      imageStyle={{ opacity: 0.85 }}
      resizeMode="cover"
    >
      <View style={styles.overlay}>
        <Text style={styles.title}>Welcome to Our App</Text>
        <Text style={styles.subtitle}>Discover amazing things around you</Text>
      </View>
    </ImageBackground>
  );
};

const styles = StyleSheet.create({
  banner: {
    width: '100%',
    height: 300,
    justifyContent: 'center',
    alignItems: 'center',
  },
  overlay: {
    backgroundColor: 'rgba(0, 0, 0, 0.4)', // Optional dark overlay
    padding: 20,
    borderRadius: 16,
    alignItems: 'center',
  },
  title: {
    fontSize: 32,
    fontWeight: 'bold',
    color: 'white',
    textAlign: 'center',
  },
  subtitle: {
    fontSize: 18,
    color: 'white',
    marginTop: 8,
    textAlign: 'center',
  },
});

export default App;
```

---

### **Important Props**

| Prop            | What it does                                      | Common Values |
|-----------------|---------------------------------------------------|---------------|
| `source`        | Image source (local or remote)                    | `require()` or `{ uri: '...' }` |
| `style`         | Style for the container                           | Width, height, flex, etc. |
| `imageStyle`    | Style applied directly to the image               | `opacity`, `resizeMode` |
| `resizeMode`    | How image fits inside container                   | `"cover"`, `"contain"`, `"stretch"` |

## 4. SVG Images

**Description**: Scalable Vector Graphics – perfect for icons, logos, and illustrations.

### **What are SVG Images?**

SVG (Scalable Vector Graphics) is a vector image format that can scale to any size without losing quality. 

Unlike regular images (PNG, JPG), SVGs are perfect for icons, logos, and illustrations because they remain crisp at any resolution and have very small file sizes.

---

### **Installation**

For **Expo** projects:

```bash
npx expo install react-native-svg
```

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, StyleSheet } from 'react-native';
import Svg, { Circle, Rect, Path } from 'react-native-svg';

const App = () => {
  return (
    <View style={styles.container}>
      <Svg width="140" height="140" viewBox="0 0 100 100">
        {/* Background Circle */}
        <Circle 
          cx="50" 
          cy="50" 
          r="45" 
          fill="#3b82f6" 
        />
        
        {/* Inner Square */}
        <Rect 
          x="25" 
          y="25" 
          width="50" 
          height="50" 
          fill="#ffffff" 
          rx="8" 
        />
        
        {/* Checkmark */}
        <Path 
          d="M35 50 L45 65 L70 35" 
          stroke="#3b82f6" 
          strokeWidth="10" 
          strokeLinecap="round" 
          strokeLinejoin="round"
        />
      </Svg>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#f8fafc',
  },
});

export default App;
```

---

### **Advantages of SVG**

| Advantage                    | Description |
|-----------------------------|-----------|
| Scalable                    | No quality loss at any size |
| Small file size             | Much smaller than PNG/JPG |
| Customizable                | Easily change colors and parts |
| Performance                 | Great for icons and animations |
| Resolution independent      | Perfect for different screen densities |

## 5. Resize Modes

**Description**: Controls how the image fits inside its container.

### **What are Resize Modes?**

`resizeMode` (or `contentFit` in `expo-image`) determines how an image should be scaled and positioned within its given dimensions.

Choosing the right resize mode is important for good visual presentation and avoiding distortion.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, Image, StyleSheet, Text } from 'react-native';

const App = () => {
  return (
    <View style={styles.container}>
      <Text style={styles.label}>resizeMode = "cover"</Text>
      <Image
        source={{ uri: 'https://picsum.photos/id/1015/600/400' }}
        style={styles.image}
        resizeMode="cover"
      />

      <Text style={styles.label}>resizeMode = "contain"</Text>
      <Image
        source={{ uri: 'https://picsum.photos/id/1015/600/400' }}
        style={styles.image}
        resizeMode="contain"
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8fafc',
  },
  label: {
    fontSize: 16,
    fontWeight: 'bold',
    marginTop: 20,
    marginBottom: 8,
  },
  image: {
    width: '100%',
    height: 200,
    backgroundColor: '#e2e8f0',
    borderRadius: 12,
  },
});

export default App;
```

---

### **Resize Modes Comparison**

| Resize Mode   | Description                              | Best Use Case                  | Behavior |
|---------------|------------------------------------------|--------------------------------|----------|
| `cover`       | Scales image to cover entire area        | Profile pictures, backgrounds  | Crops if needed |
| `contain`     | Scales image to fit without cropping     | Product images, logos          | Shows full image with letterboxing |
| `stretch`     | Stretches image to fill container        | Rarely used                    | May distort image |
| `center`      | Centers image without resizing           | Icons, small graphics          | Original size + centered |
| `repeat`      | Tiles the image                          | Background patterns            | Repeats image |

---

### **More Examples**

#### **1. Using with expo-image (Recommended)**
```jsx

<Image
  source={{ uri: imageUrl }}
  style={{ width: 300, height: 200 }}
  contentFit="cover"        // New name in expo-image

/>
```

#### **2. Avatar with Contain**

## 6. Image Optimization & Best Practices

**Description**: Optimizing images is crucial for app performance, faster loading times, and better user experience. Poorly optimized images can make your app slow and increase bundle size.

### **What is Image Optimization?**

Image optimization involves reducing image file sizes, choosing the right formats, and using efficient components to display images without compromising quality.

Good optimization leads to:
- Faster app loading
- Reduced memory usage
- Better battery life
- Smoother scrolling

---

### **Basic Complete Example with Best Practices**

```jsx
import React from 'react';
import { View, StyleSheet } from 'react-native';
import { Image } from 'expo-image';

const App = () => {
  return (
    <View style={styles.container}>
      {/* Using expo-image (Recommended) */}
      <Image
        source={{ uri: 'https://picsum.photos/id/1015/600/400' }}
        style={styles.image}
        contentFit="cover"
        transition={300}
        placeholder={require('./assets/placeholder.png')}
        onError={() => console.log('Image failed to load')}
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#f8fafc',
  },
  image: {
    width: 300,
    height: 200,
    borderRadius: 16,
  },
});

export default App;
```

---

### **Optimization Tips**

1. **Compress images** before adding them to your project.
2. Use **WebP format** when possible (smaller size with good quality).
3. Prefer **local images** for critical assets (logos, icons).
4. Use `expo-image` instead of the default `Image` for remote images.
5. Always specify `width` and `height` for remote images.
6. Use appropriate `resizeMode` / `contentFit`.

---

### **Using `expo-image` (Recommended for Remote Images)**

## 7. Loading States & Error Handling

**Description**: Always show proper feedback to users while data is loading or when something goes wrong. Good loading and error handling improves user experience significantly.

### **What are Loading States & Error Handling?**

In mobile apps, users often wait for data from APIs. You should:
- Show a loading indicator while fetching data
- Handle errors gracefully
- Provide meaningful messages when something fails

This section covers how to implement smooth loading and error states using `ActivityIndicator`, state management, and proper UX patterns.

---

### **Basic Complete Example**

```jsx
import React, { useState, useEffect } from 'react';
import { View, Text, ActivityIndicator, StyleSheet, Image } from 'react-native';

const App = () => {
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(false);
  const [data, setData] = useState(null);

  useEffect(() => {
    fetchData();
  }, []);

  const fetchData = async () => {
    setLoading(true);
    setError(false);
    
    try {
      const response = await fetch('https://api.example.com/user');
      const json = await response.json();
      setData(json);
    } catch (err) {
      setError(true);
    } finally {
      setLoading(false);
    }
  };

  return (
    <View style={styles.container}>
      {loading && (
        <View style={styles.center}>
          <ActivityIndicator size="large" color="#007AFF" />
          <Text style={styles.loadingText}>Loading...</Text>
        </View>
      )}

      {error && (
        <View style={styles.center}>
          <Text style={styles.errorText}>Failed to load data</Text>
          <Text style={styles.retryText} onPress={fetchData}>
            Tap to Retry
          </Text>
        </View>
      )}

      {data && (
        <View style={styles.content}>
          <Image 
            source={{ uri: data.avatar }} 
            style={styles.avatar}
            onLoadStart={() => setLoading(true)}
            onLoadEnd={() => setLoading(false)}
            onError={() => setError(true)}
          />
          <Text style={styles.name}>{data.name}</Text>
        </View>
      )}
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#f8fafc',
  },
  center: {
    alignItems: 'center',
  },
  loadingText: {
    marginTop: 12,
    fontSize: 16,
    color: '#64748b',
  },
  errorText: {
    fontSize: 18,
    color: '#ef4444',
    marginBottom: 8,
  },
  retryText: {
    fontSize: 16,
    color: '#3b82f6',
    textDecorationLine: 'underline',
  },
  avatar: {
    width: 120,
    height: 120,
    borderRadius: 60,
    marginBottom: 16,
  },
  name: {
    fontSize: 22,
    fontWeight: 'bold',
  },
});

export default App;
```

---

### **Important Components & Patterns**

| Component / Pattern     | What it does                                 | When to Use |
|-------------------------|----------------------------------------------|-------------|
| `ActivityIndicator`     | Shows spinning loader                        | During API calls |
| `onLoadStart` / `onLoadEnd` | Track image loading                       | For remote images |
| `onError`               | Handle image or network failures             | Error states |
| `try...catch...finally` | Proper error handling in async functions     | API calls |

---
