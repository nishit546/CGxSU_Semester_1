
# React Native - Environment Setup

Setting up the development environment correctly is crucial for a smooth React Native journey. This module covers everything you need to know when working with **Expo**.

---

## 1. Node.js

**What is Node.js?**  
Node.js is a JavaScript runtime built on Chrome’s V8 engine. It allows JavaScript to run outside the browser. It is the foundation of React Native and Expo development.

**Why is it needed?**
- Installing packages via npm
- Running development servers (Metro Bundler)
- Executing Expo CLI commands
- Managing project dependencies

### Installation & Verification
1. Download the **LTS version** from [nodejs.org](https://nodejs.org)
2. Install it on your system
3. Verify installation:

```bash
node --version
npm --version
```

**Recommended Version**: Node.js 18.x or 20.x (LTS)

---

## 2. Android Studio (Optional for Expo)

**Android Studio** is Google’s official IDE for Android development.  

Even when using Expo, it becomes useful for:
- Running Android Emulator
- Managing Android SDK
- Creating production builds (APK/AAB)
- Advanced debugging

### Key Components to Install:
- Android SDK Platform
- Android SDK Build-Tools
- Android SDK Command-line Tools
- Android Emulator
- Google USB Driver (for physical device)

> **Note**: For beginners using Expo, you can skip full Android Studio setup initially and use your physical device with Expo Go.

---

## 3. Java Development Kit (JDK)

**What is JDK?**  
JDK provides the Java environment required for Android development.

### Verification Command:
```bash
java -version
```

**Recommended**: JDK 17 or JDK 21 (OpenJDK preferred).

**Note**: Not required during early Expo development, but needed when you build production apps or use Android Studio.

---

## 4. Environment Variables

Environment variables help your system locate Java and Android tools.

### Important Variables:
- `JAVA_HOME` — Path to JDK installation
- `ANDROID_HOME` — Path to Android SDK

#### Example (Windows):
- `JAVA_HOME`: `C:\Program Files\Eclipse Adoptium\jdk-17...`
- `ANDROID_HOME`: `C:\Users\YourName\AppData\Local\Android\Sdk`

Add to **PATH**:
- `%JAVA_HOME%\bin`
- `%ANDROID_HOME%\platform-tools`
- `%ANDROID_HOME%\emulator`

#### For macOS / Linux (add to `~/.zshrc`):
```bash
export JAVA_HOME=/path/to/jdk
export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/platform-tools
export PATH=$PATH:$ANDROID_HOME/emulator
```

**Verify**:
```bash
adb --version
```

## 5. Expo CLI

Expo CLI is the command-line tool that makes React Native development much easier.

### Common Commands:
```bash
# Create new project
npx create-expo-app@latest myApp

# Navigate to project
cd myApp

# Start development server
npx expo start
```

---

## 6. Metro Bundler

**What is Metro Bundler?**  
Metro is the JavaScript bundler used by React Native and Expo. It combines all your JavaScript files into a single bundle.

**Responsibilities**:
- Bundling JavaScript code
- Watching for file changes
- Enabling Fast Refresh
- Serving the app to your device

You will see this message when starting the project:
```
Starting Metro Bundler...
```

---

## 7. Expo Go

**Expo Go** is a free mobile app that lets you run Expo projects instantly without building native apps.

### How to Use:
1. Install **Expo Go** from:
   - Play Store (Android)
   - App Store (iOS)
2. Connect your phone and computer to the **same Wi-Fi**
3. Run `npx expo start`
4. Scan the QR code displayed in the terminal

**Advantages**:
- Instant testing
- No need to build APK/IPA during development
- Supports most Expo modules

---

## 8. Fast Refresh

**Fast Refresh** is a powerful feature that improves developer productivity.

### What it does:
- Automatically updates the app when you save a file
- Preserves component state (text input, scroll position, etc.)
- Much faster than full reload

**How to trigger**: Simply save your file (`Ctrl + S`).

**Note**: Works best with functional components and React Hooks.

---

## 9. Hot Reload vs Fast Refresh

| Feature         | Hot Reload               | Fast Refresh                  |
|-----------------|--------------------------|-------------------------------|
| Generation      | Older feature            | Modern & Recommended          |
| State Handling  | Often loses state        | Preserves state when possible |
| Reliability     | Less reliable            | Highly Reliable               |
| Usage           | Legacy                   | Default in modern projects    |

---

## 10. Clearing Cache

Sometimes Metro or Expo cache causes issues (blank screen, old code, errors).

### Useful Commands:

```bash
# Best command for Expo
npx expo start --clear

# Alternative
npx expo start -c

# Clear npm cache
npm cache clean --force

# Full clean (when facing major issues)
rm -rf node_modules
rm -rf .expo
npm install
npx expo start --clear
```
