

# React Native - Introduction

**React Native** is a popular open-source framework developed by **Meta (Facebook)** that allows developers to build high-performance, native-like mobile applications for **Android and iOS** using **JavaScript** and **React**.

Instead of maintaining two separate codebases, React Native enables you to write **one single codebase** that works seamlessly across both platforms.

---

## Features at a Glance

- Cross-platform development (Android + iOS)
- Uses JavaScript and React
- Renders real **native UI components** (not WebViews)
- Excellent performance and smooth user experience
- Large community and rich ecosystem

---

## Reasons to Learn React Native

React Native empowers developers to build fast, scalable, and high-performance mobile apps using a single JavaScript codebase while delivering a truly native user experience.

### Key Benefits:

- Build mobile apps using familiar **web technologies** (JavaScript & React).
- Create **cross-platform apps** with a native look and feel on both iOS and Android.
- Use **JSX** syntax — no need to learn complex platform-specific languages like Kotlin, Java, Swift, or Objective-C.
- Maintain a **single codebase** for both platforms — easier development and maintenance.
- Benefit from **Hot Reloading** (Fast Refresh) for instant code updates.
- Easy testing and state management using tools like **Expo, ESLint, Jest, and Redux**.
- Deliver high performance by leveraging **native components** for rendering and execution.

---

## How React Native Works

React Native bridges the gap between JavaScript and native mobile platforms by allowing JavaScript code to communicate with native platform code.

### Working Flow:

1. **JavaScript Code Execution**
   - App logic is written in JavaScript.
   - Runs inside a JavaScript engine (**Hermes** or **JavaScriptCore**).

2. **JavaScript Bridge**
   - Acts as a communication layer between JavaScript and native code.
   - Works asynchronously to avoid blocking the UI thread.

3. **Native Modules**
   - Provide access to device features (Camera, GPS, File Storage, Notifications, etc.).
   - Android: Written in **Kotlin/Java**
   - iOS: Written in **Swift/Objective-C**

4. **Native UI Components**
   - React Native components (`View`, `Text`, `ScrollView`, etc.) are mapped to real native UI elements.
   - Ensures consistent and platform-native behavior.

5. **Rendering**
   - UI is rendered directly using native components on the device.
   - Results in smooth 60fps performance and native user experience.

## React Native vs Flutter

| Aspect                    | React Native                          | Flutter                              |
|--------------------------|---------------------------------------|--------------------------------------|
| **Language**             | JavaScript / TypeScript               | Dart                                 |
| **UI Rendering**         | Real Native Components                | Custom Skia Engine                   |
| **Performance**          | Excellent (Native)                    | Excellent (Near Native)              |
| **Learning Curve**       | Easy for React/JavaScript developers  | Requires learning Dart               |
| **Code Reusability**     | High                                  | High                                 |
| **Community**            | Very Large & Mature                   | Growing Rapidly                      |
| **Best For**             | Web developers, Existing React teams  | Beautiful custom UIs, Animations     |

**When to choose React Native?**  
Choose React Native if you already know JavaScript/React and want faster development with native components.

---

## Expo vs React Native CLI

| Feature                     | **Expo**                                 | **React Native CLI**                     |
|----------------------------|------------------------------------------|------------------------------------------|
| **Best For**               | Beginners, Prototyping, Small Apps       | Large-scale, Production Apps             |
| **Setup Complexity**       | Very Easy                                | More Complex                             |
| **Native Access**          | Limited (but improving)                  | Full Native Access                       |
| **Development Speed**      | Very Fast                                | Moderate                                 |
| **Requirements**           | No Android Studio / Xcode needed initially | Requires Android Studio + Xcode          |
| **Customization**          | Good for most apps                       | Best for advanced native modules         |
| **Testing**                | Expo Go App (Scan QR Code)               | Direct device/emulator                   |

**Recommendation**: Start with **Expo** and later eject to bare workflow if needed.

## Project Folder Structure (Expo)

```bash
MyFirstApp/
├── assets/               # Images, fonts, icons
├── node_modules/         # Installed dependencies
├── .expo/                # Expo configuration
├── App.js                # Main entry file
├── app.json              # App configuration
├── babel.config.js       # Babel settings
├── package.json          # Project dependencies
├── yarn.lock / package-lock.json
└── README.md
```

**Note**: In larger projects, you will create folders like:
- `components/`
- `screens/`
- `navigation/`
- `utils/`
- `services/` (API calls)
