
# React Native - Introduction

## What is React Native?

**React Native** is a popular open-source framework developed by **Meta (Facebook)** that allows developers to build high-performance, **native-like mobile applications** for **Android and iOS** using **JavaScript** and **React**.

Instead of maintaining two separate codebases, React Native enables you to write **one single codebase** that works seamlessly across both platforms.

---

## Features at a Glance

- Cross-platform development (Android + iOS)
- Uses JavaScript and React
- Renders real **native UI components** (not WebViews)
- Excellent performance and smooth user experience
- Large, mature community and rich ecosystem

---

## Why React Native?

React Native empowers developers to build fast, scalable, and high-performance mobile apps using a single JavaScript codebase while delivering a truly native user experience.

### Key Benefits:

- Build mobile apps using familiar **web technologies** (JavaScript & React).
- Create **cross-platform apps** with a native look and feel on both iOS and Android.
- Use **JSX** syntax — no need to learn complex platform-specific languages like Kotlin, Java, Swift, or Objective-C.
- Maintain a **single codebase** — easier development, updates, and maintenance.
- Benefit from **Hot Reloading** (Fast Refresh) for instant code updates.
- Excellent developer tools: Expo, ESLint, Jest, Redux, etc.
- Strong community support and vast library ecosystem.

## React Native vs React.js

| Feature          | React.js (React Web)              | React Native                          |
|------------------|-----------------------------------|---------------------------------------|
| **Platform**     | Web Browsers                      | Android & iOS                         |
| **Language**     | JavaScript + React                | JavaScript + React                    |
| **Styling**      | CSS / CSS-in-JS                   | StyleSheet (JavaScript objects)       |
| **Components**   | HTML elements (`div`, `span`)     | Native components (`View`, `Text`)    |
| **Navigation**   | React Router                      | React Navigation                      |
| **Rendering**    | DOM                               | Real Native UI Components             |

**Summary**: React.js is for building websites, while React Native is for building mobile apps. Both share the same component-based thinking and developer experience.

## How React Native Works

React Native bridges the gap between JavaScript and native mobile platforms.

### Working Flow:

1. **JavaScript Code Execution**  
   App logic runs in JavaScript using **Hermes** (recommended) or **JavaScriptCore** engine.

2. **JavaScript Bridge**  
   Enables communication between JavaScript and native code asynchronously.

3. **Native Modules**  
   Allow access to device features (Camera, GPS, Storage, Notifications, etc.).

4. **Native UI Components**  
   Components like `<View>`, `<Text>`, `<ScrollView>` are mapped to real native UI elements.

5. **Rendering**  
   UI is rendered using actual native components → smooth 60fps performance.

---

## React Native Architecture

### Old Architecture (Bridge-based)
JavaScript communicated with Native code through a **serialized Bridge**. This created performance bottlenecks in complex apps.

### New Architecture (2023 onwards)

- **JSI (JavaScript Interface)**: Direct communication between JavaScript and Native code (Bridge is removed for most cases).
- **Fabric**: New modern rendering engine for better performance, smoother animations, and improved scrolling.
- **Turbo Modules**: Lazy loading of native modules — only loads what is needed, resulting in faster app startup.

**Benefits of New Architecture**:
- Significantly better performance
- Faster app startup time
- Improved memory usage
- Smoother animations and gestures

## React Native vs Flutter

| Aspect               | React Native                          | Flutter                              |
|----------------------|---------------------------------------|--------------------------------------|
| **Language**         | JavaScript / TypeScript               | Dart                                 |
| **UI Rendering**     | Real Native Components                | Custom Skia Engine                   |
| **Performance**      | Excellent (Native)                    | Excellent (Near Native)              |
| **Learning Curve**   | Easy for React developers             | Requires learning Dart               |
| **Community**        | Very Large & Mature                   | Growing Rapidly                      |
| **Best For**         | Web developers, Existing React teams  | Highly custom UIs & Animations       |

**When to choose React Native?**  
Choose React Native if you already know JavaScript/React and want faster development with a huge ecosystem and easier hiring.

## Expo vs React Native CLI

| Feature              | **Expo**                                 | **React Native CLI**                     |
|----------------------|------------------------------------------|------------------------------------------|
| **Best For**         | Beginners, Prototyping, MVPs             | Large-scale, Production Apps             |
| **Setup Complexity** | Very Easy                                | More Complex                             |
| **Native Access**    | Limited (but improving)                  | Full Native Access                       |
| **Development Speed**| Very Fast                                | Moderate                                 |
| **Requirements**     | No Xcode / Android Studio initially      | Requires Full IDEs                       |
| **Customization**    | Good for most use cases                  | Best for advanced native modules         |

**Recommendation**: Start with **Expo**.

## Creating Your First Expo App

```bash
# Create new project
npx create-expo-app

# Navigate to project
cd ....

# Start development server
npx expo start
```

Scan the QR code with the **Expo Go** app on your phone to run the app instantly.

---

## Project Folder Structure (Expo)

```bash
MyFirstApp/
├── assets/                 # Images, fonts, icons
├── node_modules/           # Installed dependencies
├── .expo/                  # Expo configuration
├── App.js                  # Main entry file
├── app.json                # App metadata
├── babel.config.js         # Babel configuration
├── package.json
├── yarn.lock / package-lock.json
└── README.md
```

**For larger projects**, organize your code with these folders:
- `components/` – Reusable UI components
- `screens/` – Screen components
- `navigation/` – Navigation setup
- `utils/` – Helper functions
- `services/` – API calls and backend logic
- `assets/` – Static files
