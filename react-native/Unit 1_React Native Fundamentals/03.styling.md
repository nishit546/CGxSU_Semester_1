
# React Native – Styling

**Description**: Styling is one of the most important aspects of building beautiful and professional mobile apps.

### **What is Styling in React Native?**

Unlike React.js (web development), **React Native does not use CSS files** (like `.css` or `.scss`).

Instead, all styling is done using **JavaScript objects** that are passed to the `style` prop of components.

```jsx
<View style={{ backgroundColor: 'blue', padding: 20 }}>
  <Text style={{ color: 'white', fontSize: 18 }}>Hello World</Text>
</View>
```

### **Why No CSS?**

React Native works differently from the web:

- It renders **real native components** (Android `View` and iOS `UIView`).
- It does **not** use a web browser or HTML/CSS engine.
- React Native takes your JavaScript style objects and converts them into native platform styles.

This approach gives better performance and a more native look and feel.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

const App = () => {
  return (
    <View style={styles.container}>
      <Text style={styles.welcomeText}>
        Welcome to React Native Styling
      </Text>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f0f9ff',
    justifyContent: 'center',
    alignItems: 'center',
  },
  welcomeText: {
    fontSize: 24,
    fontWeight: 'bold',
    color: '#1e40af',
    textAlign: 'center',
  },
});

export default App;
```

---

### **Key Differences from Web CSS**

| Feature                  | Web (React.js)          | React Native                  |
|-------------------------|-------------------------|-------------------------------|
| Styling Method          | CSS files / modules     | JavaScript objects            |
| File Extension          | `.css`, `.scss`         | No separate style file        |
| Unit                    | `px`, `rem`, `%`        | No unit (numbers only)        |
| Layout System           | Flexbox + CSS Grid      | Flexbox only                  |
| Style Names             | `background-color`      | `backgroundColor` (camelCase) |

## 1. Inline Styles

**Description**: Writing styles directly inside the component using the `style` prop.

### **What are Inline Styles?**

Inline styles mean writing your styles **directly** as a JavaScript object inside the `style` prop of a component. 

This is the simplest and quickest way to style in React Native, especially for beginners or when you need dynamic styling.

---

### **Basic Complete Example**

```jsx
import React, { useState } from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';

const App = () => {
  const [isPressed, setIsPressed] = useState(false);

  return (
    <View style={{
      flex: 1,
      backgroundColor: '#f8fafc',
      justifyContent: 'center',
      alignItems: 'center',
    }}>
      <Text style={{
        fontSize: 32,
        fontWeight: 'bold',
        color: '#1e2937',
        marginBottom: 30,
      }}>
        Inline Styles Demo
      </Text>

      <TouchableOpacity
        onPress={() => setIsPressed(!isPressed)}
        style={{
          backgroundColor: isPressed ? '#22c55e' : '#3b82f6',
          paddingVertical: 16,
          paddingHorizontal: 40,
          borderRadius: 12,
        }}
      >
        <Text style={{
          color: 'white',
          fontSize: 18,
          fontWeight: 'bold',
        }}>
          {isPressed ? 'Pressed!' : 'Press Me'}
        </Text>
      </TouchableOpacity>
    </View>
  );
};

export default App;
```

---

### **Important Points**

| Aspect                | Description                                      | Example |
|-----------------------|--------------------------------------------------|--------|
| `style` prop          | Accepts a JavaScript object                      | `style={{ color: 'blue' }}` |
| Dynamic Styles        | Change style based on state                      | `backgroundColor: isActive ? 'green' : 'blue'` |
| Multiple styles       | Use array to combine styles                      | `style={[styles.base, { color: 'red' }]}` |

---

### **More Examples**

#### **1. Simple Inline Text**
```jsx
<Text style={{
  fontSize: 24,
  fontWeight: 'bold',
  color: '#ef4444',
  textAlign: 'center'
}}>
  Hello World
</Text>
```

#### **2. Dynamic Button Style**
```jsx
<TouchableOpacity style={{
  backgroundColor: isEnabled ? '#22c55e' : '#64748b',
  padding: 15,
  borderRadius: 10,
}}>
  <Text style={{ color: 'white' }}>Toggle Button</Text>
</TouchableOpacity>
```

#### **3. Card with Inline Styles**
```jsx
<View style={{
  backgroundColor: 'white',
  padding: 20,
  borderRadius: 16,
  shadowColor: '#000',
  shadowOffset: { width: 0, height: 4 },
  shadowOpacity: 0.1,
  elevation: 5,
  margin: 16,
}}>
  <Text style={{ fontSize: 20, fontWeight: 'bold' }}>Card Title</Text>
</View>
```

## 2. StyleSheet (Recommended Approach)

**Description**: `StyleSheet.create()` is the official and recommended way to define styles in React Native.

### **What is StyleSheet?**

`StyleSheet` is a built-in React Native API that helps you create and organize styles in a clean, performant way. 

Instead of writing inline styles (which are not recommended for large apps), you define all your styles in one place using `StyleSheet.create()`.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

const App = () => {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Welcome to My App</Text>
      <Text style={styles.subtitle}>This is styled using StyleSheet</Text>
      
      <View style={styles.button}>
        <Text style={styles.buttonText}>Press Me</Text>
      </View>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8fafc',
    justifyContent: 'center',
    alignItems: 'center',
    padding: 20,
  },
  title: {
    fontSize: 28,
    fontWeight: 'bold',
    color: '#1e2937',
    marginBottom: 8,
  },
  subtitle: {
    fontSize: 18,
    color: '#64748b',
    marginBottom: 40,
    textAlign: 'center',
  },
  button: {
    backgroundColor: '#3b82f6',
    paddingVertical: 16,
    paddingHorizontal: 40,
    borderRadius: 12,
  },
  buttonText: {
    color: 'white',
    fontSize: 18,
    fontWeight: 'bold',
  },
});

export default App;
```

---

### **Important Concepts**

| Concept              | What it does                                      | Benefit |
|----------------------|---------------------------------------------------|---------|
| `StyleSheet.create()`| Creates a stylesheet object                       | Better performance |
| `styles.container`   | Reference to a specific style                     | Clean code |
| Inline styles        | Writing styles directly in `style` prop           | Only for dynamic styles |

---

### **More Examples**

#### **1. Reusable Styles**
```jsx
const styles = StyleSheet.create({
  card: {
    backgroundColor: 'white',
    borderRadius: 16,
    padding: 20,
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 2 },
    shadowOpacity: 0.1,
    elevation: 5,
  },
  heading: {
    fontSize: 22,
    fontWeight: 'bold',
    color: '#1e2937',
  },
});
```

#### **2. Conditional Styles**
```jsx
<View style={[styles.card, isActive && styles.activeCard]}>
  <Text>Content</Text>
</View>
```

#### **3. Common Pattern – Button Styles**
```jsx
const styles = StyleSheet.create({
  primaryButton: {
    backgroundColor: '#2563eb',
    paddingVertical: 14,
    paddingHorizontal: 32,
    borderRadius: 12,
  },
  secondaryButton: {
    backgroundColor: 'transparent',
    borderWidth: 2,
    borderColor: '#2563eb',
    paddingVertical: 14,
    paddingHorizontal: 32,
    borderRadius: 12,
  },
});
```

## 3. Flexbox – The Layout System

**Description**: React Native uses **Flexbox** for all layouts. It is the most important concept for arranging components on the screen.

### **What is Flexbox in React Native?**

Flexbox is a layout system that helps you arrange and align items within a container. It is similar to CSS Flexbox but with some small differences.

In React Native, **every `View` is a flex container by default**. Mastering Flexbox is the key to building beautiful and responsive layouts.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

const App = () => {
  return (
    <View style={styles.container}>
      <View style={styles.box1}>
        <Text style={styles.text}>Box 1</Text>
      </View>
      <View style={styles.box2}>
        <Text style={styles.text}>Box 2</Text>
      </View>
      <View style={styles.box3}>
        <Text style={styles.text}>Box 3</Text>
      </View>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8fafc',
    flexDirection: 'column',      // Main direction
    justifyContent: 'center',     // Main axis
    alignItems: 'center',         // Cross axis
    gap: 20,                      // Space between items
  },
  box1: {
    width: 100,
    height: 100,
    backgroundColor: '#ef4444',
    justifyContent: 'center',
    alignItems: 'center',
    borderRadius: 12,
  },
  box2: {
    width: 100,
    height: 100,
    backgroundColor: '#3b82f6',
    justifyContent: 'center',
    alignItems: 'center',
    borderRadius: 12,
  },
  box3: {
    width: 100,
    height: 100,
    backgroundColor: '#22c55e',
    justifyContent: 'center',
    alignItems: 'center',
    borderRadius: 12,
  },
  text: {
    color: 'white',
    fontWeight: 'bold',
  },
});

export default App;
```

---

### **Key Properties**

| Property            | What it does                                      | Common Values |
|---------------------|---------------------------------------------------|---------------|
| `flexDirection`     | Main axis direction                               | `'row'`, `'column'` (default) |
| `justifyContent`    | Alignment along main axis                         | `'flex-start'`, `'center'`, `'flex-end'`, `'space-between'`, `'space-around'`, `'space-evenly'` |
| `alignItems`        | Alignment along cross axis                        | `'stretch'`, `'center'`, `'flex-start'`, `'flex-end'` |
| `flex`              | How much space item should take                   | `1`, `2`, `3` |
| `gap`               | Space between items (newer versions)              | `16`, `20` |

---

### **More Examples**

#### **1. Row Layout (Horizontal)**
```jsx
<View style={{ flexDirection: 'row', justifyContent: 'space-between' }}>
  <Text>Left</Text>
  <Text>Center</Text>
  <Text>Right</Text>
</View>
```

#### **2. Centered Content**
```jsx
<View style={{
  flex: 1,
  justifyContent: 'center',
  alignItems: 'center'
}}>
  <Text>Welcome to My App</Text>
</View>
```

#### **3. Equal Height Sections**
```jsx
<View style={{ flex: 1, flexDirection: 'row' }}>
  <View style={{ flex: 1, backgroundColor: 'red' }} />
  <View style={{ flex: 1, backgroundColor: 'blue' }} />
</View>
```

#### **4. Advanced Layout (Header + Content + Footer)**
```jsx
<View style={{ flex: 1 }}>
  <View style={{ height: 60 }}>Header</View>
  
  <View style={{ flex: 1 }}>Main Scrollable Content</View>
  
  <View style={{ height: 70 }}>Footer</View>
</View>
```

## 4. Position (Relative & Absolute)

**Description**: Controls how elements are positioned on the screen. React Native supports `relative` (default) and `absolute` positioning.

### **What is Position in React Native?**

The `position` style property determines how a component is placed within its parent container.

- **`relative`** (default): Component follows normal document flow.
- **`absolute`**: Component is removed from normal flow and positioned relative to its nearest positioned ancestor.

This is very useful for creating floating elements, overlays, and custom layouts.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, Text, StyleSheet, TouchableOpacity } from 'react-native';

const App = () => {
  return (
    <View style={styles.container}>
      {/* Main Content */}
      <View style={styles.card}>
        <Text style={styles.title}>Product Details</Text>
        <Text style={styles.description}>
          This is a sample product description. Scroll down to see more.
        </Text>
      </View>

      {/* Floating Action Button using Absolute Positioning */}
      <TouchableOpacity style={styles.fab}>
        <Text style={styles.fabText}>+</Text>
      </TouchableOpacity>

      {/* Badge Example */}
      <View style={styles.badge}>
        <Text style={styles.badgeText}>New</Text>
      </View>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8fafc',
    padding: 20,
  },
  card: {
    backgroundColor: 'white',
    padding: 20,
    borderRadius: 16,
    marginTop: 40,
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 10,
  },
  description: {
    fontSize: 16,
    color: '#64748b',
  },
  fab: {
    position: 'absolute',        // Absolute positioning
    bottom: 30,
    right: 20,
    width: 60,
    height: 60,
    backgroundColor: '#3b82f6',
    borderRadius: 30,
    justifyContent: 'center',
    alignItems: 'center',
    elevation: 8,
    zIndex: 10,
  },
  fabText: {
    color: 'white',
    fontSize: 28,
    fontWeight: 'bold',
  },
  badge: {
    position: 'absolute',
    top: 50,
    right: 30,
    backgroundColor: '#ef4444',
    paddingHorizontal: 10,
    paddingVertical: 4,
    borderRadius: 12,
  },
  badgeText: {
    color: 'white',
    fontSize: 12,
    fontWeight: 'bold',
  },
});

export default App;
```

---

### **Important Props**

| Prop          | What it does                                      | Common Values |
|---------------|---------------------------------------------------|---------------|
| `position`    | Sets positioning type                             | `'relative'`, `'absolute'` |
| `top`         | Distance from top                                 | `20`, `50` |
| `bottom`      | Distance from bottom                              | `30` |
| `left`        | Distance from left                                | `10` |
| `right`       | Distance from right                               | `20` |
| `zIndex`      | Stacking order (used with absolute)               | `1`, `5`, `10` |

---

### **More Examples**

#### **1. Relative Positioning (Default)**
```jsx
<View style={{ position: 'relative' }}>
  <Text>Normal flow</Text>
</View>
```

#### **2. Absolute Position - Top Right Badge**
```jsx
<View style={{
  position: 'absolute',
  top: 15,
  right: 15,
  backgroundColor: 'red',
  borderRadius: 10,
  padding: 4,
}}>
  <Text style={{ color: 'white', fontSize: 12 }}>9+</Text>
</View>
```

#### **3. Centered Absolute Element**
```jsx
<View style={{
  position: 'absolute',
  top: '50%',
  left: '50%',
  transform: [{ translateX: -50 }, { translateY: -50 }]
}}>
  <Text>Centered Overlay</Text>
</View>
```

## 5. zIndex

**Description**: Controls the stacking order of overlapping components. Higher number appears on top.

### **What is zIndex?**

`zIndex` is a style property used to control which component appears **on top** when multiple components overlap. 

It works similarly to layers in design tools like Figma or Photoshop. The component with the **higher zIndex value** will be displayed above others.

**Note**: `zIndex` works best with `position: 'absolute'` or when components naturally overlap.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

const App = () => {
  return (
    <View style={styles.container}>
      {/* Background Card */}
      <View style={styles.card1}>
        <Text style={styles.text}>Card 1 (zIndex: 1)</Text>
      </View>

      {/* Overlapping Card */}
      <View style={styles.card2}>
        <Text style={styles.text}>Card 2 (zIndex: 5)</Text>
      </View>

      {/* Floating Button */}
      <View style={styles.floatingButton}>
        <Text style={styles.buttonText}>+</Text>
      </View>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8fafc',
    justifyContent: 'center',
    alignItems: 'center',
  },
  card1: {
    position: 'absolute',
    width: 200,
    height: 120,
    backgroundColor: '#64748b',
    borderRadius: 16,
    zIndex: 1,
  },
  card2: {
    position: 'absolute',
    width: 200,
    height: 120,
    backgroundColor: '#3b82f6',
    borderRadius: 16,
    top: 60,
    left: 60,
    zIndex: 5,           // Higher zIndex → appears on top
  },
  floatingButton: {
    position: 'absolute',
    bottom: 40,
    right: 30,
    width: 60,
    height: 60,
    backgroundColor: '#ef4444',
    borderRadius: 30,
    justifyContent: 'center',
    alignItems: 'center',
    zIndex: 10,          // Highest zIndex
    elevation: 8,        // For Android shadow
  },
  text: {
    color: 'white',
    fontWeight: 'bold',
    padding: 20,
  },
  buttonText: {
    color: 'white',
    fontSize: 28,
    fontWeight: 'bold',
  },
});

export default App;
```

---

### **Important Props**

| Prop       | What it does                                      | Common Values |
|------------|---------------------------------------------------|---------------|
| `zIndex`   | Controls stacking order                           | Numbers: `1`, `5`, `10`, `100` |
| `position` | Required for zIndex to work effectively           | `'absolute'` or `'relative'` |
| `elevation`| Android alternative for shadow + stacking         | `5`, `10`, `20` |

---

### **More Examples**

#### **1. Simple Overlapping Texts**
```jsx
<Text style={{ zIndex: 2, position: 'absolute', top: 50 }}>On Top</Text>
<Text style={{ zIndex: 1, position: 'absolute', top: 50 }}>Behind</Text>
```

#### **2. Modal with Backdrop**
```jsx
<View style={styles.backdrop} />           // zIndex: 1
<View style={styles.modal}>               // zIndex: 10
  {/* Modal Content */}
</View>
```

#### **3. Floating Action Button (FAB)**
```jsx
<View style={{
  position: 'absolute',
  bottom: 30,
  right: 20,
  zIndex: 20,
}}>
  <Text style={styles.fab}>+</Text>
</View>
```

## 6. Responsive Design

**Description**: Making your app look good on different screen sizes and devices (phones, tablets, various Android & iOS models).

### **What is Responsive Design in React Native?**

Responsive Design means your UI automatically adapts to different screen sizes, orientations (portrait/landscape), and device types. 

Unlike the web, React Native doesn’t have media queries, so we use Flexbox, percentages, and dimension hooks to achieve responsiveness.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, Text, StyleSheet, useWindowDimensions } from 'react-native';

const App = () => {
  const { width, height } = useWindowDimensions();

  return (
    <View style={styles.container}>
      <View style={styles.card}>
        <Text style={styles.title}>Responsive Card</Text>
        <Text style={styles.subtitle}>
          Screen Width: {Math.round(width)} px
        </Text>
        <Text style={styles.subtitle}>
          Screen Height: {Math.round(height)} px
        </Text>
      </View>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8fafc',
    justifyContent: 'center',
    alignItems: 'center',
    padding: 20,
  },
  card: {
    width: '90%',                    // Responsive width
    maxWidth: 400,                   // Maximum limit
    backgroundColor: 'white',
    padding: 24,
    borderRadius: 16,
    alignItems: 'center',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 12,
  },
  subtitle: {
    fontSize: 16,
    color: '#64748b',
    textAlign: 'center',
  },
});

export default App;
```

---

### **Important Tools**

| Tool                        | What it does                                      | When to Use |
|-----------------------------|---------------------------------------------------|-----------|
| `Dimensions.get('window')`  | Get screen dimensions (static)                    | Simple cases |
| `useWindowDimensions()`     | Get dimensions + updates on rotation              | Recommended |
| `flex`                      | Makes components flexible                         | Most layouts |
| `Percentage values`         | `width: '90%'`, `height: '50%'`                   | Common |

---

### **More Examples**

#### **1. Responsive Image**
```jsx
<Image 
  source={{ uri: 'https://picsum.photos/600' }}
  style={{
    width: '100%',
    height: 220,
    borderRadius: 16,
  }}
  resizeMode="cover"
/>
```

#### **2. Responsive Grid Layout**
```jsx
<View style={styles.grid}>
  {items.map(item => (
    <View key={item.id} style={styles.gridItem}>
      <Text>{item.name}</Text>
    </View>
  ))}
</View>
```

```jsx
grid: {
  flexDirection: 'row',
  flexWrap: 'wrap',
  justifyContent: 'space-between',
},
gridItem: {
  width: '48%',           // Two columns
  marginBottom: 16,
},
```

#### **3. Using useWindowDimensions for Dynamic Sizing**
```jsx
const { width } = useWindowDimensions();

const cardWidth = width > 600 ? 500 : '90%';

<View style={{ width: cardWidth }}>
  {/* Content */}
</View>
```

## 7. Platform-Specific Styling

**Description**: Writing different styles for Android and iOS to match each platform’s design guidelines and behavior.

### **What is Platform-Specific Styling?**

React Native allows you to write code that behaves differently on Android and iOS using the `Platform` module. This is useful because:

- iOS and Android have different design languages (Human Interface Guidelines vs Material Design)
- Status bar, navigation, padding, and shadows behave differently
- Some UI elements need platform-specific adjustments

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, Text, StyleSheet, Platform } from 'react-native';

const App = () => {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Platform Specific Styling</Text>
      <Text style={styles.description}>
        This screen looks slightly different on iOS and Android.
      </Text>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8fafc',
    paddingHorizontal: 20,
    paddingTop: Platform.OS === 'ios' ? 60 : 40, // Different padding
  },
  title: {
    fontSize: 28,
    fontWeight: 'bold',
    color: '#1e2937',
    marginBottom: 10,
  },
  description: {
    fontSize: 16,
    color: '#64748b',
    lineHeight: 24,
  },
});

export default App;
```

---

### **Important Methods & Props**

| Method / Property      | What it does                                      | Example |
|------------------------|---------------------------------------------------|--------|
| `Platform.OS`          | Returns current platform                          | `'ios'` or `'android'` |
| `Platform.select()`    | Returns value based on platform                   | `Platform.select({ ios: ..., android: ... })` |
| `Platform.Version`     | Returns OS version                                | Useful for version-specific logic |
| `Platform.constants`   | Gives more platform details                       | Device info |

---

### **More Examples**

#### **1. Using Platform.select() (Recommended)**

```jsx
const styles = StyleSheet.create({
  container: {
    flex: 1,
    paddingTop: Platform.select({
      ios: 60,
      android: 40,
      default: 30,        // for other platforms (web, etc.)
    }),
    paddingHorizontal: 20,
  },
  button: {
    paddingVertical: Platform.select({
      ios: 14,
      android: 12,
    }),
    borderRadius: Platform.OS === 'ios' ? 12 : 8,
  }
});
```

#### **2. Platform-Specific Shadow**

```jsx
const styles = StyleSheet.create({
  card: {
    backgroundColor: 'white',
    padding: 16,
    borderRadius: 12,
    ...Platform.select({
      ios: {
        shadowColor: '#000',
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.1,
        shadowRadius: 8,
      },
      android: {
        elevation: 6,
      },
    }),
  },
});
```

#### **3. Conditional Rendering**

```jsx
{Platform.OS === 'ios' ? (
  <Text style={styles.iosText}>iOS Specific Text</Text>
) : (
  <Text style={styles.androidText}>Android Specific Text</Text>
)}
```

## 8. Fonts

**Description**: Custom fonts enhance the visual appeal and branding of your React Native app. React Native supports loading custom font files (`.ttf`, `.otf`).

### **What are Custom Fonts in React Native?**

By default, React Native uses system fonts. However, most modern apps use **custom fonts** to match their brand identity. 

You can load font files from your `assets/fonts` folder and use them throughout your app using the `fontFamily` style property.

**Note**: For Expo projects, `expo-font` is the easiest way to load custom fonts.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';
import { useFonts } from 'expo-font';
import AppLoading from 'expo-app-loading';

const App = () => {
  const [fontsLoaded] = useFonts({
    'Inter-Regular': require('./assets/fonts/Inter-Regular.ttf'),
    'Inter-Bold': require('./assets/fonts/Inter-Bold.ttf'),
    'Inter-SemiBold': require('./assets/fonts/Inter-SemiBold.ttf'),
  });

  if (!fontsLoaded) {
    return <AppLoading />;
  }

  return (
    <View style={styles.container}>
      <Text style={styles.heading}>Welcome to My App</Text>
      <Text style={styles.subheading}>This is a beautiful custom font</Text>
      <Text style={styles.bodyText}>
        This paragraph uses Inter Regular font which gives a clean and modern look to the app.
      </Text>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8fafc',
    padding: 20,
    justifyContent: 'center',
  },
  heading: {
    fontFamily: 'Inter-Bold',
    fontSize: 32,
    color: '#1e2937',
    marginBottom: 8,
  },
  subheading: {
    fontFamily: 'Inter-SemiBold',
    fontSize: 20,
    color: '#475569',
    marginBottom: 16,
  },
  bodyText: {
    fontFamily: 'Inter-Regular',
    fontSize: 16,
    lineHeight: 24,
    color: '#64748b',
  },
});

export default App;
```

---

### **Important Concepts & Props**

| Concept / Prop       | What it does                                      | Example |
|----------------------|---------------------------------------------------|--------|
| `useFonts`           | Hook to load custom fonts                         | `useFonts({ 'FontName': require(...) })` |
| `fontFamily`         | Applies the loaded font                           | `'Inter-Bold'` |
| `fontWeight`         | Controls boldness (when using variable fonts)     | `'bold'`, `'600'` |
| `fontSize`           | Size of the text                                  | `18`, `24` |
| `AppLoading`         | Shows loading screen until fonts are ready        | `<AppLoading />` |

---

### **More Examples**

#### **1. Basic Font Usage**
```jsx
<Text style={{ fontFamily: 'Inter-Bold', fontSize: 24 }}>
  Bold Heading
</Text>
```

#### **2. Font with Different Weights**
```jsx
<Text style={{ fontFamily: 'Inter-Regular' }}>Regular Text</Text>
<Text style={{ fontFamily: 'Inter-SemiBold' }}>Semi Bold Text</Text>
<Text style={{ fontFamily: 'Inter-Bold' }}>Bold Text</Text>
```

#### **3. Reusable Text Component**
```jsx
const AppText = ({ children, style, variant = "regular" }) => {
  const fontFamily = {
    regular: 'Inter-Regular',
    medium: 'Inter-Medium',
    bold: 'Inter-Bold',
  }[variant];

  return (
    <Text style={[{ fontFamily }, style]}>
      {children}
    </Text>
  );
};
```

**Usage:**
```jsx
<AppText variant="bold" style={{ fontSize: 20 }}>Hello World</AppText>
```

## 9. Icons

**Description**: Icons are an essential part of modern mobile apps. They improve user experience, make interfaces more intuitive, and enhance visual appeal.

### **What are Icons in React Native?**

React Native does not have built-in icons. We use third-party libraries like `@expo/vector-icons` to easily add beautiful, customizable icons to our app.

---

### **Installation**

For **Expo** projects:

```bash
npx expo install @expo/vector-icons
```

For bare React Native projects, additional setup may be required.

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';
import { Ionicons, MaterialIcons, Feather } from '@expo/vector-icons';

const App = () => {
  return (
    <View style={styles.container}>
      <Ionicons name="heart" size={40} color="red" />
      <MaterialIcons name="favorite" size={40} color="#FF5252" />
      <Feather name="home" size={40} color="#3b82f6" />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    gap: 30,
  },
});

export default App;
```

---

### **Most Popular Icon Packs**

| Icon Pack       | Best For                        | Example Names                     |
|-----------------|---------------------------------|-----------------------------------|
| `Ionicons`      | Most popular, modern look       | `heart`, `home`, `settings`, `star` |
| `MaterialIcons` | Google’s Material Design        | `favorite`, `notifications`, `menu` |
| `Feather`       | Clean, minimalist icons         | `edit`, `trash-2`, `user`, `search` |
| `FontAwesome5`  | Classic & detailed icons        | `user-circle`, `bell`, `smile` |
| `AntDesign`     | Modern UI                       | `like1`, `shoppingcart` |

---

### **Common Props**

| Prop     | Description                    | Example |
|----------|--------------------------------|---------|
| `name`   | Name of the icon               | `"heart"` |
| `size`   | Size of the icon (in pixels)   | `24`, `30`, `48` |
| `color`  | Color of the icon              | `"red"`, `"#3b82f6"` |
| `style`  | Additional styling             | `{ marginRight: 10 }` |

---

### **Practical Examples**

#### **1. Icon Button**
```jsx

  <Ionicons name="heart-outline" size={28} color="#ef4444" />
```

```jsx
  <Ionicons name="person-outline" size={24} color="#64748b" />
```


## 10. Themes & Dark Mode

**Description**: Creating a consistent and beautiful design system is essential for modern mobile apps. Implementing **themes** and **dark mode** makes your app more professional and user-friendly.

### **What is Theming?**

Theming means creating a centralized set of colors, spacing, fonts, and other design values that you can reuse across your entire app.

This approach makes your code cleaner, easier to maintain, and consistent throughout the application.

---

### **Basic Theme Setup**

```jsx
// theme.js
export const theme = {
  colors: {
    primary: '#2563eb',
    primaryDark: '#1e40af',
    background: '#ffffff',
    surface: '#f8fafc',
    text: '#1f2937',
    textSecondary: '#64748b',
    border: '#e2e8f0',
    success: '#22c55e',
    danger: '#ef4444',
  },
  spacing: {
    xs: 4,
    sm: 8,
    md: 16,
    lg: 24,
    xl: 32,
  },
  borderRadius: {
    sm: 8,
    md: 12,
    lg: 16,
    full: 9999,
  },
  fontSize: {
    xs: 12,
    sm: 14,
    base: 16,
    lg: 18,
    xl: 20,
    '2xl': 24,
  },
};
```

---

### **Using the Theme**

```jsx
import { theme } from './theme';

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: theme.colors.background,
    padding: theme.spacing.lg,
  },
  card: {
    backgroundColor: theme.colors.surface,
    borderRadius: theme.borderRadius.lg,
    padding: theme.spacing.md,

    shadowOpacity: 0.1,
  },
  title: {
    fontSize: theme.fontSize.xl,
    color: theme.colors.text,
    fontWeight: 'bold',
  },
});
```

---

### **Dark Mode Support**

React Native provides a built-in hook to detect the user’s system preference.

#### Basic Dark Mode Example

```jsx
import React from 'react';
import { useColorScheme, View, Text, StyleSheet } from 'react-native';
import { theme } from './theme';

const App = () => {
  const colorScheme = useColorScheme(); // "light" | "dark" | null
  const isDark = colorScheme === 'dark';

  const currentTheme = {
    background: isDark ? '#0f172a' : theme.colors.background,
    text: isDark ? '#f1f5f9' : theme.colors.text,
    card: isDark ? '#1e2937' : theme.colors.surface,
    border: isDark ? '#334155' : theme.colors.border,
  };

  return (
    <View style={[styles.container, { backgroundColor: currentTheme.background }]}>
      <Text style={[styles.title, { color: currentTheme.text }]}>
        Hello, Dark Mode!
      </Text>

      <View style={[styles.card, { backgroundColor: currentTheme.card }]}>
        <Text style={{ color: currentTheme.text }}>
          This card adapts to light and dark mode.
        </Text>
      </View>
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20 },
  title: { fontSize: 28, fontWeight: 'bold', marginBottom: 20 },
  card: { padding: 20, borderRadius: 16, marginTop: 20 },
});

export default App;
```

---

### **Advanced Theme Hook (Recommended)**

```jsx
// useTheme.js
import { useColorScheme } from 'react-native';
import { theme } from './theme';

export const useTheme = () => {
  const colorScheme = useColorScheme();
  const isDark = colorScheme === 'dark';

  const colors = {
    ...theme.colors,
    background: isDark ? '#0f172a' : theme.colors.background,
    surface: isDark ? '#1e2937' : theme.colors.surface,
    text: isDark ? '#f8fafc' : theme.colors.text,
    textSecondary: isDark ? '#cbd5e1' : theme.colors.textSecondary,
    border: isDark ? '#475569' : theme.colors.border,
  };

  return { colors, isDark, spacing: theme.spacing };
};
```

**Usage:**

```jsx
const { colors, isDark } = useTheme();

<View style={{ backgroundColor: colors.background }}>
  <Text style={{ color: colors.text }}>Hello</Text>
</View>
```

## 11. NativeWind (Tailwind CSS for React Native)

**Description**: NativeWind brings the power of Tailwind CSS to React Native using the `className` prop.

### **What is NativeWind?**

**NativeWind** allows you to style React Native components using familiar **Tailwind CSS** utility classes instead of writing `StyleSheet.create()` for every small thing.

It combines the speed and simplicity of Tailwind with the power of React Native.

### **Why Use NativeWind?**

- Much faster development
- Cleaner and more readable code
- Consistent design system
- Huge Tailwind community and ecosystem
- Excellent for rapid prototyping

**Recommendation**: Master `StyleSheet` first, then adopt NativeWind for larger projects.

---

### **Installation**

```bash
# For Expo
npx expo install nativewind
npx expo install tailwindcss

# For bare React Native
npm install nativewind
npm install tailwindcss
```

> After installation, configure `tailwind.config.js` and `babel.config.js` (follow official docs).

---

### **Basic Complete Example**

```jsx
import React from 'react';
import { View, Text } from 'react-native';

const App = () => {
  return (
    <View className="flex-1 bg-gray-100 justify-center items-center">
      <Text className="text-4xl font-bold text-blue-600 mb-6">
        Hello NativeWind!
      </Text>
      <Text className="text-gray-600 text-center px-8 text-lg">
        Styling has never been this fast and clean.
      </Text>
    </View>
  );
};

export default App;
```

---

| Category           | Examples                                      |
|--------------------|-----------------------------------------------|
| Spacing            | `p-4`, `m-5`, `px-6`, `py-8`, `mt-10`         |
