# 1️⃣ What is Local Storage?

Local storage allows your app to save data on the user's device.

Common examples:

- Login session
- Theme preference
- Language
- User settings
- Cached data

Unlike API data, local storage works even when the internet is unavailable.

---

# 2️⃣ AsyncStorage

AsyncStorage is a simple key-value storage system.

### Install

```bash
npx expo install @react-native-async-storage/async-storage
```

### Store Data

```jsx
import AsyncStorage from "@react-native-async-storage/async-storage";

await AsyncStorage.setItem("username", "Rajesh");
```

### Read Data

```jsx
const value = await AsyncStorage.getItem("username");
console.log(value);
```

### Update Data

```jsx
await AsyncStorage.setItem("username", "Amit");
```

### Delete Data

```jsx
await AsyncStorage.removeItem("username");
```

### Clear Storage

```jsx
await AsyncStorage.clear();
```

---

# 3️⃣ Working with Objects

Objects must be converted into JSON.

### Save

```jsx
const user = {
  id: 1,
  name: "Rajesh",
};

await AsyncStorage.setItem(
  "user",
  JSON.stringify(user)
);
```

### Read

```jsx
const data = await AsyncStorage.getItem("user");

const user = JSON.parse(data);
```

# 4️⃣ SecureStore

Sensitive data should not be stored in AsyncStorage.

Expo provides **SecureStore**.

### Install

```bash
npx expo install expo-secure-store
```

### Save Secure Data

```jsx
import * as SecureStore from "expo-secure-store";

await SecureStore.setItemAsync(
  "token",
  "abc123"
);
```

### Read Secure Data

```jsx
const token =
  await SecureStore.getItemAsync("token");
```

### Delete Secure Data

```jsx
await SecureStore.deleteItemAsync("token");
```

# 5️⃣ Token Storage

Typical authentication flow:

```text
Login
   │
Receive JWT Token
   │
Store in SecureStore
   │
Use in API Headers
   │
Logout
   │
Delete Token
```

Example:

```jsx
await SecureStore.setItemAsync(
  "accessToken",
  token
);
```

Never hardcode or expose tokens.

---

# 6️⃣ Theme Storage

Persist the user's preferred theme.

```jsx
await AsyncStorage.setItem(
  "theme",
  "dark"
);
```

Read theme on app launch:

```jsx
const theme =
  await AsyncStorage.getItem("theme");
```

Use this to automatically restore Light or Dark mode.

---

# 7️⃣ Choosing the Right Storage

| Data | Storage |
|------|---------|
| Theme | AsyncStorage |
| Language | AsyncStorage |
| User Preferences | AsyncStorage |
| JWT Token | SecureStore |
| Refresh Token | SecureStore |
| Password | Never Store |

---

# 8️⃣ Best Practices

- Use AsyncStorage for non-sensitive data
- Use SecureStore for tokens
- Store objects as JSON
- Remove tokens on logout
- Handle missing values safely
- Avoid storing huge datasets

---

#  Common Mistakes

- Saving passwords in AsyncStorage
- Forgetting JSON.stringify()
- Forgetting JSON.parse()
- Not clearing data on logout
- Storing sensitive data insecurely
