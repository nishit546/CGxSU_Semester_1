#  What are Components?

A **component** is a reusable piece of UI.

Everything you see in a React Native app is built using components.

Examples:

- Login Screen
- Header
- Button
- Card
- Profile Screen

---

# 1️⃣ View

`View` is the most fundamental component in React Native.

It works like a **`div`** in React.js.

### Example

```jsx
import { View, Text } from "react-native";

export default function App() {
  return (
    <View>
      <Text>Hello React Native</Text>
    </View>
  );
}
```

### Common Props

- style
- accessible
- testID

---

# 2️⃣ Text

`Text` is used to display text.

> In React Native, plain text **must** be wrapped inside a `Text` component.

```jsx
<Text>Hello Students </Text>
```

### Styling Example

```jsx
<Text style={{ fontSize: 22, fontWeight: "bold" }}>
  Welcome
</Text>
```

---

# 3️⃣ Image

Displays local or remote images.

## Local Image

```jsx
<Image source={require("./assets/logo.png")} />
```

## Remote Image

```jsx
<Image
  source={{
    uri: "https://picsum.photos/200"
  }}
  style={{ width: 200, height: 200 }}
/>
```

> Images require `width` and `height`.

---

# 4️⃣ ScrollView

Used when content needs scrolling.

```jsx
<ScrollView>
  <Text>Item 1</Text>
  <Text>Item 2</Text>
  <Text>Item 3</Text>
</ScrollView>
```

### Use When

- Small amount of data
- Static content

---

# 5️⃣ FlatList

Efficiently renders large lists.

```jsx
const DATA = [
  { id: "1", name: "Rajesh" },
  { id: "2", name: "Amit" },
];

<FlatList
  data={DATA}
  keyExtractor={(item) => item.id}
  renderItem={({ item }) => (
    <Text>{item.name}</Text>
  )}
/>
```

### Advantages

- Better performance
- Lazy rendering
- Memory efficient

---

# 6️⃣ SectionList

Displays grouped data.

```jsx
<SectionList
  sections={[
    { title: "A", data: ["Aman", "Ankit"] },
    { title: "B", data: ["Bharat"] }
  ]}
  renderItem={({ item }) => <Text>{item}</Text>}
  renderSectionHeader={({ section }) => (
    <Text>{section.title}</Text>
  )}
/>
```

---

# 7️⃣ Pressable

Recommended component for handling touch interactions.

```jsx
<Pressable onPress={() => alert("Pressed!")}>
  <Text>Click Me</Text>
</Pressable>
```

Common Events

- onPress
- onLongPress
- onPressIn
- onPressOut

---

# 8️⃣ TouchableOpacity

Creates a fading effect while pressing.

```jsx
<TouchableOpacity onPress={() => {}}>
  <Text>Login</Text>
</TouchableOpacity>
```

### Pressable vs TouchableOpacity

| Pressable | TouchableOpacity |
|------------|------------------|
| Modern | Older |
| More flexible | Opacity effect only |
| Recommended | Still widely used |

---

# 9️⃣ SafeAreaView

Keeps content away from notches and status bars.

```jsx
<SafeAreaView>
  <Text>Safe Content</Text>
</SafeAreaView>
```

---

# 🔟 StatusBar

Controls the device status bar.

```jsx
<StatusBar
  barStyle="light-content"
  backgroundColor="black"
/>
```

---

# 1️⃣1️⃣ Modal

Displays content above the current screen.

```jsx
<Modal visible={true} transparent>
  <Text>Hello Modal</Text>
</Modal>
```

Use Cases

- Confirmation dialogs
- Login popups
- Bottom sheets

---

# 1️⃣2️⃣ ActivityIndicator

Shows a loading spinner.

```jsx
<ActivityIndicator size="large" />
```

Usually shown while fetching APIs.

---

# 1️⃣3️⃣ Switch

Toggle ON/OFF state.

```jsx
const [enabled, setEnabled] = useState(false);

<Switch
  value={enabled}
  onValueChange={setEnabled}
/>
```

---

# 1️⃣4️⃣ RefreshControl

Provides pull-to-refresh functionality.

```jsx
<ScrollView
  refreshControl={
    <RefreshControl
      refreshing={false}
      onRefresh={() => {}}
    />
  }
/>
```

---

#  Choosing the Right Component

| Requirement | Component |
|-------------|-----------|
| Layout | View |
| Text | Text |
| Image | Image |
| Scroll Page | ScrollView |
| Large List | FlatList |
| Grouped List | SectionList |
| Button | Pressable |
| Fade Button | TouchableOpacity |
| Safe Layout | SafeAreaView |
| Loader | ActivityIndicator |
| Popup | Modal |
| Toggle | Switch |
| Pull Refresh | RefreshControl |

---

#  Common Mistakes

- Forgetting `width` and `height` on `Image`
- Using `ScrollView` for huge lists
- Rendering text outside `Text`
- Using many nested `Views`
- Forgetting `keyExtractor` in `FlatList`
