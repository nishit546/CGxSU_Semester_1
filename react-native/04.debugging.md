## 1Ô∏è‚É£ Console Logging (Basic Debugging)

The simplest and most commonly used debugging technique is logging.

```js
console.log("App executed");
console.warn("This is a warning");
console.error("Something went wrong");
```

### Why use console logging?

- Helps track app flow and execution order  
- Useful for checking variable values  
- Quick and easy during development  
- Should be removed before production to avoid unnecessary overhead  

#### Example

```js
import React from "react";
import { View, Text } from "react-native";

export default function App() {
  console.log("App executed");

  return (
    <View>
      <Text>Hello World</Text>
    </View>
  );
}
```

### Logs appear in:

- Metro Bundler terminal  
- Android Logcat  
- Xcode console  
- Expo DevTools

## 5Ô∏è‚É£ Network Debugging (API Issues)

When working with APIs (`fetch` / `axios`), always debug network calls.

### Check:

- Request URL  
- Headers  
- Status codes (200, 401, 500)  
- Response data  

### Common mistakes:

- Incorrect endpoint  
- Missing headers  
- Async/await not handled properly  
- State not updated after response  


üí° **Interview Tip:**  
 Network debugging is especially important when handling authentication, pagination, and error responses.

## 6Ô∏è‚É£ Debugging with Chrome (Remote Debugging)

React Native supports **remote debugging**, where JavaScript runs inside a **Chrome browser**.

### Steps to Enable

1. Run the app:

```bash
npm start
```

- Open Developer Menu  
- Select **Open JS Debugger** (or **Debug JS Remotely**)  

This opens a **Chrome tab** running your app‚Äôs JavaScript code.

### What you can do in Chrome DevTools:

- **Console Tab** ‚Üí View logs and errors  

- **Sources Tab**
  - Enable caught exceptions  
  - Set breakpoints  
  - Step through code line by line


‚ö†Ô∏è **Important:**  

Remote debugging slows down the app. Disable it after debugging.

üí° **Interview Tip:**  

Remote debugging runs JavaScript outside the device, which is why it affects performance.

## 7Ô∏è‚É£ Debugging with Visual Studio Code

Using **VS Code React Native Tools**, you can debug React Native apps directly inside the editor.

### Steps

#### Install Extension:
- **React Native Tools**

#### Create `launch.json`:
- Open **Run & Debug**
- Create `launch.json`
- Select **React Native**
- Choose **Attach to Packager**

#### Enable Debugging:
- Run the app
- Enable **Debug JS Remotely**
- Click **Attach to Packager**

#### Breakpoints:
- Click next to any code line
- Execution pauses at the breakpoint
- Logs appear in the **Debug Console**

‚úÖ Best for structured debugging and interviews.

## üîü Error Boundaries

Error Boundaries help catch JavaScript errors during rendering and prevent full app crashes.

### Benefits:

- Graceful error handling  
- Prevents white screen crashes  
- Improves user experience  

üí° **Note:**  
Error Boundaries catch **rendering errors**, not event handlers or async code.

## 1Ô∏è‚É£1Ô∏è‚É£ Performance Debugging (Intro)

### Signs of performance issues:

- Slow UI  
- Lag while scrolling  
- Unnecessary re-renders  

### Basic tips:

- Avoid inline functions in JSX  
- Use `useCallback` & `useMemo`  
- Keep components small and reusable

## üîó Best Practices for Debugging

- Start with `console.log`  
- Read error messages carefully  
- Debug one issue at a time  
- Use proper tools (DevTools, Flipper, VS Code)  
- Never ignore warnings during learning
