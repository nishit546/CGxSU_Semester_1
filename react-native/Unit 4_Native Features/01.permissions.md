
# React Native – Permissions

**Description**: Permissions are a critical part of modern mobile development. They protect user privacy while allowing apps to access device features like the camera, location, and microphone.

---

## 1. What are Permissions?

Permissions are security mechanisms used by Android and iOS to control access to sensitive device features and personal data. Apps must explicitly request permission from the user before accessing hardware (camera, microphone, GPS) or personal data (contacts, photos, location).

### **Why Permissions Matter**
- **Privacy & Compliance**: Violating permission rules can lead to App Store / Play Store rejection and legal issues (GDPR, CCPA).
- **User Trust**: Poor handling leads to app uninstalls. Always request permissions **in context** (e.g., when the user taps the camera button).
- **Platform Differences**:
  - **iOS**: Strict, user-centric. Permissions are prompted once; users can change them in Settings. "Limited" access is common for photos.
  - **Android**: More granular. Supports one-time permissions, foreground/background distinctions, and rationales for repeated requests.

### **Permission Statuses**

| Status       | Meaning |
|--------------|---------|
| `GRANTED`    | Access approved |
| `DENIED`     | User rejected (can ask again) |
| `BLOCKED`    | User denied + "Don't ask again" (redirect to Settings) |
| `LIMITED`    | Partial access (e.g., selected photos on iOS) |
| `UNAVAILABLE`| Not supported on this device |

---

## 2. Permission Flow

**Description**: A well-designed permission flow is essential for good user experience and high approval rates. Always request permissions **in context** rather than at app startup.

---

### **Standard Permission Flow**

```text
User tries to use a feature
          │
   Check Current Permission Status
          │
   ┌──────┴──────┐
Granted?        No
   │             │
Access     Request Permission
   │             │
  Yes     ┌──────┴──────┐
          Granted?     Denied?
             │           │
          Access     Show Friendly Message
                             │
                       Option to go to Settings
```

---

### **Implementation Pattern**
```jsx
import { Alert, Linking } from 'react-native';
import * as Permissions from 'react-native-permissions'; // or expo-permissions

const requestPermission = async (permissionType) => {
  const status = await Permissions.check(permissionType);

  if (status === 'granted') {
    return true;
  }

  if (status === 'blocked') {
    Alert.alert(
      'Permission Blocked',
      'Please enable permission in Settings',
      [
        { text: 'Cancel' },
        { text: 'Open Settings', onPress: () => Linking.openSettings() },
      ]
    );
    return false;
  }

  // Request permission
  const requestStatus = await Permissions.request(permissionType);

  if (requestStatus === 'granted') {
    return true;
  }

  // User denied
  Alert.alert(
    'Permission Denied',
    'This feature requires permission to work properly.',
    [{ text: 'OK' }]
  );
  return false;
};
```

## 3. Android Permissions

**Description**: Android uses **runtime permissions**. Unlike older versions where permissions were granted at install time, modern Android requires explicit user approval at runtime.

---

### **Key Points**

- Permissions are requested **at runtime** (when needed)
- Users can revoke permissions anytime from Settings
- You should always explain **why** you need the permission
- Android 11+ supports **one-time permissions**
- Different permission groups exist (Location, Camera, Storage, etc.)

---

### **Common Android Permissions**

| Permission                  | Use Case |
|-----------------------------|----------|
| `CAMERA`                    | Camera access |
| `READ_EXTERNAL_STORAGE`     | Read photos/files |
| `WRITE_EXTERNAL_STORAGE`    | Save files |
| `ACCESS_FINE_LOCATION`      | Precise GPS |
| `ACCESS_COARSE_LOCATION`    | Approximate location |
| `RECORD_AUDIO`              | Microphone |
| `READ_CONTACTS`             | Access contacts |

---

### **Implementation Example (with `react-native-permissions`)**
```jsx
import { PermissionsAndroid, Alert } from 'react-native';

const requestCameraPermission = async () => {
  try {
    const granted = await PermissionsAndroid.request(
      PermissionsAndroid.PERMISSIONS.CAMERA,
      {
        title: "Camera Permission",
        message: "This app needs camera access to take photos",
        buttonNeutral: "Ask Me Later",
        buttonNegative: "Cancel",
        buttonPositive: "OK",
      }
    );

    if (granted === PermissionsAndroid.RESULTS.GRANTED) {
      console.log("Camera permission granted");
      return true;
    } else {
      console.log("Camera permission denied");
      return false;
    }
  } catch (err) {
    console.warn(err);
    return false;
  }
};
```

## 4. iOS Permissions

**Description**: iOS is stricter about permissions compared to Android. The system is user-centric, and once a permission is denied, the user must manually enable it in Settings.

---

### **Key Points**

- Permission prompts appear only **once** per permission type
- If denied, the user must go to **Settings → App → Permissions** to enable it
- You **must** provide a clear reason in `Info.plist`
- iOS supports "Limited" access for some permissions (e.g., Photos)
- Background permissions require additional configuration

---

### **Common iOS Permissions**

| Permission                    | Use Case |
|-------------------------------|----------|
| `NSCameraUsageDescription`    | Camera |
| `NSPhotoLibraryUsageDescription` | Photos / Gallery |
| `NSMicrophoneUsageDescription` | Microphone |
| `NSLocationWhenInUseUsageDescription` | Foreground location |
| `NSLocationAlwaysAndWhenInUseUsageDescription` | Background location |
| `NSContactsUsageDescription`  | Contacts |
| `NSAppleMusicUsageDescription`| Music / Media |

---

### **Configuration in `Info.plist`**

```xml
<key>NSCameraUsageDescription</key>
<string>This app needs camera access to take photos and scan QR codes.</string>

<key>NSPhotoLibraryUsageDescription</key>
<string>This app needs access to your photos to let you select images.</string>

<key>NSLocationWhenInUseUsageDescription</key>
<string>This app needs location access to show nearby places.</string>
```

---

### **Requesting Permission in Code**
```jsx
import { Alert, Linking } from 'react-native';
import * as Permissions from 'react-native-permissions';

const requestCameraPermission = async () => {
  const status = await Permissions.request(Permissions.PERMISSIONS.IOS.CAMERA);

  if (status === 'granted') {
    // Access granted
    return true;
  } else if (status === 'blocked') {
    Alert.alert(
      'Camera Access Denied',
      'Please enable camera access in Settings',
      [
        { text: 'Cancel' },
        { text: 'Open Settings', onPress: () => Linking.openSettings() },
      ]
    );
  }
  return false;
};
```

## 5. Camera Permission (`expo-camera`)

**Description**: Accessing the camera requires explicit user permission. `expo-camera` provides a simple and consistent way to request and handle camera permissions across iOS and Android.

---

### **Installation**
```bash
npx expo install expo-camera
```

---

### **Basic Complete Example**
```jsx
import React from 'react';
import { View, Text, Pressable, StyleSheet, Alert } from 'react-native';
import { CameraView, useCameraPermissions } from 'expo-camera';

const CameraScreen = () => {
  const [permission, requestPermission] = useCameraPermissions();
  const [facing, setFacing] = React.useState('back');

  if (!permission) {
    return <View />; // Still loading
  }

  if (!permission.granted) {
    return (
      <View style={styles.center}>
        <Text style={styles.message}>
          We need your permission to use the camera
        </Text>
        <Pressable style={styles.button} onPress={requestPermission}>
          <Text style={styles.buttonText}>Grant Camera Permission</Text>
        </Pressable>
      </View>
    );

  return (
    <View style={styles.container}>
      <CameraView
        style={styles.camera}
        facing={facing}
      >
        <View style={styles.controls}>
          <Pressable 
            style={styles.flipButton}

          >
            <Text style={styles.flipText}>Flip Camera</Text>
          </Pressable>

    </View>
```

## 6. Gallery Permission (`expo-image-picker`)

**Description**: Accessing the device’s photo gallery requires explicit permission. `expo-image-picker` provides a simple interface to request gallery permission and launch the photo picker.

---

### **Installation**
```bash
npx expo install expo-image-picker
```

---

### **Basic Complete Example**
```jsx
import React, { useState } from 'react';
import { View, Image, Pressable, Text, StyleSheet, Alert } from 'react-native';
import * as ImagePicker from 'expo-image-picker';

const GalleryPickerScreen = () => {
  const [image, setImage] = useState(null);

  const pickImage = async () => {
    try {
      // Request permission
      const { status } = await ImagePicker.requestMediaLibraryPermissionsAsync();

      if (status !== 'granted') {
        Alert.alert(
          'Permission Denied',
          'We need access to your photo gallery to continue.'
        );
        return;
      }

      // Launch image picker
      const result = await ImagePicker.launchImageLibraryAsync({
        mediaTypes: ImagePicker.MediaTypeOptions.Images,
        allowsEditing: true,
        aspect: [4, 3],
        quality: 0.8,
      });

      if (!result.canceled) {
        setImage(result.assets[0].uri);
        console.log('Selected Image:', result.assets[0].uri);
      }
    } catch (error) {
      Alert.alert('Error', 'Failed to pick image');
      console.error(error);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Gallery Picker</Text>

      {image && (
        <Image source={{ uri: image }} style={styles.preview} />
      )}

      <Pressable style={styles.button} onPress={pickImage}>
        <Text style={styles.buttonText}> Pick from Gallery</Text>
      </Pressable>

      <Text style={styles.note}>
        Note: First time use will show permission prompt
      </Text>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
    alignItems: 'center',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 30,
  },
  preview: {
    width: 280,
    height: 280,
    borderRadius: 20,
    marginBottom: 30,
  },
  button: {
    backgroundColor: '#007AFF',
    paddingVertical: 16,
    paddingHorizontal: 32,
    borderRadius: 12,
    marginTop: 20,
  },
  buttonText: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },

    marginTop: 40,

    textAlign: 'center',

  },
});

export default GalleryPickerScreen;
```

### **Key Options**

| Option                 | Description |

## 7. Location Permission (`expo-location`)

**Description**: Accessing device location requires explicit user permission. `expo-location` provides methods to request foreground and background location permissions.

---

### **Installation**
```bash
npx expo install expo-location
```

---

### **Basic Complete Example**
```jsx
import React, { useState } from 'react';
import { View, Text, Pressable, StyleSheet, Alert } from 'react-native';
import * as Location from 'expo-location';

const LocationPermissionScreen = () => {
  const [location, setLocation] = useState(null);
  const [loading, setLoading] = useState(false);

  const requestForegroundLocation = async () => {
    setLoading(true);

    try {
      const { status } = await Location.requestForegroundPermissionsAsync();

      if (status !== 'granted') {
        Alert.alert(
          'Permission Denied',
          'This app needs location access to show nearby places.'
        );
        return;
      }

      const currentLocation = await Location.getCurrentPositionAsync({
        accuracy: Location.Accuracy.High,
      });

      setLocation(currentLocation.coords);
      console.log('Location:', currentLocation.coords);
    } catch (error) {
      Alert.alert('Error', 'Failed to get location');
      console.error(error);
    } finally {
      setLoading(false);
    }
  };

  const requestBackgroundLocation = async () => {
    const { status } = await Location.requestBackgroundPermissionsAsync();

    if (status === 'granted') {
      Alert.alert('Success', 'Background location permission granted');
    } else {
      Alert.alert('Permission Denied', 'Background location access was denied');
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Location Permissions</Text>

      <Pressable style={styles.button} onPress={requestForegroundLocation} disabled={loading}>
        <Text style={styles.buttonText}>
          {loading ? 'Getting Location...' : 'Request Foreground Location'}
        </Text>
      </Pressable>

      <Pressable style={[styles.button, styles.backgroundButton]} onPress={requestBackgroundLocation}>
        <Text style={styles.buttonText}>Request Background Location</Text>
      </Pressable>

      {location && (
        <View style={styles.result}>
          <Text style={styles.label}>Latitude: {location.latitude}</Text>
          <Text style={styles.label}>Longitude: {location.longitude}</Text>
          <Text style={styles.label}>Accuracy: {location.accuracy} meters</Text>
        </View>
      )}
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    textAlign: 'center',
    marginBottom: 40,
  },
  button: {
    backgroundColor: '#007AFF',
    padding: 16,
    borderRadius: 12,
    alignItems: 'center',
    marginVertical: 10,
  },
  backgroundButton: {
    backgroundColor: '#34C759',
  },
  buttonText: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },
  result: {
    marginTop: 30,
    padding: 20,
    backgroundColor: '#fff',
    borderRadius: 12,
  },
  label: {
    fontSize: 16,
    marginVertical: 4,
  },
});

export default LocationPermissionScreen;
```

---

### **Foreground vs Background**

| Type              | Permission Method | Use Case |
|-------------------|-------------------|----------|
| **Foreground**    | `requestForegroundPermissionsAsync()` | When app is in use |
| **Background**    | `requestBackgroundPermissionsAsync()` | Tracking while app is closed |

## 8. Microphone Permission (`expo-av`)

**Description**: Recording audio requires microphone permission. The `expo-av` library provides a clean way to request microphone access and handle audio recording.

---

### **Installation**
```bash
npx expo install expo-av
```

---

### **Basic Complete Example**
```jsx
import React, { useState } from 'react';
import { View, Pressable, Text, StyleSheet, Alert } from 'react-native';
import { Audio } from 'expo-av';

const MicrophoneScreen = () => {
  const [recording, setRecording] = useState(null);
  const [isRecording, setIsRecording] = useState(false);
  const [permissionStatus, setPermissionStatus] = useState(null);

  const requestMicrophonePermission = async () => {
    const { status } = await Audio.requestPermissionsAsync();
    setPermissionStatus(status);
    return status === 'granted';
  };

  const startRecording = async () => {
    try {
      const hasPermission = await requestMicrophonePermission();

      if (!hasPermission) {
        Alert.alert('Permission Denied', 'Microphone access is required to record audio');
        return;
      }

      await Audio.setAudioModeAsync({
        allowsRecordingIOS: true,
        playsInSilentModeIOS: true,
      });

      const { recording } = await Audio.Recording.createAsync(
        Audio.RecordingOptionsPresets.HIGH_QUALITY
      );

      setRecording(recording);
      setIsRecording(true);
      console.log('Recording started...');
    } catch (error) {
      Alert.alert('Error', 'Failed to start recording');
      console.error(error);
    }
  };

  const stopRecording = async () => {
    try {
      await recording.stopAndUnloadAsync();
      const uri = recording.getURI();
      console.log('Recording stopped. File saved at:', uri);

      setRecording(null);
      setIsRecording(false);
    } catch (error) {
      console.error(error);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Microphone Recording</Text>

      <Pressable 
        style={[styles.button, isRecording && styles.recordingButton]} 
        onPress={isRecording ? stopRecording : startRecording}
      >
        <Text style={styles.buttonText}>
          {isRecording ? ' Stop Recording' : ' Start Recording'}
        </Text>
      </Pressable>

      {isRecording && (
        <Text style={styles.recordingText}>Recording in progress...</Text>
      )}
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    backgroundColor: '#f8f9fa',
    justifyContent: 'center',
    alignItems: 'center',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 50,
  },
  button: {
    backgroundColor: '#007AFF',
    paddingVertical: 18,
    paddingHorizontal: 40,
    borderRadius: 12,
  },
  recordingButton: {
    backgroundColor: '#FF3B30',
  },
  buttonText: {
    color: '#fff',
    fontSize: 18,
    fontWeight: '600',
  },
  recordingText: {
    marginTop: 30,
    color: '#FF3B30',
    fontSize: 16,
    fontWeight: '600',
  },
});

export default MicrophoneScreen;
```

---

### **Key Methods**

| Method                              | Purpose |
|-------------------------------------|---------|
| `Audio.requestPermissionsAsync()`   | Request microphone permission |
| `Audio.Recording.createAsync()`     | Start recording |
| `recording.stopAndUnloadAsync()`    | Stop and save recording |
