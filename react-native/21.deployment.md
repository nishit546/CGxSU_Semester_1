# 1️⃣ What is Deployment?

Deployment is the process of converting your source code into a production-ready mobile application and publishing it for users.

Deployment Process:

```text
Develop App
      │
Testing
      │
Build
      │
Signing
      │
Store Upload
      │
Users Download App
```

---

# 2️⃣ APK Build

APK (Android Package Kit) is used to install Android applications directly.

### Advantages

- Easy testing
- Share with clients
- Internal distribution

### Limitations

- Google Play prefers AAB
- Larger file size

Generate APK

```bash
eas build --platform android
```

---

# 3️⃣ AAB Build

AAB (Android App Bundle) is Google's recommended format.

### Advantages

- Smaller downloads
- Optimized APK generation
- Required by Google Play

Generate AAB

```bash
eas build --platform android
```

---

# 4️⃣ EAS Build

Expo Application Services (EAS) builds apps in the cloud.

Install

```bash
npm install -g eas-cli
```

Login

```bash
eas login
```

Configure

```bash
eas build:configure
```

Android Build

```bash
eas build --platform android
```

iOS Build

```bash
eas build --platform ios
```

Build Both

```bash
eas build --platform all
```

# 7️⃣ Versioning

Every release should have a new version.

Example

```json
{
  "version": "1.2.0",
  "android": {
    "versionCode": 12
  },
  "ios": {
    "buildNumber": "12"
  }
}
```

Meaning

- version → User-visible version
- versionCode → Android internal version
- buildNumber → iOS internal version

---

# 8️⃣ App Signing

App Signing proves that the application is created by the original developer.

Android

- Keystore
- SHA Keys
- Signing Certificate

iOS

- Certificates
- Provisioning Profiles

Expo EAS can manage credentials automatically.

---

# 9️⃣ Release Checklist

Before publishing:

- Remove debug logs
- Test Release Build
- Check Permissions
- Optimize Images
- Verify API URLs
- Update Version
- Generate Signed Build
- Test on Real Device

#  Best Practices

- Use EAS Build
- Keep version numbers organized
- Always test release builds
- Store signing keys safely
- Automate builds with CI/CD
- Maintain release notes

- APK is useful for testing and direct installation.
