# 1️⃣ What is Testing?

Testing verifies that your application behaves correctly.

Instead of manually checking everything after every change, automated tests verify your code.

Benefits:

- Detect bugs early
- Improve code quality
- Refactor confidently
- Prevent regressions
- Improve maintainability

---

# 2️⃣ Types of Testing

| Type | Purpose |
|------|---------|
| Unit Testing | Test individual functions/components |
| Integration Testing | Test multiple components together |
| Snapshot Testing | Detect unexpected UI changes |
| End-to-End Testing | Test complete user flows |

This module focuses on:

- Jest
- React Native Testing Library
- Unit Testing

# 3️⃣ Jest

Jest is the default testing framework used with React Native.

### Features

- Fast
- Easy Setup
- Mocking Support
- Snapshot Testing
- Code Coverage

Install (if needed):

```bash
npm install --save-dev jest
```

Run tests:

```bash
npm test
```

or

```bash
npm run test
```

# 4️⃣ Writing Your First Test

Example Function

```jsx
export function add(a, b) {
  return a + b;
}
```

Test File

```jsx
import { add } from "./math";

test("adds two numbers", () => {
  expect(add(2, 3)).toBe(5);
});
```

---

# 5️⃣ React Native Testing Library (RNTL)

RNTL helps test components the way users interact with them.

Install

```bash
npm install --save-dev @testing-library/react-native
```

Example

```jsx
import { render } from "@testing-library/react-native";
import App from "./App";

test("renders title", () => {
  const { getByText } = render(<App />);

  expect(getByText("Welcome")).toBeTruthy();
});
```

# 6️⃣ Unit Testing

Unit testing verifies a single unit of code.

Example

```jsx
function multiply(a, b) {
  return a * b;
}
```

Test

```jsx
test("multiply numbers", () => {
  expect(multiply(2, 5)).toBe(10);
});
```

### What Should Be Unit Tested?

- Utility Functions
- Custom Hooks
- Validation Logic
- Reducers
- Business Logic

# 7️⃣ Component Testing

Example

```jsx
<Button title="Login" />
```

Test

```jsx
const { getByText } = render(
  <Button title="Login" />
);

expect(getByText("Login")).toBeTruthy();
```

---

# 8️⃣ Snapshot Testing

Snapshot testing captures the rendered UI.

```jsx
import renderer from "react-test-renderer";

test("matches snapshot", () => {
  const tree = renderer
    .create(<App />)
    .toJSON();

  expect(tree).toMatchSnapshot();
});
```

### Benefits

- Detect accidental UI changes
- Easy regression testing
- Quick visual verification

---

# 9️⃣ Mocking

Sometimes APIs should not be called during tests.

Example

```jsx
jest.mock("axios");
```

Mock Function

```jsx
axios.get.mockResolvedValue({
  data: [],
});
```

Use Cases

- API Testing
- Async Functions
- Database Calls

#  Best Practices

- Write small tests
- Keep tests independent
- Mock external services
- Test user behavior
- Use descriptive test names
- Keep snapshots updated

---

#  Common Mistakes

- Testing implementation instead of behavior
- Writing huge test files
- Ignoring failing tests
- Not mocking APIs
- Overusing snapshot tests
