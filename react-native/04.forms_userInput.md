# 1️⃣ What is User Input?

User input allows users to interact with your app by entering data such as:

- Name
- Email
- Password
- Phone Number
- OTP
- Search Queries

The primary component used is **TextInput**.

---

# 2️⃣ TextInput

```jsx
import { TextInput } from "react-native";

<TextInput
  placeholder="Enter your name"
/>
```

## Common Props

- placeholder
- value
- onChangeText
- keyboardType
- secureTextEntry
- maxLength
- editable
- multiline
- autoFocus

### Controlled Input

```jsx
const [name, setName] = useState("");

<TextInput
  value={name}
  onChangeText={setName}
/>
```

# 3️⃣ Keyboard Handling

When the keyboard opens, UI can be hidden.

Use `KeyboardAvoidingView`:

```jsx
import { KeyboardAvoidingView } from "react-native";

<KeyboardAvoidingView behavior="padding">
  {/* Form */}
</KeyboardAvoidingView>
```

Dismiss keyboard:

```jsx
import { Keyboard } from "react-native";

Keyboard.dismiss();
```

# 4️⃣ Search Bar

```jsx
const [query, setQuery] = useState("");

<TextInput
  placeholder="Search..."
  value={query}
  onChangeText={setQuery}
/>
```

## Best Practices

- Debounce API calls
- Trim whitespace
- Show empty state

# 5️⃣ Login Form

Typical fields:

- Email
- Password

```jsx
const [email, setEmail] = useState("");
const [password, setPassword] = useState("");

<TextInput
  placeholder="Email"
  value={email}
  onChangeText={setEmail}
/>

<TextInput
  placeholder="Password"
  secureTextEntry
  value={password}
  onChangeText={setPassword}
/>
```

# 6️⃣ OTP Input

```jsx
<TextInput
  keyboardType="number-pad"
  maxLength={1}
/>
```

### Tips

- Auto focus next box
- Auto submit after last digit
- Paste support

# 7️⃣ Validation

## Why Validation?

Validation prevents invalid data from reaching the server.

### Required Field

```jsx
if (!email) {
  alert("Email is required");
}
```

### Email Validation

```jsx
const emailRegex =
/^[^\s@]+@[^\s@]+\.[^\s@]+$/;
```

### Password Rules

- Minimum 8 characters
- Uppercase
- Lowercase
- Number
- Special character

# 9️⃣ Best Practices

- Use controlled inputs
- Validate before API calls
- Show clear error messages
- Disable submit while loading
- Trim user input
- Use secureTextEntry for passwords

---

#  Common Mistakes

- Not validating input
- Forgetting secureTextEntry
- Allowing empty submissions

- Using wrong keyboardType
