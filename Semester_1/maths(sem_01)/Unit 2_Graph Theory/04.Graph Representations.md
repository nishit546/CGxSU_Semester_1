---

# 🌐 **Graph Representations**

A **Graph Representation** means how we store or visualize the **connections (edges)** between **vertices (nodes)** in a graph.
Graphs are represented mainly in three standard formats:
1️⃣ **Adjacency Matrix**
2️⃣ **Incidence Matrix**
3️⃣ **Adjacency List**

Each format has its **own advantages** depending on the type and size of graph.

---

## 🧩 **1️⃣ Adjacency Matrix**

### 📘 **Definition**

An **Adjacency Matrix** is a **square matrix (V × V)** used to represent a graph.

* Each **row** and **column** represents a **vertex**.
* Entry **1 (or weight)** → there is an edge between vertices.
* Entry **0** → no connection.
* For **undirected graphs**, the matrix is **symmetric**.
* For **directed graphs**, the direction is represented by **row → column** entry.

---

### ⚙️ **Case 1: Undirected Graph**

#### 🧮 **Example 1 — Simple Connection**

**Vertices:** A, B, C
**Edges:** (A–B), (B–C)

**Visual Representation:**

```
A —— B —— C
```

**Adjacency Matrix:**

|       | A | B | C |
| ----- | - | - | - |
| **A** | 0 | 1 | 0 |
| **B** | 1 | 0 | 1 |
| **C** | 0 | 1 | 0 |

💡 **Explanation:**

* A connected with B → A,B entries are 1.
* B connected with C → B,C entries are 1.
* Symmetric because the graph is undirected.

---

#### 🧮 **Example 2 — 4 Vertices**

**Vertices:** A, B, C, D
**Edges:** (A–B), (A–C), (B–D)

**Visual Representation:**

```
     A —— B
     |     \
     C      D
```

**Adjacency Matrix:**

|       | A | B | C | D |
| ----- | - | - | - | - |
| **A** | 0 | 1 | 1 | 0 |
| **B** | 1 | 0 | 0 | 1 |
| **C** | 1 | 0 | 0 | 0 |
| **D** | 0 | 1 | 0 | 0 |

💬 **Explanation:**

* A connects to B and C.
* B connects to A and D.
* Matrix is again symmetric ✅

---

#### 🧮 **Example 3 — Weighted Graph**

**Edges:** (A–B, 3), (A–C, 5), (B–C, 2)

**Visual Representation:**

```
A
|\ 
| \
|  \
B——C
Weights: (A,B)=3, (A,C)=5, (B,C)=2
```

**Adjacency Matrix:**

|       | A | B | C |
| ----- | - | - | - |
| **A** | 0 | 3 | 5 |
| **B** | 3 | 0 | 2 |
| **C** | 5 | 2 | 0 |

🧠 **Explanation:**
Each entry holds the **edge weight** instead of 1.

---

#### 🧮 **Example 4 — Disconnected Graph**

**Edges:** (A–B)

**Visual:**

```
A —— B    C    D
```

**Matrix:**

|       | A | B | C | D |
| ----- | - | - | - | - |
| **A** | 0 | 1 | 0 | 0 |
| **B** | 1 | 0 | 0 | 0 |
| **C** | 0 | 0 | 0 | 0 |
| **D** | 0 | 0 | 0 | 0 |

💡 **Only A and B connected**, others isolated (0s show no links).

---

### ⚙️ **Case 2: Directed Graph**

#### 🧮 **Example 1**

**Edges:** A→B, B→C

**Visual:**

```
A → B → C
```

|       | A | B | C |
| ----- | - | - | - |
| **A** | 0 | 1 | 0 |
| **B** | 0 | 0 | 1 |
| **C** | 0 | 0 | 0 |

🎯 **Explanation:**

* A→B → A-row, B-column = 1
* B→C → B-row, C-column = 1

---

#### 🧮 **Example 2**

**Edges:** A→B, A→C, B→A

**Visual:**

```
A → B
↑   ↓
C ← 
```

|       | A | B | C |
| ----- | - | - | - |
| **A** | 0 | 1 | 1 |
| **B** | 1 | 0 | 0 |
| **C** | 0 | 0 | 0 |

📊 **Explanation:**
A sends edges to B and C, while B points back to A.

---

#### 🧮 **Example 3 — Weighted Directed**

**Edges:** A→B(2), B→C(4), A→C(6)

|       | A | B | C |
| ----- | - | - | - |
| **A** | 0 | 2 | 6 |
| **B** | 0 | 0 | 4 |
| **C** | 0 | 0 | 0 |

💬 **Explanation:**
Weights show directed connection strengths.

---

#### 🧮 **Example 4 — Self-loop**

**Edges:** A→A, A→B

|       | A | B |
| ----- | - | - |
| **A** | 1 | 1 |
| **B** | 0 | 0 |

🔁 **Self-loop** represented as diagonal 1.

---

## 🧮 **2️⃣ Incidence Matrix**

### 📘 **Definition**

An **Incidence Matrix** shows the relationship between **vertices** (rows) and **edges** (columns).

* For **undirected graphs**:

  * 1 if the vertex is connected to the edge
  * 0 otherwise
* For **directed graphs**:

  * -1 → edge starts from vertex
  * +1 → edge ends at vertex

---

### ⚙️ **Case 1: Undirected Graph**

#### 🧮 **Example 1**

**Edges:** e₁(A–B), e₂(B–C)

**Visual:**

```
A —— B —— C
```

|       | e₁ | e₂ |
| ----- | -- | -- |
| **A** | 1  | 0  |
| **B** | 1  | 1  |
| **C** | 0  | 1  |

💡 **B** connects both edges.

---

#### 🧮 **Example 2**

**Edges:** e₁(A–B), e₂(A–C), e₃(C–D)

**Visual:**

```
A——B
|
C——D
```

|       | e₁ | e₂ | e₃ |
| ----- | -- | -- | -- |
| **A** | 1  | 1  | 0  |
| **B** | 1  | 0  | 0  |
| **C** | 0  | 1  | 1  |
| **D** | 0  | 0  | 1  |

---

#### 🧮 **Example 3**

**Edges:** e₁(A–B), e₂(B–D), e₃(C–D)

|       | e₁ | e₂ | e₃ |
| ----- | -- | -- | -- |
| **A** | 1  | 0  | 0  |
| **B** | 1  | 1  | 0  |
| **C** | 0  | 0  | 1  |
| **D** | 0  | 1  | 1  |

💬 **Explanation:**
Each edge touches two vertices → both get 1s.

---

#### 🧮 **Example 4 — Disconnected**

**Edges:** e₁(A–B)**

|       | e₁ |
| ----- | -- |
| **A** | 1  |
| **B** | 1  |
| **C** | 0  |
| **D** | 0  |

📌 Only A & B are connected.

---

### ⚙️ **Case 2: Directed Graph**

#### 🧮 **Example 1**

**Edges:** e₁: A→B, e₂: B→C

|       | e₁ | e₂ |
| ----- | -- | -- |
| **A** | -1 | 0  |
| **B** | 1  | -1 |
| **C** | 0  | 1  |

📈 **Outgoing = -1**, **Incoming = +1**

---

#### 🧮 **Example 2**

**Edges:** e₁: A→B, e₂: C→A

|       | e₁ | e₂ |
| ----- | -- | -- |
| **A** | -1 | 1  |
| **B** | 1  | 0  |
| **C** | 0  | -1 |

---

#### 🧮 **Example 3**

**Edges:** e₁: A→B, e₂: B→A, e₃: A→C**

|       | e₁ | e₂ | e₃ |
| ----- | -- | -- | -- |
| **A** | -1 | 1  | -1 |
| **B** | 1  | -1 | 0  |
| **C** | 0  | 0  | 1  |

---

#### 🧮 **Example 4 — Self-loop**

**Edges:** e₁: A→A**

|       | e₁ |
| ----- | -- |
| **A** | 0  |

🔁 Self-loop counted as 0 in directed incidence form.

---

## 📜 **3️⃣ Adjacency List**

### 📘 **Definition**

An **Adjacency List** stores each vertex along with a **list of its neighbors**.

* It uses **linked lists or arrays**.
* Ideal for **sparse graphs** (few edges).
* Saves memory compared to adjacency matrix.

---

### ⚙️ **Case 1: Undirected Graph**

#### 🧮 **Example 1**

```
A —— B —— C
```

**Adjacency List:**

```
A → B  
B → A, C  
C → B
```

---

#### 🧮 **Example 2**

```
A —— B
|     \
C      D
```

**Adjacency List:**

```
A → B, C  
B → A, D  
C → A  
D → B
```

---

#### 🧮 **Example 3 (Weighted)**

```
A——(3)——B——(4)——C
 \        /
  (2)   (1)
    \  /
      D
```

**Adjacency List:**

```
A → B(3), D(2)  
B → A(3), C(4), D(1)  
C → B(4)  
D → A(2), B(1)
```

---

#### 🧮 **Example 4 — Disconnected**

```
A —— B     C     D
```

**Adjacency List:**

```
A → B  
B → A  
C →  
D → 
```

---

### ⚙️ **Case 2: Directed Graph**

#### 🧮 **Example 1**

```
A → B → C
```

**Adjacency List:**

```
A → B  
B → C  
C → 
```

---

#### 🧮 **Example 2**

```
A → B
A → C
B → D
```

**Adjacency List:**

```
A → B, C  
B → D  
C →  
D → 
```

---

#### 🧮 **Example 3 (Weighted Directed)**

```
A → B(2), C(5)
B → C(3)
```

**Adjacency List:**

```
A → B(2), C(5)  
B → C(3)  
C → 
```

---

#### 🧮 **Example 4 (Self-loop)**

```
A → A
```

**Adjacency List:**

```
A → A
```
