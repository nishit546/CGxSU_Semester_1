

# 🧭 **Shortest Path Overview & Dijkstra’s Algorithm**

---

## 🌉 **1️⃣ Shortest Path Overview**

### 💡 **Concept**

The **Shortest Path Problem** in graph theory is about finding a path between two vertices so that the **total sum of edge weights is minimum**.
Edge weights represent **distance, cost, or time**, depending on the context.

---

### 🧠 **Key Points**

🔹 **Weighted Graph:** Edges have numerical values (distance, cost, etc.)
🔹 **Unweighted Graph:** All edges treated equally
🔹 **Single Source Shortest Path:** One starting node → all others
🔹 **All Pairs Shortest Path:** All-to-all shortest paths
🔹 **Real-World Uses:**

* 🚗 GPS navigation
* 🌐 Internet routing
* ✈️ Airline planning
* 🤖 Robot pathfinding
* 📦 Delivery logistics

## 🧩 **Example 1: Simple Weighted Graph**

**Find the shortest path from A → D**

Paths:

* A → B → D = 2 + 6 = 8
* A → C → D = 4 + 3 = 7
* A → B → C → D = 2 + 1 + 3 = 6 ✅

🎯 **Shortest Path:** `A → B → C → D`
🧮 **Total Cost = 6**

## 🧮 **2️⃣ Dijkstra’s Algorithm**

### 💡 **Concept**

**Dijkstra’s Algorithm** is a **Greedy Algorithm** that finds the **shortest path from a single source** to all other nodes in a **weighted graph** (with non-negative weights).

📜 Invented by **Edsger W. Dijkstra (1956)** — the foundation of modern pathfinding algorithms used in networks, maps, and AI.

---

### ⚙️ **Key Properties**

🔸 Works only for **non-negative weights**
🔸 Uses a **Priority Queue (Min-Heap)**
🔸 Two sets:

* ✅ **Visited nodes** – distance finalized
* ⏳ **Unvisited nodes** – still to process
  🔸 Continuously updates **tentative distances**
  🔸 Stops when all vertices are finalized

---

### 🧭 **Step-by-Step Idea**

1. Initialize: `distance[start] = 0`, others = ∞
2. Select vertex with **minimum distance**
3. Update neighbor distances
4. Mark node as **visited**
5. Repeat until all are visited

---

### 🖼️ **Visual Flow**

```
        (A)
       /   \
   4 /       \ 2
     /         \
   (B)----5----(C)
      \        /
       \10   /3
        (D)
```

🎯 **Find Shortest Path A → D**

1️⃣ Start at A → distance 0
2️⃣ B = 4, C = 2
3️⃣ Choose smaller → C (2)
4️⃣ From C → D = 2 + 3 = 5 ✅
🎯 **Shortest Path:** `A → C → D`
🧮 **Total = 5**
